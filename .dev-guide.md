# CRM 三系统本地开发环境指南

> 最后更新: 2026-04-10  
> Node.js 要求: >= 22.0.0 (当前使用 v24.x)

---

## 一、服务启动方法

### 端口分配

| 服务 | 端口 | 目录 | 启动命令 |
|------|------|------|----------|
| 后端 API | 3000 | `backend/` | `npm run dev` |
| CRM 前端 | 5173 | 项目根目录 | `npm run dev` |
| Admin 管理后台 | 5174 | `admin/` | `npm run dev` |
| Website 官网 | 8080 | `website/` | `npm run dev` |

### 快速启动（IDE 终端）

在 IDE 中打开 **4 个终端标签页**，分别执行：

```bash
# 终端1 - 后端（必须先启动，其他服务依赖它的 API）
cd backend && npm run dev

# 终端2 - CRM前端
npm run dev

# 终端3 - Admin管理后台
cd admin && npm run dev

# 终端4 - Website官网
cd website && npm run dev
```

### 后端启动原理

后端使用 `nodemon` + `ts-node` 运行 TypeScript：
- 入口文件: `backend/src/app.ts`
- nodemon 配置: `backend/nodemon.json`
- 执行命令: `ts-node --transpile-only src/app.ts`
- 环境配置: 开发环境自动加载 `backend/.env.local`
- 数据库: MySQL 8.0+ (端口 3306)

### 停止服务

```powershell
# 停止所有 node 进程
Get-Process -Name "node" -ErrorAction SilentlyContinue | Stop-Process -Force
```

### 端口检查

```powershell
netstat -ano | findstr "LISTENING" | findstr "3000 5173 5174 8080"
```

## 二、常见启动问题排查

### 1. 后端静默崩溃（无错误输出）

**诊断命令**（能看到完整错误）：
```bash
cd backend
node -e "try{require('ts-node/register/transpile-only');require('./src/app')}catch(e){console.error(e.message);console.error(e.stack);process.exit(1)}"
```

### 2. 常见错误: `xxx is not defined`

原因: 路由文件存在但忘记在 `index.ts` 中 import。  
检查: `backend/src/routes/admin/index.ts` 和 `backend/src/app.ts` 的 import 是否完整。

### 3. 端口被占用

```powershell
# 查看占用端口的进程
netstat -ano | findstr ":3000 " | findstr "LISTENING"
# 根据 PID 杀进程
taskkill /PID <PID> /F
```

### 4. ESM/CJS 兼容问题 (Node v24+)

Node v24 默认开启 `--experimental-require-module`，部分 ESM-only 包与 ts-node 的 CJS 模式冲突。  
解决方案: 确保依赖包同时支持 CJS (如 `uuid` 使用 v9 而非 v13+)。

## 三、开发规范（必须严格遵守）

### JSON 转义规范 ⚠️ 重要

**所有涉及 JSON 的场景必须遵守以下规范，违反将导致运行时错误：**

1. **禁止手动拼接 JSON 字符串**
   ```typescript
   // ❌ 错误 - 手动拼接容易转义出错
   const json = '{"name": "' + name + '", "value": "' + value + '"}';
   
   // ✅ 正确 - 使用 JSON.stringify
   const json = JSON.stringify({ name, value });
   ```

2. **JSON 字段值中的特殊字符必须正确转义**
   ```
   双引号 " → \"
   反斜杠 \ → \\
   换行符   → \n
   制表符   → \t
   ```

3. **数据库存取 JSON 必须使用 JSON.stringify / JSON.parse**
   ```typescript
   // ❌ 错误
   await db.query(`UPDATE config SET value = '${jsonStr}' WHERE key = 'xxx'`);
   
   // ✅ 正确
   await db.query(`UPDATE config SET value = ? WHERE key = ?`, [JSON.stringify(data), 'xxx']);
   ```

4. **API 返回 JSON 让框架处理，不要手动序列化**
   ```typescript
   // ❌ 错误
   res.send(JSON.stringify({ success: true }));
   
   // ✅ 正确
   res.json({ success: true });
   ```

5. **前端模板中的 JSON 使用 v-bind 而非内联**
   ```vue
   <!-- ❌ 错误 -->
   <component :data="'{"key": "value"}'">
   
   <!-- ✅ 正确 -->
   <component :data="{ key: 'value' }">
   ```

### 路由注册规范

**新增路由文件后，必须完成以下两步，否则后端会静默崩溃：**

1. 在路由目录的 `index.ts` 中添加 `import` 语句
2. 在 `router.use()` 中注册路由

```typescript
// 示例: 新增 backend/src/routes/admin/xxx.ts
// 必须在 backend/src/routes/admin/index.ts 中:
import xxxRouter from './xxx';          // 第1步: import
router.use('/xxx', xxxRouter);          // 第2步: 注册
```

### TypeScript 编码规范

1. **禁止使用 `any` 类型**（除非有充分理由并写注释说明）
2. **所有 import 必须有对应的使用**（未使用的 import 会增加启动时间）
3. **Entity 字段类型必须显式声明**（TypeORM 需要 `emitDecoratorMetadata`）
4. **禁止循环引用**（Entity 之间用 `() => TargetEntity` 延迟加载）

### 依赖管理规范

1. **后端是 CommonJS 项目**（`tsconfig.json` → `"module": "commonjs"`）
2. **新增依赖前检查是否纯 ESM**（纯 ESM 包在后端 CJS 环境下会报错）
3. **检查方法**: 查看包的 `package.json` 是否有 `"type": "module"` 且无 CJS 入口
4. **已知不兼容包**: `uuid@13+`（纯 ESM），应使用 `uuid@9`
