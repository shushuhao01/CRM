# CRM系统呼入功能技术设计文档

> 版本：v1.0  
> 日期：2026-05-02  
> 状态：待评审

---

## 一、背景概述

CRM系统已完成外呼功能开发，支持：
- **工作手机外呼**：CRM端发起 → 后端通过WebSocket通知APP → APP拨号 → 状态同步回CRM
- **网络电话外呼**：CRM端发起 → 后端调用云通信API（阿里云等） → Webhook回调更新状态
- **传统线路外呼**：通过SIP中继/PBX发起

**现需解决呼入（Inbound）场景**：客户拨打成员绑定的工作手机号码或传统分机号时，CRM系统能实时感知、显示来电信息并记录通话。

---

## 二、呼入场景分类

| 场景 | 触发方式 | 当前状态 | 需要做的工作 |
|------|---------|---------|-------------|
| **场景1：传统分机/SIP呼入** | 企业PBX/SIP网关回调Webhook | ✅ 后端Webhook已有 / ❌ CRM前端未接收 | CRM前端接收CALL_INCOMING事件 |
| **场景2：工作手机呼入** | 客户拨打成员绑定手机号 | ❌ 完全未开发 | APP检测呼入→上报后端→推送CRM |
| **场景3：网络电话呼入** | 客户拨打云通信DID号码 | ⚠️ 阿里云Webhook可复用 | 配置云端DID+呼入路由 |

---

## 三、整体架构设计

### 3.1 场景1：传统分机/SIP呼入流程

```
客户拨打企业号码
    ↓
企业PBX/SIP网关
    ↓ POST /api/v1/calls/webhook/sip/incoming
CRM后端（callWebhook.ts）
    ├─ 1. 根据主叫号码查找客户信息
    ├─ 2. 根据被叫号码/DID找到负责人
    ├─ 3. 创建inbound类型call_records记录
    └─ 4. WebSocket推送 CALL_INCOMING 给CRM端
         ↓
CRM前端（CallManagement.vue）
    ├─ 弹出来电提醒弹窗
    ├─ 显示客户信息（姓名、号码、等级、历史）
    └─ 操作：接听确认 / 拒绝 / 查看详情 / 快速跟进
         ↓
PBX/SIP网关
    ↓ POST /api/v1/calls/webhook/sip/status
CRM后端
    ├─ 更新call_records（answered/hangup/missed）
    ├─ 推送状态变化给CRM前端
    └─ 如有录音URL，更新记录
```

### 3.2 场景2：工作手机呼入流程

```
客户拨打成员工作手机号码
    ↓
手机系统响铃（APP在后台运行）
    ↓ 原生API检测来电
APP端（原生模块）
    ├─ 检测到来电状态（RINGING）
    ├─ 获取来电号码
    └─ 通过WebSocket发送 INCOMING_CALL_DETECTED
         ↓
CRM后端（MobileWebSocketService.ts）
    ├─ 1. 解析来电号码
    ├─ 2. 查找匹配客户信息
    ├─ 3. 创建inbound类型call_records记录
    └─ 4. WebSocket推送 CALL_INCOMING 给CRM端
         ↓
CRM前端（CallManagement.vue）
    ├─ 弹出来电提醒弹窗
    └─ 显示客户信息
         ↓
APP端检测通话状态变化
    ├─ 接听(OFFHOOK) → 发送 CALL_STATUS {status: 'connected'}
    ├─ 结束(IDLE) → 发送 CALL_ENDED {duration, hasRecording}
    └─ 录音文件 → POST /api/v1/mobile/recording/upload
```

### 3.3 场景3：网络电话（云通信DID）呼入流程

```
客户拨打企业DID号码（如 400-xxx-xxxx）
    ↓
云通信服务商（阿里云/腾讯云）
    ↓ 呼入路由：回调到CRM Webhook
CRM后端（callWebhook.ts）
    ├─ 复用 /aliyun/status 或新增 /aliyun/incoming
    ├─ 识别呼入方向（inbound）
    ├─ 查找客户 + 匹配负责人
    └─ 推送 CALL_INCOMING
         ↓
（后续同场景1）
```

## 四、数据库变更

### 4.1 call_records 表 - 无需变更

现有表结构已支持呼入：
- `call_type` 已有 `'inbound'` 枚举值
- `call_status` 已有 `'connected' | 'missed' | 'busy' | 'failed' | 'rejected'` 等状态
- `caller_number` 可存储主叫号码
- `call_method` 可存储呼入来源（`'sip'` / `'mobile'` / `'voip'`）

### 4.2 call_records 表 - 建议新增字段

```sql
-- 新增呼入相关字段（可选，增强呼入管理能力）
ALTER TABLE call_records
  ADD COLUMN `call_direction` ENUM('outbound', 'inbound') DEFAULT 'outbound'
    COMMENT '通话方向（与call_type冗余，用于更清晰的查询）' AFTER call_type,
  ADD COLUMN `ring_duration` INT DEFAULT 0
    COMMENT '振铃时长（秒），用于分析接听效率' AFTER duration,
  ADD COLUMN `queue_time` INT DEFAULT 0
    COMMENT '排队等待时长（秒）' AFTER ring_duration,
  ADD COLUMN `transfer_from` VARCHAR(50) NULL
    COMMENT '转接来源（其他成员ID）' AFTER queue_time,
  ADD COLUMN `transfer_to` VARCHAR(50) NULL
    COMMENT '转接目标（其他成员ID）' AFTER transfer_from,
  ADD COLUMN `inbound_source` VARCHAR(50) NULL
    COMMENT '呼入来源标识：sip/mobile/did/ivr' AFTER transfer_to;

-- 索引
ALTER TABLE call_records ADD INDEX idx_call_direction (call_direction);
ALTER TABLE call_records ADD INDEX idx_inbound_source (inbound_source);
```

> **说明**：以上字段为可选增强，最小可行方案不需要这些字段。现有 `call_type='inbound'` + `call_method` 字段已够用。

### 4.3 新增表：呼入路由配置表（可选）

### 8.1 MobileWebSocketService.ts — 添加 INCOMING_CALL_DETECTED 处理

在 `handleMessage` 的 `switch` 中新增：

新增方法：

```typescript
private async handleIncomingCallDetected(
  deviceId: string, userId: number, data: any
): Promise<void> {
  try {
    const dataSource = getDataSource();
    if (!dataSource || !data.callerNumber) return;

    const callerNumber = data.callerNumber;
    logger.info(`[MobileWS] APP检测到来电: ${callerNumber}, deviceId: ${deviceId}`);

    // 1. 查找客户信息
    let customerId = null;
    let customerName = '未知来电';
    let customerLevel = null;
    let company = null;
    let lastCallTime = null;
    let tags = [];

    const customers = await dataSource.query(
      `SELECT c.id, c.name, c.level, c.company, c.tags,
              (SELECT MAX(start_time) FROM call_records
               WHERE customer_id = c.id) as last_call
       FROM customers c
       WHERE c.phone = ? OR c.mobile = ?
       LIMIT 1`,
      [callerNumber, callerNumber]
    );

    if (customers.length > 0) {
      const cust = customers[0];
      customerId = cust.id;
      customerName = cust.name;
      customerLevel = cust.level;
      company = cust.company;
      lastCallTime = cust.last_call;
      tags = cust.tags ? JSON.parse(cust.tags) : [];
    }

    // 2. 获取工作手机信息
    const phones = await dataSource.query(
      `SELECT id, phone_number, device_name FROM work_phones
       WHERE device_id = ? LIMIT 1`, [deviceId]
    );
    const phoneName = phones[0]?.device_name || phones[0]?.phone_number || '';

    // 3. 创建呼入通话记录
    const callId = `IN-${Date.now()}-${Math.random().toString(36).slice(2, 9)}`;
    const userInfo = await dataSource.query(
      `SELECT name, real_name FROM users WHERE id = ? LIMIT 1`, [userId]
    );
    const userName = userInfo[0]?.real_name || userInfo[0]?.name || '';

    await dataSource.query(
      `INSERT INTO call_records
       (id, customer_id, customer_name, customer_phone, call_type, call_status,
        call_method, user_id, user_name, start_time, created_at, tenant_id)
       VALUES (?, ?, ?, ?, 'inbound', 'ringing', 'mobile', ?, ?, NOW(), NOW(),
               (SELECT tenant_id FROM work_phones WHERE device_id = ? LIMIT 1))`,
      [callId, customerId, customerName, callerNumber,
       String(userId), userName, deviceId]
    );

    // 4. 推送来电通知给CRM端
    if (global.webSocketService) {
      global.webSocketService.sendToUser(userId, 'CALL_INCOMING', {
        callId,
        callerNumber,
        callSource: 'mobile',
        customerInfo: {
          customerId,
          customerName,
          customerLevel,
          company,
          lastCallTime,
          tags
        },
        deviceInfo: {
          deviceId,
          phoneName
        },
        timestamp: new Date().toISOString()
      });
    }

    // 5. 回传callId给APP
    this.sendToDevice(deviceId, {
      type: 'INCOMING_CALL_CONFIRMED',
      data: {
        callId,
        callerNumber,
        customerName,
        customerId
      }
    });

    logger.info(`[MobileWS] 来电已处理: callId=${callId}, customer=${customerName}`);
  } catch (error: any) {
    logger.error('[MobileWS] 处理来电检测失败:', error.message);
  }
}
```

### 9.2 Android 来电检测实现

#### 核心代码（Kotlin/Java）

```kotlin
// 电话状态监听器
class CallStateReceiver : BroadcastReceiver() {
    override fun onReceive(context: Context, intent: Intent) {
        val state = intent.getStringExtra(TelephonyManager.EXTRA_STATE)
        val incomingNumber = intent.getStringExtra(
            TelephonyManager.EXTRA_INCOMING_NUMBER
        )

        when (state) {
            TelephonyManager.EXTRA_STATE_RINGING -> {
                // 检测到来电 - 上报CRM
                WebSocketManager.send(JsonObject().apply {
                    addProperty("type", "INCOMING_CALL_DETECTED")
                    add("data", JsonObject().apply {
                        addProperty("callerNumber", incomingNumber)
                        addProperty("ringStartTime", System.currentTimeMillis())
                    })
                })
            }
            TelephonyManager.EXTRA_STATE_OFFHOOK -> {
                // 已接听
                WebSocketManager.send(JsonObject().apply {
                    addProperty("type", "CALL_STATUS")
                    add("data", JsonObject().apply {
                        addProperty("callId", currentCallId)
                        addProperty("status", "connected")
                    })
                })
                // 开始录音
                CallRecorder.startRecording(currentCallId)
            }
            TelephonyManager.EXTRA_STATE_IDLE -> {
                // 通话结束
                CallRecorder.stopRecording()
                WebSocketManager.send(JsonObject().apply {
                    addProperty("type", "CALL_ENDED")
                    add("data", JsonObject().apply {
                        addProperty("callId", currentCallId)
                        addProperty("duration", callDuration)
                        addProperty("hasRecording", true)
                    })
                })
            }
        }
    }
}
```

### 9.3 iOS 来电检测实现

```swift
import CallKit

class CallObserver: NSObject, CXCallObserverDelegate {
    let callObserver = CXCallObserver()

    func startMonitoring() {
        callObserver.setDelegate(self, queue: nil)
    }

    func callObserver(_ callObserver: CXCallObserver,
                      callChanged call: CXCall) {
        if call.isOutgoing { return } // 忽略外呼

        if call.hasConnected && !call.hasEnded {
            // 来电已接听
            WebSocketManager.shared.send(type: "CALL_STATUS",
                data: ["callId": currentCallId, "status": "connected"])
        } else if call.hasEnded {
            // 来电结束
            WebSocketManager.shared.send(type: "CALL_ENDED",
                data: ["callId": currentCallId, "duration": duration])
        } else if !call.isOutgoing && !call.hasConnected && !call.hasEnded {
            // 来电响铃中
            // 注意：iOS无法直接获取来电号码，需要用CallDirectory Extension
            WebSocketManager.shared.send(type: "INCOMING_CALL_DETECTED",
                data: ["ringStartTime": Date().timeIntervalSince1970])
        }
    }
}
```

> **iOS限制**：iOS不允许后台直接获取来电号码。需要使用 `CallDirectory Extension` 结合 `CXCallDirectoryProvider`，或者通过通话结束后查询通话记录获取号码。

### 9.4 通话录音方案

#### Android

```kotlin
class CallRecorder {
    private var mediaRecorder: MediaRecorder? = null

    fun startRecording(callId: String) {
        val outputFile = File(cacheDir, "recording_$callId.mp3")

        mediaRecorder = MediaRecorder().apply {
            setAudioSource(MediaRecorder.AudioSource.VOICE_COMMUNICATION)
            setOutputFormat(MediaRecorder.OutputFormat.MPEG_4)
            setAudioEncoder(MediaRecorder.AudioEncoder.AAC)
            setAudioSamplingRate(44100)
            setAudioEncodingBitRate(128000)
            setOutputFile(outputFile.absolutePath)
            prepare()
            start()
        }
    }

    fun stopRecording(): File? {
        mediaRecorder?.apply {
            stop()
            release()
        }
        mediaRecorder = null
        return outputFile
    }
}
```

> **Android录音限制**：Android 9+ 部分厂商限制通话录音。建议使用 `AudioSource.VOICE_COMMUNICATION` 或 `VOICE_CALL`（需系统权限）。某些定制ROM（小米、华为等）可能需要额外适配。

#### iOS

iOS系统不允许第三方APP录制电话通话音频。替代方案：
1. **三方通话录音**：通话接通后拨入录音服务号码形成三方通话
2. **手动备忘录**：通话结束后通过APP记录文字备注
3. **VoIP模式**：使用VoIP通话（如SIP软电话）可以录音

## 十二、注意事项

1. **隐私合规**：通话录音需获得用户明确授权，建议通话开始时播放提示音
2. **多租户**：所有查询需加租户隔离（已有 `tenantRawSQL()` 工具）
3. **并发来电**：同一用户同时收到多个来电时，需队列处理
4. **APP后台保活**：Android需要前台服务（Foreground Service）保持后台运行
5. **iOS推送**：当APP被杀死时，需要通过APNs推送唤醒APP（VoIP Push）
6. **录音存储**：录音文件可能较大，需要考虑存储限制（已有 `checkStorageLimit` 中间件）
7. **号码归一化**：来电号码可能有不同格式（+86、区号等），查找客户时需归一化处理
