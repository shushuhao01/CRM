## 一、软件著作权概述

### 1.1 什么是软件著作权

计算机软件著作权（简称"软著"）是指软件的开发者或者其他权利人依据有关著作权法律的规定，对于软件作品所享有的各项专有权利。

### 1.2 为什么要申请软著

| 作用 | 说明 |
|------|------|
| **法律保护** | 明确权利归属，发生侵权时可作为维权依据 |
| **企业资质** | 申请高新技术企业、双软认证的前置条件 |
| **税收优惠** | 软件产品增值税即征即退、企业所得税减免 |
| **项目招投标** | 政府项目、企业采购招标时的加分项 |
| **融资估值** | 知识产权是企业无形资产，有助于融资和估值 |
| **技术入股** | 可作为技术出资入股凭证 |

### 1.3 登记机关

- **中国版权保护中心**（国家版权局认定的软件登记机构）
- 官方网站：https://www.ccopyright.com.cn
- 线上系统：https://register.ccopyright.com.cn

## 二、申请前准备工作

### 2.1 确认著作权人信息

| 项目 | 需确认内容 |
|------|-----------|
| 著作权人名称 | 公司全称（与营业执照一致） |
| 统一社会信用代码 | 18位代码 |
| 联系人 | 经办人姓名 |
| 联系电话 | 手机号码 |
| 联系地址 | 公司注册地址 |
| 邮政编码 | 对应邮编 |

### 2.2 确认软件基本信息

每个软件需要确认：

| 项目 | 说明 |
|------|------|
| 软件全称 | 建议格式：`XX公司XX系统 V1.0` |
| 软件简称 | 可选，不超过10个汉字 |
| 版本号 | 如 V1.0、V1.8.0 等 |
| 开发完成日期 | 软件开发完成的具体日期 |
| 首次发表日期 | 软件首次对外发布的日期（可选"未发表"） |
| 开发方式 | 独立开发 / 合作开发 / 委托开发 / 下达任务开发 |
| 原始取得权利 | 开发者为初始著作权人 |
| 权利范围 | 全部权利 |
| 软件用途 | 行业应用类别 |
| 技术特点 | 主要技术亮点概述 |
| 硬件环境 | 运行所需的硬件配置 |
| 软件环境 | 运行所需的操作系统、数据库等 |
| 编程语言 | 主要开发语言 |
| 源程序量 | 总代码行数（单位：行） |
| 主要功能 | 500字以内的功能描述 |

### 2.3 所需证明材料

| 材料 | 说明 |
|------|------|
| 营业执照副本复印件 | 加盖公章 |
| 法人身份证复印件 | 正反面 |
| 经办人身份证复印件 | 如非法人本人办理 |
| 授权委托书 | 委托代理人办理时需要 |

## 五、源代码文档编制规范

### 5.1 基本要求

- **格式**：A4纸，单面打印
- **页码**：每页右上角标注页码
- **页眉**：标注软件全称及版本号
- **字体**：宋体 / 等宽字体，小四号（12pt）
- **行数**：每页不少于50行

## 九、常见问题与注意事项

### 9.1 常见驳回原因

| 问题 | 解决方案 |
|------|----------|
| 源代码行数不足50行/页 | 调整格式，确保每页至少50行 |
| 源代码包含注释过多 | 保留必要注释，去除大段说明性注释 |
| 说明书缺少截图 | 补充完整的界面截图 |
| 软件名称与材料不一致 | 确保所有材料中名称完全一致 |
| 版本号格式不规范 | 使用 V1.0 格式（大写V + 数字） |
| 开发完成日期不合理 | 确保日期早于首次发表日期 |
| 功能描述与截图不符 | 核对说明书与实际软件功能 |

### 9.2 重要注意事项

1. **软件名称一致性**
   - 申请表中的软件名称、源代码页眉、说明书页眉，三处名称必须完全一致
   - 包括版本号也要一致

2. **源代码要求**
   - 必须是源程序（如 .ts、.vue、.js），不能是编译后的代码
   - 不要包含大段注释说明
   - 代码应该是有效的、可运行的代码
   - 不要包含空白页

3. **说明书截图要求**
   - 截图必须是软件实际运行画面
   - 不要使用设计稿或原型图代替
   - 截图需要清晰可读
   - 建议使用测试/演示数据

4. **版本号规范**
   - 格式为 V + 主版本号.次版本号，如 V1.0
   - 不建议使用过于复杂的版本号（如V1.8.0），可简化为V1.8

5. **开发完成日期**
   - 必须早于或等于首次发表日期
   - 必须晚于公司成立日期
   - 建议填写实际完成的合理日期

6. **多个软件分别申请**
   - 每个软件需要单独填写一份申请表
   - 每个软件的源代码和说明书各自独立
   - 可以同时提交多个申请

### 9.3 提高通过率的技巧

1. **源代码选取**
   - 选择核心业务逻辑代码，避免配置文件和依赖声明
   - 前30页从程序入口开始，后30页选取核心功能实现
   - 确保代码有一定的逻辑复杂度

2. **说明书编写**
   - 截图要丰富，每2-3页至少一张截图
   - 操作步骤要详细，带有编号
   - 功能描述要与实际截图对应
   - 避免大段纯文字描述

3. **信息填写**
   - 功能描述尽量详细但不超500字
   - 技术特点突出独特性
   - 编程语言如实填写所有使用的语言

## 十二、源代码行数统计方法

在项目根目录执行以下命令统计各系统代码行数：

### Windows PowerShell

```powershell
# CRM外呼助手APP代码行数
(Get-ChildItem -Path "crmAPP/src" -Recurse -Include *.ts,*.vue,*.js | Get-Content | Measure-Object -Line).Lines

# CRM主系统前端代码行数
(Get-ChildItem -Path "src" -Recurse -Include *.ts,*.vue,*.js | Get-Content | Measure-Object -Line).Lines

# CRM主系统后端代码行数
(Get-ChildItem -Path "backend/src" -Recurse -Include *.ts,*.js | Get-Content | Measure-Object -Line).Lines

# 管理后台代码行数
(Get-ChildItem -Path "admin/src" -Recurse -Include *.ts,*.vue,*.js | Get-Content | Measure-Object -Line).Lines

# 官方网站代码行数
(Get-ChildItem -Path "website/src" -Recurse -Include *.ts,*.vue,*.js | Get-Content | Measure-Object -Line).Lines
```

# CRM外呼助手APP

# 管理后台
find admin/src -name "*.ts" -o -name "*.vue" -o -name "*.js" | xargs wc -l | tail -1

find website/src -name "*.ts" -o -name "*.vue" -o -name "*.js" | xargs wc -l | tail -1
