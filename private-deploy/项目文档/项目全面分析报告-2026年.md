### 2.3 权限体系

系统拥有**四级权限体系**：

| 层级 | 控制粒度 | 实现方式 |
|------|----------|----------|
| **路由级** | 菜单可见/不可见 | `meta.requiresAuth/Admin/SuperAdmin/Manager` + `meta.permissions` |
| **功能级** | 按钮/操作可见 | `v-permission` 指令 + 角色权限JSON |
| **数据级** | 看哪些数据 | 部门/个人/全部 + 客服权限 + 敏感信息权限 |
| **字段级** | 看到/看不到哪些字段 | 敏感信息脱敏(手机号/身份证/工资/银行卡) |

## 六、微信小程序 — 新增功能 详细分析

### 6.1 定位

**这是一个「客户自助填写资料」微信小程序**，属于普通微信小程序（非企微），注册在微信公众平台。

客户通过微信小程序自主填写姓名、手机号、地址等个人资料，提交后自动同步到CRM系统，并归属到发送卡片的成员名下。

### 6.2 技术栈

原生微信小程序开发（JavaScript + WXML + WXSS），无需 Vue/React 框架。

### 6.3 页面构成（3个页面）

| 页面 | 路径 | 功能 |
|------|------|------|
| 表单填写页 | `pages/form/form` | 动态字段渲染、智能识别粘贴、微信一键获取手机号、省市区三级联动、协议勾选、提交校验 |
| 提交成功页 | `pages/success/success` | 成功图标+提示、「告知顾问」分享按钮、「返回聊天」跳转按钮、微信流量主Banner广告位 |
| 链接过期页 | `pages/expired/expired` | 链接过期/签名无效提示 |

### 6.4 核心技术亮点

| 功能 | 实现 | 说明 |
|------|------|------|
| 智能化粘贴 | 正则提取 | 粘贴文本→自动识别姓名/手机号/地址并填入表单 |
| 动态表单 | 从API获取字段配置 | 每个租户可在CRM后台自定义：哪些字段小程序可见、哪些必填、哪些折叠 |
| 微信手机号 | `open-type="getPhoneNumber"` | 一键获取微信绑定手机号 + 手动输入备选方案 |
| 省市区街道 | 原生 `region` picker | 三级联动 + 街道列表选择/手动输入两种模式 |
| 多租户隔离 | URL参数 `?tenantId=xxx&memberId=xxx&ts=xxx&sign=xxx` | 一个小程序服务所有租户，通过签名隔离数据 |
| 编码转化 | util.js双编码处理 | 自动检测 URL 参数编码（原始/base64/GBK乱码/双重编码），自动修复 |
| 微信广告 | `<ad unit-id="xxx">` 原生组件 | 成功页底部展示流量主Banner广告，30秒自动刷新 |
| 开发模式兜底 | `__wxConfig.envVersion` | DevTools中自动使用Mock数据，无需完整参数即可开发调试 |
| 隐私合规 | 用户协议+隐私政策弹窗 | 明确角色定义、采集方义务、技术方免责、联系方式 |

### 6.5 后端API（10个接口，挂载于 `/api/v1/mp/`）

| 接口 | 方法 | 鉴权方式 | 功能 |
|------|------|----------|------|

| `/submit-customer` | POST | 签名验证 | 创建客户+系统消息+WebSocket推送 |
| `/get-phone` | POST | 签名验证 | 微信API解密手机号+额度管理 |
| `/generate-card` | POST | JWT登录 | 生成签名+获取AppID和卡片配置 |
| `/log-send` | POST | JWT登录 | 记录成员发送卡片日志 |
| `/collect-stats` | GET | JWT登录 | 统计小程序来源客户数 |
| `/collect-records` | GET | JWT登录 | 分页列表+敏感信息脱敏 |

### 6.6 多租户架构（关键创新）

一个小程序服务所有租户，完全透明：

```
你(运营方)
  ├─ SaaS租户A (corpId=A, tenantId=aaa) → 成员张三发送卡片
  │   page: /pages/form/form?tenantId=aaa&memberId=u1&sign=xxx
  │   → 客户填写后数据归属到租户A→成员张三
  ├─ SaaS租户B (corpId=B, tenantId=bbb) → 成员李四发送卡片
  │   → 数据归属到租户B→成员李四
  └─ 私有部署客户C (tenantId=ccc) → 成员王五发送卡片
      → 数据归属到私有客户C→成员王五
```

**租户零操作**：成员在企微侧边栏点一个按钮即自动携带参数。

## 十二、总结

### 12.1 核心优势

1. **功能极为全面**：14个模块覆盖销售全链路（客户→订单→物流→财务→业绩→售后），从获客到回款一站式

3. **微信小程序创新**：客户自助填写资料，零泄露、自动同步，隐私合规设计完善
4. **双部署模式**：SaaS+私有化，灵活的商业模式
5. **安全合规标杆**：四级权限体系、敏感信息脱敏、操作全审计
6. **帮助体系完善**：40+帮助内容组件，用户上手无门槛
