# 云客CRM 演示系统数据重置方案

> **适用环境**: 宝塔面板部署的演示站
> **演示租户ID**: `bf5385e4-e7fb-4ff3-8ac1-1f5e1330662c`
> **演示地址**: https://demo.yunkes.com
> **演示账号**: demo / demo123

---

## 一、整体流程

```
在演示站手动添加/调整数据，直到满意
        ↓
SSH/宝塔终端，mysqldump 导出该租户数据
        ↓
组装重置脚本（DELETE旧数据 + INSERT基线数据）
        ↓
手动执行一次验证
        ↓
宝塔「计划任务」定时执行（每天/每周）
        ↓
以后想更新基线？重复以上步骤即可
```

---

## 二、操作步骤

### 第1步：在演示站手动添加数据

登录演示站，正常操作添加客户、订单、商品等，调整到你满意的状态。这些数据就是以后每次重置恢复的「基线数据」。

### 第2步：导出演示租户数据

在宝塔面板中操作：**终端** → 执行以下命令：

```bash
mysqldump -u root -p crm_db \
  --where="tenant_id='bf5385e4-e7fb-4ff3-8ac1-1f5e1330662c'" \
  --no-create-info \
  --complete-insert \
  --skip-triggers \
  departments users products customers orders \
  > /www/server/crm/demo-seed-data-live.sql
```

> **参数说明**：
> - `crm_db` — 替换为你实际的数据库名
> - `--where` — 只导出演示租户的数据，不影响其他租户
> - `--no-create-info` — 只导出 INSERT 语句，不导出建表语句
> - `--complete-insert` — INSERT 语句包含列名，更安全
> - 最后的表名列表按实际需要增减（如有跟进记录表等也一并加上）

执行后输入 MySQL 密码，会在 `/www/server/crm/` 下生成 `demo-seed-data-live.sql`。

### 第3步：创建重置脚本

在宝塔 **文件管理** 中创建 `/www/server/crm/demo-reset.sql`，内容如下：

```sql
-- ==========================================
-- 云客CRM 演示环境数据重置脚本
-- 每次执行：先删除旧数据，再插入基线数据
-- ==========================================

SET @tid = 'bf5385e4-e7fb-4ff3-8ac1-1f5e1330662c';

-- 1. 清除旧数据（注意顺序：先删有外键依赖的表）
DELETE FROM `orders` WHERE `tenant_id` = @tid;
DELETE FROM `customers` WHERE `tenant_id` = @tid;
DELETE FROM `products` WHERE `tenant_id` = @tid;
DELETE FROM `users` WHERE `tenant_id` = @tid;
DELETE FROM `departments` WHERE `tenant_id` = @tid;

-- 2. 插入基线数据（来自 mysqldump 导出）
SOURCE /www/server/crm/demo-seed-data-live.sql;
```

> **注意**：如果 `SOURCE` 命令在你的环境不生效，可以把 `demo-seed-data-live.sql` 的内容直接粘贴到 DELETE 语句下方，合并成一个文件。

### 第4步：手动测试一次

在宝塔终端执行：

```bash
mysql -u root -p crm_db < /www/server/crm/demo-reset.sql
```

然后去演示站登录验证：
- 数据是否恢复到基线状态 ✓
- 所有账号能否正常登录（不弹修改密码） ✓
- 客户、订单数据是否完整 ✓

### 第5步：宝塔设置定时重置

在宝塔面板操作：

1. 进入 **计划任务**
2. 点击 **添加任务**
3. 配置如下：

| 配置项 | 值 |
|--------|------|
| 任务类型 | Shell脚本 |
| 任务名称 | 演示环境数据重置 |
| 执行周期 | 每天凌晨 3:00（或按需调整） |
| 脚本内容 | 见下方 |

**脚本内容**：

```bash
#!/bin/bash
echo "===== 演示环境数据重置开始: $(date) ====="

# 执行重置SQL
mysql -u root -p'你的MySQL密码' crm_db < /www/server/crm/demo-reset.sql

if [ $? -eq 0 ]; then
  echo "✅ 数据重置成功"
else
  echo "❌ 数据重置失败"
fi

echo "===== 演示环境数据重置结束: $(date) ====="
```

> **注意**：
> - `-p'密码'` 中 `-p` 和密码之间**没有空格**
> - `crm_db` 替换为你实际的数据库名
> - 宝塔会自动记录任务执行日志，可在「计划任务」页面查看

---

## 三、更新基线数据

当你想调整演示数据时：

1. 在演示站手动修改数据
2. 重新执行第2步的 `mysqldump` 命令，覆盖 `demo-seed-data-live.sql`
3. 下次定时任务执行时自动使用新基线

无需修改重置脚本或计划任务。

---

## 四、密码不弹窗的关键

演示账号插入时必须满足以下两个条件，否则登录后会弹出「强制修改密码」弹窗：

| 字段 | 要求 | 说明 |
|------|------|------|
| `need_change_password` | `0` | 不要求修改密码 |
| `password_last_changed` | 非NULL（如 `NOW()`） | 系统不会判定为「首次登录」 |

如果当前演示环境已经在弹窗，在宝塔终端执行以下SQL立即修复：

```sql
UPDATE `users`
SET `need_change_password` = 0, `password_last_changed` = NOW()
WHERE `tenant_id` = 'bf5385e4-e7fb-4ff3-8ac1-1f5e1330662c';
```

这样设置后，导出的 `demo-seed-data-live.sql` 会自动包含这两个字段的正确值，以后每次重置都不会再弹窗。
