# 打完电话不算完，从选品下单到签收回款，怎么让每一单都清清楚楚？

> **公众号推文 · 产品功能深度解读 · 发布日期：2026年6月**

---

"那个客户说昨天已经打款了，钱呢？"

"发货了吗？发哪个快递？单号多少？"

"这个月做了多少单、卖了多少货、回款多少，谁能给我一个数？"

如果上一篇文章（通话管理）解决的是"怎么打"的问题，那这篇文章要解决的是"打了之后怎么办"。

电销的终点不是挂电话——是订单、是发货、是回款。如果这一环断了，前面的外呼、跟进、客户维护全是白干。

今天，我们来拆解云客CRM的**订单管理与产品管理**——从选品到下单、从审核到发货、从签收到回款，全链路覆盖。

## 一、一张图看懂：产品怎么管、订单怎么走

如果你是老板，你只需要看懂两条线：

**第一条线：产品管理（仓库里有什么）**

```
建好产品库 → 分好类 → 设好价格 → 管好库存 → 卖得清楚
```

**第二条线：订单管理（卖了什么、怎么走的）**

```
销售下单 → 领导审批 → 仓库发货 → 客户签收 → 回款完成
```

两条线合在一起，就是云客CRM的订单系统。

最核心的设计就一句话：**一个客户从打电话进来、到收到货、到付完款，整个过程中，谁在什么时间干了什么，系统全部记下来，随时能查。**

## 二、产品管理：先把"卖什么"理清楚

很多团队用Excel管产品：一个表格，不同规格挤在备注栏里，库存靠手记，三个月后自己都看不懂。

### 2.1 一个产品，一个档案

录入产品时，该填的一次填完，以后就不乱了：

- **产品名称和编码**：系统自动生成编号，不会重复
- **分到哪个类别**：保健品、护肤品、医疗器械……按你的业务自定义，后面报表可以按类别统计
- **卖多少钱、成本多少**：成本价只有老板和财务能看到，销售只看售价
- **库存有多少**：发货后自动扣减，不需要人手动去记
- **低于多少要补货**：设一个警戒线，库存快没了系统自动标红提醒
- **产品图片、品牌、规格**：该有的信息都有
- **是不是主推款**：可以标"推荐""新品""热销"，首页优先展示

### 2.2 一个产品多个规格？一个页面管完

这是真实场景：

> 某保健品公司卖"中老年钙片"，分60粒装、120粒装、礼盒装。三个规格价格不同、库存不同。如果当成三个产品去管，月底对账对到崩溃，促销活动也经常搞错规格。

云客CRM的做法是：

```
产品：中老年钙片
  ├─ 60粒装    ¥68    库存200瓶
  ├─ 120粒装   ¥128   库存150瓶
  └─ 礼盒装    ¥198   库存80瓶
```

一个产品下面挂多个规格，每个规格独立定价、独立管库存。销售下单时选哪个规格，价格和库存自动出来，绝对不会搞混。

总库存（350瓶）- 单个规格库存（200/150/80）- 系统自动帮你算好，不用心算。

### 2.3 库存管理：货少了系统告诉你

**自动扣库存**：只要订单发了货，对应产品的库存就自动减少。不需要仓库手动去改Excel。

**手动也可以调**：采购到货入库、退货入库、样品出库、月底盘点修正——都可以手动调，而且每次调整都记录"谁调的、什么时间、从多少改成多少、为什么调"。不是拍脑袋改个数就完了。

**缺货预警**：产品列表里，库存低于警戒线的自动标红。老板看一眼前端就知道哪些货该补了，不用等仓库卖空了才知道。

### 2.4 不怕超卖：下单时库存就扣住了

做过电销的都知道，最怕的就是"客户付了钱，仓库没货了"。

云客CRM在下单那一刻就锁定库存。如果审核没通过、订单取消了，库存自动还回去。从机制上杜绝超卖。

### 2.5 不发货也能卖：课程卡、会员码、电子券

不是所有生意都要发快递。如果你卖的是课程激活码、会员兑换码、电子资料——

- **卡密模式**：提前导入卡密，客户下单后系统自动分配一个未使用过的激活码
- **网盘模式**：客户下单后系统自动发送资源链接
- **纯服务**：不需要发货，确认了就完成

这些东西的库存单独管理，不会和实物库存混在一起。

### 2.6 Excel导入，不用一个一个敲

下载系统模板，把产品数据填进去，上传，系统自动识别列名并导入。几百个产品几分钟搞定。

### 2.7 什么部门能卖什么产品，老板说了算

一个产品可以设置"只允许某些部门下单"。比如高端保健品只让VIP客服部卖，普通销售团队在下单页面根本看不到——从源头管住产品线。

## 三、订单管理：每一步都清清楚楚

产品是"弹药库"，订单就是"流水线"。

### 3.1 一个订单到底记了哪些东西？

打开一条订单，你能看到的信息：

- **基本信息**：订单号、下单时间、哪个销售创建的
- **客户信息**：谁买的、电话多少、地址在哪（自动从客户档案带出来）
- **买了什么**：产品名称、规格、数量、单价、总金额
- **怎么付的**：现金/微信/支付宝/银行转账/信用卡、付了多少、有没有定金、什么时候付的
- **怎么发的**：哪个快递、单号多少、什么时候发的、到哪了
- **发票信息**：需不需要发票、发票抬头
- **退款信息**（如果有）：退了多少钱、为什么退、什么时候退的
- **谁碰过这个订单**：每一条操作记录，时间、人物、做了什么
- **自定义字段**：如果你有特殊需求，比如"客户生日""礼品备注"，系统支持7个自定义字段

一条订单，从头到尾所有信息都在一个页面里，不用到处翻。

### 3.2 订单状态：不是想改就能改

云客CRM的订单有严格的"路线图"，每一步能走到哪一步，系统管着：

```
订单创建 → 待流转
    ↓ 销售提交
待审核 → 领导审批
    ├─ 审核通过 → 待发货 → 已发货 → 已签收 → 已完成
    ├─ 审核拒绝 → 可以重新修改再提审，也可以取消
    └─ 虚拟商品 → 审核通过直接完成（不用发货）
```

**如果销售想把"待审核"的订单直接改成"已发货"——系统会直接拒绝。**

不是不信任人，而是防止手滑。一个按钮点错，可能发错货、重复发货、没付款就发货。系统把路规定好，你就不会走错。

### 3.3 订单的时间线：谁干了什么，一眼看穿

打开任意一条订单，你能看到完整的时间线：

```
6月5日 10:23 — 销售部-小王
   创建订单 ORD-20260605-001

6月5日 10:25 — 销售部-小王
   提交审核

6月5日 11:05 — 管理部-李经理
   审核通过

6月5日 14:30 — 仓库-老张
   发货：顺丰速运 SF1234567890

6月7日 09:15 — 系统自动
   物流更新：已签收

6月7日 09:16 — 系统自动
   订单完成
```

**出问题了，谁在什么时间做了什么，一分钟就定位。** 不用扯皮，不用翻聊天记录。

### 3.4 每个环节自动通知，不用盯着看

订单状态一变，相关的人自动收到通知：

| 发生了什么事 | 通知谁 |
|:----|:----|
| 订单创建了 | 告诉销售"订单创建成功" |
| 提交审核了 | 通知领导"有订单等你审批" |
| 审核通过/拒绝了 | 告诉销售结果 |
| 发货了 | 告诉销售 +（可选）发短信给客户 |

销售不需要反复刷页面看状态，系统会主动告诉你。

### 3.5 怎么找订单：一搜就有

一万条订单里找你要的那一条：

- 搜订单号、搜客户姓名、搜手机号、搜快递单号——输什么都能搜
- 按状态筛选：只看待审核的、只看已发货的、只看已取消的……
- 按时间筛选：这个月的、上周的、某天到某天的
- 按金额筛选：大额订单、小额订单
- 按商品筛选：搜一款产品，看谁买了
- 按支付方式筛选：现金单、微信单、转账单
- 按部门/人员筛选：看看每个团队各自做了多少

所有条件可以组合使用，老板想看"销售一部这个月所有已签收、金额超过500元的订单"——几秒钟出结果。

### 3.6 数据权限：什么人能看到什么

- **老板和管理员**：看全公司所有订单
- **部门经理**：看自己部门所有人的订单
- **销售员**：只看自己创建的订单

不同角色看到的数据范围不同，既保护隐私，又方便管理。

## 四、审核与审批：大额订单不能一个人说了算

### 4.1 订单需要领导审批

销售下了单，金额大的、新客户的、有特殊折扣的——这些订单应该先经过领导审核再发货。

流程很简单：

```
销售下单 → 提交审核 → 领导看到 → 通过/拒绝
```

审核通过才流转到仓库发货，审核拒绝则退回给销售修改。

### 4.2 给部门上"紧箍咒"

如果你担心某个部门或者新人超限下单，可以设规则：

- 这个部门每天最多下50单
- 这个部门单笔订单不能超过5000元
- 这个部门当月累计不能超过20万
- 低于100元的订单不让下（没利润）

**下单前系统自动检查，超了就直接拦下来。** 不是限制人，是保护公司。

### 4.3 延时流转：给销售一个"反悔窗口"

可以设置订单创建后延迟3分钟再进入审核流程。万一销售选错了产品、填错了数量——这3分钟内还能改。过了时间就自动提交，不能反悔。

## 五、支付与财务：钱的事，不能马虎

### 5.1 每种收款都记清楚

不依赖第三方支付接口，开箱即用：

- 现金：记谁收的、什么时候收的
- 微信/支付宝：记交易号，回头可以对账
- 银行转账：上传转账截图做凭证
- 信用卡：记卡号后四位
- 其他方式：自己备注

**支付状态分四档**：未付款 → 已付定金 → 已付全款 → 已退款。每个订单的收款状态一目了然。

### 5.2 定金 + 尾款模式

先收定金锁单，发货前或签收后补齐尾款。定金截图可以上传保留凭证。财务月底算账，应收多少、实收多少、欠多少——数字不会对不上。

### 5.3 退款有记录

订单退款时，退了多少、为什么退、什么时间退的——全部记录在订单里。不是"退款了就算了"，每笔退款都有据可查。

## 六、发货与物流：从仓库到客户手上

### 6.1 发货五步走

```
仓库看到"待发货"订单
  → 备货
  → 选快递公司、填运单号
  → 点"发货"
  → 系统自动扣库存 + 通知销售
```

### 6.2 物流跟踪

支持主流的快递100查询，运单号填进去，客户签收了系统自动更新状态。拒收、退回、包裹异常的订单也会自动标记，方便及时处理。
