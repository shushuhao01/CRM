## 一、数据范围(data_scope)机制分析

### 1.1 角色数据范围配置

| 角色 | data_scope | 说明 |
|------|-----------|------|
| super_admin（超级管理员） | `all` | 全部数据，无需设置 |
| admin（管理员） | `all` | 全部数据，无需设置 |
| department_manager（部门经理） | `department` | 本部门数据 |
| sales_staff（销售员） | `self` | 仅个人数据 |
| customer_service（客服） | `self` | 由客服权限表单独控制 |

### 1.2 数据范围生效方式

当前系统采用**角色名称硬编码匹配**方式在后端各业务路由中强制执行数据过滤，而非通过 `roles.data_scope` 字段动态判断。各核心模块的过滤逻辑一致：

```
if (adminRoles.includes(role)) {
    // 管理员/超管 → 全部数据，不做过滤
} else if (managerRoles.includes(role)) {
    // 部门经理 → 按 createdByDepartmentId 过滤
} else {
    // 销售员/其他 → 按 createdBy = userId 过滤
}
```

### 1.3 各业务模块数据范围生效确认

| 业务模块 | 路由文件 | admin/super_admin | department_manager | sales_staff | customer_service |
|---------|---------|-------------------|-------------------|-------------|-----------------|
| 订单列表 | `routes/orders/orderCrud.ts` | ✅ 全部 | ✅ 部门 | ✅ 个人 | ✅ 全部* |
| 客户列表 | `routes/customers/core.ts` | ✅ 全部 | ✅ 部门 | ✅ 个人 | N/A |
| 资料列表 | `routes/data.ts` | ✅ 全部 | ✅ 部门 | ✅ 个人 | ✅ 按CS权限配置 |
| 绩效数据 | `routes/finance.ts` | ✅ 全部 | ✅ 部门 | ✅ 个人 | ✅ 全部* |
| 绩效管理 | `routes/finance.ts` | ✅ 全部 | ✅ 部门 | ✅ 个人 | ✅ 全部* |
| 商品分析 | `controllers/ProductController.ts` | ✅ 全部 | ✅ 部门标识 | ✅ 个人标识 | N/A |

> *客服角色在订单/绩效等模块被归为 `allowAllRoles`，可查看全部数据。

### 1.4 客服角色特殊说明

客服角色的数据范围由独立的 `customer_service_permissions` 表控制（非 `roles.data_scope`），支持：
- `all` - 全部数据
- `department` - 部门数据
- `self` - 个人数据
- `custom` - 自定义部门范围

```

  ├── loadPermissionTree()
  │     └── permissionApiService.getPermissionTree()  → GET /api/v1/permissions/tree
  │           └── 降级: permissionService.getAllPermissions() → 本地硬编码

```

## 四、权限生效全流程

```
管理员打开"权限设置"弹窗
  → 前端调用 GET /api/v1/permissions/tree 获取权限树（256节点）
  → 前端调用 GET /api/v1/roles/:id/permissions 获取角色已有权限（code数组）
  → el-tree 渲染权限树，node-key="code"，勾选匹配的code节点
  → 管理员修改勾选状态
  → 点击"确定"
  → 前端调用 PUT /api/v1/roles/:id/permissions 保存新权限（code数组）
  → 前端调用 PUT /api/v1/roles/:id 保存dataScope
  → 该角色下所有用户的功能权限立即生效（下次API请求时校验）
```
