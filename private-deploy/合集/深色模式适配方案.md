# 云客 CRM 深色模式（Dark Mode）适配方案

> 版本：1.0 | 编写日期：2026-06-29  
> 适用项目：云客 CRM v1.8.0（Vue 3 + Element Plus 2.x + Sass + Vite）

---

## 1. 背景与现状

### 1.1 为什么要做深色模式

- 降低用户在弱光环境下的视觉疲劳，CRM 客服/销售团队经常加班使用
- 减少 OLED 屏幕设备功耗
- 已成为主流 SaaS 产品的标配功能（钉钉、飞书、Notion、Linear 均已支持）
- 提升产品质感和用户体验评分

### 1.2 当前技术栈

| 组件 | 版本 | 深色模式支持 |
|------|------|-------------|
| Vue 3 | 3.5.x | 无直接关系 |
| Element Plus | 2.3.x | **原生支持** `dark` CSS 命名空间 |
| Sass (sass-embedded) | 1.93.x | 支持 CSS 变量 / mixin |
| Vite | 5.4.x | 无直接关系 |
| ECharts (vue-echarts) | 7.x | 内置 `dark` 主题 |

### 1.3 难度评估

| 维度 | 评估 | 说明 |
|------|------|------|
| Element Plus 组件 | **低** | 官方提供 `dark` 命名空间，引入一行 CSS 即可 |
| 自定义组件/页面 | **中** | 约 120+ 个 `.vue` 文件使用了硬编码颜色值 |
| ECharts 图表 | **低** | 切换 `theme: 'dark'` 即可 |
| 全局样式/布局 | **中** | 侧边栏、顶栏、卡片等使用了固定背景色 |
| 第三方组件 | **中** | wangeditor 富文本编辑器、日期选择器等需要单独适配 |
| 图片/图标 | **低** | Element Plus Icons 基于 SVG，自动跟随颜色 |
| 总工作量 | **约 3-5 人天** | 渐进式实施可分阶段完成 |

## 2. 实施方案

### 2.1 方案对比

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **A. CSS 变量 + Element Plus dark** | 通过 CSS 变量切换色值 | 性能最好、Element Plus 原生支持、可扩展 | 需要将硬编码颜色迁移为变量 |
| B. 双样式表 | 编译两份完整 CSS | 隔离彻底 | 打包体积翻倍、维护成本高 |
| C. CSS filter invert | `filter: invert(1)` 全局反色 | 零开发成本 | 图片反色、颜色失真、不可控 |
| D. Tailwind dark: 前缀 | `class="dark:bg-gray-900"` | 颗粒度细 | 项目未使用 Tailwind，引入成本大 |

**推荐方案 A**：CSS 变量 + Element Plus dark 命名空间。

### 2.2 核心实现步骤

#### 步骤 1：引入 Element Plus Dark CSS

在 `src/main.ts` 中添加一行：

```typescript
import 'element-plus/theme-chalk/dark/css-vars.css'
```

这会注册 `.dark` 命名空间下的 CSS 变量覆盖，当 `<html>` 标签有 `class="dark"` 时自动生效。

#### 步骤 2：定义全局 CSS 变量

创建 `src/styles/dark-variables.scss`：

```scss
// 亮色主题变量（默认）
:root {
  --crm-bg-primary: #ffffff;
  --crm-bg-secondary: #f5f7fa;
  --crm-bg-card: #ffffff;
  --crm-bg-sidebar: #304156;
  --crm-bg-header: #ffffff;
  --crm-text-primary: #303133;
  --crm-text-regular: #606266;
  --crm-text-secondary: #909399;
  --crm-text-placeholder: #c0c4cc;
  --crm-border-color: #dcdfe6;
  --crm-border-light: #e4e7ed;
  --crm-shadow-base: 0 2px 12px rgba(0, 0, 0, 0.1);
  --crm-shadow-light: 0 2px 4px rgba(0, 0, 0, 0.04);
}

// 深色主题变量
html.dark {
  --crm-bg-primary: #141414;
  --crm-bg-secondary: #1d1d1d;
  --crm-bg-card: #1d1d1d;
  --crm-bg-sidebar: #111111;
  --crm-bg-header: #1d1d1d;
  --crm-text-primary: #e5eaf3;
  --crm-text-regular: #cfd3dc;
  --crm-text-secondary: #a3a6ad;
  --crm-text-placeholder: #8d9095;
  --crm-border-color: #4c4d4f;
  --crm-border-light: #414243;
  --crm-shadow-base: 0 2px 12px rgba(0, 0, 0, 0.4);
  --crm-shadow-light: 0 2px 4px rgba(0, 0, 0, 0.2);
}
```

#### 步骤 3：创建主题切换 Composable

创建 `src/composables/useTheme.ts`：

```typescript
import { ref, watch } from 'vue'

type Theme = 'light' | 'dark' | 'auto'

const STORAGE_KEY = 'crm-theme-preference'

const currentTheme = ref<Theme>(
  (localStorage.getItem(STORAGE_KEY) as Theme) || 'auto'
)

const isDark = ref(false)

function applyTheme(theme: Theme) {
  let shouldBeDark = false

  if (theme === 'dark') {
    shouldBeDark = true
  } else if (theme === 'auto') {
    shouldBeDark = window.matchMedia('(prefers-color-scheme: dark)').matches
  }

  isDark.value = shouldBeDark
  document.documentElement.classList.toggle('dark', shouldBeDark)
}

// 监听系统主题变化
const mediaQuery = window.matchMedia('(prefers-color-scheme: dark)')
mediaQuery.addEventListener('change', () => {
  if (currentTheme.value === 'auto') {
    applyTheme('auto')
  }
})

export function useTheme() {
  function setTheme(theme: Theme) {
    currentTheme.value = theme
    localStorage.setItem(STORAGE_KEY, theme)
    applyTheme(theme)
  }

  // 初始化
  applyTheme(currentTheme.value)

  return {
    currentTheme,
    isDark,
    setTheme
  }
}
```

#### 步骤 4：添加主题切换 UI

在顶栏用户菜单旁添加切换按钮，推荐三态切换（亮色 / 深色 / 跟随系统）：

```vue
<template>
  <el-dropdown @command="setTheme">
    <el-button circle>
      <el-icon>
        <Sunny v-if="!isDark" />
        <Moon v-else />
      </el-icon>
    </el-button>
    <template #dropdown>
      <el-dropdown-menu>
        <el-dropdown-item command="light" :class="{ active: currentTheme === 'light' }">
          亮色模式
        </el-dropdown-item>
        <el-dropdown-item command="dark" :class="{ active: currentTheme === 'dark' }">
          深色模式
        </el-dropdown-item>
        <el-dropdown-item command="auto" :class="{ active: currentTheme === 'auto' }">
          跟随系统
        </el-dropdown-item>
      </el-dropdown-menu>
    </template>
  </el-dropdown>
</template>
```

#### 步骤 5：迁移硬编码颜色

将所有 `.vue` 和 `.scss` 中的硬编码颜色替换为 CSS 变量：

```scss
// 修改前
.sidebar {
  background: #304156;
}
.main-content {
  background: #f0f2f5;
  color: #333;
}

// 修改后
.sidebar {
  background: var(--crm-bg-sidebar);
}
.main-content {
  background: var(--crm-bg-secondary);
  color: var(--crm-text-primary);
}
```

#### 步骤 6：ECharts 图表适配

```typescript
import { useTheme } from '@/composables/useTheme'

const { isDark } = useTheme()

// 在 ECharts 选项中
const chartOption = computed(() => ({
  // 根据主题自动切换
  backgroundColor: isDark.value ? '#1d1d1d' : '#ffffff',
  textStyle: {
    color: isDark.value ? '#e5eaf3' : '#303133'
  },
  // ... 其他选项
}))

// 或使用 vue-echarts 的 theme prop
// <v-chart :theme="isDark ? 'dark' : ''" />
```

## 4. 需要注意的坑

### 4.1 首屏闪烁（FOUC）

用户选择了深色模式，但页面加载时会先显示亮色再切换。解决方案：

在 `index.html` 的 `<head>` 中加入内联脚本：

```html
<script>
  ;(function() {
    var theme = localStorage.getItem('crm-theme-preference') || 'auto'
    var isDark = theme === 'dark' ||
      (theme === 'auto' && window.matchMedia('(prefers-color-scheme: dark)').matches)
    if (isDark) document.documentElement.classList.add('dark')
  })()
</script>
```

### 4.2 打印样式

深色模式下用户打印页面会变成深色背景，浪费墨水。需要：

```scss
@media print {
  html.dark {
    --crm-bg-primary: #ffffff !important;
    --crm-bg-secondary: #ffffff !important;
    --crm-text-primary: #000000 !important;
    // 强制还原为亮色
  }
}
```

### 4.3 图片与 Logo

- SVG 图标：自动跟随 `color` 属性，无需处理
- PNG/JPG Logo：需准备深色版本，或使用 CSS filter 调整

```scss
html.dark {
  .app-logo img {
    filter: brightness(1.2);
  }
}
```

### 4.4 第三方富文本编辑器

wangeditor 有独立的样式体系，需要在其容器上添加覆盖：

```scss
html.dark {
  .w-e-text-container {
    background-color: var(--crm-bg-card) !important;
    color: var(--crm-text-primary) !important;
  }
  .w-e-toolbar {
    background-color: var(--crm-bg-secondary) !important;
    border-color: var(--crm-border-color) !important;
  }
}
```

### 4.5 颜色对比度

深色模式下需确保文字与背景的对比度符合 WCAG AA 标准（至少 4.5:1）。推荐使用 Chrome DevTools 的对比度检查功能逐页验证。

## 5. 可选的增强功能

### 5.1 定时切换

允许用户设置时间段自动切换：

```
日间模式：07:00 - 18:00
夜间模式：18:00 - 07:00
```

### 5.2 主题色自定义

结合 Element Plus 的 CSS 变量系统，允许用户自定义主色调：

```scss
:root {
  --el-color-primary: #409eff; // 默认蓝色
}

// 用户选择绿色主题
:root[data-theme-color="green"] {
  --el-color-primary: #67c23a;
}
```

### 5.3 与系统设置联动

在系统设置中添加"外观设置"选项卡，提供：
- 主题模式选择（亮色 / 深色 / 自动）
- 侧边栏样式（深色 / 浅色 / 跟随主题）
- 主色调选择（蓝 / 绿 / 紫 / 橙 / 自定义）
- 字体大小偏好

## 7. 总结

| 结论 | 说明 |
|------|------|
| **实施难度** | 中等偏低，Element Plus 原生支持是最大利好 |
| **预估工期** | 3-5 人天（含测试） |
| **推荐方案** | CSS 变量 + Element Plus dark 命名空间 |
| **收益** | 提升用户体验、产品竞争力 |
| **风险点** | 第三方组件适配、硬编码颜色遗漏、首屏闪烁 |
| **建议** | 分阶段实施，先做基础框架 + 全局布局，再逐页适配 |

> 深色模式是一个"做起来不难，做好需要细心"的功能。借助 Element Plus 的原生支持，基础适配可以在半天内完成，但要做到每个页面、每个弹窗、每个图表都完美适配，需要逐一排查和调整。建议先实施阶段一和阶段二，让用户能立即使用，后续再逐步完善细节。
