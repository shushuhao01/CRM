# 云客CRM 开源社区版策略：功能切割 × 授权设计 × 引流转化

> **版本**: v1.0 | **创建时间**: 2026年5月
> **定位**: 开源社区版（Community Edition）的完整落地方案
> **目标**: 通过 GitHub/Gitee 开源引流，转化为私有部署付费客户和 SaaS 订阅客户
> **保密级别**: 内部决策参考

---

## 一、核心策略：开源不是免费送代码，是最高效的获客漏斗

```
  开源社区版（GitHub/Gitee 免费下载）
         │
         │  Star / Fork / 部署体验
         ↓
  用户部署 → 输入免费授权码激活（≤3用户永久免费）
         │
         │  业务增长，3人不够用了
         ↓
  ┌──────┴──────────────────────────┐
  │                                 │
  ↓                                 ↓
  私有部署付费版                    SaaS 云端版
  ¥14,800起（买断）                ¥149/月起（订阅）
  解锁全功能+更多用户              零运维，开箱即用
```

**关键原则**：
1. 开源版**必须好用**——足够让 1-3 人的微型团队日常使用，这样才有口碑传播
2. 开源版**必须有限制**——用户增长到 3 人以上时自然触碰天花板，产生付费动力
3. 授权码**必须从你这里拿**——即使代码开源，没有授权码无法使用，你始终掌握入口

## 二、版本体系设计

### 2.1 三版本架构

| 维度 | 社区版（Community） | 私有部署版（Professional） | SaaS 云端版 |
|------|:-------------------:|:-------------------------:|:-----------:|
| **价格** | 免费（永久） | ¥14,800 起（买断） | ¥149/月 起 |
| **代码** | GitHub/Gitee 开源 | 源码交付（不开源） | 无需关心源码 |
| **授权** | 免费授权码（≤3用户） | 付费授权码 | 账号订阅 |
| **部署** | 用户自行部署 | 提供部署协助 | 无需部署 |
| **用户上限** | **3人** | 30/100/不限 | 5-50席位 |
| **技术支持** | GitHub Issues + 社区 | 邮件/在线/专属 | 在线客服 |
| **更新频率** | 延迟 1-2 个版本 | 实时更新 | 实时更新 |

### 2.2 为什么是 3 个用户？

| 数量 | 效果 |
|------|------|
| **1 人** | 太少，连带老板+1个销售都不够，没有真实使用场景 |
| **3 人** | ✅ 甜蜜点：老板+2个销售刚好够用，能体验完整流程（创建客户→下单→发货→跟进） |
| **5 人** | 太多，小团队 5 人够用了，付费动力不足 |

**3 人 = 刚好能用 + 刚好不够用 = 最大化转化动力**

## 三、功能切割方案——开源版 vs 付费版

### 3.1 切割原则

1. **核心 CRM 流程完整保留**：客户→订单→物流→售后，社区版必须跑通
2. **去掉高价值差异化模块**：企微集成、通话系统、高级报表、批量操作
3. **去掉规模化功能**：多部门、复杂权限、业绩阶梯、佣金计算
4. **保留足够的"痒点"**：用户看到付费版有这些功能但自己没有，产生升级欲望

## 四、技术实现方案——如何区分社区版和付费版

### 4.1 现有授权架构（无需重写）

你的系统已经有完善的授权基础设施，社区版只需**复用现有机制**：

```
已有组件：
├── LicenseService.ts        ← 授权验证 + 功能模块检查（isFeatureEnabled）
├── License 实体              ← features JSON 字段，存储可用模块列表
├── checkLicenseWrite.ts     ← 过期后阻止写入
├── checkTenantLimits.ts     ← 用户数上限检查
├── deploy.ts                ← 部署模式（private/saas）
└── SaaSGuardService.ts      ← SaaS 模式 RSA 签名守卫
```

### 4.2 社区版授权码设计

#### 授权码格式

```
COMMUNITY-XXXX-XXXX-XXXX-XXXX
```

在 admin 管理后台生成时：

| 字段 | 社区版值 | 说明 |
|------|---------|------|
| `license_type` | `'community'` | **新增枚举值** |
| `max_users` | `3` | 固定 3 人 |
| `expires_at` | `NULL` | **永不过期** |
| `features` | `["dashboard","customer","order","logistics","product","service","performance_basic"]` | 限定可用模块 |
| `status` | `'active'` | 激活状态 |

#### License 实体变更

在 `License` 实体的 `licenseType` 枚举中新增 `'community'`：

你已有 `isFeatureEnabled(featureCode)` 方法，社区版的功能限制**完全通过 features 字段控制**：

#### 前端菜单控制

在前端路由/菜单渲染时，根据后端返回的 features 列表控制：

```typescript
// 前端获取授权信息后
const features = userStore.licenseFeatures; // ['dashboard','customer','order',...]

// 菜单项判断
function isMenuEnabled(moduleId: string): boolean {
  if (!features || features.includes('all')) return true;
  return features.includes(moduleId);
}

// 渲染效果：
// - 有权限的模块：正常显示和使用
// - 无权限的模块：显示但灰色 + 锁图标 + 点击弹出「升级到付费版解锁此功能」
```

在导出相关路由中增加社区版判断：

```typescript

```

### 4.6 完整的版本判断工具函数

新建一个工具函数供全局使用：

```typescript
// backend/src/utils/editionHelper.ts

import { licenseService } from '../services/LicenseService';

export type Edition = 'community' | 'standard' | 'professional' | 'enterprise' | 'dev';

export async function getCurrentEdition(): Promise<Edition> {
  const info = await licenseService.getLicenseInfo();
  if (!info || !info.activated) return 'dev';    // 未激活 = 开发模式
  const type = info.licenseType;
  if (type === 'community') return 'community';
  if (type === 'trial') return 'community';       // 试用期等同社区
  if (type === 'perpetual' || type === 'annual' || type === 'monthly') {
    if (info.maxUsers <= 30) return 'standard';
    if (info.maxUsers <= 100) return 'professional';
    return 'enterprise';
  }
  return 'dev';
}

export async function isCommunityEdition(): Promise<boolean> {
  const edition = await getCurrentEdition();
  return edition === 'community';
}
```

## 七、双仓库架构——主仓库不动，开源仓库独立

### 7.1 核心原则：绝不在主仓库删代码

```
你现在的仓库结构：

d:\kaifa\CRM - 1.8.0\          ← 主仓库（私有，完整功能，绝不修改）
├── backend/                     ← 后端全功能
├── src/                         ← 前端全功能（Vue 3）
├── admin/                       ← 管理后台
├── h5/                          ← H5 移动端
├── website/                     ← 官网
├── miniprogram/                 ← 小程序
├── crmAPP/                      ← 移动 APP
├── wecom-program/               ← 企微小程序
├── private-deploy/              ← 私有部署相关（含本文档）
├── scripts/                     ← 工具脚本
└── ...

开源版的做法：

① 在 GitHub 创建一个全新的独立仓库：yunke-crm（公开）
② 写一个 "导出脚本"，从主仓库复制代码到临时目录
③ 脚本自动删除付费模块、修改配置、清理敏感信息
④ 把处理后的干净代码推送到开源仓库
⑤ 主仓库完全不受影响，一行代码都不用改
```

### 7.2 整体流程图

```
主仓库（d:\kaifa\CRM - 1.8.0）
    │
    │  运行导出脚本 build-community.ps1
    │  （在 scripts/ 目录下）
    ↓
临时目录（d:\kaifa\yunke-crm-build\）
    │
    │  脚本自动执行：
    │  ① 复制 backend/ src/ h5/ 到临时目录
    │  ② 删除付费模块（企微/通话/短信/财务/admin等）
    │  ③ 修改 app.ts（注释掉付费路由的 import 和 use）
    │  ④ 修改 router/index.ts（付费路由指向 Upgrade.vue）
    │  ⑤ 注入 Upgrade.vue 升级引导页
    │  ⑥ 清理 .env 中的敏感信息
    │  ⑦ 生成社区版专用的 README.md / docker-compose.yml / LICENSE
    ↓
开源仓库（GitHub: yunke-crm）
    │
    │  git add + commit + push
    ↓
GitHub/Gitee 公开可见
```

**关键**：主仓库始终是完整的，开源仓库是"加工品"。就像工厂生产全配置的车，然后拆掉某些配件作为基础款出售——工厂产线不变，只是出货的版本不同。

## 十二、风险控制

### 12.1 防止绕过授权

| 风险 | 应对 |
|------|------|
| 修改前端隐藏升级提示 | 后端 `isFeatureEnabled` 做二次校验，前端只是 UI 层 |
| 直接改 `max_users` | 授权码绑定 `machine_id`，管理后台校验一致性 |
| 不输授权码硬用 | 未激活状态 `canCreateUser` 返回 dev 模式，功能可用但**定期弹窗**提醒激活 |
| 伪造授权码 | 授权码格式有签名校验（HMAC），伪造的无法通过 `verifyLicense` |
| 把付费版代码合并到社区版 | 付费版不开源，拿不到代码 |

### 12.2 社区版不放心的话可以加的保险

```typescript
// 社区版启动时检查（可选）
if (licenseInfo.licenseType === 'community') {
  // 每 24 小时向 yunkes.com 报告一次心跳（不传敏感数据，只传授权码+用户数）
  // 如果授权码被吊销，下次重启时拒绝启动
  setInterval(() => heartbeat(licenseInfo.licenseKey), 24 * 60 * 60 * 1000);
}
```

### 15.3 授权码生成逻辑细节

#### 社区版授权码生成规则

| 要素 | 规则 |
|------|------|
| **前缀** | `COMMUNITY-` |
| **格式** | `COMMUNITY-XXXX-XXXX-XXXX-XXXX`（4段4字符，大写字母+数字） |
| **生成位置** | `Tenant.generateCommunityLicenseKey()`（后端实体方法） |
| **触发时机** | 官网注册时 `register.ts` 判断 `packages[0].type === 'community'` |
| **授权状态** | 直接为 `active`（免费，无需支付激活） |
| **到期时间** | `expire_date = now + 36500天`（≈100年，等同永不过期） |
| **日志标记** | `'官网注册-开源社区版'` |

#### 与其他授权码的区分

```
社区版:    COMMUNITY-A1B2-C3D4-E5F6-G7H8  （永久免费，3用户）
私有部署:  PRIVATE-A1B2-C3D4-E5F6-G7H8     （付费，按套餐配置）
SaaS租户:  TENANT-A1B2-C3D4-E5F6-G7H8      （SaaS模式专用）
```

#### 后端判断社区版的三种方式

```typescript
// 方式1：通过 license_type 字段（推荐）
const isCommunity = licenseInfo.licenseType === 'community'

// 方式2：通过授权码前缀
const isCommunity = licenseKey.startsWith('COMMUNITY-')

// 方式3：通过 editionHelper 工具函数
import { isCommunityEdition, isCommunityLicenseKey } from './utils/editionHelper'
const isCommunity = isCommunityEdition(licenseType)
const isCommunity2 = isCommunityLicenseKey(licenseKey)
```

### 15.5 前端菜单锁定方案（社区版功能门禁）

社区版部署后，前端菜单需对付费模块做锁定展示：

#### 锁定策略

| 模块 | 社区版行为 | 实现方式 |
|------|-----------|---------|
| 客户/订单/物流/售后/看板/商品 | 正常使用 | 无特殊处理 |
| 财务/企微/通话/短信 | 菜单灰色 + 🔒图标，点击弹升级提示 | `isFeatureEnabled()` 判断 |
| 批量操作（发货/审核/导入） | 按钮禁用，hover 提示升级 | 同上 |
| 数据导出 | 限制100条，超出弹升级提示 | `getExportLimit()` 判断 |
