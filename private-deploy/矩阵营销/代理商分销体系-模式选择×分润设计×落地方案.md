> **版本**: v1.0 | **创建时间**: 2026年5月

> **保密级别**: 内部决策参考

## 一、代理商是什么？一句话讲清楚

**代理商 = 帮你卖产品的人/公司，你给他分钱。**

他们不写代码、不开发产品，他们有的是**客户资源**和**行业关系**。比如：
- 做企微代运营的公司，手里有一堆需要 CRM 的客户
- 本地的 IT 服务商，给中小企业装软件、卖服务器
- 做财务/ERP 的公司，客户也需要 CRM
- 行业协会、商会里有影响力的人

**你的核心问题：代理商拿到东西后怎么卖？**

## 二、三种代理模式对比——先搞清楚再选

### 2.1 模式全景

```
模式A：推荐返佣           模式B：分销定价           模式C：独立部署
（最简单）                （中等难度）              （最复杂）
                                                    
代理商介绍客户 →          代理商自己卖 →            代理商拿源码 →
客户注册你的SaaS           客户用你的SaaS            部署到自己服务器
你收款 → 分佣给代理        代理收款 → 分账给你       代理自己收款 → 年费给你
                                                    
你控制一切               你控制平台，他控制价格      他控制一切，你只卖授权
```

### 2.2 详细对比

| 维度 | 模式A：推荐返佣 | 模式B：分销定价 | 模式C：独立部署 |
|------|:---:|:---:|:---:|
| **一句话** | 代理介绍客户，你卖你分钱 | 代理用你的平台卖，自己定价 | 代理买断源码，自己当老板 |
| **客户用谁的系统** | 你的 SaaS | 你的 SaaS（贴代理品牌） | 代理自己部署的系统 |
| **谁收客户的钱** | 你收，返佣给代理 | 代理收，分账给你 | 代理收，年费/授权费给你 |
| **代理能自己定价吗** | ❌ 不能，你统一定价 | ✅ 能，但有底价 | ✅ 完全自主 |
| **技术难度** | ⭐ 几乎为零 | ⭐⭐⭐ 中等 | ⭐⭐⭐⭐⭐ 很高 |
| **你的开发工作量** | 0.5天（加个推荐码） | 2-3周（多级租户+白标） | 0（已有私有部署能力） |
| **代理门槛** | 低（任何人都能推荐） | 中（需要售前能力） | 高（需要技术+服务器+运维） |
| **你的控制力** | 🟢 最强（客户在你手里） | 🟡 中等（平台在你手里） | 🔴 最弱（代理完全独立） |
| **适合的代理** | 个人、兼职、KOL | IT服务商、行业方案商 | 有技术团队的集成商 |
| **风险** | 低 | 中（代理乱定价损害品牌） | 高（代理可能跑路/烂服务） |

### 2.3 推荐策略

> **现阶段（0-100客户）：主推模式A，试点模式B，暂不做模式C。**

理由：
1. 你现在最缺的是**客户数量**，模式A门槛最低，能快速铺开
2. 模式B需要开发多级租户和白标功能，投入产出比要等客户量上来后再做
3. 模式C本质就是卖私有部署版，你已经在做了，只是换个名字叫"代理"

## 三、模式A：推荐返佣（立即可做）

### 3.1 运作方式

```
代理商（张三）
    │
    │ 分享专属推荐链接：yunkes.com/r/zhangsan
    ↓
客户点击 → 注册SaaS试用 → 付费订阅
    │
    │ 系统自动记录「此客户由张三推荐」
    ↓
你收到客户付款 ¥699/月
    │
    │ 每月自动结算
    ↓
返佣给张三：¥699 × 30% = ¥209.7/月（首年）
```

### 3.2 返佣规则

| 产品 | 首年返佣 | 续费返佣 | 说明 |
|------|:---:|:---:|------|
| SaaS 订阅 | **30%** | **15%** | 首年高返佣激励拉新，续费降低（客户已稳定） |
| 私有部署 | **25%** | **20%**（年度授权） | 一次性大单，返佣比例略低 |
| 增值服务 | **20%** | **20%** | 短信包、AI包、会话存档等 |

### 3.3 代理等级

| 等级 | 条件 | 首年返佣 | 额外权益 |
|------|------|:---:|------|
| 🥉 **铜牌** | 注册即可 | 25% | 推荐码 + 基础物料 |
| 🥈 **银牌** | 累计推荐 5 个付费客户 | 30% | + 专属客服 + 优先结算 |
| 🥇 **金牌** | 累计推荐 20 个付费客户 | 35% | + 联合品牌 + 线下培训资格 |
| 💎 **钻石** | 累计推荐 50 个付费客户 | 40% | + 区域独家 + 定制开发优惠 |

### 3.4 技术实现（极简）

只需要做 3 件事：

**① 推荐码机制**

```
注册链接：https://www.yunkes.com/register?ref=AGENT001
```

后端注册接口新增：
```typescript
// 注册时记录推荐来源
if (req.query.ref || req.body.referralCode) {
  tenant.referralCode = req.query.ref || req.body.referralCode;
  tenant.referralSource = 'agent';
}
```

在 `tenants` 表新增两个字段：
```sql
ALTER TABLE tenants ADD COLUMN referral_code VARCHAR(50) NULL COMMENT '推荐人代码';
ALTER TABLE tenants ADD COLUMN referral_source VARCHAR(20) NULL COMMENT '推荐来源';
```

**② 代理商管理页面（Admin 后台）**

在管理后台新增一个"代理商管理"菜单：
- 代理商列表（姓名、手机、推荐码、累计推荐数、累计返佣）
- 查看某代理商推荐的全部客户
- 返佣结算记录

**③ 返佣结算**

前期手动结算即可（Excel 导出 → 微信/银行转账），客户量大了再做自动化。

### 3.5 开发工时估算

| 任务 | 工时 |
|------|------|
| 数据库新增字段 | 0.5h |
| 注册接口加推荐码逻辑 | 1h |
| Admin 后台代理商管理页面 | 4h |
| 代理商注册/申请页面（官网） | 2h |
| **合计** | **约 1 天** |

## 四、模式B：分销定价（中期规划）

### 4.1 运作方式

```
你的 SaaS 平台（yunkes.com）
    │
    ├── 直客：客户直接在 yunkes.com 注册
    │
    └── 代理商入口：
        │
        ├── 代理A（专注教育行业）
        │   域名：edu-crm.agent-a.com（CNAME 到你的服务器）
        │   品牌：「教培通 CRM」（代理自己的品牌名）
        │   定价：基础版 ¥599/月（比你贵，因为他加了行业服务）
        │   他的客户注册后 → 数据在你的平台 → 属于代理A的"子租户"
        │
        └── 代理B（专注电商）
            域名：shop-crm.agent-b.com
            品牌：「电商管家 CRM」
            定价：专业版 ¥899/月
```

### 4.2 核心概念：二级租户

```
你的平台（超级管理员）
    │
    ├── 直客租户 A（直接付费给你）
    ├── 直客租户 B
    │
    ├── 代理商 X（一级账号）
    │   ├── 代理X 的客户1（二级租户）
    │   ├── 代理X 的客户2（二级租户）
    │   └── 代理X 的客户3（二级租户）
    │
    └── 代理商 Y（一级账号）
        ├── 代理Y 的客户1
        └── 代理Y 的客户2
```

代理商拥有一个"代理管理后台"（比普通租户多一层），可以：
- 创建/管理自己的下级租户（客户）
- 给客户分配套餐和用户数
- 查看自己所有客户的使用情况
- 设定自己的定价（但不能低于你的底价）

### 4.3 白标能力

| 可自定义项 | 说明 |
|-----------|------|
| Logo | 代理的品牌 Logo |
| 系统名称 | 如"教培通 CRM" |
| 登录页背景 | 代理自己的视觉风格 |
| 域名 | agent.yunkes.com 或自定义域名 CNAME |
| 客服联系方式 | 代理的客服，而不是你的 |
| 颜色主题 | 主色调自定义 |
| 底部版权 | 代理的公司名 |

**不可自定义**（你的底线）：
- 功能模块（不能自己加功能）
- 数据库（数据在你的服务器）
- 后端逻辑（不能改代码）

### 4.4 分账规则

```
客户付费 ¥899/月（代理定价）
    │
    ├── 你的底价：¥399/月（不可低于此价）
    │   → 你拿 ¥399
    │
    └── 代理加价部分：¥500/月
        → 代理拿 ¥500
```

即：**代理商赚差价，你赚底价。** 底价 = 你对应版本的内部成本价。

| 版本 | 你的统一零售价 | 代理商底价 | 代理最低售价 |
|------|:---:|:---:|:---:|
| 入门版 | ¥149/月 | ¥80/月 | ¥149/月（不能比你便宜） |
| 基础版 | ¥399/月 | ¥200/月 | ¥399/月 |
| 专业版 | ¥699/月 | ¥350/月 | ¥699/月 |
| 企业版 | ¥1,299/月 | ¥650/月 | ¥1,299/月 |

代理可以卖得比你贵（加服务费），但**不能比你便宜**（保护品牌价值）。

### 4.5 技术实现要点

| 模块 | 说明 | 难度 |
|------|------|:---:|
| `agents` 表 | 代理商账号、等级、域名配置、品牌配置 | ⭐⭐ |
| 二级租户关联 | `tenants` 表加 `agent_id` 字段 | ⭐ |
| 代理商管理后台 | 类似现在的 admin 后台，但只能管理自己的客户 | ⭐⭐⭐⭐ |
| 白标渲染 | 前端根据域名/agent_id 动态加载 Logo、名称、主题色 | ⭐⭐⭐ |
| 自定义域名 | Nginx 配置 + SSL 证书自动签发 | ⭐⭐⭐ |
| 分账系统 | 记录每笔订单的代理归属，月底生成结算单 | ⭐⭐⭐ |

### 4.6 开发工时估算

| 任务 | 工时 |
|------|------|
| 数据库设计（agents 表、关联关系） | 2h |
| 代理商管理后台（前端+后端） | 40h（约 1 周） |
| 白标功能（Logo/名称/主题色/版权） | 16h |
| 自定义域名 + SSL | 8h |
| 分账结算模块 | 16h |
| 测试 + 联调 | 16h |
| **合计** | **约 2-3 周** |

### 4.7 什么时候做

**不急。** 等你有了 50+ 直客、5+ 个模式A代理商之后，再投入开发模式B。因为：
1. 白标功能开发量不小，没有代理商用就是浪费
2. 你需要先通过模式A验证"代理商真的能带来客户"
3. 模式A的代理如果做得好，自然会要求升级到模式B

## 五、模式C：独立部署（本质就是卖私有部署）

### 5.1 运作方式

```
代理商从你这里买"私有部署版"
    │
    ↓
部署到代理商自己的服务器
    │
    ↓
代理商配置为 SaaS 模式（DEPLOY_MODE=saas）
    │
    ↓
但是！SaaS 模式需要 SAAS_LICENSE_TOKEN（RSA 签名）
    │
    ↓
代理商必须从你这里购买 SaaS 运营授权
    │
    ↓
代理商自己招租户、自己收钱、自己定价、自己运维
```

### 5.2 你已有的安全保障

你的系统已经做了完美的防护（`SaaSGuardService.ts`）：

| 安全机制 | 作用 |
|---------|------|
| `SAAS_LICENSE_TOKEN` | RSA 签名的 JWT，必须你的私钥签发 |
| 硬编码公钥验证 | 代理无法伪造 Token |
| 域名绑定 | Token 中指定 `allowedDomains` |
| 最大租户数 | Token 中指定 `maxTenants` |
| 定期验证 | 每小时重验，防止篡改 |
| 过期机制 | Token 过期后自动降级为私有部署模式 |

**所以：代理就算拿到了全部源码，没有你的私钥，也开不了 SaaS 模式。**

### 5.3 定价方案

| 授权类型 | 价格 | 包含 | 适合 |
|---------|------|------|------|
| **SaaS 运营授权（基础）** | ¥59,800/年 | 源码 + SaaS Token（≤50租户） | 小型代理 |
| **SaaS 运营授权（标准）** | ¥99,800/年 | 源码 + SaaS Token（≤200租户） | 中型代理 |
| **SaaS 运营授权（企业）** | ¥199,800/年 | 源码 + SaaS Token（不限租户） | 大型代理 |

**或者按租户数阶梯收费**：
- 基础费 ¥39,800/年（含 20 租户额度）
- 超出部分 ¥500/租户/年

### 5.4 给代理商的 SaaS Token

你签发的 Token 内容示例：

```json
{
  "type": "saas_platform_license",
  "mode": "saas",
  "allowedDomains": ["crm.agent-company.com"],
  "maxTenants": 50,
  "licenseId": "AGENT-2026-001",
  "version": 1,
  "iat": 1717200000,
  "exp": 1748736000
}
```

关键控制点：
- **域名绑定**：代理只能在指定域名下运营
- **租户数上限**：超过需要续费升级
- **有效期**：年付模式，到期需续费，否则自动降级

### 5.5 风险与应对

| 风险 | 应对 |
|------|------|
| 代理拿了源码不续费 | Token 过期后 SaaS 功能自动关闭，降级为单租户私有部署 |
| 代理改代码去掉验证 | 公钥硬编码在代码中，改了就会导致整个安全机制链失效，他们不敢改 |
| 代理服务差砸你招牌 | 合同约束 + 不授权使用"云客"品牌 + 代理需改品牌名 |
| 代理把源码泄露/转卖 | 合同约束 + 每份源码内嵌代理ID水印（可追溯） |

### 5.6 什么时候做

**最后做。** 理由：
1. 你把整套系统给了代理商，就等于培养了一个"小竞对"
2. 模式C的代理商需要有技术团队、服务器、运维能力——这种公司很少
3. 一旦交出源码就无法收回，需要非常信任的合作关系
4. 你的 SaaS 平台还没做大，先把自己的直客做好

## 六、代理商招募与管理

### 6.1 理想的代理商画像

| 类型 | 适合模式 | 为什么合适 |
|------|:---:|------|
| **企微代运营公司** | A/B | 手里有大量需要 CRM 的客户，企微模块是强卖点 |
| **本地 IT 服务商** | A | 给中小企业装系统的，信任度高，能做售后 |
| **行业 SaaS 公司** | A/B | 做 ERP/财务/进销存的，客户有 CRM 需求但他们没产品 |
| **自媒体/KOL** | A | 有粉丝，一篇推文能带来几十个注册 |
| **独立顾问/自由职业者** | A | 做企业咨询的，推荐一下就能赚佣金 |

### 6.2 代理商合作流程

```
代理商申请 → 你审核 → 签合作协议 → 发放推荐码/代理账号
    ↓
培训（产品知识 + 销售话术 + Demo 演示）
    ↓
代理商开始推广 → 客户注册/成交 → 系统自动记录归属
    ↓
月底结算 → 确认返佣金额 → 打款
    ↓
季度复盘 → 调整策略 → 优秀代理升级
```

### 6.3 给代理商的支持物料

| 物料 | 说明 |
|------|------|
| 📋 **产品手册 PDF** | 功能介绍 + 截图 + 版本对比 + 定价表 |
| 🎬 **产品演示视频** | 3-5 分钟核心流程演示（代理可以转发给客户看） |
| 🖥️ **Demo 账号** | 代理商专属 Demo 环境，可以带客户现场演示 |
| 📝 **销售话术** | 常见客户问题 Q&A + 竞品对比话术 |
| 🎨 **宣传素材** | 朋友圈图文、公众号文章模板、海报模板 |
| 💰 **返佣计算器** | 代理看到推荐 X 个客户能赚多少钱 |
| 📞 **专属对接人** | 银牌以上代理配专属对接人，及时解答问题 |

### 6.4 代理商协议关键条款

| 条款 | 内容 |
|------|------|
| **品牌使用** | 模式A：可以说"推荐云客CRM"；模式B/C：不能使用"云客"品牌 |
| **最低售价** | 不能低于你的官方零售价（保护价格体系） |
| **独家/非独家** | 默认非独家；钻石代理可谈区域独家 |
| **结算周期** | 月结，次月 15 日前打款 |
| **退出条款** | 任何一方可提前 30 天书面通知终止合作 |
| **客户归属** | 推荐的客户永远归属该代理（不会被其他代理抢走） |
| **竞业限制** | 模式B/C代理不得同时代理直接竞品 CRM |

## 七、分阶段落地路线

### 第一阶段：推荐返佣启动（本月内可做完）

| 周 | 任务 | 说明 |
|---|------|------|
| **W1** | 数据库加字段 + 注册接口改造 | `referral_code` + `referral_source` |
| **W1** | Admin 后台加"代理商管理"页面 | CRUD + 推荐统计 |
| **W1** | 官网加"成为合作伙伴"入口 | 申请表单 |
| **W2** | 准备销售物料包 | PDF + 视频 + Demo 账号 |
| **W2** | 招募首批 5 个代理 | 从现有人脉中找 |
| **W2** | 手动结算流程跑通 | Excel 导出推荐数据 → 计算佣金 → 打款 |

### 第二阶段：验证跑通（1-3个月）

| 任务 | 目标 |
|------|------|
| 代理商数量达到 10-20 个 | 覆盖 3-5 个行业 |
| 代理带来首批 20+ 付费客户 | 验证模式可行 |
| 优化返佣规则 | 根据实际数据调整比例 |
| 自动化结算 | 开发自动结算模块（如果手动太累） |

### 第三阶段：模式B 开发（客户数 50+ 后）

| 任务 | 工时 |
|------|------|
| 代理商管理后台开发 | 1 周 |
| 白标功能开发 | 1 周 |
| 分账系统 | 0.5 周 |
| 招募 3-5 个 B 模式代理 | 持续 |

### 第四阶段：模式C 开放（客户数 200+ 后）

## 九、常见问题

### Q1：代理商会不会把客户抢走？
**不会。** 模式A/B中客户注册在你的平台上，数据在你的服务器。代理商没有客户的数据库访问权限。

### Q2：代理商卖得比我便宜怎么办？
合作协议中明确写"不得低于官方零售价"。违反即终止合作并取消返佣。

### Q3：代理商不积极怎么办？
正常。20个代理中可能只有 3-5 个是活跃的。把精力放在活跃代理身上，给他们更好的支持。不活跃的代理不需要管理成本——他们不卖也不花你钱。

### Q4：模式C代理拿了源码自己改不给我钱了？
SaaS Token 有期限（年付）。到期不续费 → Token 过期 → SaaS 功能自动关闭 → 他的所有租户无法使用。你的 `SaaSGuardService` 已经做了这个保护。

### Q5：先做哪种代理？
**先做企微代运营公司的推荐返佣。** 理由：
1. 他们的客户 100% 需要 CRM
2. 你的企微集成是强差异化卖点
3. 他们推荐起来最自然（"我帮你做企微运营，顺便用这个 CRM 管客户"）

## 十、一句话总结

> **代理商分三档干：**
> - **现在**：做推荐返佣（模式A），1天开发完，招5个代理试水，验证模式
> - **半年后**：做分销定价（模式B），2周开发完，给优秀代理升级为白标分销
> - **一年后**：开放独立部署（模式C），卖 SaaS 运营授权，年费 6-20 万/家
>
> **核心原则：客户数据在你手里、授权密钥在你手里、品牌定价在你手里。代理商赚的是服务费和销售佣金，不是你的产品。**
