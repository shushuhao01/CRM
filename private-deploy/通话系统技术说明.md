# CRM 通话系统技术说明

> 版本: 1.8.0 | 更新日期: 2026-06-15

---

## 一、系统架构总览

```
                   ┌─────────────┐
                   │   PBX/SIP   │
                   │  FreePBX    │
                   │  Asterisk   │
                   └──────┬──────┘
                          │ Webhook (HTTP POST)
                          ▼
┌──────────┐     ┌──────────────────┐     ┌──────────────┐
│ APP 端   │────▶│   CRM 后端       │◀────│  云呼叫平台  │
│ Android  │ WS  │  (Express.js)    │ API │  阿里云/腾讯 │
│ iOS      │     │                  │     └──────────────┘
└──────────┘     └───────┬──────────┘
                         │ WebSocket
                         ▼
                ┌──────────────────┐
                │  CRM 前端 (Vue3) │
                │  CallManagement  │
                └──────────────────┘
```

**通信链路**:
- **外呼**: CRM前端 → 后端API → 云呼叫平台/APP端
- **呼入(SIP)**: PBX → Webhook → 后端 → WebSocket → CRM前端
- **呼入(手机)**: APP原生来电检测 → WebSocket → 后端 → WebSocket → CRM前端

## 二、外呼功能

### 2.1 外呼方式

| 方式 | 标识 | 说明 |
|------|------|------|
| 系统线路 | `system` | 通过云呼叫平台(阿里云/腾讯云/华为云)发起 |
| 工作手机 | `mobile` | 通过APP控制手机原生拨号 |
| 网络电话 | `voip` | WebRTC / SIP软电话 |
| SIP线路 | `sip` | SIP中继/IP-PBX |
| 手动拨号 | `manual` | 手动拨打，仅记录 |

### 2.2 外呼配置管理

- **路由前缀**: `/api/v1/call-config`
- **全局配置**: `GET/PUT /call-config/global` (管理员)
- **线路管理**: `CRUD /call-config/lines` (管理员)
- **线路分配**: `CRUD /call-config/assignments` (管理员)
- **用户偏好**: `GET/PUT /call-config/preference`
- **用户可用线路**: `GET /call-config/my-lines`
- **工作手机**: `GET /call-config/work-phones`, `POST /call-config/work-phones/qrcode`

### 2.3 阿里云外呼集成

- **服务文件**: `backend/src/services/AliyunCallService.ts`
- **发起外呼**: 调用阿里云 VMS/CCC API
- **状态回调**: `POST /api/v1/calls/webhook/aliyun/status`
- **录音回调**: `POST /api/v1/calls/webhook/aliyun/recording`

## 三、呼入功能

### 3.1 呼入场景

| 场景 | 触发方式 | 关键接口 |
|------|----------|----------|
| SIP/PBX呼入 | PBX调用Webhook | `/calls/webhook/sip/incoming` |
| 工作手机呼入 | APP检测来电→WebSocket | `INCOMING_CALL_DETECTED` 消息 |
| 手机HTTP上报 | APP调用HTTP API | `/mobile/call/incoming` |

### 3.2 SIP/PBX 呼入链路

```
外线来电 → PBX → AGI/Lua脚本 → POST /sip/incoming
                                        │
                    ┌───────────────────┘
                    ▼
             CRM后端处理:
             1. 客户匹配(phone/mobile)
             2. 呼入路由查找(inbound_routes)
             3. 线路分配查找(user_line_assignments)
             4. 客户专属销售匹配
             5. 坐席状态检查(agent_status)
             6. 创建call_records(inbound/ringing)
             7. WebSocket推送CALL_INCOMING
                    │
                    ▼
             CRM前端:
             - 来电弹窗(客户信息+接听/拒绝)
             - 来电铃声
             - 桌面通知
```

**Webhook 密钥验证**: 
- 环境变量: `SIP_WEBHOOK_SECRET`
- 支持 Header(`X-Webhook-Secret`) 或 Body(`secret`) 传递
- 使用 `crypto.timingSafeEqual` 防时序攻击
- 未配置密钥时跳过验证（开发模式）

### 3.3 工作手机呼入链路

```
来电 → APP原生检测(PhoneStateListener/CXCallObserver)
     → MobileWebSocket: INCOMING_CALL_DETECTED
     → MobileWebSocketService.handleIncomingCallDetected()
       - 客户匹配
       - 创建call_records(inbound/ringing)
       - WebSocket推送CALL_INCOMING到CRM前端
       - 回传callId给APP
```

### 3.4 呼入路由分配

**优先级**（从低到高）:
1. `inbound_routes` 表 DID 路由匹配
2. `user_line_assignments` + `call_lines` 线路分配
3. `customers.sales_rep_id` 客户专属销售 ← **最终覆盖**

**坐席状态过滤**: 当坐席 `agent_status = 'busy'` 时，仅创建通话记录，不推送来电弹窗。

## 四、坐席状态管理

### 4.1 API

| 接口 | 方法 | 说明 |
|------|------|------|
| `/call-config/agent-status` | GET | 获取当前用户坐席状态 |
| `/call-config/agent-status` | PUT | 更新坐席状态 |
| `/call-config/agent-status/list` | GET | 管理员查看所有坐席状态 |

**状态值**: `ready`(就绪) / `busy`(忙碌) / `offline`(离线)

### 4.2 存储

- **后端**: `users` 表 `agent_status`、`status_changed_at`、`status_reason` 字段
- **前端**: 优先从后端获取，失败时降级到 `localStorage`

### 4.3 状态切换流程

```
前端 toggleCallStatus()
  → callConfigApi.updateAgentStatus(newStatus)
    → PUT /call-config/agent-status
      → UPDATE users SET agent_status = ?
      → WebSocket: AGENT_STATUS_CHANGED
  → localStorage 同步
  → UI 更新
```

## 五、通话记录管理

### 5.1 数据表

**主表**: `call_records`

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | VARCHAR(50) PK | 通话ID |
| `tenant_id` | VARCHAR(36) | 租户ID |
| `call_type` | ENUM | `outbound` / `inbound` |
| `call_status` | VARCHAR(20) | `ringing`/`connected`/`missed`/`failed`/`rejected`/`busy` |
| `call_method` | VARCHAR(20) | `system`/`mobile`/`voip`/`sip`/`manual` |
| `call_direction` | VARCHAR(10) | `in`/`out` (冗余字段) |
| `inbound_source` | VARCHAR(50) | `sip`/`mobile`/`voip`/`pbx` |
| `inbound_route_id` | VARCHAR(50) | 关联呼入路由 |
| `ring_duration` | INT | 振铃时长(秒) |
| `queue_time` | INT | 排队等待时长(秒) |
| `transfer_from` | VARCHAR(100) | 转接来源坐席 |
| `transfer_to` | VARCHAR(100) | 转接目标坐席 |
| `caller_number` | VARCHAR(20) | 主叫号码 |
| `provider_call_id` | VARCHAR(100) | 服务商侧通话ID |
| `recording_url` | VARCHAR(500) | 录音文件URL |
| `duration` | INT | 通话时长(秒) |

### 5.2 API

| 接口 | 路由前缀 | 说明 |
|------|----------|------|
| 通话记录CRUD | `/api/v1/calls/records` | 列表/详情/更新 |
| 状态更新 | `PUT /calls/:id/status` | 接听/拒绝/未接 |
| 录音管理 | `/api/v1/calls/recordings` | 录音列表/播放 |
| 跟进记录 | `/api/v1/calls/followups` | 通话后跟进 |
| 通话任务 | `/api/v1/calls/tasks` | 批量外呼任务 |

## 六、WebSocket 消息协议

### 6.1 呼入相关消息

| 消息类型 | 方向 | 说明 |
|----------|------|------|
| `INCOMING_CALL_DETECTED` | APP → 后端 | APP检测到来电 |
| `CALL_INCOMING` | 后端 → CRM前端 | 来电通知（弹窗） |
| `CALL_STATUS` | 后端 → 前端 | 通话状态变更 |
| `CALL_ENDED` | 后端 → 前端 | 通话结束 |
| `AGENT_STATUS_CHANGED` | 后端 → 前端 | 坐席状态变更广播 |

## 九、数据库变更汇总 (v1.8.0)

### 9.1 users 表新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `agent_status` | VARCHAR(20) DEFAULT 'ready' | 坐席状态 |
| `status_changed_at` | DATETIME NULL | 状态变更时间 |
| `status_reason` | VARCHAR(200) NULL | 状态变更原因 |

### 9.2 call_records 表新增字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `call_direction` | VARCHAR(10) | in/out |
| `ring_duration` | INT DEFAULT 0 | 振铃时长 |
| `queue_time` | INT DEFAULT 0 | 排队时长 |
| `transfer_from` | VARCHAR(100) | 转接来源 |
| `transfer_to` | VARCHAR(100) | 转接目标 |
| `inbound_route_id` | VARCHAR(50) | 呼入路由ID |
| `inbound_source` | VARCHAR(50) | 呼入来源 |

### 9.3 新增表: inbound_routes

呼入路由配置表，支持 DID→坐席 的映射，支持直接分配、排队、振铃组等模式。

### 9.4 迁移脚本

- **文件**: `backend/database-migrations/20260615-inbound-call-enhancement.sql`
- **兼容性**: MySQL 5.7+ / phpMyAdmin（使用 PREPARE/EXECUTE 实现幂等）
- **执行方式**: phpMyAdmin → SQL → 粘贴执行 或 `mysql -u abc789 -p crm_local < 迁移脚本.sql`
