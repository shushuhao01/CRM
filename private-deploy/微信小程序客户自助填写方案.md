# 微信小程序 — 客户自助填写资料方案

> 版本：v1.1 | 日期：2026-05-04 | 适用：私有部署 + 租户版本

---

## ⭐ 核心架构问题一：这是什么类型的小程序？

### 答案：普通微信小程序，不是企微服务商应用

**这是在微信公众平台 (mp.weixin.qq.com) 注册的普通微信小程序**，和你现有的「云客CRM企微服务商应用」是完全不同的两个东西。

```
┌──────────────────────────────────┐     ┌──────────────────────────────────┐
│  云客CRM 企微服务商应用 (已有)     │     │  客户资料填写 微信小程序 (新增)    │
│  ─────────────────────────────   │     │  ─────────────────────────────   │
│  注册平台: 企微开放平台            │     │  注册平台: 微信公众平台            │
│      open.work.weixin.qq.com     │     │      mp.weixin.qq.com            │
│  使用者: 企业成员(销售员)          │────▶│  使用者: 外部客户(普通微信用户)    │
│  运行在: 企业微信App              │发送  │  运行在: 普通微信App              │
│  ID: SuiteID (ww开头)            │卡片  │  ID: AppID (wx开头)              │
│  功能: 侧边栏/通讯录/会话存档      │     │  功能: 客户自助填写姓名电话地址    │
└──────────────────────────────────┘     └──────────────────────────────────┘
         企业微信里运行                            普通微信里运行
```

**为什么必须是普通微信小程序？** 因为填资料的是**客户**，客户用的是**普通微信**，不是企业微信。企微的 `sendChatMessage` API 原生支持发送普通微信小程序卡片到聊天，客户在微信中直接打开。

**两者的关系**：
- `云客CRM服务商应用`：你已有的企微第三方应用，提供侧边栏、通讯录等能力
- `客户资料填写小程序`：新注册的普通微信小程序，只需在企微服务商后台**关联**一下AppID即可
- 关联后，成员在企微侧边栏就能通过 `sendChatMessage` 把这个微信小程序卡片发给客户

### 你需要做的事（一次性）

| 步骤 | 在哪里做 | 做什么 | 耗时 |
|------|---------|--------|------|
| ① | 微信公众平台 mp.weixin.qq.com | 用你的企业主体注册一个新的微信小程序，获取 AppID + AppSecret | 10分钟 |
| ② | 微信公众平台 | 提交小程序认证（300元/年，如需获取手机号功能） | 1-3天审核 |
| ③ | 企微服务商后台 open.work.weixin.qq.com | 在你的「云客CRM」应用中关联这个小程序AppID | 5分钟 |
| ④ | 你的Admin管理后台 | SuiteApp页面新增小程序配置（AppID + AppSecret） | 配置1分钟 |

做完这4步，**所有已授权的租户自动可用**，不需要租户做任何操作。

## ⭐ 核心架构问题二：多租户如何共用一个小程序？

### 答案：租户无需任何额外操作，侧边栏自动携带小程序

**小程序只有一个，属于你（运营方/服务商）。** 所有租户共用同一个小程序，通过URL参数（`tenantId` + `memberId`）来区分数据归属。

#### 工作原理

```
你(运营方) 拥有小程序 AppID: wxXXXXXXXX
    │
    ├─ 租户A (corpId=A, tenantId=aaa)
    │   ├─ 成员张三 (userId=u1) → 侧边栏发送小程序卡片
    │   │   page: /pages/index?tenantId=aaa&memberId=u1&sign=xxx
    │   │   → 客户填写后数据归属到 租户A → 成员张三
    │   │
    │   └─ 成员李四 (userId=u2) → 侧边栏发送小程序卡片
    │       page: /pages/index?tenantId=aaa&memberId=u2&sign=xxx
    │       → 客户填写后数据归属到 租户A → 成员李四
    │
    ├─ 租户B (corpId=B, tenantId=bbb)
    │   └─ 成员王五 (userId=u3) → 侧边栏发送小程序卡片
    │       page: /pages/index?tenantId=bbb&memberId=u3&sign=xxx
    │       → 客户填写后数据归属到 租户B → 成员王五
    │
    └─ 私有部署客户C (tenantId=ccc)
        └─ 同理，tenantId=ccc，数据完全隔离
```

#### 为什么租户不需要额外操作？

你的系统已有完整的侧边栏认证链路：

1. **租户管理员已在CRM后台的企微管理中绑定了 `corpId`** → 存储在 `wecom_configs` 表，每个 `corpId` 对应一个 `tenantId`
2. **成员已在侧边栏H5中绑定了CRM账号** → 通过 `sidebar/bind-account` 接口，JWT token 中已包含 `tenantId` + `userId`
3. **侧边栏H5页面中点击「发送资料收集」时**，直接从当前 sidebar JWT token 中取 `tenantId` 和 `crmUserId`，拼接到小程序页面路径参数中
4. **客户打开小程序提交数据时**，后端根据路径中的 `tenantId` 设置租户上下文，数据自动写入正确租户的数据库空间

**整个流程：租户成员只需在侧边栏点一个按钮，完全无感知，不需要扫码、不需要授权、不需要绑定小程序。**

#### 三种使用场景对比

| 场景 | 租户需要做什么 | 小程序归属 |
|------|---------------|-----------|
| **SaaS租户** | 什么都不用做，侧边栏自动可用 | 运营方(你)的小程序 |
| **私有部署** | 什么都不用做，侧边栏自动可用 | 运营方(你)的小程序 |
| **租户想用自己的小程序(可选)** | 在CRM后台配置自己的AppID（高级功能，后期可扩展） | 租户自己的小程序 |

## 一、需求概述

### 1.1 背景与痛点

当前CRM系统新增客户流程：**成员(销售员)手动填写客户资料** → 需要客户将姓名、电话、地址等敏感信息通过微信聊天发送给成员 → 成员复制粘贴到CRM表单。

**痛点：**
- 客户资料通过聊天传递，**存在泄露风险**
- 成员掌握客户全部敏感信息（手机号、地址等），即使CRM配置了敏感信息权限也无法防止聊天记录中的信息泄露
- 手动录入效率低，容易出错

### 1.2 方案目标

开发一个**微信小程序**，实现：
1. 成员通过**企业微信侧边栏一键转发小程序卡片**给客户
2. 小程序卡片**携带租户ID + 成员ID**，确保资料归属
3. 客户**自主填写个人资料并提交**
4. 资料提交后**自动同步到该租户该成员的CRM客户管理**
5. CRM配置了敏感信息权限的情况下，成员**无法查看客户手机号等敏感字段**
6. 表单字段由CRM后台**客户设置**统一管控，支持**小程序专用的必填/折叠配置**

### 2.2 数据流

```

     ↓
3. 转发小程序卡片到客户微信聊天
     ↓
4. 客户打开小程序 → 请求后端获取表单配置 (GET /api/v1/mp/form-config)
     ↓
5. 客户填写资料 → 可选微信授权获取手机号
     ↓
6. 客户提交 → POST /api/v1/mp/submit-customer
     ↓
7. 后端验证签名 → 创建客户 → 发送系统消息通知 → WebSocket推送
     ↓
8. 小程序显示成功页 → 客户可点击「告知顾问」或「返回聊天」
     ↓
9. CRM客户列表实时显示新客户 (来源标记: 小程序自助提交)
```

#### 3.1.4 微信获取手机号接口说明

微信小程序获取用户手机号需要：
1. **小程序必须已认证**（企业主体，认证费300元/年）
2. 使用 `<button open-type="getPhoneNumber" @getphonenumber="onGetPhoneNumber">` 获取加密数据
3. 将加密数据（`code`）发送到后端，后端调用微信 `phonenumber.getPhoneNumber` 接口解密获得真实手机号
4. **费用**：每次调用该接口微信收费 **0.03元/次**（2024年起）

**备选方案：** 如果不想付费，可以只提供手动输入手机号的方式，但会降低用户体验。建议至少保留手动输入作为备选。

#### 3.2.2 接口详细设计

##### ① GET /api/v1/mp/form-config

**功能**：获取该租户的小程序表单字段配置

**请求参数**：
```
?tenantId=xxx&memberId=xxx&ts=1714800000&sign=xxx
```

**签名算法**：
```
sign = MD5(tenantId + memberId + ts + SECRET_KEY)
```
- `SECRET_KEY` 从 `backend/.env` 的 `MP_FORM_SECRET` 读取
- `ts` 有效期7天，超过返回链接过期

**响应**：
```json
{
  "success": true,
  "data": {
    "tenantName": "XX品牌",
```

##### ② POST /api/v1/mp/submit-customer

**功能**：客户提交资料，创建客户记录

**请求体**：
```json
{
  "tenantId": "租户ID",
  "memberId": "成员ID",
  "ts": 1714800000,
  "sign": "签名",
  "customerData": {
    "name": "张三",
    "phone": "13800138000",
    "gender": "male",
    "age": 30,
    "province": "广东省",
    "city": "深圳市",
    "district": "南山区",
    "street": "粤海街道",
    "detailAddress": "科技园xxx号",
    "customFields": {
      "customer_custom_field1": "无"
    }
  }
}
```

**后端处理流程**：
1. 验证签名和时间戳有效性
2. 设置租户上下文 `TenantContextManager.setContext({ tenantId, userId: memberId })`
3. 检查手机号是否已存在（复用现有逻辑）
4. 创建客户记录，`source` 设为 `'miniprogram'`，`createdBy` 和 `salesPersonId` 设为 `memberId`
5. 生成客户编号
6. **发送系统消息通知**给该成员（复用 `messageService.sendSystemMessage`）
7. 通过 **WebSocket** 实时推送通知
8. 记录客户创建日志 `createCustomerLog`（来源标记为小程序自助提交）
9. 返回成功

**响应**：
```json
{
  "success": true,
  "message": "资料提交成功",
  "data": { "customerId": "xxx" }
}
```

##### ③ POST /api/v1/mp/get-phone

**功能**：解密微信手机号

**请求体**：
```json
{
  "code": "微信返回的code",
  "tenantId": "租户ID",
  "sign": "签名"
}
```

**后端处理**：
1. 使用小程序 `appId` + `appSecret` 调用微信 API `https://api.weixin.qq.com/wxa/business/getuserphonenumber`
2. 传入 `code` 获取用户手机号
3. 返回解密后的手机号

**关键点：租户不需要做任何额外操作。** 侧边栏H5在发送小程序卡片时，自动从已有的认证上下文获取 `tenantId` 和 `memberId`。

```

    └─ sidebar/bind-account → JWT Token: { tenantId: 'tenant_aaa', crmUserId: 'user_001', corpId: 'A_CORP_ID' }
```

#### 3.4.7 发送机制

侧边栏H5页面已有企微JS-SDK初始化（你的 `sidebar/js-sdk-config` 接口已实现）。用户在「客户资料收集」页面点击按钮时调用：

```javascript
// h5/src/views/app/MpCollect.vue 中的发送逻辑

async function sendCustomerFormMP() {
  // 1. 从已有的auth store获取身份信息(登录时已存在)
  const tenantId = authStore.tenantId     // 从JWT token中获取
  const memberId = authStore.crmUserId    // 从JWT token中获取
  const ts = Math.floor(Date.now() / 1000)

  // 2. 调用后端接口计算签名(防止前端暴露密钥) + 获取卡片配置
  const { data } = await api.post('/api/v1/mp/generate-card', { tenantId, memberId, ts })
  // data = { sign, cardTitle, cardCoverUrl, appId }

  // 3. 通过企微JS-SDK发送小程序卡片到当前聊天窗口
  wx.invoke('sendChatMessage', {
    msgtype: 'miniprogram',
    miniprogram: {
      appid: data.appId,          // 全局小程序AppID
      title: data.cardTitle,      // 卡片标题(租户配置 > 全局默认)
      imgUrl: data.cardCoverUrl,  // 封面图(租户配置 > 全局默认)
      page: `/pages/index/index?tenantId=${tenantId}&memberId=${memberId}&ts=${ts}&sign=${data.sign}`
    }
  }, function(res) {
    if (res.err_msg === 'sendChatMessage:ok') {
      // 4. 发送成功，记录发送日志到后端
      api.post('/api/v1/mp/log-send', { tenantId, memberId, ts })
      showToast('小程序卡片已发送')
    }
  })
}
```

**后端 `generate-card` 接口**返回的数据：
```json
{
  "sign": "a1b2c3...",
  "appId": "wxXXXXXXXX",
  "cardTitle": "XX品牌邀您填写个人信息",
  "cardCoverUrl": "https://xxx.com/cover.png"
}
```
此接口会读取全局配置 + 当前租户配置，合并后返回。

## 五、安全设计

### 5.1 防篡改签名

```
sign = MD5(tenantId + memberId + ts + MP_FORM_SECRET)
```

- 每个租户的 `MP_FORM_SECRET` 存在 `.env` 或 `tenant_settings` 中
- 时间戳 `ts` 有效期默认7天，可配置
- 签名不匹配或过期则拒绝请求

### 5.2 防重复提交

- 同一 `tenantId + phone` 组合，30秒内只能提交一次（Redis/内存缓存限流）
- 如果该租户已存在相同手机号的客户，返回友好提示「您的资料已提交过，无需重复提交」

### 5.3 敏感信息保护

- 小程序提交的客户数据，在CRM端遵循现有**敏感信息权限**策略
- 手机号在系统消息中**脱敏显示**
- 小程序端使用 **HTTPS** 传输
- 后端接口增加 **速率限制**（每IP每分钟10次）

### 5.4 小程序安全

- 小程序需通过微信审核发布
- 后端 `MP_FORM_SECRET` 不暴露到前端
- 微信获取手机号的解密在后端完成，前端只传 `code`

### 11.5 后端API

#### 11.5.1 额度数据结构

存储在 `tenant_settings` 表中：

```json
{
  "settingKey": "mp_phone_quota",
  "settingType": "json",
  "settingGroup": "miniprogram",
  "settingValue": {
    "total": 1000,
    "used": 200,
    "purchases": [
      {
        "packageId": "mp_phone_1",
        "packageName": "基础包",
        "quota": 1000,
        "price": 50,
        "purchaseTime": "2026-05-04T10:00:00Z"
      }
    ]
  }
}
```

#### 11.5.2 API接口

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 查询额度 | GET | `/api/v1/wecom/h5/app/mp-phone-quota` | 返回total/used/remaining + 可购买套餐列表 |
| 购买额度 | POST | `/api/v1/wecom/h5/app/mp-phone-quota-purchase` | 购买套餐，total += quota |
| 扣减额度 | 内部调用 | `POST /api/v1/mp/get-phone` 中自动扣减 | 每次成功获取手机号后 used += 1 |

#### 11.5.3 额度扣减逻辑

在 `backend/src/routes/miniprogram/index.ts` 的 `POST /get-phone` 接口中：

```
1. 查询 tenant_settings.mp_phone_quota
2. 检查 remaining = total - used
3. remaining <= 0 → 返回 { code: 'QUOTA_EXHAUSTED', message: '手机号获取额度已用完' }
4. remaining > 0 → 调用微信API获取手机号 → used += 1 → 保存
5. 返回手机号
```

小程序端收到 `QUOTA_EXHAUSTED` 后：
- 隐藏"微信授权获取手机号"按钮
- 显示手动输入手机号输入框

### 11.6 数据库改动

**无需新增表或字段**。所有数据存储在现有的 `tenant_settings` 表（JSON类型 `setting_value` 字段）和 `system_configs` 表中。

### 11.7 安全考虑

- 购买接口需要 **sidebar JWT鉴权**，防止未授权调用
- 额度扣减使用 **数据库事务**，防止并发超扣
- 套餐配置只有Admin管理后台（运营方）可以修改，租户只能查看和购买
