# CRM系统支付方式完整配置指南

> 本文档详细说明微信支付、支付宝、对公转账三种支付方式的配置步骤，包括第三方平台操作流程。
> 更新时间：2026-04-27

---

## 一、前置条件

| 项目 | 要求 |
|------|------|
| 企业营业执照 | 已完成工商注册 |
| 对公银行账户 | 已开户 |
| 域名 | 已完成ICP备案 |
| SSL证书 | 已部署HTTPS |
| 后端API地址 | 如 `https://api.yunkes.com` |

**回调地址格式（由系统自动生成）：**
- 微信回调：`https://api.yunkes.com/api/v1/public/payment/wechat/notify`
- 支付宝回调：`https://api.yunkes.com/api/v1/public/payment/alipay/notify`

> 回调地址在管理后台「支付配置」页面会自动根据 `SITE_CONFIG.API_URL` 生成并显示，无需手动输入。

## 二、微信支付配置

### 2.1 微信支付商户平台注册

1. 访问 [微信支付商户平台](https://pay.weixin.qq.com/) 点击「成为商家」
2. 准备材料：
   - 营业执照照片
   - 法人身份证正反面
   - 对公银行账户信息
   - 联系人信息
3. 按照引导完成「企业/个体工商户」入驻
4. 等待微信审核（通常1-3个工作日）
5. 审核通过后，打款验证激活账户

### 2.2 获取商户号和AppID

1. 登录 [微信支付商户平台](https://pay.weixin.qq.com/)
2. **商户号(MchId)**：首页右上角或「账户中心 → 商户信息 → 微信支付商户号」
3. **AppID**：
   - 如果关联公众号：登录 [微信公众平台](https://mp.weixin.qq.com/) → 「设置与开发 → 基本配置」获取AppID
   - 如果关联小程序：登录 [微信小程序平台](https://mp.weixin.qq.com/) → 「设置 → 基本设置」获取AppID
4. **关联AppID**：商户平台 → 「产品中心 → AppID账号管理 → 关联AppID」→ 输入公众号/小程序AppID → 确认关联

### 2.3 V3接口配置（推荐）

#### 2.3.1 设置APIv3密钥

1. 登录商户平台 → 「账户中心 → API安全」
2. 点击「设置APIv3密钥」
3. 设置32位字符密钥（建议使用密码生成器），**记录此密钥**
4. 将密钥填入管理后台的「V3密钥」输入框

#### 2.3.2 获取证书序列号

1. 商户平台 → 「账户中心 → API安全 → API证书」
2. 点击「申请证书」，按照指引下载证书工具
3. 使用证书工具生成证书请求，获取证书文件
4. 证书序列号在证书详情中查看，格式如 `43_0178b463M6023025121100101KJWM2G01`
5. 将序列号填入管理后台「证书序列号」输入框

#### 2.3.3 上传商户证书

1. 下载的证书包含两个重要文件：
   - `apiclient_cert.pem` → 上传到管理后台「商家支付证书」
   - `apiclient_key.pem` → 上传到管理后台「微信支付证书密钥」
2. 如需V3支付公钥：商户平台 → 「账户中心 → API安全 → 申请支付公钥」

### 2.4 V2接口配置（即将废弃）

1. 商户平台 → 「账户中心 → API安全 → API密钥(V2)」
2. 设置32位API密钥，**记录此密钥**
3. 将密钥填入管理后台「API密钥(V2)」输入框

### 2.5 开通Native支付

1. 商户平台 → 「产品中心 → 我的产品」
2. 找到「Native支付」，点击「开通」
3. 审核通过后即可使用扫码支付

### 2.6 配置支付回调地址

回调地址由系统自动生成，显示在管理后台支付配置页面的「支付回调地址」字段中。

**确认事项：**
- 回调地址必须是 HTTPS 协议
- 服务器必须可被外网访问
- Nginx 反向代理需正确转发 `/api/v1/public/payment/wechat/notify` 路径

### 2.7 在管理后台填写配置

1. 登录管理后台 → 「支付管理 → 支付配置 → 微信支付配置」
2. 选择「V3 (安全性较高/推荐)」
3. 填写以下字段：

| 字段 | 来源 |
|------|------|
| 商户号 | 商户平台首页 |
| AppID | 公众号/小程序后台 |
| 证书序列号 | 证书详情页 |
| V3密钥 | API安全设置页 |
| V3支付证书 | 证书编号 |
| 商家支付证书 | apiclient_cert.pem |
| 微信支付证书密钥 | apiclient_key.pem |

4. 点击「保存微信配置」
5. 点击「测试连接」验证配置是否正确

## 三、支付宝配置

### 3.1 支付宝开放平台注册

1. 访问 [支付宝开放平台](https://open.alipay.com/) 注册/登录
2. 完成企业实名认证（需要营业执照、法人信息）
3. 进入「控制台」

### 3.2 创建应用

1. 开放平台控制台 → 「我的应用 → 网页&移动应用 → 创建应用」
2. 应用类型选择「网页应用」
3. 填写应用名称（如「XX公司CRM系统」）、应用图标等
4. 提交审核（通常1个工作日）

### 3.3 获取AppID

1. 应用审核通过后，在应用详情页获取 **AppID**
2. 记录此AppID，填入管理后台「支付应用Appid」

### 3.4 配置密钥

#### 3.4.1 使用密钥工具生成密钥对

1. 下载 [支付宝密钥生成工具](https://opendocs.alipay.com/common/02kipk)
2. 打开工具，选择：
   - 密钥格式：PKCS1(非Java适用)
   - 密钥长度：RSA2(2048位)
3. 点击「生成密钥」
4. **应用私钥**：复制保存（需填入管理后台「支付宝商家私钥」）
5. **应用公钥**：复制保存

#### 3.4.2 上传公钥到支付宝

1. 回到开放平台 → 应用详情 → 「开发设置 → 接口加签方式」
2. 点击「设置」
3. 加签模式选择「公钥」
4. 将上一步生成的**应用公钥**粘贴到输入框
5. 点击「保存设置」
6. 系统会返回**支付宝公钥**，复制保存（需填入管理后台「支付宝公钥」）

> **注意**：应用公钥和支付宝公钥是不同的！
> - 应用公钥：你上传给支付宝的
> - 支付宝公钥：支付宝返回给你的，用于验证回调签名

#### 3.4.3 使用公钥证书模式（可选，安全性更高）

1. 开放平台 → 应用详情 → 「开发设置 → 接口加签方式」
2. 加签模式选择「公钥证书」
3. 上传CSR文件（由密钥工具生成）
4. 下载三个证书文件：
   - `appCertPublicKey_xxx.crt`（应用公钥证书）
   - `alipayCertPublicKey_RSA2.crt`（支付宝公钥证书）
   - `alipayRootCert.crt`（支付宝根证书）
5. 在管理后台支付配置选择「公钥证书」模式，分别上传三个证书

### 3.5 开通电脑网站支付

1. 开放平台 → 应用详情 → 「产品绑定」
2. 绑定「电脑网站支付」产品
3. 签约（需要营业执照信息）
4. 等待审核通过

### 3.6 配置回调地址

回调地址由系统自动生成，显示在管理后台支付配置页面中。

**在支付宝开放平台配置（可选）：**
1. 应用详情 → 「开发设置 → 应用网关」
2. 填入系统域名：`https://api.yunkes.com`
3. 「授权回调地址」填入：`https://api.yunkes.com/api/v1/public/payment/alipay/notify`

### 3.7 在管理后台填写配置

1. 登录管理后台 → 「支付管理 → 支付配置 → 支付宝支付配置」
2. 填写以下字段：

| 字段 | 来源 |
|------|------|
| 支付应用Appid | 开放平台应用详情页 |
| 支付宝商家私钥 | 密钥工具生成的应用私钥 |
| 支付宝公钥 | 开放平台设置加签后返回的支付宝公钥 |
| 签名类型 | RSA2（推荐） |

3. 点击「保存支付宝配置」
4. 点击「测试连接」验证配置是否正确

## 四、对公转账配置

### 4.1 前置准备

对公转账无需接入第三方平台，仅需配置公司的银行账户信息。

### 4.2 准备银行账户信息

向公司财务部门获取以下信息：
- **开户银行**：如「中国工商银行」
- **账户名称**：公司全称，如「XX科技有限公司」
- **银行账号**：对公账户号码
- **开户支行**：如「中国工商银行XX支行」

### 4.3 在管理后台填写配置

1. 登录管理后台 → 「支付管理 → 支付配置 → 对公转账配置」
2. 填写以下字段：

| 字段 | 说明 |
|------|------|
| 开户银行 | 银行名称 |
| 账户名称 | 公司全称 |
| 银行账号 | 对公账户号码 |
| 开户支行 | 具体支行名称 |
| 转账备注说明 | 引导客户正确备注，如「请在备注中填写公司名称和联系电话」 |

3. 点击「保存对公转账配置」

### 4.4 对公转账业务流程

1. 客户在官网选择「对公转账」方式下单
2. 系统生成待支付订单，展示银行账户信息给客户
3. 客户通过银行转账将款项汇入
4. 管理员在后台「支付管理 → 支付列表」中找到对应订单
5. 点击「确认到账」按钮，填写银行流水号
6. 系统自动完成租户激活和授权码生成

## 五、Nginx配置要点

确保 Nginx 正确转发支付回调请求：

```nginx
# 支付回调路由
location /api/v1/public/payment/ {
    proxy_pass http://127.0.0.1:3000;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;
    
    # 微信支付回调使用 XML 格式
    proxy_set_header Content-Type $content_type;
}
```

## 六、环境变量配置（可选）

在 `backend/.env` 或 `.env.local` 中可配置以下环境变量：

```env
# API地址（用于自动生成回调地址）
API_URL=https://api.yunkes.com

# 微信支付回调地址（如需自定义覆盖自动生成的值）
PAYMENT_NOTIFY_URL=https://api.yunkes.com/api/v1/public/payment/wechat/notify

# 支付宝回调地址
PAYMENT_NOTIFY_URL_ALIPAY=https://api.yunkes.com/api/v1/public/payment/alipay/notify

# 支付成功跳转页面
PAYMENT_RETURN_URL=https://yunkes.com/pay-success
```

## 七、常见问题

### Q1: 微信支付测试连接显示"签名失败"
- 检查 V3 密钥是否正确
- 检查 apiclient_key.pem 是否已正确上传
- 证书序列号是否匹配

### Q2: 支付宝测试连接返回"isv.invalid-signature"
- 检查私钥格式是否正确（PKCS1格式，不含 BEGIN/END 标签）
- 确认使用的是应用私钥而不是支付宝公钥
- RSA2签名类型是否一致

### Q3: 支付回调未生效
- 确认回调地址的域名已通过 HTTPS 访问
- 检查 Nginx 是否正确转发 `/api/v1/public/payment/` 路径
- 检查后端日志中是否收到回调请求

### Q4: 对公转账如何确认到账
- 管理员需手动在「支付列表」中操作
- 找到状态为「待支付」且支付方式为「对公转账」的订单
- 点击「确认到账」并填写银行流水号

## 八、安全提醒

1. **密钥保密**：所有密钥和证书仅在管理后台填写，系统使用 AES-256-CBC 加密存储
2. **HTTPS必须**：回调地址必须使用 HTTPS，否则微信/支付宝不会发送回调
3. **定期检查**：微信支付证书有效期约5年（V3），建议设置到期提醒
4. **IP白名单**：如支付宝开放平台有 IP 白名单设置，需添加服务器公网 IP
