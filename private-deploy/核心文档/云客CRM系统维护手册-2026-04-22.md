### 1.3 多租户架构

系统采用共享数据库、租户ID隔离的多租户模式：
- 每个租户拥有唯一的 `tenant_id`
- 所有业务表均包含 `tenant_id` 字段
- 中间件 `tenantAuth.ts` 自动注入租户上下文
- API查询自动添加租户条件过滤

## 四、后端子系统（Backend）

### 4.1 应用启动流程

```
backend/src/app.ts
  ├── 1. 加载环境变量（dotenv）
  ├── 2. 初始化Express应用
  ├── 3. 注册全局中间件
  │   ├── CORS
  │   ├── JSON解析 / URL编码
  │   ├── 请求日志
  │   ├── 静态文件服务
  │   └── 速率限制（登录接口）
  ├── 4. 初始化数据库连接（TypeORM）
  ├── 5. 注册API路由（50+顶级路由）
  ├── 6. 注册404和错误处理中间件
  ├── 7. 启动WebSocket服务
  ├── 8. 启动定时任务调度器
  └── 9. 监听端口启动HTTP服务
```

## 九、数据库维护

### 9.1 数据库概览

- **数据库类型**：MySQL 8.0+（推荐8.0.30+）
- **字符集**：utf8mb4 + utf8mb4_unicode_ci
- **初始化脚本**：`database/schema.sql`（v4.2.1，6080行）
- **总表数量**：**190张表**
- **辅助脚本**：`migrate_packages_v2.sql`（套餐重构），`migrate_agent_status.sql`（坐席状态）

### 9.2 初始化步骤

1. 创建数据库：
```sql
CREATE DATABASE crm_local CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. 导入schema.sql：
```bash
mysql -u root -p crm_local < database/schema.sql
```

3. 或在phpMyAdmin中导入：
   - 选择数据库 `crm_local`
   - 点击「导入」→ 选择 `schema.sql` → 执行

### 9.3 预设账号

| 账号 | 密码 | 角色 |
|------|------|------|
| superadmin | super123456 | 超级管理员 |
| admin | admin123 | 管理员 |
| manager | manager123 | 部门经理 |
| sales | sales123 | 销售员 |
| service | service123 | 客服 |

### 9.4 重要注意事项

1. **外键检查**：schema.sql开头关闭了外键检查（`SET FOREIGN_KEY_CHECKS = 0`），末尾恢复
2. **DROP TABLE**：脚本包含 `DROP TABLE IF EXISTS`，在已有数据的库上执行会**删除数据**
3. **字符集**：确保MySQL服务器支持 utf8mb4
4. **时区**：脚本设置了 `+08:00` 时区
5. **JSON列**：部分表使用了JSON类型，需MySQL 5.7.8+
6. **函数索引**：`api_configs` 表使用了 `JSON_LENGTH` 函数索引，需MySQL 8.0.13+

### 9.5 数据库备份建议

```bash
# 完整备份（推荐每日执行）
mysqldump -u root -p --single-transaction --routines --triggers crm_local > backup_$(date +%Y%m%d).sql

# 仅结构
mysqldump -u root -p --no-data crm_local > structure_$(date +%Y%m%d).sql

# 增量备份（binlog）
mysqlbinlog --start-datetime="2026-05-29 00:00:00" /var/log/mysql/mysql-bin.000001 > incremental.sql

# 恢复
mysql -u root -p crm_local < backup_20260530.sql

# 定时备份脚本建议（crontab）
# 0 2 * * * /usr/local/bin/backup-crm.sh >> /var/log/crm-backup.log 2>&1
```

### 10.3 Nginx 配置参考

```nginx
# CRM主系统（含API）
server {
    listen 443 ssl;
    server_name crm.your-domain.com;

    ssl_certificate /etc/ssl/certs/your-domain.pem;
    ssl_certificate_key /etc/ssl/private/your-domain.key;

    location / {
        root /var/www/crm/dist;
        try_files $uri $uri/ /index.html;
        expires 7d;
    }

    # 管理后台
    location /admin {
        alias /var/www/admin/dist;
        try_files $uri $uri/ /admin/index.html;

    location /h5 {
        alias /var/www/h5/dist;
        try_files $uri $uri/ /h5/index.html;
    }

    # API接口
    location /api {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        client_max_body_size 50m;
    }

    # WebSocket (Socket.IO)
    location /socket.io {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_read_timeout 86400;
    }

    # 上传文件
    location /uploads {
        alias /var/www/crm-uploads;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
}

# 官方网站/会员中心
server {
    listen 443 ssl;
    server_name www.your-domain.com;

    ssl_certificate /etc/ssl/certs/your-domain.pem;

    location / {

    }
}

# HTTP → HTTPS 重定向
server {
    listen 80;

}
```

- 每个站点独立域名、独立品牌配置
- 共享一套后端API和数据库
- 通过 `sites` 配置区分站点（site_id隔离）
- Nginx为每个站点配置独立server块

### 10.10 系统更新流程

```bash
# 1. 备份当前版本
```

## 十一、常见问题与排查

### 11.1 后端启动失败

**问题1：数据库连接失败**
```
Error: connect ECONNREFUSED 127.0.0.1:3306
```
排查步骤：
1. 检查MySQL服务是否运行：`systemctl status mysql`
2. 检查 `.env` 中数据库配置是否正确
3. 检查用户名密码是否正确
4. 检查数据库是否已创建

**问题2：端口被占用**
```
Error: listen EADDRINUSE: address already in use :::3000
```
排查步骤：
1. 查看占用进程：`lsof -i :3000`
2. 终止进程：`kill -9 <PID>`
3. 或修改 `.env` 中的 PORT

**问题3：TypeORM 实体不匹配**
```
QueryFailedError: Unknown column 'xxx' in 'field list'
```
排查步骤：
1. 检查 schema.sql 是否已包含该字段
2. 重新导入 schema.sql（注意备份数据）
3. 或手动执行 ALTER TABLE 添加缺失字段

### 11.2 前端构建失败

**问题1：依赖安装失败**
```bash
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

**问题2：TypeScript 编译错误**
```bash
npm run type-check     # 检查类型错误
```

**问题3：Vite 构建内存不足**
```bash
export NODE_OPTIONS="--max-old-space-size=4096"
npm run build
```

### 11.3 企微相关问题

**问题1：回调验证失败**
- 检查回调URL是否可从外网访问
- 检查Token和EncodingAESKey是否一致
- 检查域名SSL证书是否正常

**问题2：通讯录同步失败**
- 检查应用可见范围是否包含目标部门
- 检查通讯录读取权限是否已申请
- 查看 `wecom_configs` 表中的token是否过期

**问题3：会话存档无数据**
- 确认已购买会话存档许可
- 确认存档密钥配置正确
- 检查 `wecom_archive_settings` 和 `wecom_archive_members` 表

### 11.4 短信发送失败

- 检查短信额度是否充足
- 检查模板是否已通过审核（状态为 active）
- 检查阿里云短信配置（AccessKey、SecretKey、签名）
- 查看 `sms_records` 表中的 `status` 和 `error_message` 字段

### 11.5 支付相关问题

- 检查支付宝/微信支付证书是否正确
- 确认回调URL已配置且可访问
- 查看 `PaymentService.ts` 和 `payment` 路由的日志
- 对账时检查 `capacity_orders` 或 `sms_quota_orders` 表

### 11.6 WebSocket 连接失败

- 检查Nginx是否配置了 `/socket.io` 代理（不是 `/ws`）
- 确认 `proxy_http_version 1.1` 和 `Upgrade` 头已设置
- 检查防火墙是否放行了WebSocket端口
- 查看浏览器控制台是否有跨域错误
- 确认Socket.IO客户端版本与服务端兼容（均为4.x）

### 11.7 移动APP连接问题

**问题1：扫码绑定失败**
- 确认手机与PC在同一网络（或都可访问服务器）
- 检查 `/api/v1/qr-connection` 接口是否正常
- WebSocket长连接需要稳定网络

**问题2：录音上传失败**
- APP需要麦克风和文件读取权限
- 检查手机品牌录音路径是否已适配
- 上传接口 `/api/v1/mobile/upload-recording` 限制50MB

**问题3：外呼指令无响应**
- 检查WebSocket连接状态（APP设置页可查看）
- 确认CRM端呼出配置→工作手机已绑定
- Android后台保活设置（电池优化白名单）

### 11.8 企微H5工作台问题

**问题1：免登录跳转失败**
- 检查 `WECOM_SUITE_ID`、`WECOM_SUITE_SECRET` 配置
- 确认应用可见范围包含当前用户
- 检查 Token 交换接口 `/wecom/h5/exchange-token`
- JS-SDK签名需要配置可信域名

**问题2：侧边栏无法加载**
- 检查企微后台「侧边栏」配置的URL
- 确认HTTPS证书有效且域名已备案
- 查看 `wecom_sidebar_auth_codes` 是否有对应授权码

### 11.9 小程序问题

**问题1：表单提交失败**
- 检查小程序AppSecret是否配置正确
- 确认 `mini-program-config` 接口返回正确字段配置
- 手机号授权需在微信开放平台开通

**问题2：链接已过期**
- 默认短令牌有效期7天，过期后显示expired页
- 检查 `token_expiry` 字段是否合理

### 11.10 性能优化建议

**后端**：
- MySQL添加慢查询日志（`slow_query_log = 1; long_query_time = 2`）
- 大表添加复合索引（`tenant_id` + 时间字段）
- 通话录音/大文件上传使用OSS直传

**前端**：
- 开启Gzip压缩（Nginx: `gzip on`）
- 静态资源CDN加速
- 图片懒加载 + WebP格式

**数据库**：
- `innodb_buffer_pool_size` 设为可用内存的60-70%
- 定期执行 `ANALYZE TABLE` 更新统计信息
- 超大表考虑按月分区

## 十二、安全注意事项

### 12.1 敏感信息保护
- `.env` 文件不要提交到版本库
- JWT密钥使用强随机字符串
- 数据库密码定期更换
- 支付证书严格保密

### 12.2 API安全
- 所有业务API都经过JWT认证
- 登录接口有频率限制（防暴力破解）
- 敏感信息（手机号、身份证等）脱敏显示
- 文件上传有类型和大小限制

### 12.3 数据安全
- 用户密码使用 bcrypt 加密存储
- 定期备份数据库
- 操作日志完整记录
- 多租户数据严格隔离

### 12.4 防护建议
- Nginx配置HTTPS强制跳转（301重定向）
- 启用CORS白名单（仅允许已知域名）
- 配置WAF防火墙（推荐阿里云WAF或CloudFlare）
- 定期更新依赖包修复安全漏洞（`npm audit fix`）
- 使用Helmet中间件添加安全HTTP头
- 配置CSP（Content-Security-Policy）防止XSS
- 限制文件上传类型（白名单：jpg/png/pdf/xlsx/mp3/wav）
- 对外暴露的API开启限流（express-rate-limit）

### 12.5 敏感信息权限体系

系统内置分级敏感信息保护：

| 级别 | 字段 | 脱敏方式 | 控制位置 |
|------|------|---------|---------|
| 手机号 | phone | `138****1234` | 客户敏感信息权限 |
| 微信号 | wechat | `wx****89` | 同上 |
| 地址 | address | 完整/隐藏 | 客户敏感信息权限 |
| 身份证 | idCard | `4401****1234` | 同上 |

配置路径：系统设置 → 权限管理 → 客户敏感信息权限

### 12.6 企微安全配置

- **回调Token/AESKey**：妥善保管，泄漏需立即更换
- **会话存档密钥**：仅存储在安全环境，不提交代码库
- **企微专区程序**：运行在隔离沙箱，数据不出专区
- **侧边栏授权码**：定期清理过期授权码

| HTTP状态码 | 含义 | 常见原因 |
|-----------|------|---------|
| 200 | 成功 | 正常响应 |
| 201 | 创建成功 | POST请求成功 |
| 400 | 参数错误 | 请求参数缺失或格式错误 |
| 401 | 未认证 | Token缺失或过期 |
| 403 | 无权限 | 角色权限不足 |
| 404 | 未找到 | 资源不存在 |
| 409 | 冲突 | 数据重复（如用户名已存在） |
| 429 | 请求过多 | 触发频率限制 |
