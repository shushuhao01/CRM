# 本地开发环境启动指南

## 📋 前置要求

### 必需软件

1. **Node.js** (推荐 v16 或更高版本)
   - 下载地址: https://nodejs.org/
   - 验证安装: `node -v`

2. **npm** (随Node.js自动安装)
   - 验证安装: `npm -v`

3. **MySQL** (本地数据库)
   - 推荐使用 MySQL 8.0
   - 或使用 XAMPP/WAMP 等集成环境

### 可选软件

- **Git** (用于版本控制)
- **VS Code** (推荐的代码编辑器)

---

## 🚀 快速开始

### 第一步：安装依赖

首次使用或更新代码后，需要安装依赖：

```cmd
# 双击运行或命令行执行
install-all-dependencies.bat
```

这个脚本会自动安装所有四个项目的依赖：
- 后端 (backend)
- CRM前端 (根目录)
- Admin后台 (admin)
- Website官网 (website)

---

### 第二步：配置环境变量

确保以下环境变量文件存在并配置正确：

#### 1. 后端配置 (backend/.env)

```env
# 数据库配置
DB_HOST=localhost
DB_PORT=3306
DB_USERNAME=root
DB_PASSWORD=你的密码
DB_DATABASE=crm_local

# 服务端口
PORT=3000

# JWT密钥
JWT_SECRET=your-secret-key

# 环境
NODE_ENV=development
```

#### 2. CRM前端配置 (.env.development)

```env
# API地址
VUE_APP_API_BASE_URL=http://localhost:3000

# 其他配置
VUE_APP_TITLE=CRM系统
```

#### 3. Admin后台配置 (admin/.env.development)

```env
# API地址
VITE_API_BASE_URL=http://localhost:3000

# 其他配置
VITE_APP_TITLE=Admin后台
```

#### 4. Website官网配置 (website/.env.development)

```env
# API地址
VITE_API_BASE_URL=http://localhost:3000

# 其他配置
VITE_APP_TITLE=官方网站
```

---

### 第三步：初始化数据库

```cmd
# 进入backend目录
cd backend

# 运行数据库初始化脚本
node init-crm-local-db.js
```

---

### 第四步：启动服务

#### 方式一：独立窗口启动（推荐）

```cmd
# 双击运行
start-dev-local.bat
```

这会打开4个独立的命令窗口，分别运行：
- 后端服务
- CRM前端
- Admin后台
- Website官网

优点：
- 每个服务独立运行，互不干扰
- 可以单独关闭某个服务
- 日志清晰，便于调试

#### 方式二：并发启动

```cmd
# 双击运行
start-dev-concurrent.bat
```

所有服务在同一个窗口中运行。

优点：
- 只有一个窗口，界面简洁
- 按 Ctrl+C 可同时停止所有服务

缺点：
- 日志混在一起，不便于查看

## 🐛 常见问题

### Q1: 端口被占用

**错误信息**: `Port 8080 is already in use`

**解决方法**:
```cmd
# 查找占用端口的进程
netstat -ano | findstr :8080

# 结束进程（替换PID为实际进程ID）
taskkill /PID 进程ID /F
```

### Q2: 依赖安装失败

**解决方法**:
```cmd
# 清除npm缓存
npm cache clean --force

# 删除node_modules和package-lock.json
rmdir /s /q node_modules
del package-lock.json

# 重新安装
npm install
```

### Q3: 数据库连接失败

**检查项**:
1. MySQL服务是否启动
2. 数据库配置是否正确 (backend/.env)
3. 数据库是否已创建
4. 用户权限是否正确

### Q4: 后端启动报错

**常见原因**:
1. 依赖未安装: `cd backend && npm install`
2. 数据库未初始化: `node init-crm-local-db.js`
3. 环境变量配置错误: 检查 backend/.env

### Q5: 前端页面空白

**检查项**:
1. 后端API是否正常运行
2. API地址配置是否正确
3. 浏览器控制台是否有错误
4. 网络请求是否成功

## 📊 性能优化建议

### 1. 使用SSD硬盘

将项目放在SSD上可以显著提升启动速度。

### 2. 关闭不必要的服务

如果只开发某个系统，可以只启动对应的服务：

```cmd
# 只启动后端和CRM前端
cd backend
start cmd /k "npm run dev"

cd ..
start cmd /k "npm run serve"
```

### 3. 使用淘宝镜像

```cmd
# 设置npm镜像
npm config set registry https://registry.npmmirror.com

# 或使用cnpm
npm install -g cnpm --registry=https://registry.npmmirror.com
cnpm install
```
