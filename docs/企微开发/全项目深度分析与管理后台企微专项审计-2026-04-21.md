## 一、项目全景架构

### 1.1 工程组织

本项目为"多前端 + 单后端"单体仓库：

| 模块 | 路径 | 角色 |
|------|------|------|
| CRM系统 | 根目录 `src/` | 租户业务前台（主产品） |
| 管理后台 | `admin/src/` | 平台运营管理 |
| 官网会员中心 | `website/src/` | 对外展示 + 会员服务 |
| 后端API | `backend/src/` | 统一业务服务 |
| 移动APP | 仅接口文档 | 独立仓库或未落地 |

### 1.2 技术栈

- 前端：Vue 3.4 + Vite 5 + TypeScript + Vue Router 4 + Pinia + Element Plus
- 后端：Node.js ≥22 + Express 4 + TypeORM 0.3 + MySQL + Socket.IO + JWT
- 官网：Vue 3 + Vite 5 + 自定义样式（无 Element Plus）

### 1.3 鉴权与多租户

- CRM：`auth_token` + 路由守卫 + 业务权限码
- Admin：`admin_token` + 菜单权限 + `v-permission` 指令
- 会员中心：`member_token` + memberAuth 中间件
- 后端数据隔离：`tenantId` + `getTenantRepo()` + AsyncLocalStorage

## 三、代码质量与工程治理

### 3.1 正面评价

- 三端TypeScript化，有明确项目结构和路由组织
- 后端安全基础完善（helmet/cors/rate-limit/JWT/全局错误处理）
- 多租户隔离机制成熟
- 编码标准文档齐全（`.dev-guide.md`/`.dev-rules.md`/`.coding-standards.md`）

### 3.2 需改进问题

1. TypeScript `strict: false`（后端），`any` 使用较多
2. 路由层承载过多业务逻辑，部分文件3000+行
3. 迁移体系不统一（migration脚本与运行时DDL并存）
4. ESLint配置分裂（根目录ESLint 9 vs Admin用`.eslintrc.cjs`）
5. 临时脚本泛滥（根目录和backend下大量`_*.js/_*.cjs`）
6. CRM构建`minify: false`，生产包体积偏大
7. 集成测试严重不足
