## 一、服务商应用整体架构

### 1.1 应用在企微中的呈现形式

```
企业微信中我们的应用有3个可见入口:

  入口1: 工作台应用(主入口)
    企微底部Tab -> 工作台 -> [云客CRM] 应用图标
    点击后: 打开应用主页面(H5)
    展示: 应用首页,可查看企微数据+CRM数据

  入口2: 聊天侧边栏(聊天时使用)
    企微聊天窗口 -> 右侧工具栏 -> [云客CRM] 图标
    点击后: 在侧边栏中打开H5页面
    展示: 当前聊天客户的CRM画像/快捷操作

  入口3: 企微消息推送(被动触发)
    我们的应用可以向企微成员发送应用消息
    如: 新客户提醒/待办通知/审批提醒等

三个入口的关系:
  - 入口1(工作台): 综合管理入口,功能最全
  - 入口2(侧边栏): 聊天场景专用,轻量快捷
  - 入口3(消息): 主动触达,引导用户进入应用
```

### 4.2 企微JS-SDK鉴权流程

```
=== H5页面加载时的鉴权流程 ===

  1. 页面加载
       |
       v
  2. 调用后端获取JS-SDK签名:
     GET /api/v1/wecom/jssdk/config?url=当前页面URL
     返回: { corpId, agentId, timestamp, nonceStr, signature }
       |
       v
  3. 初始化企微JS-SDK:
     wx.config({
       beta: true,
       appId: corpId,         // 或 企业CorpID
       timestamp: timestamp,
       nonceStr: nonceStr,
       signature: signature,
       jsApiList: [
         'getCurExternalContact',  // 获取当前外部联系人
         'sendChatMessage',        // 发送消息
         'getContext',             // 获取进入应用的上下文
         'openUserProfile',       // 打开用户资料页
       ]
     })
       |
       v
  4. wx.ready(() => {
       // SDK初始化成功
       // 获取当前登录用户身份
       wx.invoke('getContext', {}, (res) => {
         // res.entry: 进入来源(normal/contact_profile/single_chat等)
         // res.shareTicket: 分享票据
       })
     })
       |
       v
  5. 获取用户身份(免登录):
     // 企微内置浏览器自带身份信息
     // 通过OAuth2获取当前用户userId
     // 或通过 JS-SDK getContext 获取
       |
       v
  6. 用userId查询绑定关系:
     GET /api/v1/wecom/sidebar/verify-binding
     -> 已绑定: 返回CRM用户信息,正常使用
     -> 未绑定: 跳转绑定页面(输入租户编码+CRM账号)

=== 后端签名生成 ===

  // wecom-jssdk.ts
  router.get('/jssdk/config', async (req, res) => {
    const url = req.query.url as string
    const corpId = wecomConfig.corpId

    // 获取jsapi_ticket(需缓存,2小时有效)
    const ticket = await getJsApiTicket(corpId)

    const timestamp = Math.floor(Date.now() / 1000)
    const nonceStr = generateNonceStr()

    // 签名算法: SHA1(jsapi_ticket=xxx&noncestr=xxx&timestamp=xxx&url=xxx)
    const signature = sha1(
      'jsapi_ticket=' + ticket +
      '&noncestr=' + nonceStr +
      '&timestamp=' + timestamp +
      '&url=' + url
    )

    res.json({
      corpId, timestamp, nonceStr, signature,
      agentId: wecomConfig.agentId
    })
  })
```

### 4.3 侧边栏获取当前聊天客户

```
// sidebar/Portrait.vue

import { onMounted, ref } from 'vue'

const customerInfo = ref(null)
const loading = ref(true)

onMounted(async () => {
  try {
    // 1. 通过JS-SDK获取当前聊天的外部联系人
    const externalUserId = await new Promise((resolve, reject) => {
      wx.invoke('getCurExternalContact', {}, (res) => {
        if (res.err_msg === 'getCurExternalContact:ok') {
          resolve(res.userId)  // 外部联系人userId
        } else {
          reject(new Error('获取外部联系人失败'))
        }
      })
    })

    // 2. 调用CRM后端查询客户画像
    const { data } = await axios.get('/api/v1/wecom/sidebar/customer-portrait', {
      params: { externalUserId },
      headers: { 'Authorization': 'Bearer ' + sidebarToken }
    })

    customerInfo.value = data
    // data包含: 企微信息 + CRM客户信息 + 跟进记录 + 订单等

  } catch (error) {
    // 可能是: 当前不是外部联系人会话(如内部群聊)
    // 显示: "请在与外部客户的聊天中使用"
  } finally {
    loading.value = false
  }
})
```

### 5.2 我们后端需要实现的回调服务

```
=== 文件: backend/src/routes/wecom/suite-callback.ts ===

需要实现的回调处理:

1. suite_ticket 接收(每10分钟):
   POST /wecom/suite/callback
   解密 -> 提取suite_ticket -> 存入缓存/数据库
   必须返回 "success"

2. 授权通知(create_auth):
   企业安装我们应用时触发
   解密 -> 提取auth_code -> 换取permanent_code
   -> 创建WecomSuiteAuth记录
   -> 自动创建WecomConfig(如果关联了租户)

3. 授权变更(change_auth):
   企业修改授权范围时触发
   更新本地授权信息

4. 取消授权(cancel_auth):
   企业取消安装时触发
   标记授权状态为cancelled
   通知Admin和租户

5. 回调URL验证(GET请求):
   企微配置回调时会发GET验证请求
   需要正确解密echostr并返回

代码框架:

  router.post('/suite/callback', (req, res) => {
    const { msg_signature, timestamp, nonce } = req.query
    const xmlBody = req.body  // XML格式

    // 1. 验证签名
    const verified = verifySignature(token, timestamp, nonce, msg_signature, xmlBody)
    if (!verified) return res.status(403).send('invalid signature')

    // 2. 解密消息
    const decrypted = decryptMessage(encodingAESKey, xmlBody)

    // 3. 根据InfoType处理
    switch (decrypted.InfoType) {
      case 'suite_ticket':
        handleSuiteTicket(decrypted.SuiteTicket)
        break
      case 'create_auth':
        handleCreateAuth(decrypted.AuthCode)
        break
      case 'change_auth':
        handleChangeAuth(decrypted.AuthCorpId)
        break
      case 'cancel_auth':
        handleCancelAuth(decrypted.AuthCorpId)
        break
    }

    // 4. 必须返回success
    res.send('success')
  })

  // GET验证
  router.get('/suite/callback', (req, res) => {
    const { msg_signature, timestamp, nonce, echostr } = req.query
    const decrypted = decryptEchoStr(encodingAESKey, echostr)
    res.send(decrypted)
  })
```

## 九、服务商向企微支付的具体方式

> 客户在我们平台内购后，我们需要向企微付款为客户开通服务
> 这笔钱从哪出？怎么付？需要预充值还是绑定对公账户？

### 9.1 企微服务商结算体系

```
=== 企微服务商的付款方式 ===

企微开放平台为服务商提供了专门的结算账户体系:

  步骤1: 绑定对公账户(一次性操作)
    登录企微开放平台 -> 服务商管理 -> 财务管理
    绑定企业的对公银行账户(用于结算/提现)
    需要: 开户行/银行账号/户名(必须与企业认证名称一致)
    验证: 企微会向该账户打一笔小额验证金(如0.01元)
          收到后在平台确认金额即完成绑定

  步骤2: 预充值到服务商钱包(下单前)
    绑定对公账户后，向服务商钱包充值
    充值方式:
      a. 对公转账: 从绑定的对公账户转入(推荐，无手续费)
      b. 在线支付: 部分平台支持微信支付/网银(可能有手续费)
    充值到账: 通常1-3个工作日(对公转账)

  步骤3: 下单时从钱包余额扣款
    为客户企业下单接口许可/会话存档等服务时
    直接从服务商钱包余额中扣除
    余额不足时无法下单

  完整流程:
  对公账户 --充值--> 服务商钱包 --下单扣款--> 企微开通服务

=== 资金流图 ===

  ┌──────────────┐
  │ 客户支付      │  ¥45,000 (微信/支付宝)

  └──────┬───────┘

  └──────┬───────┘
```

```
    const wallet = await getProviderWalletBalance()
    if (wallet.available < purchaseOrder.costPrice) {
      // 余额不足,标记订单为待充值
```

```
=== 数据范围权限(DataScope) ===

  与CRM系统完全一致的三级权限:

  管理员(admin/super_admin):
    数据范围: 全部
    可见: 所有成员的客户/会话/统计数据

  部门主管(department_manager):
    数据范围: 本部门及下属部门
    可见: 本部门所有成员的客户/会话/统计

  普通销售(sales_staff):
    数据范围: 仅自己
    可见: 只能看到自己跟进的客户/自己的会话/自己的统计

=== 权限来源 ===

  企微H5页面登录后获取:
    1. 企微JS-SDK -> 获取当前企微userId
    2. 通过userId查询 WecomUserBinding -> 获取crmUserId
    3. 通过crmUserId查询CRM用户 -> 获取角色(role)和部门(departmentId)
    4. 根据角色确定数据范围(dataScope)

  数据范围枚举:
    'all'        -> 全部数据(管理员)
    'department' -> 本部门数据(主管)
    'self'       -> 仅自己数据(普通销售)
```

```
=== 统一的数据范围过滤中间件 ===

// utils/data-scope.ts

interface DataScopeContext {
  userId: string          // CRM用户ID
  role: string            // 角色
  departmentId: string    // 部门ID
  dataScope: 'all' | 'department' | 'self'
}

// 从token中提取权限上下文
function getDataScope(req): DataScopeContext {
  const user = req.currentUser  // 从JWT提取
  let dataScope: string

  if (['super_admin', 'admin'].includes(user.role)) {
    dataScope = 'all'
  } else if (user.role === 'department_manager') {
    dataScope = 'department'
  } else {
    dataScope = 'self'
  }

  return {
    userId: user.id,
    role: user.role,
    departmentId: user.departmentId,
    dataScope
  }
}

// 构建SQL WHERE条件
function buildScopeWhere(ctx: DataScopeContext, ownerColumn: string): string {
  switch (ctx.dataScope) {
    case 'all':
      return ''  // 无额外条件(已有tenantId过滤)
    case 'department':
      return ` AND ${ownerColumn} IN (
        SELECT id FROM users
        WHERE tenant_id = ? AND department_id = ?
      )`
    case 'self':
      return ` AND ${ownerColumn} = ?`
  }
}

=== 在API中使用 ===

// 工作台首页API
router.get('/app/home', async (req, res) => {
  const tenantId = req.tenantId
  const scope = getDataScope(req)

  // 客户数统计
  let customerCountSQL = `
    SELECT COUNT(*) as cnt FROM wecom_customers
    WHERE tenant_id = ?
  `
  const params = [tenantId]

  if (scope.dataScope === 'department') {
    customerCountSQL += ` AND follow_user_id IN (
      SELECT wecom_user_id FROM wecom_user_bindings
      WHERE tenant_id = ? AND crm_user_id IN (
        SELECT id FROM users WHERE department_id = ?
      )
    )`
    params.push(tenantId, scope.departmentId)
  } else if (scope.dataScope === 'self') {
    customerCountSQL += ` AND follow_user_id = (
      SELECT wecom_user_id FROM wecom_user_bindings
      WHERE tenant_id = ? AND crm_user_id = ? LIMIT 1
    )`
    params.push(tenantId, scope.userId)
  }

  const result = await AppDataSource.query(customerCountSQL, params)
  // ...返回数据
})

// 客户列表API
router.get('/app/customers', async (req, res) => {
  const tenantId = req.tenantId
  const scope = getDataScope(req)

  let sql = `
    SELECT wc.*, u.real_name as followUserName
    FROM wecom_customers wc
    LEFT JOIN wecom_user_bindings wb ON wc.follow_user_id = wb.wecom_user_id
    LEFT JOIN users u ON wb.crm_user_id = u.id
    WHERE wc.tenant_id = ?
  `
  const params = [tenantId]

  // 数据范围过滤
  switch (scope.dataScope) {
    case 'department':
      sql += ` AND u.department_id = ?`
      params.push(scope.departmentId)
      break
    case 'self':
      sql += ` AND wb.crm_user_id = ?`
      params.push(scope.userId)
      break
    // all: 无额外条件
  }

  sql += ` ORDER BY wc.created_at DESC LIMIT ? OFFSET ?`
  // ...分页+返回
})
```
