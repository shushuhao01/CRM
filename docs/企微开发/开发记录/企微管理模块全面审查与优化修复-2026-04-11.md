## 四、已知限制与待完善项

### 4.1 会话存档 - 消息内容拉取需Finance SDK

**当前状态**: 元数据同步模式
- ✅ 可获取开通成员列表
- ✅ 可检查同意状态
- ✅ 可保存/查询会话记录
- ✅ RSA解密方法已实现
- ⚠️ 实际消息拉取需企微Finance SDK（C/C++库）
- ⚠️ Node.js需通过native addon或外部桥接服务调用

**影响**: 会话存档列表页虽然UI完整（650行），包含记录列表、全文搜索、质检标记、VAS购买流程等，但实际聊天记录数据需要SDK拉取。

**替代方案**: 可通过企微HTTP API方式（2023年后支持）获取部分数据，无需C++ SDK。

### 4.2 侧边栏应用 - 仅前端存储

**当前状态**: 使用localStorage存储应用配置
- 轻量级实现，适用于当前阶段
- 未来可升级为服务端存储

**优先级**: P3，可保持现状

### 4.3 回调消息处理 - 仅日志模式

**当前状态**: POST /callback 已实现消息解密和事件分发框架
- ✅ 签名验证
- ✅ AES解密
- ✅ 事件类型识别（change_external_contact/change_external_chat/msgaudit_notify）
- ⚠️ 具体事件处理为日志输出，未执行业务操作（如自动同步客户）

**建议**: 在实际部署时，补充以下事件处理逻辑：
- `add_external_contact` → 自动创建WecomCustomer记录
- `del_external_contact` → 更新客户状态为deleted
- `msgaudit_notify` → 触发会话存档同步

### 4.4 V2.0 设计方案待实现

以下内容在设计文档中已规划，当前为P2/P3优先级：
- 会话存档Tab化布局（聊天会话/敏感词/质检/绑定账号/设置）
- 敏感词库管理（wecom_sensitive_words表）
- 质检记录管理（wecom_quality_inspections表）
- 增值服务订单管理（wecom_vas_orders/wecom_vas_configs表）
- 企微侧边栏客户详情应用
- 第三方应用授权流程

## 九、补充修复 - 侧边栏应用后端路由 (2026-04-11 续)

### 9.1 问题描述

侧边栏应用(Sidebar.vue)前端调用4个API但后端无对应路由，导致404错误：
- `GET /api/v1/wecom/sidebar-apps` → 404
- `POST /api/v1/wecom/sidebar-apps` → 404
- `PUT /api/v1/wecom/sidebar-apps` → 404
- `DELETE /api/v1/wecom/sidebar-apps/:id` → 404

### 9.2 修复方案

使用 `tenant_settings` 表存储侧边栏应用配置（key: `wecom_sidebar_apps`，类型: JSON数组），
利用已有的 `TenantSettings` 实体实现CRUD，每个应用对象包含：
`{ id, name, url, description, isEnabled, createdAt }`

### 9.3 修改文件

| 文件 | 修改内容 |
|------|---------|
| backend/src/routes/wecom.ts | 新增TenantSettings导入 + SIDEBAR_APPS_KEY常量 + 4个路由（GET/POST/PUT/DELETE） |
| src/views/Wecom/Sidebar.vue | 更新info提示文案，反映服务端存储 |

### 9.4 新增API端点

| API路径 | 方法 | 认证 | 功能 |
|---------|------|------|------|
| /sidebar-apps | GET | token | 获取侧边栏应用列表 |
| /sidebar-apps | POST | admin | 添加侧边栏应用 |
| /sidebar-apps | PUT | admin | 批量保存（整体覆盖） |
| /sidebar-apps/:id | DELETE | admin | 删除侧边栏应用 |
