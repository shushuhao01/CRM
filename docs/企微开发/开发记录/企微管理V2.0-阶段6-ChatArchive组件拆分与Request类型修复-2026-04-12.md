### 2.1 ChatArchive.vue 拆分策略

将原 2,038 行单体组件拆分为 **1 个父组件 + 5 个子组件 + 1 个类型文件 + 1 个工具函数文件**：

```
src/views/Wecom/
├── ChatArchive.vue          ← 父组件（409行，含Tab1消息记录 + 子组件编排）
├── types.ts                 ← 共享TypeScript接口定义（114行）
├── utils.ts                 ← 共享工具函数（137行）
└── components/
    ├── ConversationView.vue     ← Tab2 聊天会话三栏布局（549行）
    ├── PurchaseDialog.vue       ← 购买增值服务弹窗（353行）

    ├── FullTextSearchDrawer.vue ← 全文搜索抽屉（97行）
    └── AuditDialog.vue          ← 质检标记弹窗（66行）
```

### 2.2 全局Request类型修复方案

在 `utils/request.ts` 中：
- 新增 `CustomAxiosInstance` 接口，覆写 `get/post/put/delete/patch` 方法返回 `Promise<T>` 而非 `Promise<AxiosResponse<T>>`
- 将 `export default service` 改为 `export default service as CustomAxiosInstance`
- 修复响应拦截器中 `error.response.data` 的类型推断（`as any`）
- 修复请求拦截器的 `config` 参数类型（`any` 以兼容自定义 `RequestConfig` 属性）

| AuditDialog | `modelValue` (v-model) | `update:modelValue`, `audit-success` | `open(row)` |

### 4.2 共享代码提取

**utils.ts** 中的 11 个函数被 3+ 个组件复用：
- `formatMsgTime` → ChatArchive, ConversationView, FullTextSearchDrawer
- `getMsgTypeText` → ChatArchive, ConversationView
- `getTextContent/getMetaSummary/getMetaAgreed` → ChatArchive, ConversationView
- `formatToUsers/getFirstToUser` → ChatArchive, ConversationView
- `highlightText` → FullTextSearchDrawer
- `formatConvTime/formatMsgTimeShort/getDateDividerText` → ConversationView

**types.ts** 中的接口替代了大量 `any` 类型：
- `ref<any[]>([])` → `ref<ChatRecord[]>([])`、`ref<Conversation[]>([])`、`ref<ConvMessage[]>([])`
- `ref<any>(null)` → `ref<ArchiveStatus | null>(null)`、`ref<VasPricing>({})`
- `ref<any>({})` → `ref<PurchaseResult>({...})`

### 4.3 Request类型修复效果

**修复前**（8个API文件受影响）：
```typescript
// api/wecom.ts
import request from '@/utils/request'  // AxiosInstance → .get() returns Promise<AxiosResponse>
const res = await getChatRecords(params)
res.list   // ❌ TS2339: Property 'list' does not exist on type 'AxiosResponse'
```

**修复后**：
```typescript
// utils/request.ts
interface CustomAxiosInstance extends AxiosInstance {
  get<T = any>(url: string, config?: AxiosRequestConfig): Promise<T>
  // ...
}
export default service as CustomAxiosInstance  // .get() returns Promise<T>

// api/wecom.ts
const res = await getChatRecords(params)
res.list   // ✅ Promise<any> → any.list
```

同时修复了：
- `error.response.data` 的 `{}` 类型推断 → `as any`
- 请求拦截器 `config: RequestConfig` vs `InternalAxiosRequestConfig` 不兼容 → `config: any`
