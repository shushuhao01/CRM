# 订单列表金额显示修复

**日期**: 2026-04-24  
**模块**: CRM端 - 订单管理 - 订单列表  
**类型**: Bug修复  

## 问题描述

1. **订单金额显示为0**: 新增订单时产品有金额，提交保存后，订单列表的"订单金额"列始终显示 ¥0
2. **代收金额被标记"已改"**: 订单首次提交后，代收金额列就出现"已改"标记，而正常逻辑是只有取消代收或编辑订单后代收金额发生变化才应标记

## 根因分析

**文件**: `backend/src/routes/orders/orderCrud.ts` 第261行（订单列表GET接口的数据映射）

```javascript
// ❌ BUG: totalAmount 被写在了单行注释 // 之后，被 JavaScript 引擎当作注释忽略
totalQuantity,  // 🔥 新增：总数量        totalAmount: Number(order.totalAmount) || 0,
```

- `totalQuantity,` 后面的 `// 🔥 新增：总数量` 是单行注释
- 注释延伸到行末，所以 `totalAmount: Number(order.totalAmount) || 0,` 也被当成了注释的一部分
- 导致后端返回的订单列表数据中 **没有 `totalAmount` 字段**

### 影响链路

1. **订单金额为0**: 前端 `row.totalAmount` 为 `undefined`，`(row.totalAmount || 0)` 渲染为 ¥0
2. **代收金额"已改"**: 前端判断条件 `codAmount !== (totalAmount - depositAmount)` 变成 `codAmount !== (0 - depositAmount)`，恒不等，"已改"标记恒显示

## 修复方案

将 `totalAmount` 拆分到独立行：

```javascript
// ✅ 修复后
totalQuantity,  // 🔥 新增：总数量
totalAmount: Number(order.totalAmount) || 0,
```

## 影响范围

- **仅修改1行代码**: `backend/src/routes/orders/orderCrud.ts` 第261行
- **不影响**:
  - 订单创建（后端正确存储 totalAmount 和 codAmount）
  - 订单详情（详情接口的 totalAmount 在独立行，不受影响）
  - 其他列表（物流、审核等接口均无此问题）
  - 订单编辑的 codAmount 同步逻辑

## 验证方法

1. 刷新订单列表页面，订单金额应正确显示
2. 新建订单后，代收金额列不应出现"已改"标记
3. 只有通过代收管理修改金额或取消代收后，才应出现"已改"标记
