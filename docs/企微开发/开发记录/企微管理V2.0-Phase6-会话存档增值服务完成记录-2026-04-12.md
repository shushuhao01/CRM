## 五、核心业务逻辑说明

### 5.1 席位管控

- 席位上限 `max_users` 在购买时写入 `wecom_archive_settings`
- 成员管理通过 `wecom_archive_members` 表记录
- 更新成员时校验启用数不超过 `max_users`
- 自动同步 `used_users` 到设置表

### 5.2 购买类型处理

| 类型 | max_users 处理 | expire_date 处理 |
|------|---------------|------------------|
| new | 直接设置 | NOW() + 1年 |
| renew | GREATEST(当前, 新值) | 从到期日延长1年 |
| upgrade | 当前 + 新值 | 不变 |

### 5.3 可见性过滤

| 级别 | 规则 |
|------|------|
| self | 只看自己的聊天记录 |
| department | 看同部门同事的记录 |
| all | 全部可见 |
| admin | 管理员始终全部可见 |

### 5.4 套餐配置化

- 预设套餐包从 `system_config` 表 `wecom_vas_config` 键中的 `presetPackages` 字段获取
- 续费折扣从 `renewalDiscount` 字段获取
- 前端回退到默认6个套餐包（API获取失败时）
