# 企微独立窗口权限修复记录

**日期**: 2026-04-24  
**模块**: CRM端 - 企微管理 - 独立窗口  

---

## 问题描述

企微管理支持在独立窗口中打开（点击各页面右上角的 ↗ 箭头按钮），但独立窗口存在以下问题：

1. **无用户角色权限过滤**：独立窗口显示全部11个菜单项（企微授权、通讯录、企业客户等），不区分当前登录用户的角色，任何用户都能看到管理员级别的完整菜单
2. **预览模式未移除**：独立窗口中仍显示"示例预览模式"横幅（WecomDemoBanner）
3. **WecomHeader 嵌套问题**：独立窗口内的子页面仍显示"全屏"和"独立窗口"操作按钮，可能导致嵌套打开

## 根因分析

- `WecomStandalone.vue` 中的 `menuItems` 是**硬编码**的全部菜单数组，仅通过 `tenantPackage.menuPermissions`（套餐权限）过滤，完全缺失用户角色权限过滤逻辑
- CRM 主应用通过 `DynamicMenu.vue` → `getUserAccessibleMenus()` → `hasMenuPermission()` → `config/menu.ts` 实现角色权限过滤，但独立窗口没有复用这一机制
- 认证本身没问题：同源 `localStorage` 共享 auth_token，路由守卫正常触发 `initUser()` 恢复用户会话

## 修复方案

### 1. WecomStandalone.vue - 角色权限菜单过滤

**文件**: `src/views/Wecom/WecomStandalone.vue`

- 导入 `useUserStore`、`menuConfig`、`hasMenuPermission` 
- 将硬编码的 `menuItems` 改为 `allMenuItems`（带 `configId` 字段映射到 `config/menu.ts` 中的配置）
- 新增 `visibleMenuItems` 计算属性，根据用户角色和权限动态过滤
- 管理员/超管显示全部，普通角色根据 `config/menu.ts` 中的 `roles` 和 `permissions` 配置过滤
- 函数 `hasMenuPermission` 更名为 `hasPackagePermission`，明确其为套餐权限检查（区别于角色权限）
- `onMounted` 中增加用户会话二次保障，并根据可见菜单智能设置默认激活标签
- 通过 `provide('isWecomStandalone', true)` 向子组件传递独立窗口标识

### 2. WecomDemoBanner.vue - 独立窗口隐藏预览模式

**文件**: `src/views/Wecom/components/WecomDemoBanner.vue`

- 通过 `inject('isWecomStandalone', false)` 检测独立窗口模式
- 模板条件改为 `v-if="isDemoMode && !isStandalone"`

### 3. WecomHeader.vue - 隐藏嵌套操作按钮

**文件**: `src/views/Wecom/components/WecomHeader.vue`

- 通过 `inject('isWecomStandalone', false)` 检测独立窗口模式
- 独立窗口中隐藏"全屏模式"和"独立窗口"按钮，防止嵌套打开

### 4. WecomStandaloneLayout.vue - 显示当前用户信息

**文件**: `src/layouts/WecomStandaloneLayout.vue`

- 导入 `useUserStore`，在顶栏右侧显示当前登录用户名
- 让用户确认独立窗口的认证状态正确
