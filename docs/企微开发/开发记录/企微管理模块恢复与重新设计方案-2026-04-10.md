## 四、全新设计方案 - 企微管理 V2.0
### 4.1 整体架构
#### 4.1.1 授权接入方案
采用 **企业微信第三方应用(服务商模式)** + **自建应用** 双模式:
- **SaaS多租户**: 我们作为服务商开发第三方应用，租户通过扫码授权绑定自己的企业微信主体
- **私有部署**: 客户在自己的企业微信后台创建自建应用，填入配置信息
`
                      +------------------+
                      |  企业微信开放平台  |
                      +--------+---------+
                               |
              +----------------+----------------+
              |                                 |
    +---------v---------+           +-----------v---------+
    | 第三方应用(SaaS)    |           | 自建应用(私有部署)    |
    | 服务商模式          |           | 客户自行配置          |
    | 租户扫码授权        |           | 填写corpId/secret   |
    +--------+----------+           +-----------+----------+
             |                                  |
             +----------------------------------+
                               |
                    +----------v----------+
                    |   云客CRM后端服务     |
                    |   统一API接口层      |
                    +----------+----------+
                               |
          +--------------------+--------------------+
          |                    |                    |
+---------v------+  +----------v-------+  +---------v--------+
| CRM前端        |  | Admin管理后台     |  | Website会员中心   |
| 企微管理模块    |  | 企微管理模块      |  | 增值服务开通       |
+----------------+  +------------------+  +------------------+
`
#### 4.1.2 第三方应用授权流程
`
租户操作:
1. CRM系统 -> 企微管理 -> 企微授权
2. 系统生成授权链接(含预授权码)
3. 租户企微管理员扫码
4. 选择授权范围(部门/员工)
5. 确认授权
6. 企微回调通知我们的服务器
7. 系统自动创建WecomConfig记录
8. 授权完成，租户可使用企微功能
技术流程:
[前端] -> [后端生成安装链接] -> [企微授权页] -> [管理员扫码]
-> [企微回调/callback] -> [获取永久授权码] -> [存储配置] -> [完成]
`

## 六、会话存档技术实现方案
### 6.1 企微会话存档原理
`
1. 企业在企微后台开通会话存档
2. 设置存档密钥(RSA公钥)
3. 企微服务器加密存储聊天内容
4. 我们通过API拉取加密消息
5. 用RSA私钥解密获得明文
关键技术:
- 企微Finance SDK (C++ 动态库)
- Node.js通过node-ffi调用C++ SDK
- 或使用企微新版HTTP API(推荐)
`
### 6.2 推荐方案: HTTP API方式(无需C++ SDK)
`
企微2023年后支持HTTP API方式获取会话存档:
- 接口: POST /cgi-bin/msgaudit/get_permit_user_list
- 接口: POST /cgi-bin/msgaudit/groupchat/get
- 拉取密文消息后用RSA私钥解密
优点:
- 无需编译C++ SDK
- 跨平台兼容(Linux/Windows)
- Node.js原生crypto即可解密
- 部署简单
后端实现:
1. WecomChatArchiveService.ts - 会话存档服务
2. 定时任务每5分钟拉取新消息
3. RSA解密存入wecom_chat_records
4. 敏感词匹配写入wecom_sensitive_hits
5. 媒体文件下载存储到OSS
`

## 九、关键问题回答
### Q1: 是不是我们自己开发一个企业微信应用，然后让其他租户扫码授权?
**是的，推荐方案:**
1. 我们在企业微信服务商平台注册成为服务商
2. 创建一个第三方应用(如"云客CRM助手")
3. 配置好回调URL、加解密密钥
4. 租户在CRM系统中点击"授权绑定"
5. 系统生成安装链接，租户企微管理员扫码授权
6. 授权成功后自动创建配置，即可使用
对于私有部署客户，他们可以选择自建应用模式(手动填写配置信息)。
### Q2: 增值服务价格如何管理?
- Admin管理后台新增"增值服务配置"页面
- 支持设置默认价格、阶梯定价
- 会话存档: 100元/人/年(可配置)
- 企微官方费用由租户自行承担
- 我们的费用通过会员中心收取
### Q3: 数据权限如何控制?
| 角色 | 范围 |
|------|------|
| 普通员工 | 仅看自己的聊天记录 |
| 部门经理 | 看本部门所有聊天记录 |
| 管理员/质检员/客服 | 不受限制 |
