## 一、快捷话术区优化

### 1.1 变更内容

**文件**: `src/views/Wecom/Sidebar.vue`（侧边栏页面 → 快捷话术标签）

**之前**: 使用卡片式布局（`.script-card`），每条话术占用大量垂直空间（标题行+多行内容+附件缩略图），话术多时需大量滚动，左侧分组和头部筛选区不固定。

**之后**: 改为紧凑双行列表布局（`.script-row`）：

| 行 | 内容 |
|----|------|
| 第一行 | 序号 + 标题（支持标题颜色）+ 内容单行预览 + 附件标记📎 + 编辑/删除/复制按钮 |
| 第二行 | 类型标签（公共/个人）+ 所属分组标签 + 使用次数 + 时间 |

### 1.2 滚动与冻结优化

- **左侧分组面板**：`position: sticky` + `height: 100%`，滚动话术时分组面板固定不动
- **头部筛选区**：位于 `scripts-split-layout` 之外，自然固定在顶部
- **右侧话术区**：`overflow-y: auto` 独立滚动，高度 `calc(100vh - 340px)` 自适应屏幕
- **超30条分页**：已有分页组件，支持 30/50/100 条切换 + "展开全部/收起" 功能

### 1.3 复制功能增强

`handleCopyScriptContent` 方法升级：
- 复制文字内容到剪贴板
- 如果有附件（`attachments` 字段），将附件信息一并追加复制
- 使用 `navigator.clipboard.writeText` API

### 1.4 样式变更

- **移除**: `.script-card`, `.script-card__*` 系列卡片样式
- **新增**: `.script-compact-list`, `.script-row`, `.script-row__line1`, `.script-row__line2`, `.script-row__index`, `.script-row__title`, `.script-row__content`, `.script-row__att-badge`, `.script-row__actions`, `.script-row__meta`
- **修改**: `.scripts-split-layout` 增加固定高度；`.scripts-group-panel` 增加 sticky 定位

## 二、管理后台企微总览分析

### 2.1 文件位置
- **前端**: `admin/src/views/wecom/Overview.vue`
- **API**: `GET /wecom-management/summary` (统计摘要) + `GET /wecom-management/overview` (列表)

### 2.2 统计汇总数据

总览页顶部展示 6 个统计卡片：
| 指标 | 字段 | 说明 |
|------|------|------|
| 企微配置 | `totalConfigs` | 所有企微配置数量 |
| 已连接 | `connectedConfigs` | 连接状态为 connected 的配置数 |
| 接入租户 | `tenantCount` | 关联了租户的配置数量 |
| 企微客户 | `totalCustomers` | 所有企微客户总数 |
| 会话记录 | `totalChatRecords` | 所有会话存档消息数 |
| 存档已授权 | `chatArchiveAuthCount` | 已开启会话存档的租户数 |

### 2.3 CRM端租户授权扫码后的数据流

**是的，CRM端租户授权扫码后，该租户的企微配置数据会出现在管理后台的企微总览中。** 流程如下：
1. CRM端租户扫码授权 → 创建 `WecomConfig` 记录（含 `tenantId`, `corpId`, `connectionStatus` 等）
2. 管理后台 Overview 调用 `/wecom-management/overview` → 查询所有 `WecomConfig` 记录
3. 列表展示租户名称、企业ID、连接状态、客户数、绑定数、会话记录数等

### 2.4 会话存档开关的作用

列表每行有"会话存档"开关（`chatArchiveAuth`），其作用是：
- **授权层面**: 控制该租户是否被允许使用会话存档功能
- **调用接口**: `PUT /wecom-management/toggle-archive-auth/{tenantId}` 切换 `enabled` 布尔值
- **同步效果**: 同时影响租户的 CRM 端和私有客户详情页的会话存档展示开关

**重要**: 管理后台开启会话存档授权 ≠ 租户可以使用会话存档。完整的使用条件为：
1. ✅ 管理后台授权开启（`chatArchiveAuth = true`）
2. ✅ 租户购买了会话存档套餐（通过 PurchaseOrders 代购或直接开通）
3. ✅ 购买了足够的使用人数（`usedUsers ≤ maxUsers`）
4. ✅ 套餐未过期（`expireDate > now`）

## 三、会话存档管理分析

### 3.1 文件位置
- **前端**: `admin/src/views/wecom/ChatArchiveManagement.vue`
- **API**: `GET /wecom-management/chat-archive-overview` (列表+统计)

### 3.2 功能特点

| 功能 | 说明 |
|------|------|
| 统计摘要 | 已授权租户数、总消息数、活跃租户数、总开通人数、使用中人数 |
| 配置管理 | 存储方式（数据库/OSS）、保留天数、最大人数、到期日期、服务状态 |
| OSS配置 | 支持阿里云 OSS 对象存储，配置 Bucket/前缀/Endpoint/Key |
| 数据清理 | 按保留天数清理过期数据，不可恢复 |
| 状态管理 | active(生效中) / expired(已过期) / disabled(未启用) |

### 3.3 与管理后台总览的关系

- Overview 的会话存档开关：**控制授权**（是否允许该租户使用）
- ChatArchiveManagement：**管理详细配置**（存储方式、人数上限、到期日期等）
- 两者共同决定了一个租户是否真正拥有会话存档服务

## 四、会话存档代购分析

### 4.1 文件位置
- **前端**: `admin/src/views/wecom/PurchaseOrders.vue`
- **API**: 多个接口（`getPurchaseOrders`, `fulfillPurchaseOrder`, `refundPurchaseOrder` 等）

### 4.2 功能模块

| Tab | 功能 | 说明 |
|-----|------|------|
| 代购订单 | 订单管理 | 展示订单号、租户、类型（会话存档/获客助手）、人数、售价、成本、利润、状态 |
| 采购成本 | 成本配置 | 默认折扣比例 + 阶梯折扣（不同人数区间不同折扣） |
| 供应商配置 | 供应商 | 服务商CorpID、下单API地址；支持手动履约模式 |
| 自动履约 | 自动化 | 待企微服务商开放代购API后启用 |

### 4.3 订单状态流转
```
pending(待履约) → fulfilled(已履约) → [可选] refunded(已退款)
pending(待履约) → failed(失败)
```

### 4.4 履约方式
- **自动**: 调用企微服务商API（待开放）
- **手动**: 管理员在企微服务商后台手动操作后回来标记完成

## 五、套餐与定价分析

### 5.1 文件位置
- **前端**: `admin/src/views/wecom/Pricing.vue`
- **API**: `getPricingConfig` / `savePricingConfig`

### 5.2 定价体系

| Tab | 内容 | 关键字段 |
|-----|------|---------|
| 企微套餐 | 定义不同版本的企微服务 | 套餐名称、企微数、功能范围(基础/标准/全部)、年费 |
| 会话存档定价 | 按人数区间定价 | 区间标签、官方参考价、我们售价、采购成本 → 自动算利润率 |
| AI额度定价 | AI功能套餐 | 套餐ID、名称、Token额度/调用次数、售价 |
| 支付与计费 | 支付方式配置 | 微信支付、支付宝、对公转账；AI计费单位(按次/按Token) |
| 试用与折扣 | 营销策略 | 试用天数、试用功能范围、试用AI额度、年付折扣、首购优惠、续费优惠 |
| 获客助手定价 | 待定 | 当前为免费基础功能 |

### 5.3 各模块关系图

```
套餐与定价 (Pricing)
  ├── 定义企微套餐 → 租户在会员中心购买
  ├── 定义会话存档定价 → 决定代购订单中的售价
  └── 定义AI额度定价 → CRM前端「订单与使用量」展示

会话存档代购 (PurchaseOrders)
  ├── 接收租户购买请求 → 生成代购订单
  ├── 采购成本配置 → 计算利润 = 售价 - 成本
  └── 履约管理 → 完成后同步到 ChatArchiveManagement

会话存档管理 (ChatArchiveManagement)
  ├── 管理每个租户的存档配置
  ├── 控制使用人数上限、到期日期
  └── 存储方式和数据清理

企微总览 (Overview)
  ├── 全局视角查看所有企微配置
  ├── 会话存档授权开关 → 决定租户能否使用
  └── 统计汇总数据
```

## 七、总结

1. **快捷话术UI优化**：将原来的多列表格改为简洁的双行列表，第一行展示序号+标题+内容+操作，第二行展示元信息，更加紧凑易用
2. **复制功能**：支持文字+附件一并复制到剪贴板
3. **管理后台企微总览**：正确统计汇总了企微配置、连接状态、租户数、客户数、会话记录数、存档授权数。CRM端租户扫码授权后数据确实会出现在这里
4. **会话存档完整使用条件**：管理后台授权 + 购买套餐 + 购买人数 + 未过期，四个条件缺一不可
5. **商业闭环**：Pricing定价 → 租户购买 → PurchaseOrders代购 → 履约 → ChatArchiveManagement管理配置 → Overview全局监控
