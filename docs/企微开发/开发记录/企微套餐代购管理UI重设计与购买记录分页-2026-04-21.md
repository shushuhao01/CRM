## 二、后端变更

### 文件：`backend/src/routes/admin/wecom-management.ts`

**删除旧的 TODO 存根路由**（原在约2812行）：
- `GET /purchase-orders` - TODO占位
- `GET /purchase-orders/:id` - TODO占位
- `POST /purchase-orders/:id/fulfill` - TODO占位
- `POST /purchase-orders/:id/refund` - TODO占位

- `PUT /purchase-cost` - TODO占位
- `GET /supplier-config` - TODO占位
- `PUT /supplier-config` - TODO占位

**新增真实实现路由**（约2159行）：

#### `GET /wecom-management/purchase-orders`
- 扫描所有 `tenant_billing_records_*` system_config 条目
- 聚合所有租户账单记录
- 支持 status/keyword/type 过滤
- 支持分页（默认10条/页）
- 返回 stats（pending/fulfilled/failed/monthCost/totalRevenue）

#### `POST /wecom-management/purchase-orders/:id/fulfill`
- 参数：tenantId, method(manual/auto), note
- 更新 billing record 状态为 fulfilled/active
- 会话存档类型自动激活 wecom_archive_settings

#### `POST /wecom-management/purchase-orders/:id/refund`
- 参数：tenantId, reason
- 更新 billing record 状态为 refunded

#### `GET /wecom-management/purchase-cost`
- 从 system_config key `wecom_purchase_cost` 读取
- 默认值：officialBasePrice=300, defaultDiscount=0.70, 4档阶梯

#### `PUT /wecom-management/purchase-cost`
- 保存到 system_config key `wecom_purchase_cost`

#### `GET /wecom-management/supplier-config`
- 从 system_config key `wecom_supplier_config` 读取
- 包含：providerCorpId, providerSecret, orderApiUrl, autoFulfillEnabled, prechargeBalance, notifyEmail, notifyWebhook

#### `PUT /wecom-management/supplier-config`
- 保存到 system_config key `wecom_supplier_config`

## 五、业务逻辑设计（会话存档套餐类型）

| 套餐类型 | 场景 | 流程 |
|---------|------|------|
| **代购套餐** (proxy) | 客户不自购，由我们代购企微官方 | 客户支付 → 生成代购订单 → 运营手动/自动API代购 → 上传确认函 → 激活席位 |
| **服务套餐** (service) | 客户已自购企微会话存档，购买我们的管理服务 | 客户支付服务费 → 我们直接激活管理权限 → 客户配置存档范围成员 |

---

## 六、注意事项

1. **企微代购API**：目前依赖服务商套件，建议使用手动履约，后续开发服务商应用后对接自动履约
2. **供应商配置安全**：providerSecret 字段存储在 system_config，生产环境建议加密存储
3. **预充值**：代购资金需提前充值到企微服务商账户，系统仅记录配置值供参考
