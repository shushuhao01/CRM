## 二、修复内容

### 修复 1：DataStats.vue — 租户排行表加服务端翻页

**前端变更**：
- 新增 `rankPage`（默认1）、`rankPageSize`（默认10）、`rankTotal` 状态
- 将 `fetchRankings` 抽取为独立函数，传入 `page`/`pageSize` 参数
- 新增 `el-pagination` 控件（`layout="total, sizes, prev, pager, next, jumper"`）
- 排名列正确计算跨页偏移（前3名显示奖牌 🥇🥈🥉，后续显示序号）
- 切换 `period` 时 `rankPage` 重置为 1

**后端变更**（`/data-stats/rankings`）：
- 新增 `page`、`pageSize` 参数（默认 `page=1, pageSize=10`）
- 新增 `SELECT COUNT(*) as cnt` 查询返回总数
- SQL 改为 `LIMIT ? OFFSET ?` 支持真分页
- 响应格式增加 `total`、`page`、`pageSize` 字段

### 修复 2：AuditLog.vue — 默认分页 20→10，统计准确化

**分页修复**：
```diff
- page: 1, pageSize: 20
+ page: 1, pageSize: 10

- :page-sizes="[20, 50, 100]"
+ :page-sizes="[10, 20, 50, 100]"

- layout="total, sizes, prev, pager, next"
+ layout="total, sizes, prev, pager, next, jumper"
```

**统计准确化**：
- 新增 `todayCountVal`、`actionTypeCountVal` 响应式变量
- 后端接口返回 `todayCount`（当天日志数）和 `actionTypeCount`（操作类型种类数）
- 前端优先使用后端返回值，降级到当前页估算

**后端变更**（`/audit-log`）：
- 新增 `SELECT COUNT(*) FROM ${tableName} WHERE DATE(created_at) = CURDATE()` 返回 `todayCount`
- 新增 `SELECT COUNT(DISTINCT action_type) FROM ${tableName}` 返回 `actionTypeCount`
- 响应格式增加这两个字段，兼容原有结构

## 四、系统配置说明（WecomSystemConfig 各字段用途）

| 配置节 | 字段 | 说明 |
|--------|------|------|
| `global` | `enableWecom` | 全局开关，关闭后所有企微 API 停用 |
| `global` | `maxConfigsPerTenant` | 单租户最大企微绑定数量上限 |
| `global` | `apiRateLimit / apiRateLimitWindow` | 全局 API 速率限制 |
| `chatArchive` | `defaultRetentionDays` | 新开租户默认消息保留天数（365天） |
| `chatArchive` | `syncIntervalMinutes` | 消息同步定时任务间隔 |
| `chatArchive` | `maxSyncBatchSize` | 单次拉取最大消息量（防止超时） |
| `oss` | `provider/region/bucket/...` | 会话存档媒体文件的 OSS 存储配置 |
| `notification` | `expireReminderDays` | 到期前多少天发提醒（支持多个节点） |
| `notification` | `quotaAlertThreshold` | 席位使用率告警阈值（默认90%） |
| `notification` | `alertWebhook` | 企微机器人/钉钉/飞书 Webhook 告警 |
| `rateLimit` | `perTenantMaxRequests` | 单租户 API 调用频次上限 |
