# CRM密码安全策略优化 - 首次登录强制修改与分级提醒

## 任务日期
2026-04-17（初版）、2026-04-18（提示词优化+Bug修复x4，架构修正）

## 任务概述
优化CRM系统的账号密码安全策略，实现首次登录强制修改密码（不可关闭弹窗），并建立30/60/90天分级提醒机制。

## 修改前状态（问题分析）

### 原有策略
1. **首次登录**：检测到 `forcePasswordChange = true` 或默认密码时，会弹出修改密码弹窗，但代码中将 `isForcePasswordChange` 设为 `false`，**允许用户关闭弹窗不修改密码**
2. **密码过期**：使用系统配置中的 `passwordExpireDays` 作为过期天数，固定7天提醒
3. **不再提醒**：用户可以设置"不再提醒"，在期间内跳过所有密码提醒

### 核心问题
- `App.vue` 第878行：`isForcePasswordChange.value = false` 导致强制修改变成了可选修改
- 没有分级提醒机制（30天/60天/90天）
- 过期策略依赖系统配置，未固定90天强制周期

## 修改后状态（新策略）

### 新密码安全策略
| 阶段 | 天数 | 行为 | 弹窗可关闭 |
|------|------|------|-----------|
| 首次登录 | - | 强制修改密码 | ❌ 不可关闭 |
| 默认密码 | - | 强制修改密码 | ❌ 不可关闭 |
| 第一次提醒 | 30天 | 弹窗提醒修改 | ✅ 可关闭 |
| 第二次提醒 | 60天 | 弹窗提醒修改 | ✅ 可关闭 |
| 强制修改 | 90天 | 强制修改密码 | ❌ 不可关闭 |

### 修改密码后
- 重置计时，从0天重新开始
- 可以正常使用系统

## 完整数据流（后端数据库为准）

```
用户登录（任何浏览器）
  → 后端从数据库读取 password_last_changed, need_change_password
  → 返回给前端
  → 前端存入 userStore + localStorage('user') 会话缓存

刷新页面（同一浏览器）
  → 从 localStorage('user') 恢复会话缓存
  → 检查 passwordLastChanged 判断密码状态

修改密码
  → 前端调用 PUT /profile/password
  → 后端更新数据库: password_last_changed=NOW(), need_change_password=0
  → 前端更新 userStore + localStorage('user') 会话缓存

换浏览器/重新登录
  → 后端从数据库重新读取最新的 password_last_changed
  → 前端获取最新状态，不依赖旧浏览器的 localStorage

管理员重置密码
  → 后端更新数据库: need_change_password=1, password_last_changed=NULL
  → 用户下次登录时从数据库获取，强制修改密码
```

## 修改的文件

### 1. `src/services/passwordService.ts`
- **新增** `getPasswordAgeDays(user)` - 计算密码已使用天数
- **新增** `getPasswordReminderLevel(user)` - 获取提醒级别（'first'=30天, 'second'=60天, 'force'=90天）
- **修改** `isPasswordExpired(user)` - 改为90天固定过期
- **修改** `needsPasswordReminder(user)` - 改为30-90天区间提醒
- **修改** `getPasswordRemainingDays(user)` - 改为计算距90天强制修改的剩余天数
- **清理** `changePassword()` 中移除了写入 `localStorage('users')` 的遗留 mock 逻辑

### 2. `src/components/PasswordChangeModal.vue`（2026-04-18 更新）
- **新增** `isFirstLogin` prop - 区分首次登录场景
- **优化** 弹窗标题：首次登录显示"首次登录 - 修改密码"，过期显示"密码已过期 - 修改密码"
- **优化** 提示文案：
  - 首次登录："您是首次登录系统，为了您的账户安全，请立即设置新密码后才能继续使用系统。"

  - 90天过期："您的密码已超过90天未修改，已过期。请立即修改密码后才能继续使用系统。"
  - 管理员要求："管理员要求您修改密码，请设置新密码后才能继续使用系统。"
- **强调** 所有强制场景均包含"才能继续使用系统"提示
- **修复（04-18三次）** 密码修改成功后，将更新后的用户数据（含 `passwordLastChanged`）保存到 `localStorage`，避免刷新页面后丢失导致重复弹出首次登录弹窗

### 3. `src/components/PasswordReminderModal.vue`（2026-04-18 更新）
- **优化** 30天提醒文案："您的密码已使用超过30天，距离强制修改还有X天"
- **优化** 60天提醒文案："您的密码已使用超过60天，距离强制修改仅剩X天，请尽快修改密码！"
- **新增** "X天后提醒我"功能：用户可选择7/15/30天后再次提醒，期间内不弹窗
- **新增** `remindAfterDays` 事件，通知父组件设置延迟提醒

### 4. `src/App.vue` - `checkPasswordStatus()` 方法
- **清理** 移除 `passwordService.initializeUserPasswordInfo()` 调用（该方法操作的是独立的 `localStorage('users')`，与主用户 store 无关）
- **修复（04-18二次）** 新账户首次登录时 `passwordLastChanged` 为空，导致 `isPasswordExpired` 返回 `true`，绕过首次登录判断直接走到"90天过期"分支。新增 `neverChangedPassword` 判断，将"从未修改过密码"视为首次登录而非过期
- **首次登录（从未改过密码）/默认密码/管理员强制**：`isForcePasswordChange = true`，弹窗不可关闭，标题显示"首次登录 - 修改密码"
- **修复** 首次登录时将 `isPasswordExpired` 设为 `false`，避免首次登录显示"过期"提示
- **90天过期**：`isForcePasswordChange = true`，弹窗不可关闭
- **30天/60天**：显示 `PasswordReminderModal` 提醒弹窗，可关闭
- **新增** `handleRemindAfterDays` 处理函数，将延迟时间存入 localStorage
- **新增** 30/60天提醒前检查"X天后提醒我"延迟期，未到期则跳过提醒
- **移除**了"不再提醒"期间跳过所有密码提醒的逻辑

### 5. `src/services/apiService.ts`（2026-04-18 新增）
- **修复** `handleTokenExpired()` 方法：新增登录页检测，当 `window.location.pathname === '/login'` 时，只静默清除残留 token，不弹出"登录已过期"提示
- **原因**：登录页 `onMounted` 会调用租户验证API（`verifyTenantCode`/`checkPrivateActivation`），如果 localStorage 中有残留的过期 token，请求拦截器会自动附加该 token，后端返回 401，触发 `handleTokenExpired` 弹出过期提示。在无痕浏览器或首次访问登录页时不应该出现此提示

### 6. `src/stores/user/authActions.ts`（2026-04-18 新增）

- **修复** `needChangePassword` 后端返回数字 `1/0`，使用 `!!` 转为布尔值

### 7. `backend/src/entities/User.ts`（2026-04-18 新增）

### 8. `backend/src/routes/profile.ts`（2026-04-18 新增）

### 9. `backend/add-password-security-columns.js`（2026-04-18 新增）

## 测试要点
1. 新账号首次登录 → 弹出强制修改密码弹窗，无法关闭
2. 使用默认密码登录 → 弹出强制修改密码弹窗，无法关闭
3. 密码修改后正常使用系统
4. 30天后登录 → 弹出提醒弹窗，可关闭
5. 60天后登录 → 弹出提醒弹窗，可关闭
6. 90天后登录 → 弹出强制修改密码弹窗，无法关闭
7. 超级管理员不受此策略影响
8. 修改密码后刷新页面 → 不再弹出首次登录强制修改弹窗
9. 部署前需运行 `node backend/add-password-security-columns.js` 执行数据库迁移
