# CRM APP 全面审计与修复记录

**日期**: 2026-04-17  
**范围**: crmAPP 全端审计（Token管理、首页/统计、通话、录音、WebSocket）

---

## 一、审计发现的问题与修复

### 1. Token管理（严重）

**问题**: `setLoginInfo` 使用 `setEncryptedStorage('token', ...)` 加密存储token，但 `request.ts` 使用 `uni.getStorageSync('token')` 读取的是**加密后的密文**，导致所有API请求携带错误的token。

**修复**:
- `request.ts` 改为使用 `getEncryptedStorage('token')` 读取token，与存储方式一致
- `uploadFile` 函数同步使用 `getEncryptedStorage('token')` 获取最新token

**文件**: `crmAPP/src/utils/request.ts`

### 2. uploadFile 缺少401重试机制（严重）

**问题**: `uploadFile` 函数直接使用 `userStore.token`（可能为空），且没有token过期时的重试机制，导致录音上传在token过期后必然失败。

**修复**:
- 添加从加密存储获取最新token的逻辑
- 添加401状态码检测和静默重登录重试机制

**文件**: `crmAPP/src/utils/request.ts`

### 3. 录音服务权限检查逻辑bug（中等）

**问题**: `recordingService.ts` 中 `checkPermissions()` 的 Android 版本判断顺序错误：先判断 `sdkVersion >= 30`（Android 11+），导致 `sdkVersion >= 33`（Android 13+）的分支永远不会执行。Android 13+ 设备会错误地请求 `MANAGE_EXTERNAL_STORAGE` 权限而非 `READ_MEDIA_AUDIO`。

**修复**: 调整判断顺序，先判断 `sdkVersion >= 33`，再判断 `sdkVersion >= 30`。

**文件**: `crmAPP/src/services/recordingService.ts`

### 4. 页面token状态判断不健壮（中等）

**问题**: 首页、统计页、通话记录页的 `onShow` 和数据加载函数仅检查 `userStore.token` 是否存在，但加密存储的恢复可能有延迟，导致页面误判为未登录。

**修复**: 所有页面改为双重检查 `userStore.token || userStore.isLoggedIn`，确保从后台切回时能正确加载数据。

**文件**:
- `crmAPP/src/pages/index/index.vue`
- `crmAPP/src/pages/stats/index.vue`
- `crmAPP/src/pages/calls/index.vue`

### 5. WebSocket断开重连问题（中等）

**问题**:
1. `disconnect()` 主动断开后 `onClose` 仍会触发自动重连，导致不必要的重连
2. `reconnectAttempts` 达到上限后不会重置，用户手动点击"重新连接"也无效
3. `executeDial` 中的 `onCallEnd` 回调重复清除 `currentCall`，与 `callStateService` 内部清除产生竞态

**修复**:
1. 添加 `intentionalClose` 标记，`disconnect()` 设为 `true`，`onClose` 中判断后不触发重连
2. `disconnect()` 时重置 `reconnectAttempts = 0`
3. 移除 websocket 侧的重复 `removeStorageSync('currentCall')`

**文件**: `crmAPP/src/services/websocket.ts`

### 6. logout 未完全清理存储（低）

**问题**: `logout()` 未清理 `savedUsername`、`savedPassword` 等敏感存储。

**修复**: 添加 `token_plain`、`savedUsername`、`savedPassword` 的清理。

**文件**: `crmAPP/src/stores/user.ts`

## 四、录音功能增强（二次更新）

### 7. 系统录音跳转优化

**问题**: 点击"开启系统录音"无法正确跳转到手机的通话录音设置页。

**修复**:
- `recordingService.tryEnableSystemRecording()` 返回值从 `boolean` 改为 `{ jumped, brand, guideTips }`
- 针对小米/红米添加了 `MiuiDialerSettingsActivity` 等更多精确 Intent
- 每个品牌（小米、华为、OPPO、VIVO、三星）都有**跳转成功**和**跳转失败**两套指引文案
- 指引文案包含完整的手动操作路径（如：拨号APP → 设置 → 通话录音 → 自动录音）

**文件**: `crmAPP/src/services/recordingService.ts`, `crmAPP/src/pages/settings/index.vue`

### 8. 默认开启自动上传和自动清理

**修复**: `callSettings` 默认值改为 `autoUploadRecording: true`, `autoCleanRecording: true`。首次安装或首次登录时如果没有保存过设置，会自动初始化默认值。

**文件**: `crmAPP/src/pages/settings/index.vue`, `crmAPP/src/pages/login/index.vue`

### 9. 登录成功后录音状态检测提醒

**修复**: 登录成功跳转首页后延迟2秒，自动检测系统录音是否已开启。如果未开启，弹窗提醒用户去设置页开启。同一天只提醒一次，避免打扰。

**文件**: `crmAPP/src/pages/login/index.vue`
