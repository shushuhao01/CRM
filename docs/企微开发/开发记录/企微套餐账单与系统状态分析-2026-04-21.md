## 一、新增功能：账单记录

### 后端（`pricing.ts`）

新增接口：
- `GET /wecom/billing-records` - 获取当前租户账单记录（按时间倒序，最多100条）
- 内部函数 `appendBillingRecord()` - 在每次购买/领取时自动追加记录

各购买接口均已接入账单写入：

| 接口 | 写入字段 |
|------|---------|
| `POST /wecom/claim-package` | type=wecom, amount, status=free/pending_payment |
| `POST /wecom/purchase-ai` | type=ai, calls, status=free/pending_payment |
| `POST /wecom/purchase-archive` | type=archive, maxMembers, amount, status=pending_payment |
| `POST /wecom/purchase-acquisition` | type=acquisition, billingCycle, status=free/pending_payment |

账单数据存储：`system_config.tenant_billing_records_{tenantId}`（JSON数组，最多保存200条）

### 前端

新增组件 `BillingRecordSection.vue`：
- 集成在 `PackagePurchaseTab.vue` 底部（3个Tab共用）
- 按 `type` 字段筛选显示各自的账单
- 包含说明横幅（根据type显示不同提示）
- 状态标签：免费领取 / 已支付 / 待支付 / 待履约 / 失败

## 二、套餐权限生效机制分析

### 当前已实现（✅）

| 套餐类型 | 购买后立即生效？ | 权限存储位置 |
|---------|--------------|------------|
| 企微套餐 | ✅ 免费领取立即生效 | `system_config.tenant_wecom_package_{tenantId}` |
| 企微套餐（付费版） | 支付确认后调 `/claim-package?action=purchase` 写入 | 同上 |
| 会话存档 | 需人工在企微后台履约后手动更新 `maxUsers` | `wecom_archive_settings.max_users` |
| 获客助手 | ✅ 领取/购买后记录，前端功能控制 by `dashboardEnabled` 等字段 | `system_config.tenant_billing_records_{tenantId}` |

### 未实现（❌）- 待后续迭代

- **AI额度消耗追踪**：购买后只记录了套餐，但每次AI调用未扣减额度计数
- **会话存档套餐→席位联动**：购买后需人工在「存档设置」更新 `maxUsers`，应自动关联
- **前端菜单权限读取**：`tenant_wecom_package_{tenantId}` 的 `menuPermissions` 已写入，但 CRM 端菜单路由守卫尚未读取此字段做动态控制（当前只能看到菜单但点进去无锁定提示）

## 四、会话存档代购说明（重要）

**当前状态：手动履约，无自动化**

系统在用户支付确认后：
1. 只记录账单记录（状态=pending_payment）
2. **不会自动向企微官方API提交代购订单**
3. 企微官方的代购流程需要服务商账号和特定API权限（`POST https://api.work.weixin.qq.com/cgi-bin/externalpay/create_order`），目前未对接

**实际流程（需人工介入）：**
1. 客户支付 → 我们收到款项
2. 我们运营人员手动到企微服务商后台为客户企业申请会话存档席位
3. 企微向客户企业发送确认函
4. 客户企业管理员在企微管理后台签署确认函
5. 席位生效后，我们后台手动更新 `WecomArchiveSetting.maxUsers` 和 `expireDate`

**前端已添加说明横幅**（`BillingRecordSection.vue`）明确告知用户此流程。
