# 在线席位制全流程验收文档 - 2026-04-24

## 一、修复项总结

### 本次修复的问题
| # | 问题 | 修复方案 | 影响文件 |
|---|------|---------|---------|
| 1 | "新用户无法登录"措辞不准确 | 改为"其他用户将无法登录"，不区分新老 | `User.vue` |
| 2 | 在线数超过席位上限无自动踢出 | 新增 `enforceExcessKick()` 方法，踢出最久不活跃用户 | `OnlineSeatService.ts` |
| 3 | 调整席位上限后不立即生效 | 租户更新API后调用 `enforceExcessKick()` | `admin/tenants.ts` |
| 4 | IP地址显示 `::1` 而非真实IP | 新增 `getClientIp()` 工具函数，全局替换 `req.ip` | `getClientIp.ts`, `auth.ts`, `UserController.ts`, `onlineSeat.ts` |
| 5 | 用户详情弹窗无IP字段 | 添加"最后登录IP"显示 | `User.vue` |
| 6 | 会员中心在线统计不准确 | `COUNT(DISTINCT user_id)` 替代 `COUNT(*)` | `MemberService.ts` |
| 7 | Admin详情页在线数不实时 | 详情API实时查 `user_sessions` 而非仅用缓存值 | `admin/tenants.ts` |
| 8 | `kickedTokensCache` 可能无限增长 | 超5000条自动清理 | `OnlineSeatService.ts` |
| 9 | 登出后席位不立即释放 | `destroySession()` 添加 `syncTenantOnlineCount` | `OnlineSeatService.ts` |

### 前次修复（核心Bug）
| # | 问题 | 修复方案 |
|---|------|---------|
| A | 被踢会话被auth中间件自动重建 | `kickedTokensCache` + `ensureSessionExists` 拒绝重建 |
| B | Auth不检查被踢状态 | 同步调用 `isSessionKicked()`，返回 `401 SESSION_KICKED` |

## 二、全流程测试用例

### 测试前提
- 后端重启后服务正常
- 创建测试租户，初始 `user_limit_mode = 'online'`, `max_online_seats = 1`
- 准备3个测试用户账号（admin/user1/user2）

---

### TC-01: 创建租户并设置在线席位制
**步骤**：
1. Admin后台 → 客户管理 → 创建租户
2. 设置 `用户限制模式 = 在线席位制`，`在线席位数 = 1`
3. 创建3个用户

**预期**：
- ✅ 租户创建成功，`user_limit_mode = 'online'`, `max_online_seats = 1`

### TC-02: 首个用户登录占席位
**步骤**：
1. 用户A 在浏览器1 登录
2. 登录成功后检查CRM → 用户管理页面

**预期**：
- ✅ 登录成功，进入CRM首页

### TC-04: 同一用户多标签页不重复占席位

## 三、在线统计一致性公式

所有端统一使用以下SQL获取在线人数：

```sql
SELECT COUNT(DISTINCT user_id) as cnt
FROM user_sessions
WHERE tenant_id = ? AND status = 'active'
```

| 位置 | 查询方式 | 状态 |
|------|---------|------|
| CRM 用户管理页 | `onlineSeatService.getOnlineCount()` | ✅ 已统一 |
| CRM 在线成员弹窗 | `getOnlineSessions()` 窗口函数去重 | ✅ 已统一 |
| CRM 用户列表在线状态 | `getOnlineUserIds()` | ✅ 已统一 |
| Admin 列表页 | SQL子查询 `COUNT(DISTINCT)` | ✅ 已统一 |
| Admin 详情页 | 实时查 `user_sessions` | ✅ 已修复 |
| 会员中心概览 | `MemberService.getProfileWithUsage()` | ✅ 已修复 |
| 会员中心企业信息 | 同 `MemberService` | ✅ 已修复 |
| 登录检查 | `checkLoginAllowed()` | ✅ 已统一 |
