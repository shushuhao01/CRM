**开发者**: AI Assistant  

## 一、问题描述

虚拟商品新功能开发中，多个页面的单选（Radio）组件出现"同时选中"的异常行为，表现为所有选项都被选中，无法正常互斥切换。同时多选（Checkbox）组件也存在同样的属性兼容性问题。

### 根因分析

项目使用 Element Plus `^2.3.14`，该版本 `el-radio`、`el-radio-button`、`el-checkbox` 使用 **`label`** 属性绑定选项值。而新开发的虚拟商品相关代码错误使用了 **`value`** 属性（这是 Element Plus 2.7+ 版本才引入的新写法），导致组件无法正确识别选中值，所有选项都显示为选中状态。

### 2.5 卡密库存/资源库存500错误修复

**问题**：前端访问卡密库存和资源库存页面时，后端返回500错误（`Internal Server Error`），原因是数据库中虚拟商品相关表（`card_key_inventory`、`resource_inventory`等）尚未创建。

**修复**：在 `backend/src/routes/virtualInventory.ts` 中添加自动建表中间件：
- 首次请求时自动检查并创建 `card_key_inventory`、`resource_inventory`、`virtual_delivery_records`、`virtual_claim_settings`、`tenant_email_config` 5张表
- 使用 `CREATE TABLE IF NOT EXISTS`，幂等安全
- 只在首次请求时执行，后续请求跳过检查

### 2.6 SQL迁移脚本修复

**文件**: `database/migration-virtual-product-v1.8.0.sql`

原脚本使用 `ADD COLUMN IF NOT EXISTS` 语法，这是 **MariaDB 专有语法**，MySQL 8.0 不支持，执行会报语法错误。

修复方案：使用存储过程 + `INFORMATION_SCHEMA.COLUMNS` 查询实现幂等迁移。

## 三、注意事项

1. **Element Plus 版本兼容**：当前项目锁定 `^2.3.14`，所有 `el-radio`/`el-radio-button`/`el-checkbox` 必须使用 `label` 属性而非 `value`
2. **未来升级**：若升级到 Element Plus 2.7+，需要将 `label` 改回 `value`（届时 `label` 将被弃用）
3. **SQL执行**：迁移脚本需在 MySQL 客户端或 phpMyAdmin 的 SQL 执行界面运行，支持 `DELIMITER` 语法的环境
4. **后端重启**：修改了 `virtualInventory.ts`，需要重启后端服务使自动建表逻辑生效
