### 2. 资源配额监控（QuotaMonitor）

**现状问题**：
- 用量排行表仅在后端 SQL 限制 `LIMIT 20`，当租户增多时无法看到更多。
- 前端表格无翻页控件。
- "即将到期"和"超限租户"列表也无翻页控件。
- 后端 SQL 未返回 `totalMessages` 和 `totalStorageMB` 字段（SELECT 中缺失）。

**API 映射评估**：
- 后端路由 `GET /wecom-management/quota-monitor` 已实现，查询 `wecom_archive_settings` 表（JOIN `tenants`）。
- 返回：`summary`（整体统计）、`topUsage`（排行）、`expiringSoon`（30天内到期）、`overLimit`（超限）、`alerts`（告警条）。
- 前端正确映射了所有字段，数据流无断点。

### 3. 获客助手使用量（AcquisitionUsage）

**现状问题**：
- 租户用量列表无翻页控件，大量租户时无法翻页查看。
- 排名序号在翻页后不连续（始终从 1 开始）。

**API 映射评估**：
- 后端路由 `GET /wecom-management/acquisition-usage` 已实现，查询 `wecom_acquisition_links` 和 `wecom_acquisition_customers` 表。
- 返回字段：`tenantId`、`tenantName`、`linkCount`、`monthAdded`、`monthLost`、`netGrowth`、`conversionRate`、`quota`，以及全局 `stats`。
- 前端正确调用 `getAcquisitionUsage()` API，统计卡片（本月总添加/活跃链接/活跃租户/平均转化率）正确显示。
- 趋势折线图基于返回列表派生，符合当前数据结构。
- 结论：**统计和监控逻辑正确**，仅缺翻页。

## 三、数据准确性说明

### 资源配额监控数据来源
| 指标 | 数据库表 | 字段 |
|------|---------|------|
| 活跃租户数 | `wecom_archive_settings` | `status='active'` COUNT |
| 总席位 / 已使用 | `wecom_archive_settings` | `max_users` / `used_users` |
| 总消息数 | `wecom_archive_settings` | `total_messages` |
| 总存储 | `wecom_archive_settings` | `total_storage_mb` |
| 即将到期 | `wecom_archive_settings` | `expire_date BETWEEN NOW() AND NOW()+30d` |
| 超限租户 | `wecom_archive_settings` | `used_users > max_users` |

### 获客助手用量数据来源
| 指标 | 数据库表 | 说明 |
|------|---------|------|
| 链接数 | `wecom_acquisition_links` | 按租户 COUNT |
| 本月添加 | `wecom_acquisition_customers` | `created_at` 在30天内 |
| 本月流失 | `wecom_acquisition_customers` | `status='lost'` AND 30天内 |
| 转化率 | links.click_count | 添加数/点击数×100% |

> 注意：若 `wecom_acquisition_links` 或 `wecom_acquisition_customers` 表不存在，后端会自动返回空数据，不会报错。

## 四、待优化项（P2）→ 2026-04-22 已完成

### ✅ 已完成：获客助手配额控制

**目标**：为每个租户配置获客助手链接数上限，达到上限时前端显示进度条和告警，并阻止新增。

**改动清单**：

| 文件 | 改动 |
|------|------|
| `backend/wecom-management.ts` GET `/config-quotas` | 返回数据中新增 `acquisitionLinkQuota` 字段（默认 -1 = 无限制） |
| `backend/wecom-management.ts` GET `/acquisition-usage` | 读取 `wecom_tenant_quotas` 配置，将 `acquisitionLinkQuota` 赋给每条数据的 `quota` 字段，替代原固定 `-1` |
| `admin/AcquisitionUsage.vue` | 列表"链接数"列改为"链接数/配额"，显示 `used/quota` 数值 + el-progress 进度条（绿/橙/红三色，超限显示 danger tag） |
| `admin/AcquisitionUsage.vue` | 操作列新增"配额"按钮，点击弹出 `el-dialog`，输入 `-1`（无限）或具体数值，保存后即时更新列表 |
| `admin/ConfigQuota.vue` | 企微配额设置弹窗中新增"获客链接配额"输入项，与 `acquisitionLinkQuota` 字段双向绑定，统一配额管理入口 |

**数据存储**：配额值存于 `system_config.config_value`（key: `wecom_tenant_quotas`），JSON 格式，字段为 `acquisitionLinkQuota`，与企微绑定配额（`maxCount`）共享同一存储结构，无需新建数据库表。

**进度条颜色逻辑**：
- `< 80%` → 绿色（正常）
- `80% ~ 99%` → 橙色（即满）+ warning tag
- `>= 100%` → 红色（超限）+ danger tag

### ✅ 已完成：资源配额监控定时轮询

**目标**：由纯手动刷新改为每 5 分钟自动轮询，同时显示上次刷新时间和倒计时。

**改动清单**：

| 文件 | 改动 |
|------|------|
| `admin/QuotaMonitor.vue` | 引入 `onUnmounted`，新增 `pollTimer`（setInterval 300s）和 `countdownTimer`（setInterval 1s） |
| `admin/QuotaMonitor.vue` | 新增 `countdown` ref（倒计时秒数）、`lastRefreshedAt` ref（显示上次刷新时刻） |
| `admin/QuotaMonitor.vue` | 刷新按钮区域显示「上次刷新：HH:MM:SS」+ 「X:XX 后自动刷新」倒计时 tag + 「立即刷新」按钮 |
| `admin/QuotaMonitor.vue` | `onUnmounted` 中 clearInterval 防止内存泄漏 |

**轮询策略**：
- 初始化时立即执行第一次 fetchData
- 之后每 5 分钟（300秒）触发一次自动刷新
- 点击"立即刷新"时重置倒计时为 300 秒
- 页面卸载时清理两个定时器
