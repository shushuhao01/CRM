## 四、后端API变更

### 4.2 新增API（5个）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/member/wecom/auth-info` | 授权详情：配置列表(含每个配置的客户数/绑定数)+存档设置 |

## 五、5个标签页详细说明

### 5.1 能力介绍（WecomIntro.vue）

- **顶部横幅**：企微×CRM品牌展示，渐变绿色背景

- **会话存档卡片**：席位进度条+查看范围+存储方式

## 六、Tab容器设计

### WecomService.vue 结构

```
企微服务
├── Tab导航条（圆角pill风格，绿色active）
│   ├── 📖 能力介绍
│   ├── 🔑 开通授权    [Badge: 已绑定/未绑定]
│   ├── 💰 套餐购买
│   ├── 🔄 续费/增购   [Badge: 即将到期(≤30天)]
│   └── 📊 用量/订单
└── Tab内容区（动态组件切换）
```

### 数据流

```
WecomService (Tab容器)
  ├─ 获取 wecomData (GET /member/wecom)
  ├─ 传递 props: wecomData → Auth/Pricing/Renew
  ├─ 监听 @refresh 事件 → 重新获取数据
  ├─ WecomIntro: 自行获取tierPricing展示
  └─ WecomUsage: 自行获取usage和orders数据
```

## 七、JSON规范遵循

- 所有后端API使用 `res.json()` 返回，统一 `{ code: 0, data: {...} }` 格式
- JSON字段安全解析：`try { JSON.parse() } catch {}`
- 前端使用 `await res.json()` 解析，`json.code === 0` 判断成功
- `body: JSON.stringify({...})` 序列化请求体
- 无手动字符串拼接JSON

## 八、业务规则

| 规则 | 说明 |
|------|------|
| 续费时限 | 仅在到期前30天内可续费 |
| 续费折扣 | 从system_config读取，默认0.9（9折） |
| 增购折算 | 增购价 = 增购人数 × 单价 × (剩余天数÷365) |
| 阶梯定价 | 按总人数（含增购后）确定单价档位 |
| 订单超时 | 30分钟未支付自动过期 |
| 支付轮询 | 3秒一次，5分钟后停止 |
| 预设套餐 | 优先从API获取，降级使用前端默认值 |
