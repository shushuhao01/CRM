## 二、修改清单

### 阶段1：获客趋势图动态化 ✅

**文件**: `src/views/Wecom/components/AcquisitionOverview.vue`

**问题**: 获客趋势图的demo数据使用 `Math.random()` 生成纯随机数，每次刷新都变，不是合理的动态模拟数据。

**修改内容**:
- 新增 `generateDemoSeriesData()` 函数，根据时间范围生成有规律的模拟数据：
  - **今日/昨日**: 按小时，模拟上午10点和下午3点的工作高峰
  - **本周/本月/上月**: 按天，周末略低，工作日正常，整体呈正弦波动
  - **全部**: 按月，持续增长趋势
- `generateTrendLabels()` 函数已有正确的动态X轴逻辑（今日按2小时段、昨日按小时、本周/本月/上月按天、全部按月），保持不变

### 阶段2：留存分析曲线动态化 ✅

**文件**: `src/views/Wecom/components/AcquisitionRetention.vue`

**问题**: 留存曲线固定5个点（1日/3日/7日/14日/30日），不随时间范围切换变化；留存矩阵也是固定硬编码日期。

**修改内容**:
- 新增 `generateDemoCurve()` 函数，根据时间范围动态生成留存曲线：
  - **今日**: 按2小时段，留存从100%微降
  - **昨日**: 按小时，24h留存曲线
  - **本周**: 按天，每天留存
  - **本月/上月**: 按天，逐日变化
  - **近7天/14天/30天**: 按天
  - **全部**: 按月
- 新增 `generateDemoMatrix()` 函数，根据时间范围动态生成矩阵数据（根据距今天数自动判断哪些留存周期已到期可显示）
- API fallback也使用动态函数替代静态demo数据

### 阶段4：通讯录成员状态 + 部门树异常标红 ✅

**文件**:

**修改内容**:
1. **成员列表表格** — 新增「账号状态」列，显示"正常"或"异常(封号)"标签（绿色/红色）
2. **成员详情抽屉** — 新增「账号状态」描述项，显示"✅ 正常"或"⚠ 异常(疑似封号)"

   - 部门名称文字变红
   - 显示红色小"异常"标签
4. **Demo数据增强** — `DEMO_MEMBERS` 改为按部门ID索引的对象结构，每个成员新增 `accountStatus` 字段（`normal`/`abnormal`），王五设为异常以演示

**关于企业微信API封号检测说明**:
> 企业微信API的 `GET /cgi-bin/user/get` 接口返回的成员信息中，`status` 字段含义为：
> - 1=已激活, 2=已禁用, 4=未激活, 5=退出企业
> 
> 企业微信目前**没有直接回传账号是否被微信官方封号**的API字段。但可通过以下方式间接判断：
> - 通过消息发送失败回调检测
> - 通过客户接口异常状态推断
> - 后端定期检测成员消息收发能力
> 
> 当前实现在后端API返回 `accountStatus` 字段，由后端根据多种信号综合判断。
