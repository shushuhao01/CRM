**日期**: 2026-04-22  
**范围**: 管理后台(Admin)权限系统 + CRM端套餐购买支付流程 + 企微配额管理

## 修复清单

### 1. Admin后台套餐与定价保存403权限问题 ✅

**问题**: 超级管理员在Admin后台保存"企微套餐"配置时返回403权限不足。

**根因**: 
- `adminAuthMiddleware` 将管理员信息存入 `req.adminUser`
- JWT令牌中只包含 `{ adminId, username, role, isAdmin }`，**不包含permissions字段**
- `checkPermission()` 函数从 `req.user?.permissions` 读取权限（错误的属性路径）
- 结果是 permissions 始终为空数组 `[]`，所有权限检查都失败

**修复**:
1. **`backend/src/middleware/adminAuth.ts`**: 改为异步，在JWT验证后从数据库加载管理员权限
   - `super_admin` 角色获得 `['*']` 通配符权限
   - 其他角色从 `admin_roles` 表加载权限列表
   - 权限存入 `decoded.permissions`
2. **`backend/src/routes/admin/wecom-management.ts`**: 修复 `checkPermission()` 函数
   - 从 `req.adminUser.permissions` 读取（而非 `req.user`）
   - 支持 `*` 通配符权限匹配

### 2. CRM端套餐购买未实际验证支付就显示成功 ✅

**问题**: 点击"立即购买"后弹出支付弹窗，直接点"我已完成支付"就标记为已支付并生成账单，客户未实际付款。

**根因**:
- 前端 `confirmPayment()` 同时调用 `claimWecomPackage()` 保存套餐 + `confirmWecomPayment()` 标记已付
- 后端 `confirm-payment` 接口虽有 `orderNo` 校验逻辑，但前端从不传 `orderNo`
- 导致支付校验完全被跳过

**修复**:
1. **后端 `claim-package`**: `action='purchase'` 时只创建 billing 记录（status=pending_payment），**不再立即保存租户套餐**
2. **后端 `confirm-payment`**: 
   - 生产环境：付费套餐(amount>0)且无 `orderNo` 时拒绝确认
   - 开发环境：允许直接确认但记录警告日志
   - 确认成功后才从 billing 记录的 `packageData` 恢复并保存租户套餐
3. **前端 `PackagePurchaseTab.vue`**:
   - `handlePurchase()` 先创建订单再弹出支付窗口（分离下单与确认）
   - `confirmPayment()` 仅调用确认接口
   - 开发模式下显示黄色警告提示

### 3. 套餐菜单权限更新后CRM端同步 ✅

**问题**: Admin后台修改套餐定义的菜单权限后，已购买该套餐的租户CRM端不会同步更新。

**修复**:
1. **后端 `GET /wecom/tenant-package`**: 合并两个权限源
   - 租户套餐 `menuPermissions`（购买时快照）
   - Admin配额管理 `features`（实时配置）
   - 配额管理优先级更高：只要配额管理开启了，无论套餐是否包含都生效
   - 没有套餐但配额管理有features时，构建基础包返回
2. **CRM路由守卫**: 改为始终从服务器获取最新数据，网络失败时回退到缓存

### 4. 企微配额管理租户列显示优化 ✅

**修改**: `admin/src/views/wecom/ConfigQuota.vue`
- 租户列从 "租户名称 + tenantId" 改为 "租户名称 + tenantCode"

### 5. 企微配额管理保存403权限问题 ✅

**与问题1同源**: `checkPermission()` 修复后同时解决。
- 配额管理保存调用 `checkPermission(req, res, 'wecom-management:tenants:edit')`
- 修复后超级管理员（`*`通配符）可正常保存

### 6. 配额管理逻辑：套餐权限同步+管理员额外增减 ✅

**需求**: 
- 客户买了什么套餐，配额管理自动同步对应的菜单权限
- 配额管理开启的功能，无论套餐是否包含都生效
- 管理员可额外增减配额

**实现**:
1. 新增 `syncPackageToQuota()` 函数（`backend/src/routes/wecom/pricing.ts`）
   - 将套餐 `menuPermissions` 映射为配额管理 `features`
   - 合并策略：只增不减，管理员之前额外开启的开关保留
2. 调用时机：
   - 免费领取套餐后立即同步
   - 付费套餐支付确认后同步
3. CRM端 `tenant-package` 接口返回合并后的权限，两端一致

## 权限流转图

```
Admin登录 → JWT{adminId, role, isAdmin}
    ↓
adminAuthMiddleware → 加载permissions到req.adminUser
    ↓ super_admin → ['*']
    ↓ 其他角色 → 从admin_roles表加载
    ↓
checkPermission(req, res, permCode) → 检查adminUser.permissions
```

## 套餐购买流转图

```
CRM端"立即购买" → claimWecomPackage(action='purchase')
    ↓ 创建billing记录(pending_payment) + 存储packageData
    ↓ ≠ 保存租户套餐（等待支付）
    ↓
支付弹窗 → 用户扫码支付（或开发模式直接确认）
    ↓
"我已支付" → confirmWecomPayment()
    ↓ 生产环境：验证orderNo支付状态
    ↓ 开发环境：允许但记录警告
    ↓
billing → paid → 从packageData恢复保存租户套餐 → 同步到配额管理
```
