# 在线席位制核心Bug修复 - 2026-04-24

## 问题描述

在线席位制功能存在多个核心Bug，导致以下问题：
1. **用户管理页面所有用户显示"离线"**，即使已登录
2. **"管理在线成员"弹窗显示每个用户多条重复记录**（同一用户出现4-5条）
3. **席位限制完全无效**：席位上限为1，但3个用户都能成功登录
4. **踢人无效**：管理员踢出用户后，用户立即恢复在线，席位不释放
5. **会员中心在线席位统计不准确**：使用了不一致的统计逻辑

## 根因分析

### 核心Bug：`ensureSessionExists` 自动重建被踢出的会话

`auth.ts` 中间件在每个认证请求中异步调用 `onlineSeatService.ensureSessionExists()`。该方法的设计初衷是"服务器重启后自动补建丢失的会话记录"，但存在严重缺陷：

**踢人后的执行流程（修复前）：**
1. 管理员踢出用户 → `forceKickSession` 将会话 status 改为 `'kicked'`
2. 被踢用户的下一个API请求到达 → auth中间件执行
3. `ensureSessionExists` 查询：`SELECT id FROM user_sessions WHERE session_token = ? AND status = 'active'` → 无结果（已kicked）
4. 方法认为"会话不存在" → **自动创建新会话**
5. 用户瞬间恢复在线，席位重新被占用

这导致：
- **踢人永远无效**（下一个请求自动恢复）
- **席位计数永远不减少**（被踢 → 恢复 → 被踢 → 恢复…）
- **登录限制永远不生效**（`checkLoginAllowed` 检查在线数，但数永远不减少）

### 次要Bug：`getOnlineSessions` 不按用户去重

同一用户可能有多个 `status='active'` 的会话记录（多标签页、token刷新等），查询返回所有记录而非按用户去重。

### 次要Bug：`MemberService` 在线统计逻辑不一致

会员中心使用 `COUNT(*) ... AND last_active_at > 30分钟阈值`，而其他地方使用 `COUNT(DISTINCT user_id)` 无阈值。

## 修复方案

### 1. 后端 `OnlineSeatService.ts` — 5项修复

**a) 新增 `kickedTokensCache` 内存缓存**
- 跟踪所有被踢出/过期/登出的 session token
- 防止 `ensureSessionExists` 自动重建被踢会话
- 超过5000条自动清理（DB查询兜底）

**b) 重写 `ensureSessionExists` 方法**
- 先查内存缓存 `kickedTokensCache`，命中则直接返回
- 查DB时检查所有状态（含 kicked/expired/logged_out），非 active 的不重建
- 完全无记录时（真正的首次/重启场景）先检查席位再决定是否创建

**c) 新增 `isSessionKicked` 方法**
- 供auth中间件同步调用，判断session是否已被踢出
- 优先查内存缓存，缓存未命中查DB

**d) 重写 `getOnlineSessions` 查询**
- 使用 `ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY last_active_at DESC)` 窗口函数
- 每个在线用户只返回最新活跃的一条会话记录
- total 改为 `COUNT(DISTINCT user_id)`

**e) 修复 `forceKickSession` / `forceKickUser` / `destroySession`**
- 踢出单个会话时同时踢出该用户的所有其他活跃会话（彻底释放席位）
- 所有被踢/登出的 token 加入 `kickedTokensCache`
- 踢出/登出后立即调用 `syncTenantOnlineCount` 同步租户在线数
- 新增 `syncTenantOnlineCount(tenantId)` 单租户同步方法

### 2. 后端 `auth.ts` — 同步检查被踢状态

修改前：异步调用 `ensureSessionExists`，不检查被踢状态，所有请求放行。

修改后：
- **同步调用** `isSessionKicked(sessionToken)`
- 被踢/过期/登出 → 返回 `401 SESSION_KICKED`，强制用户重新登录
- 未被踢 → 正常更新活跃时间 + 异步确保会话存在

### 3. 前端 `request.ts` — 区分被踢和token过期

401拦截器新增 `SESSION_KICKED` 错误码处理：
- 被踢出：显示"已被下线"弹窗 + 重定向登录页
- 普通过期：显示"登录已过期"弹窗（原有逻辑）

### 4. 后端 `MemberService.ts` — 统一在线统计逻辑

- `COUNT(*)` → `COUNT(DISTINCT user_id)`
- 去掉 `last_active_at > 30分钟` 阈值
- 与 `OnlineSeatService.getOnlineCount` 保持一致

## 修改文件清单

| 文件 | 修改内容 |
|------|---------|
| `backend/src/services/OnlineSeatService.ts` | 核心修复：kickedTokensCache、ensureSessionExists、isSessionKicked、getOnlineSessions去重、forceKick释放席位、destroySession同步、syncTenantOnlineCount |
| `backend/src/middleware/auth.ts` | 同步检查被踢状态，返回401 SESSION_KICKED |
| `backend/src/services/MemberService.ts` | 统一在线统计逻辑 |
| `src/utils/request.ts` | 前端401拦截器区分SESSION_KICKED |
