### 关键修改详情

**修改1: 应用ID从数字改为UUID（M3修复）**
```typescript
// 修改前（并发冲突风险）
const maxId = apps.length > 0 ? Math.max(...apps.map((a: any) => a.id || 0)) : 0;
const newApp = { id: maxId + 1, ... };

// 修改后（UUID唯一ID）
const newApp: SidebarApp = { id: uuidv4(), ... };
```

**修改2: 手机号脱敏安全处理（S3修复）**
```typescript
// 修改前（null值.replace会报错，短号码脱敏异常）
phone: crmCustomer.phone ? crmCustomer.phone.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2') : null

// 修改后（null安全+长度校验）
phone: crmCustomer.phone && crmCustomer.phone.length >= 7
  ? crmCustomer.phone.replace(/(\d{3})\d{4}(\d{4})/, '$1****$2')
  : (crmCustomer.phone ? '****' : null)
```

**修改3: products安全解析（M7修复）**
```typescript
// 修改前
products: o.products

// 修改后
products: safeJsonParse(typeof o.products === 'string' ? o.products : null, o.products || [])
```

**修改4: 动态import消除（M1修复）**
```typescript
// 修改前（每个路由内重复动态import）
const { getCurrentTenantId } = await import('../../utils/tenantContext');

// 修改后（文件顶部静态import，4处统一引用）
import { getCurrentTenantId } from '../../utils/tenantContext';
```

## 4. V2.0需求对sidebar模块的影响评估

### 4.1 sidebar在V2.0中的角色

根据需求文档V1.0/V2.0和开发方案：

| 维度 | 现状 | V2.0目标 | 差距 |
|------|------|---------|------|
| 侧边栏应用管理 | ✅ 已完成CRUD | 改造为工具栏配置+JS-SDK | 中等改造 |
| 侧边栏H5 API | ✅ 基本完成 | 增强客户画像+快捷操作 | 功能增强 |
| JS-SDK签名 | ✅ 已实现 | 支持agentConfig双重签名 | 小幅增强 |
| 双模式支持 | ❌ 仅自建 | 支持第三方应用+自建 | 核心改造 |

### 4.2 V2.0改造要点

1. **双模式Token适配**：js-sdk-config需支持第三方应用的suite_access_token模式
2. **客户画像增强**：customer-detail需集成更多CRM维度数据（跟进记录、标签、画像）
3. **快捷操作**：侧边栏增加快捷创建跟进记录、发送话术等能力
4. **工具栏配置**：从简单的应用列表改为分类工具栏（快捷操作/客户信息/素材库）

### 4.3 改造建议

sidebar.ts建议拆分为：
```
backend/src/routes/wecom/sidebar/
  ├── index.ts           (路由入口)
  ├── sidebarApps.ts     (应用管理CRUD)
  ├── sidebarAuth.ts     (账号绑定/验证)
  ├── sidebarCustomer.ts (客户详情/画像)
  └── sidebarJsSdk.ts    (JS-SDK配置)
```

**当前暂不执行此拆分**，因为254行尚可维护，待V2.0实际开发时再拆分。
