# CRM 三端问题修复记录

**日期**: 2026-04-24
**范围**: H5话术分组拖拽排序、Dashboard快捷操作、经理角色权限弹窗

---

## 问题1：H5页面快捷话术拖拽分组排序报错

**现象**: 拖拽排序视觉上成功，但后端 `PUT /wecom/scripts/categories/sort` 返回 500 错误：`Unknown column 'NaN' in 'where clause'`

**根因**: `Sidebar.vue` 的 `handleCategoryDrop()` 直接将 `scriptCategoriesList` 中的 id 映射为数组发送到后端，未过滤 `null`/`undefined`/`NaN` 等无效值。当分类列表中存在临时或未保存的项时，id 可能为无效值，导致后端 SQL 查询出错。

**修复**:
- **前端** `src/views/Wecom/Sidebar.vue` — `handleCategoryDrop()` 增加 `.filter()` 过滤无效 id（与 `SidebarPreview.vue` 的 `sDropCatManage` 保持一致），且仅在有效 id 存在时才发送请求。
- **后端** `backend/src/routes/wecom/scripts.ts` — `PUT /scripts/categories/sort` 增强校验：
  - 提前检查 `ids` 是否为有效数组
  - 使用 `Number.isFinite(id)` 替代 `isNaN` 进行更严格的数字校验
  - 仅在 `tenantId` 有值时才加入查询条件，避免 `undefined` 污染 SQL

## 问题2：CRM数据看板快捷操作卡片显示"无可用的快捷操作"

**现象**: Dashboard 快捷操作区域提示"无可用的快捷操作"，默认的新增客户、新增订单、物流查询、客户查询四个快捷入口不显示。

**根因**: `permittedQuickActions` 计算属性仅通过 `hasMenuPermission()` 匹配菜单路径来判断快捷操作是否可用。当用户权限格式（点号/冒号）与菜单配置不完全匹配时，路径匹配失败导致所有快捷操作被过滤掉。

**修复**:
- `src/views/Dashboard.vue` — 在 `permittedQuickActions` 中增加后备机制：除菜单路径匹配外，还直接检查快捷操作自身的权限码是否与用户权限匹配（支持冒号格式、点号格式及父级权限匹配），确保拥有对应权限的用户能看到快捷操作。

## 问题3：经理角色访问业绩分析/业绩分享/资料列表/绩效数据时弹出"权限不足"

**现象**: 使用经理角色登录后，进入统计业绩的业绩分析和业绩分享、资料管理的资料列表、财务管理的绩效数据页面时弹出"权限不足"提示。

**根因**:
1. `permissionHelpers.ts` 中 `isDepartmentManager` 仅识别 `department_manager`、`部门经理`、`部门负责人` 三种角色名，但后端 `requireManagerOrAdmin` 中间件同时支持 `manager` 角色。当用户角色为 `manager` 或 `经理` 时，前端 `isManager` 判断为 `false`，导致页面级权限检查失败。
2. `Analysis.vue` 的 `checkPermission()` 仅检查 `isManager` 和 `isAdmin` 角色，不检查用户实际拥有的权限码，导致拥有 `performance:analysis` 权限但角色名不匹配的用户被拒绝。
3. `authActions.ts` 中 `getRoleIdByRole` 映射缺少 `department_manager` 和 `super_admin`，导致权限刷新时获取到错误的角色 ID。

**修复**:
- `src/stores/user/permissionHelpers.ts` — `isDepartmentManager` 增加 `'manager'` 和 `'经理'` 角色名识别，与后端保持一致。
- `src/views/Performance/Analysis.vue` — `checkPermission()` 在角色检查失败时，额外检查用户是否拥有 `performance:analysis`、`performance.analysis` 或 `performance` 权限码，有则放行。
- `src/stores/user/authActions.ts` — `getRoleIdByRole` 增加 `super_admin`→`2`、`department_manager`→`3`、`sales_staff`→`4` 映射。
