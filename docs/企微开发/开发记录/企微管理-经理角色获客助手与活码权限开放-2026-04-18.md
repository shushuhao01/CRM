# 企微管理 - 经理角色获客助手与活码管理权限开放

## 日期：2026-04-18

## 任务概述

为 `department_manager`（部门经理）角色开放企微管理中的**获客助手**和**活码管理**功能，实现基于数据所有权的权限控制：
- **管理员（admin/super_admin）**：可查看、编辑、删除所有人创建的获客链接和活码
- **部门经理（department_manager）**：只能查看、编辑、删除**自己创建**的获客链接和活码

```
┌─────────────────┬──────────────────────────────────────────────┐
│     角色         │ 数据范围                                      │
├─────────────────┼──────────────────────────────────────────────┤
│ super_admin     │ 查看/编辑/删除 所有获客链接和活码               │
│ admin           │ 查看/编辑/删除 所有获客链接和活码               │
│ department_mgr  │ 查看/编辑/删除 仅限自己创建的获客链接和活码     │
│ sales_staff     │ 无权访问                                       │
└─────────────────┴──────────────────────────────────────────────┘
```

## 关键实现细节

### 数据范围过滤函数
```typescript
function isAdminRole(req: Request): boolean {
  const role = req.user?.role || '';
  return ['admin', 'super_admin', 'superadmin'].includes(role);
}

function getCreatorFilter(req: Request): string | null {
  if (isAdminRole(req)) return null; // 管理员不过滤
  return req.currentUser?.id || null;
}
```

### 所有权校验逻辑
```typescript
// 非管理员编辑/删除时校验
const creatorFilter = getCreatorFilter(req);
if (creatorFilter && record.createdBy !== creatorFilter) {
  return res.status(403).json({ success: false, message: '只能操作自己创建的数据' });
}
```
