## 一、本次完成的修复工作

### 1. WecomStandalone 独立窗口/全屏 同步更新

**问题**：点击全屏或新窗口打开时，加载的是旧版组件映射（Binding.vue、ContactBook.vue），缺少新增的 AddressBook、AiAssistant、ContactWay 等页面。

**修复内容**：
- 更新 `WecomStandalone.vue` 的 `componentMap`，移除旧的 `binding`、`contact-book`
- 新增 `address-book`（通讯录V4）、`ai-assistant`（AI助手）、`contact-way`（渠道活码）
- 菜单项与 CRM 主站路由完全一致，共 11 个功能页面：
  - 企微授权 → 通讯录 → 企业客户 → 客户群 → 获客助手 → 渠道活码 → 会话存档 → AI助手 → 微信客服 → 侧边栏 → 对外收款

### 2. 通讯录示例数据清理

**问题**：`AddressBook.vue` 中硬编码了 `demoDeptTree`、`demoDeptMembers`、`demoBindings` 等示例数据，无配置时会自动加载假数据（如「云客科技（示例）」「张三」「李四」等）。

**修复内容**：
- 删除所有硬编码示例数据（`demoDeptTree`、`demoDeptMembers`、`demoBindings`）
- 删除 `loadDemoData()` 函数及其调用
- 删除 `handleDeptClick` 中 `selectedConfigId === -1` 的 demo 分支
- 无配置时正常显示空状态，不再注入虚假数据
- 修复编码损坏的字符串（`绑�?` → `绑定`、`发�?` → `发现`）

### 3. WecomHeader 全屏/新窗口功能验证

`WecomHeader.vue` 中全屏和独立窗口功能代码正确，通过 `/wecom-standalone?tab=xxx` 打开新窗口，现已与 `WecomStandalone.vue` 的 componentMap 同步。
