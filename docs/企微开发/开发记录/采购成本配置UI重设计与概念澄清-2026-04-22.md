# 采购成本配置UI重设计与概念澄清

**日期**：2026-04-22  
**涉及模块**：管理后台 → 企微管理 → 会话存档代购 → 采购成本配置 Tab

---

## 一、业务概念澄清（重要）

### 两个完全独立的概念，不能混淆：

| 概念 | 位置 | 说明 |
|------|------|------|
| **采购成本** | 会话存档代购 → 采购成本配置 | 我方（渠道商）向企微官方下单时的成本价，用于核算利润 |
| **租户售价** | 套餐与定价 | 租户向我们购买时支付的价格，与采购成本无关 |

### 采购成本配置的正确理解：

- **企微官方价格**：
  - 服务版：**900 元 / 人 / 年**
  - 企业版：**1800 元 / 人 / 年**
  
- **渠道激励折扣**：
  - 企微官网给渠道商/服务商的激励折扣（如 0.9 折）
  - **这是企微给我们的优惠，不是给租户的折扣**
  - 租户购买的价格在「套餐与定价」中单独维护
  
- **人数阶梯折扣**：
  - 企微官方针对不同采购人数给予的阶梯优化折扣
  - 人数越多，渠道采购价越优惠
  - 同样是企微给我们的，与租户侧价格无关
  - 系统根据订单实际人数自动匹配阶梯，计算当单的采购成本

## 二、本次变更内容

### 文件：`admin/src/views/wecom/PurchaseOrders.vue`

#### 1. 采购成本配置 Tab 全面重设计

**说明 Banner 新增**：
- 在 Tab 顶部增加 `el-alert` Warning 样式横幅
- 清晰说明"这里配置的是我方向企微官方的采购成本，与租户购买套餐的售价无关"
- 列明服务版（¥900）和企业版（¥1800）区别

**左侧基础配置卡片重设计**：
- 原来只有一个"官方基准价"字段，**改为分别配置服务版和企业版**
  - `serviceBasePrice`：服务版官方价（默认 900）
  - `enterpriseBasePrice`：企业版官方价（默认 1800）
- 原来的"默认采购折扣（defaultDiscount 0.70）" → **改为渠道激励折扣（channelDiscount 0.90）**
  - 字段名更名，默认值从 0.7 改为 0.9（匹配企微实际渠道激励）
  - 说明文字明确标注"企微给渠道商的激励，非租户折扣"
  - 实时显示两个版本的折后成本价
- 使用 `el-form` + `label-position="right"` 保证 label 右对齐
- 备注输入框宽度自适应 100%

**右侧人数阶梯表格重设计**：
- 表格 `size` 从 `small` 改为 `default`，更宽松
- 去掉所有固定 `width`，全部改用 `min-width`，自适应列宽
- 表头副标题说明阶梯用途
- **折后价列拆成两列**：服务版折后价 + 企业版折后价，分别实时计算
- 阶梯默认值更新为合理的渠道激励场景（0.90/0.88/0.85/0.80）
- 删除按钮改用 `el-icon` 图标（Delete）
- 表格下方增加提示说明

#### 2. costConfig 默认值更新

```ts
// 旧
costConfig = {
  officialBasePrice: 300,
  defaultDiscount: 0.70,
  tiers: [... discount 0.75/0.70/0.65/0.60 ...]
}

// 新
costConfig = {
  serviceBasePrice: 900,      // 服务版官方价
  enterpriseBasePrice: 1800,  // 企业版官方价
  channelDiscount: 0.90,      // 渠道激励折扣
  tiers: [... discount 0.90/0.88/0.85/0.80 ...]
}
```

#### 3. 新增图标引入

```ts
import { Refresh, InfoFilled, Delete } from '@element-plus/icons-vue'
```

## 三、注意事项

1. 如果后端 `GET /wecom-management/purchase-cost` 已经返回旧的 `officialBasePrice` / `defaultDiscount` 字段，前端 `Object.assign` 会保留旧数据。需要在后端或前端做字段映射兼容处理（将旧字段迁移到新字段名）。

2. 订单利润计算逻辑（`costPrice` 字段）后续可根据订单的企微版本类型（服务版/企业版）和人数，用新的阶梯规则重新推算。

3. 采购成本配置的变更**不会影响**「套餐与定价」模块，两者完全独立。
