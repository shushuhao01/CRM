# 订单轨迹详细化与操作人审计优化 - 2026-04-19

## 任务描述

订单详情页的"订单状态和轨迹"卡片存在以下问题：
1. 订单全部业务状态流程没有全部记录（创建、审核、发货、编辑等操作未全部记录）
2. 操作人显示不规范（显示"审核员""物流部"等笼统名称，而非具体的"某部门-某人"）
3. 编辑订单等非状态变更操作没有记录
4. 前端回退生成的轨迹使用硬编码的操作人名称

## 修改范围

### 后端修改

#### 1. `backend/src/entities/OrderStatusHistory.ts`
- 新增字段 `operatorDepartment`（VARCHAR 100）- 记录操作人所属部门
- 新增字段 `actionType`（VARCHAR 50）- 记录操作类型（create/edit/submit_audit/audit_approve/audit_reject/cancel_approve/cancel_reject/status_change）
- 新增字段 `changeDetail`（TEXT）- 记录变更详情JSON

#### 2. `backend/src/routes/orders/orderHelpers.ts`
- `saveStatusHistory` 函数新增 `options` 参数，支持传入 `operatorDepartment`、`actionType`、`changeDetail`
- 新增 `getActionTypeTitle()` 函数 - 根据操作类型返回中文标题
- 扩展 `getStatusTitle()` 函数 - 补充更多状态的中文映射

#### 3. `backend/src/routes/orders/orderCrud.ts`
- 新增 `getOperatorInfo(req)` 辅助函数 - 从请求中提取操作人完整信息（部门+姓名）
- 操作人名称格式改为"某部门-某人"，方便审计追溯
- **订单创建**：记录 actionType='create'，包含部门信息
- **订单状态变更**：记录 actionType='status_change'，描述改为"状态从X变更为Y"
- **订单编辑（非状态变更）**：新增编辑操作记录，actionType='edit'，记录修改了哪些字段
- **提交审核**：记录 actionType='submit_audit'
- **审核通过/拒绝**：记录 actionType='audit_approve'/'audit_reject'
- **取消审核**：记录 actionType='cancel_approve'/'cancel_reject'
- status-history API 返回数据增加 `operatorDepartment`、`actionType`、`changeDetail` 字段

#### 4. `backend/src/routes/logistics/logisticsStatus.ts`
- 批量物流状态更新时，状态历史记录也带上部门信息和 actionType

#### 5. `backend/add-status-history-fields.js`
- 新增数据库迁移脚本，为 order_status_history 表添加3个新字段

### 前端修改

#### 1. `src/views/Order/Detail.vue`
- `loadOrderTimeline` 函数增加 `actionType` 字段传递
- `generateTimelineFromStatus` 函数移除硬编码的"审核员""物流部"等操作人名称
- `getTimelineColor` 增加 'edit' 和 'audit_rejected' 颜色映射

#### 2. `src/views/Order/Detail/OrderTrackingSections.vue`
- 时间线标题增加操作类型标签（如"编辑""审核通过""创建"等彩色标签）
- 操作人显示增加用户图标
- 新增 `getActionTagType` 和 `getActionTagLabel` 辅助函数
- 引入 `User` 图标组件

## 数据库迁移

部署前需执行迁移脚本：
```bash
cd backend
node add-status-history-fields.js
```

或者 TypeORM synchronize 会自动同步新字段（如果开启了 synchronize: true）。

## 操作类型对照表

| actionType | 中文标题 | 标签颜色 |
|---|---|---|
| create | 订单创建 | success（绿色） |
| edit | 编辑订单 | warning（橙色） |
| submit_audit | 提交审核 | info（灰色） |
| audit_approve | 审核通过 | success（绿色） |
| audit_reject | 审核拒绝 | danger（红色） |
| cancel_approve | 取消申请通过 | danger（红色） |
| cancel_reject | 取消申请拒绝 | warning（橙色） |
| status_change | 状态变更 | 根据状态显示 |

## 操作人格式

- 格式：`部门名称-操作人姓名`（如"销售部-张三"、"审核部-李四"）
- 如果无部门信息，则只显示姓名
- 从 `req.currentUser` 中取 `departmentName` + `realName`/`name`/`username`

## 测试要点

1. 创建新订单 → 查看详情 → 应有"订单创建"记录，操作人格式正确
2. 编辑订单（修改地址/金额等）→ 应有"编辑订单"记录，说明修改了哪些字段
3. 提交审核 → 应有"提交审核"记录
4. 审核通过/拒绝 → 应有对应记录，操作人为审核人
5. 取消订单 → 应有完整记录链
6. 发货/签收等状态变更 → 全部有记录
