### 发现的问题

#### 问题 1（P0）：后端 `GET /wecom/configs` 权限过严
- **位置**：`backend/src/routes/wecom.ts` 第 105 行、第 133 行
- **原因**：配置列表和单个配置查询接口使用了 `requireAdmin` 中间件，但「企业客户」「对外收款」等页面对 `sales_staff`、`department_manager` 角色可见，这些页面在 `onMounted` 时调用 `getWecomConfigs()` 获取配置下拉列表
- **影响**：非管理员用户打开这些页面时收到 403 错误
- **修复**：将 `GET /configs` 和 `GET /configs/:id` 的权限从 `requireAdmin` 放宽为仅 `authenticateToken`（数据已脱敏，只返回安全信息）

#### 问题 2（P0）：前端全局错误弹窗干扰用户
- **位置**：`src/api/wecom.ts` + `src/utils/request.ts` 响应拦截器
- **原因**：wecom 查询类 API 未设置 `showError: false`，当后端返回非200响应时，`request.ts` 拦截器自动调用 `appStore.showError()` 弹出全局错误提示
- **影响**：用户看到多个"请求失败"弹窗，体验为"加载失败"
- **修复**：在所有 wecom 查询类 API 中添加 `showError: false`，由各组件自行处理错误

#### 问题 3（P1）：缺少独立的 wecom 建表迁移脚本
- **位置**：`database/` 目录
- **原因**：database.ts 中 `synchronize: false`，且无独立迁移脚本（表定义仅在 schema.sql 中）
- **影响**：新环境部署时可能遗漏建表步骤
- **修复**：创建 `database/migration-wecom-tables.sql` 和 `backend/create-wecom-tables.js`
