## 技术实现细节

### 后端实现机制

1. **数据存储**: 使用 `system_config` 表存储免费套餐领取记录
   - AI助手: `tenant_ai_free_claimed_{tenantId}`
   - 获客助手: `tenant_acquisition_free_claimed_{tenantId}`

2. **校验逻辑**: 
   - 在套餐购买/领取时检查是否存在领取记录
   - 如果已领取,返回400错误并提示"每个租户仅可领取一次"
   - 首次领取时写入记录,包含领取时间和套餐ID

3. **账单记录**: 
   - 所有领取/购买操作都会写入 `tenant_billing_records_{tenantId}`
   - 记录包含类型(type)、状态(status)、金额(amount)等信息
   - 前端通过查询账单记录来判断是否已领取免费套餐

### 前端实现机制

1. **状态管理**: 
   - 使用 `freePackageClaimed` ref对象追踪AI和获客助手的领取状态
   - 在组件加载时通过账单记录API查询领取状态

2. **UI交互**:
   - 未领取: 显示"免费领取"按钮,可点击
   - 已领取: 显示"已领取(每个租户仅可领取一次)"按钮,禁用状态
   - 领取成功后立即更新本地状态,无需刷新页面

3. **数据流**:
   ```
   loadData() 
   → getBillingRecords() 
   → 检查账单记录中是否有免费套餐(amount=0且status=free/paid/active)
   → 更新 freePackageClaimed 状态
   → 模板根据状态显示对应按钮
   ```
