## 二、完整开发阶段总览

基于三份需求文档+代码审查，整个企微管理V2.0重构划分为 **11个开发阶段**：

| 阶段 | 名称 | 核心内容 | 涉及端 | 依赖 |
|------|------|---------|--------|------|
| **Phase 1** | 数据库基础 + 实体扩展 | 15张表(7扩展+8新增)迁移脚本+实体 | 后端 | 无(基础) |
| **Phase 2** | 双模式授权系统 | WecomTokenService+Config.vue重构+回调处理器 | 后端+CRM前端 | Phase 1 |
| **Phase 3** | 核心CRM能力 | 通讯录+客户管理增强+客户群管理(全新) | 后端+CRM前端 | Phase 2 |
| **Phase 4** | 会话存档核心重做 | 消息拉取解密+气泡UI+4个Tab | 后端+CRM前端 | Phase 2 |
| **Phase 5** | 获客 + 客服增强 | 获客权重/渠道分析+客服会话/统计/快捷回复 | 后端+CRM前端 | Phase 2 |
| **Phase 6** | 会话存档增值服务 | 席位管控+可见性权限+套餐配置化+续费增购 | 后端+CRM前端 | Phase 4 |
| **Phase 7** | CRM深度集成 | USID字段+企微客户增强+CRM客户企微卡片 | 后端+CRM前端 | Phase 1 |
| **Phase 8** | 全屏独立窗口 + 侧边栏增强 | WecomHeader+独立窗口路由+侧边栏物流售后 | CRM前端+后端 | Phase 3 |
| **Phase 9** | Admin后台企微管理 | 6个管理页面(租户/套餐/配额/订单/系统配置) | Admin+后端 | Phase 6 |
| **Phase 10** | 会员中心企微服务(SaaS) | 5个标签页(介绍/授权/购买/续费/用量) | Website+后端 | Phase 9 |
| **Phase 11** | 全链路测试 + 文档 | 单元测试+集成测试+压力测试+文档 | 全端 | 全部 |

## 四、Phase 2：双模式授权系统

### 目标
实现SaaS(第三方应用)和私有部署(自建应用)双模式授权，这是所有企微功能的核心前提。

### 4.1 后端Token管理服务

**新建** `backend/src/services/wecom/WecomTokenService.ts`

核心方法:
- `getCorpAccessToken(config)`: 根据auth_type分支获取Token
  - self_built: corpId+secret直接获取
  - third_party: suite_access_token+permanent_code换取
- `getSuiteAccessToken()`: 用suite_id+suite_secret+suite_ticket换取
- `getContactToken(configId)`: 通讯录Token
- `getExternalToken(configId)`: 客户联系Token
- `getChatArchiveToken(configId)`: 会话存档Token

### 4.2 回调事件处理器重构

**重构** `backend/src/routes/wecom/callback.ts`

事件处理器注册:
- `create_auth`: 接收永久授权码，保存到wecom_configs
- `cancel_auth`: 标记配置为已取消
- `change_external_contact`: 客户变更同步
- `change_external_chat`: 客户群变更同步
- `kf_msg_or_event`: 客服消息处理
- `customer_acquisition`: 获客助手事件
- `msgaudit_notify`: 会话存档通知

### 4.3 Config.vue双模式UI重构

**重构** `src/views/Wecom/Config.vue`

- SaaS模式: 展示授权二维码 + 扫码授权流程 + 授权状态卡片 + 权限申请引导
- 私有部署模式: 应用配置表单(CorpID/Secret/AgentID) + 回调配置 + 连接测试
- 通过 `deployConfig.isSaaS()` 条件渲染

### 4.4 验证要点

- 双模式Token获取逻辑正确
- 回调事件能正确分发到处理器
- Config.vue双模式UI切换正确
- 私有部署模式配置+测试流程完整
- SaaS模式扫码授权流程完整(需企微测试环境验证)
