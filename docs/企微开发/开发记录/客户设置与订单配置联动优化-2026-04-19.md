## 修改内容

### 1. 修复客户列表500错误（数据库迁移）

**问题原因**: 数据库缺少 `custom_fields` 列，导致 TypeORM 查询失败  
**错误日志**: `Unknown column 'customer.custom_fields' in 'field list'`

**修复方案**:
- 创建数据库迁移脚本 `backend/add-custom-fields-column.js`
- 创建SQL迁移文件 `backend/database-migrations/add-custom-fields-column.sql`
- 执行迁移为 `customers` 表添加 `custom_fields JSON NULL` 列

**涉及文件**:
- `backend/add-custom-fields-column.js` (新建)
- `backend/database-migrations/add-custom-fields-column.sql` (新建)

### 2. 客户表单字段动态控制（系统设置联动）

**问题**: 新增客户表单的字段启用/必填是硬编码的，系统设置中的客户配置无法生效

**修复方案**:
- 将 `formRules` 从静态对象改为 `computed` 动态计算属性
- 根据 `customerFieldConfigStore.isFieldRequired()` 动态设置字段必填规则
- 模板中使用 `v-if="customerFieldConfigStore.isFieldEnabled('fieldKey')"` 控制字段显隐
- 更新默认配置匹配原有硬编码行为（gender/age/height/weight/fanAcquisitionTime/medicalHistory/improvementGoals 默认必填）

**涉及文件**:
- `src/views/Customer/Add.vue` - 模板添加v-if、formRules改为computed
- `src/stores/customerFieldConfig.ts` - 默认配置更新

### 3. 自定义字段位置调整

**修改**: 将自定义字段区域从"备注信息"区块移到"基本信息"区块（收货地址上方）

**涉及文件**:
- `src/views/Customer/Add.vue` - 模板结构调整

### 4. 企微UserID全链路支持

**新增功能**: 在客户配置中添加企微UserID字段开关，支持在以下位置显示和使用：

- **客户配置**: 添加 `wecomExternalUserid` 内置字段（默认关闭）

- **后端API**:

  - 客户详情接口返回 `wecomExternalUserid`
  - 创建客户接口支持接收 `wecomExternalUserid`

**涉及文件**:
- `src/stores/customerFieldConfig.ts` - 添加wecomExternalUserid字段定义
- `src/views/Customer/Add.vue` - 添加表单字段和提交数据

### 15. 修复区县和街道必填配置不生效

**问题**: 系统设置中开启区县/街道的必填后，新增客户表单仍显示非必填，配置没有联动生效。

**根因**:
1. 模板中区县 `el-form-item` 没有 `:required` 动态绑定
2. 模板中街道 `el-form-item` 没有 `:required` 动态绑定
3. 验证规则 `district` 只有长度校验（`min:1, max:50`），没有 required 判断
4. 验证规则 `street` 只有长度校验（`min:1, max:100`），没有 required 判断

**修复**:
- 区县和街道的 `el-form-item` 添加 `:required="!customerForm.isOverseas && customerFieldConfigStore.isFieldRequired('district/street')"`
- 验证规则改为自定义 validator，读取 `customerFieldConfigStore.isFieldRequired()` 动态判断

**默认配置**: address/province/city/detailAddress 默认必填，district/street 默认非必填，但支持租户在配置中开启必填。

**涉及文件**:
- `src/views/Customer/Add.vue` - 模板:required绑定、formRules验证规则

## 第五轮修复（2026-04-19 续）

### 16. 修复自定义字段 select/radio/checkbox 类型保存后消失

**问题**: 创建下拉选择/单选/多选类型的自定义字段，保存后返回系统设置页面记录消失。

**根因分析**:
1. 选项配置中要求同时填写"选项名称"和"选项值"，但用户通常只填名称不填值，导致验证不通过字段没有真正添加
2. checkbox 类型的自定义字段在新增客户表单中未初始化为数组，导致 `el-checkbox-group` 的 v-model 报错

**修复**:
1. 选项名称输入时自动生成选项值（将名称中的空格替换为下划线作为值）
2. 选项值输入框 placeholder 改为"选项值(自动生成)"
3. 当字段类型切换为 select/radio/checkbox 时，自动添加一个空选项行
4. 新增客户表单中 checkbox 类型自定义字段自动初始化为空数组
5. 添加 `watch` 监听 `customFieldForm.fieldType` 变化

**涉及文件**:
- `src/views/Settings/CustomerSettings.vue` - autoFillOptionValue函数、watch、选项placeholder
- `src/views/Customer/Add.vue` - enabledCustomFields中初始化checkbox类型为数组
