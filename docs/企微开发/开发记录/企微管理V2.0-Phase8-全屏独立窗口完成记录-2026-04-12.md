## 四、功能详细说明

### 4.1 WecomHeader组件

- **Props**: `tabName` (string, 可选) - 用于独立窗口URL的 ?tab= 参数
- **Slots**: 
  - 默认slot: 标题文本
  - `actions` slot: 页面特定的操作按钮（如配置选择、搜索等）
- **全屏模式**: 
  - 使用 `document.documentElement.requestFullscreen()` 浏览器全屏API
  - 添加 `wecom-fullscreen` class到body，隐藏CRM侧栏/顶栏/标签页
  - 监听 `fullscreenchange` 事件自动清理
- **独立窗口**: `window.open('/wecom-standalone?tab=xxx', '_blank', 'width=1400,height=900,...')`

### 4.2 独立窗口

- **路由**: `/wecom-standalone` 使用 `WecomStandaloneLayout` 极简布局
- **布局检测**: App.vue通过 `isStandaloneLayout` 计算属性检测当前路由
- **Tab导航**: 10个Tab对应10个企微子页面，使用 `defineAsyncComponent` 懒加载
- **URL参数**: 支持 `?tab=config|binding|customer|...` 初始Tab，切换时更新URL（replaceState）
- **共享登录态**: 同源同Cookie/Token，独立窗口与主窗口共享

### 4.3 侧边栏客户详情增强

| 新增区块 | 数据来源 | 说明 |
|---------|---------|------|
| 企微USID | crmCustomer.wecomExternalUserid | 显示+一键复制（clipboard API + fallback） |
| 租户编码 | crmCustomer.tenantCode | CRM客户信息区域新增 |
| 收货信息 | shippingAddress (脱敏) | 收货人/手机(脱敏)/地址 |
| 商品明细 | order.products[] | 每个订单下展示商品名称×数量 |
| 物流信息 | order.logistics | 快递公司+单号+状态标签 |
| 售后记录 | afterSales[] | 类型+状态+原因+金额+时间 |
| 绑定信息 | bindingInfo | CRM用户名+租户编码+绑定时间 |
| 换绑按钮 | - | 确认后清除绑定，跳转登录页重新绑定 |

### 4.4 全屏模式CSS

```css
body.wecom-fullscreen .header,
body.wecom-fullscreen .sidebar,
body.wecom-fullscreen .tabs-container,
body.wecom-fullscreen .quota-warning-banner,
body.wecom-fullscreen .mobile-overlay {
  display: none !important;
}
```

使用独立的非scoped `<style>` 块确保全局生效。

## 五、JSON规范遵循

- 所有新增代码中无手动JSON字符串拼接
- 前端数据通过 `v-if` 安全守卫访问可能为null的嵌套字段
- 物流/售后/收货数据通过API响应的标准JSON结构获取
- 所有格式化函数包含null/undefined安全检查
