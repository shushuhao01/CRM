## 全流程测试验收

### 流程一：虚拟商品-无需发货型

```
新增虚拟商品（发货类型=none）
→ 下单（无需预占库存）
→ 提审 → 审核通过
→ 订单状态直接变 signed（已签收）✅
→ completionSource = audit_auto_complete ✅
→ 不进入发货列表 ✅
→ 不进入代收管理 ✅
→ 进入绩效统计（signed状态计入） ✅
→ 订单状态轨迹显示：创建→待审核→已签收 ✅
```

### 流程二：虚拟商品-卡密/资源型

```
新增虚拟商品（发货类型=card_key/resource_link）
→ 添加卡密/资源库存（CardKeyManage/ResourceManage）
→ 下单（系统预占一条库存）
→ 提审 → 审核通过
→ 订单状态变 virtual_pending（虚拟待发货）✅
→ 进入发货列表"虚拟待发货"tab ✅
→ 点击发货 → 选卡密/资源确认发货
→ 库存状态 reserved → used，写入发货记录，生成 claimToken ✅
→ 订单状态变 signed（已签收）✅
→ 订单详情显示发货记录+领取链接（可一键复制）✅
→ 客户打开 /virtual-claim/:token
  → 输入手机号+密码/验证码
  → 验证成功查看卡密/资源
  → 点击"确认领取" → 库存状态 used → claimed ✅
→ 进入绩效统计（signed状态计入）✅
→ 不进入代收管理 ✅
→ 不进入物流列表/状态更新 ✅
```

## 业务规则汇总

| 规则 | 值 |
|------|---|
| 虚拟订单最终状态 | `signed`（已签收） |
| 无需发货型完成来源 | `completionSource = audit_auto_complete` |
| 卡密发货完成来源 | `completionSource = virtual_delivery` |
| 业绩统计包含状态 | `delivered`, `signed`, `completed` |
| 代收管理排除条件 | `order_product_type != 'virtual'` |
| 物流管理排除条件 | `trackingNumber IS NOT NULL`（虚拟订单无单号） |
| 售后类型（虚拟） | 仅显示退款/重新发放（隐藏退货/换货/维修） |
