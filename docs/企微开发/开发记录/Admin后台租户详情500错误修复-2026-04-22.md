# Admin后台租户客户详情500错误修复记录

**日期**：2026-04-22  
**模块**：管理后台 Admin - 租户管理  
**问题类型**：后端 API 500 Internal Server Error

---

## 问题描述

管理后台访问租户客户详情页时，所有租户均报 500 错误：

```
Failed to load resource: the server responded with a status of 500 (Internal Server Error)
:5174/api/v1/admin/tenants/2bf03730-cef7-498d-8400-fe795e0590cb
:5174/api/v1/admin/tenants/686e1d62-87e8-482c-a1c1-65ea28ee498f
:5174/api/v1/admin/tenants/cd6a9e46-c271-40bc-b1e9-8c6c08f7998c
```

---

## 根本原因分析

通过查看 `backend/logs/error.log`，发现两个 SQL 错误依次出现：

### 错误一（较早）：`Unknown column 'p.modules' in 'field list'`

位于 `GET /:id` 租户详情接口中，查询套餐模块的 SQL：
```sql
SELECT modules, features as pkg_features FROM tenant_packages WHERE id = ?
```
**原因**：`tenant_packages` 表中不存在 `modules` 列（只有 `features` 列），导致查询失败并抛出异常。

### 错误二（当前主因）：`Illegal mix of collations`

位于租户详情主查询：
```sql
SELECT t.*, p.name as package_name, p.features as package_features_raw 
FROM tenants t
LEFT JOIN tenant_packages p ON t.package_id = CAST(p.id AS CHAR) 
WHERE t.id = ?
```
**原因**：`t.package_id` 字段排序规则为 `utf8mb4_unicode_ci`，而 `CAST(p.id AS CHAR)` 默认排序规则为 `utf8mb4_general_ci`，两者不匹配导致 MySQL 抛出字符集排序规则冲突错误。

---

## 修复方案

**文件**：`backend/src/routes/admin/tenants.ts`

### 修复1：主查询 JOIN 条件改用 COLLATE 明确字符集

```typescript
// 修复前
LEFT JOIN tenant_packages p ON t.package_id = CAST(p.id AS CHAR) WHERE t.id = ?

// 修复后
LEFT JOIN tenant_packages p ON t.package_id COLLATE utf8mb4_unicode_ci = p.id COLLATE utf8mb4_unicode_ci WHERE t.id = ?
```

### 修复2：套餐模块查询去除不存在的 `modules` 列

```typescript
// 修复前
SELECT modules, features as pkg_features FROM tenant_packages WHERE id = ?

// 修复后
SELECT features as pkg_features FROM tenant_packages WHERE id = ?
```

同时调整对应的取值逻辑（去除 `pkgRow[0].modules ||` 的 fallback）：

```typescript
// 修复前
const raw = pkgRow[0].modules || pkgRow[0].pkg_features;

// 修复后
const raw = pkgRow[0].pkg_features;
```
