# 登录密码安全策略-数据库字段修复与错误提示优化

> **日期**: 2026-04-18  
> **关联任务**: CRM密码安全策略优化  
> **状态**: ✅ 已完成

## 问题描述

在实施密码修改安全策略后，系统登录出现以下问题：

1. **登录报"数据库操作失败"**：User实体新增了 `password_last_changed` 和 `need_change_password` 字段，但数据库表缺少对应列，导致 TypeORM 查询用户时触发 `QueryFailedError`（`ER_BAD_FIELD_ERROR`）
2. **密码错误提示不友好**：密码输入错误时返回一堆报错信息，缺少剩余尝试次数提示

## 修复内容

### 1. 数据库字段补充

**本地数据库（crm_local）** — 已自动执行迁移：

```sql
ALTER TABLE users ADD COLUMN password_last_changed DATETIME NULL COMMENT '密码最后修改时间';
ALTER TABLE users ADD COLUMN need_change_password TINYINT(1) DEFAULT 1 COMMENT '是否需要修改密码: 0=否, 1=是';
```

**生产环境MySQL（phpMyAdmin可执行）**：

```sql
-- ============================================
-- 密码安全策略字段迁移 (2026-04-18)
-- 在phpMyAdmin的SQL标签页中直接执行
-- ============================================

-- 检查并添加 password_last_changed 列
ALTER TABLE `users` ADD COLUMN `password_last_changed` DATETIME NULL COMMENT '密码最后修改时间';

-- 检查并添加 need_change_password 列
ALTER TABLE `users` ADD COLUMN `need_change_password` TINYINT(1) DEFAULT 1 COMMENT '是否需要修改密码: 0=否, 1=是';

-- 将已有用户的 need_change_password 设为0（避免所有老用户被强制修改密码）
UPDATE `users` SET `need_change_password` = 0 WHERE `need_change_password` = 1;
```

> ⚠️ **注意**：如果列已存在，ALTER TABLE 会报错 `Duplicate column name`，忽略即可。也可以先查询确认：
> ```sql
> SELECT COLUMN_NAME FROM INFORMATION_SCHEMA.COLUMNS 
> WHERE TABLE_NAME = 'users' AND COLUMN_NAME IN ('password_last_changed', 'need_change_password');
> ```

### 2. schema.sql 源文件更新

在 `database/schema.sql` 的 users 表定义中，`must_change_password` 后新增了两个字段：

```sql
`password_last_changed` DATETIME NULL COMMENT '密码最后修改时间',
`need_change_password` TINYINT(1) DEFAULT 1 COMMENT '是否需要修改密码: 0=否, 1=是(新用户默认需要)',
```

### 3. 登录查询错误处理优化

**文件**: `backend/src/controllers/UserController.ts`

- 登录查询用户时增加 try-catch 包裹，数据库错误不再暴露给前端，改为友好提示："系统登录服务暂时不可用，请稍后重试或联系管理员"
- `loginFailCount` 初始化防护：`(user.loginFailCount || 0) + 1`，避免 null 值导致 NaN

### 4. 密码错误友好提示

- 密码错误时显示剩余尝试次数："用户名或密码错误，还可尝试X次，超过将锁定账户"
- 账户因错误次数过多被锁定时专门提示："密码错误次数过多，账户已被锁定，请联系管理员解锁"
- 保存失败次数的操作也加了 try-catch 防护

### 5. 全局错误处理器优化

**文件**: `backend/src/middleware/errorHandler.ts`

- 新增 `ER_BAD_FIELD_ERROR` 数据库错误类型处理，提示"系统数据结构需要升级，请联系管理员"
- 通用数据库错误提示从"数据库操作失败"改为"数据库操作失败，请稍后重试"

### 企微管理模块权限配置（2026-04-18 追加）

为销售员和经理角色添加企微管理模块的3个二级菜单权限：企微客户、客户群、会话存档。

**数据范围规则：**
- 销售员：查看个人数据
- 经理：查看部门数据
- 会话存档：默认销售员看个人、经理看部门，管理员可在存档设置中开放全部数据

**修改文件：**

| 文件 | 修改说明 |
|------|---------|
| `src/types/permissions.ts` | PERMISSION_TREE 新增 wecom 一级节点（3个二级页面） |
| `src/services/permissionService.ts` | 权限树新增 wecom 节点 + ROLE_DEFAULT_PERMISSIONS 更新 |
| `database/schema.sql` | department_manager、sales_staff 角色 JSON_ARRAY 追加企微权限 |
| `backend/migrate-wecom-permissions.js` | 本地数据库迁移脚本 |

**权限码清单：**

| 权限码 | 说明 |
|-------|------|
| `wecom` | 企微管理模块 |
| `wecom.customer` / `wecom:customer` | 企微客户页面 |
| `wecom.customer.view` / `wecom:customer:view` | 查看企微客户 |

**生产环境SQL（phpMyAdmin执行）：**

