# APP系统录音检测与设置跳转优化

**日期**: 2026-04-17  
**模块**: crmAPP - 录音设置  
**文件**: `crmAPP/src/services/recordingService.ts`, `crmAPP/src/pages/settings/index.vue`

## 第二轮修复：检测逻辑误报问题（核心修复）

### 问题现象
手机拨号服务的通话自动录音实际是**关闭**的，但APP检测显示"已开启"。

### 根因分析

**旧检测逻辑的致命缺陷**：

| 策略 | 旧逻辑 | 为什么误报 |
|------|--------|-----------|
| 策略1 | 检查预定义路径是否存在 → 存在就返回true | `/Recordings/`、`/Record/`、`/Sounds/` 在大多数安卓手机上**默认就存在**，跟录音开关无关 |
| 策略2 | 动态发现含"record"/"sound"关键词的文件夹 | 同上，`Record`、`Recordings` 是系统默认目录 |
| 策略3 | MediaStore查询 `DISPLAY_NAME LIKE '%call%'` | 太宽泛，任何名字含"call"的音频文件都会匹配 |
| 策略4 | 读取系统Settings键值 | ✅ 可靠，但排在最后 |

**结论**：策略1（目录存在就判定开启）是误报的直接原因，几乎所有安卓手机都会命中。

### 修复方案：将录音路径分为两级

**Tier 1 - 通话录音专属路径**（只有开启录音才会被创建）：
```
/MIUI/sound_recorder/call_rec/     ← 小米通话录音专属
/Sounds/CallRecord/                ← 华为通话录音专属
/Recordings/Call Recordings/       ← OPPO通话录音专属
/Record/Call/                      ← VIVO通话录音专属
/Call/                            ← 三星通话录音专属
/CallRecordings/                  ← 通用通话录音专属
/PhoneRecord/                     ← 一加/华为通话录音专属
```

**Tier 2 - 通用路径**（系统默认就存在，不能作为判断依据）：
```
/Recordings/     ← 系统默认存在❌
/Record/         ← 系统默认存在❌
/Sounds/         ← 系统默认存在❌
/Music/Recordings/ ← 系统默认存在❌
```

### 新检测策略（严格优先级）

| 优先级 | 策略 | 逻辑 | 可靠度 |
|--------|------|------|--------|
| 1 | 系统Settings键值 | 读取系统数据库中的录音开关键 | ⭐⭐⭐⭐⭐ |
| 2 | 专属目录+文件检查 | 专属路径存在 + 里面有实际音频文件 | ⭐⭐⭐⭐⭐ |
| 2b | 专属目录+时间检查 | 专属路径存在 + 近7天内创建（刚开启还没打电话） | ⭐⭐⭐⭐ |
| 3 | 通用目录+命名规则 | 通用目录中找到符合通话录音命名规则的文件 | ⭐⭐⭐⭐ |
| 4 | MediaStore精确查询 | 路径必须在通话录音专属目录下 | ⭐⭐⭐⭐ |
| 5 | 动态发现+文件验证 | 动态发现的目录中有符合命名规则的通话录音文件 | ⭐⭐⭐ |

### 通话录音文件命名规则识别

新增 `isCallRecordingFileName()` 方法，通过以下规则判断文件是否是通话录音：
- 文件名包含手机号码（1开头11位数字）
- 文件名包含 "call"（排除callback）
- 文件名包含中文"通话"或"录音"
- 文件名为 `Record_日期_时间` 格式
- 文件名为纯时间戳格式（14位以上数字）

### MediaStore查询收紧

旧查询：`DISPLAY_NAME LIKE '%call%'` → 任何含call的文件 ❌
新查询：`DATA LIKE '%/callrecord/%'` 等 → 只匹配路径在专属目录下的文件 ✅

## 第一轮修复：设置跳转问题

1. 优化APP系统自动录音检测功能，确保所有手机型号都能正确检测录音是否开启
2. 修复"开启系统录音"点击后无法跳转到设置的问题
3. 针对市面常见安卓手机品牌，先检测机型再按正确链路跳转
4. 无法自动跳转时，提供正确的手动操作路径指引

## 问题分析

### 原有问题

1. **跳转Intent错误**：
   - 华为：跳转到 `ProtectActivity`（电池优化页面），与录音无关
   - VIVO：跳转到 `BgStartUpManagerActivity`（后台启动管理），与录音无关
   - OPPO：使用的 `CallRecordSettingActivity` 组件名不存在
   - 三星：没有独立的跳转方法，使用通用方法
   - 所有Intent缺少 `FLAG_ACTIVITY_NEW_TASK` 标志

2. **手动引导路径不准确**：提示信息过于笼统，没有针对不同品牌给出正确的设置路径

3. **品牌覆盖不全**：缺少POCO、荣耀独立品牌、魅族、联想/摩托罗拉等

## 修复内容

### 1. recordingService.ts - tryEnableSystemRecording()

**重写了整个设置跳转逻辑**：

| 品牌 | 修复前跳转目标 | 修复后跳转目标 |
|------|--------------|--------------|
| 小米/红米/POCO | CallRecordingSettingsActivity（不存在） | CallRecordSettingsActivity / MiuiCallFeaturesSetting / CallSettingsActivity |
| 华为 | ProtectActivity（电池优化❌） | CallSettingsActivity（通话设置✅） |
| 荣耀 | 同华为 | hihonor.contacts.CallSettingsActivity |
| OPPO/Realme | CallRecordSettingActivity（不存在） | OppoCallFeaturesSetting / DialerSettingsActivity |
| 一加 | 同OPPO | oneplus.dialer.DialerSettingsActivity |
| VIVO/iQOO | BgStartUpManagerActivity（后台管理❌） | CallFeaturesSetting / CallSettingsActivity |
| 三星 | 无独立方法 | samsung.android.dialer.CallSettingsActivity |
| 通用 | CALL_SETTINGS | telecom.action.SHOW_CALL_SETTINGS（优先） |

### 2. 每个品牌的Intent尝试链路

每个品牌都设置了多种跳转方式，按优先级依次尝试：
1. 品牌专属通话录音设置页面
2. 品牌电话APP通话设置页面
3. `android.telecom.action.SHOW_CALL_SETTINGS`（Android 6.0+标准接口）
4. `android.settings.CALL_SETTINGS`（通用通话设置）
5. 品牌特定的备用路径

所有Intent均添加了 `FLAG_ACTIVITY_NEW_TASK (0x10000000)` 标志。

### 3. 手动引导路径（无法自动跳转时）

为每个品牌提供了准确的手动操作路径，包含：
- 显示用户手机型号（如"您的手机型号：Redmi K60"）
- 多条备选路径（路径①②③）
- 品牌专属提示（如MIUI/HyperOS路径提示）

### 4. 新增三星独立跳转方法

新增 `enableSamsungRecording()` 方法，包含三星One UI专属的Intent路径。

### 5. 新增品牌支持

- POCO（归入小米系列）
- 荣耀独立品牌（hihonor包名）
- 魅族
- 联想/摩托罗拉

## 录音检测策略（已有，未变更）

四层检测策略保持不变，确保全机型覆盖：
1. **策略1**: 检查预定义录音文件夹是否存在（40+个路径）
2. **策略2**: 动态发现录音文件夹（递归扫描关键词匹配）
3. **策略3**: MediaStore数据库查询通话录音
4. **策略4**: 读取系统Settings中的录音开关键值
