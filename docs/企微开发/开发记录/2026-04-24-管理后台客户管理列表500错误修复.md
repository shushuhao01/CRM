# 管理后台客户管理列表 500 错误修复

## 修复日期
2026-04-24

## 问题描述
管理后台「客户管理」模块下的「所有客户」和「续费提醒」两个页面无法获取数据，请求返回 500 Internal Server Error：

- `GET /api/v1/admin/customer-management/all-customers?page=1&pageSize=10` → 500
- `GET /api/v1/admin/customer-management/renewal-reminders?page=1&pageSize=10` → 500

统计卡片（stats）数据正常显示（25个客户总数），但列表数据为空。

## 根因分析

**MySQL 排序规则（Collation）不匹配导致 SQL 执行失败。**

### 详细原因

| 表名 | 排序规则 | 说明 |
|------|----------|------|
| `tenants` | `utf8mb4_unicode_ci` | 主业务表 |
| `licenses` | `utf8mb4_unicode_ci` | 主业务表 |
| `user_sessions` | `utf8mb4_0900_ai_ci` | ❌ 与其他表不一致 |
| `tenant_packages` | `utf8mb4_0900_ai_ci` | 已有 COLLATE 处理 |

`user_sessions` 表在 `schema.sql` 中创建时**缺少 `COLLATE=utf8mb4_unicode_ci`** 子句，导致 MySQL 8.0 使用默认的 `utf8mb4_0900_ai_ci`。

在客户列表查询中，在线席位子查询：
```sql
SELECT COUNT(DISTINCT us.user_id) 
FROM user_sessions us 
WHERE us.tenant_id = t.id AND us.status = 'active'
```

`us.tenant_id`（utf8mb4_0900_ai_ci）与 `t.id`（utf8mb4_unicode_ci）比较时触发：
```
Illegal mix of collations (utf8mb4_0900_ai_ci,IMPLICIT) and (utf8mb4_unicode_ci,IMPLICIT) for operation '='
```

统计卡片正常是因为 `/stats` 接口只查 `licenses` 和 `tenants` 表，不涉及 `user_sessions` 关联。

## 修复方案

### 1. 后端路由 SQL 修复（即时生效）
**文件**: `backend/src/routes/admin/customer-management.ts`

在 `all-customers` 和 `renewal-reminders` 两个接口的 `user_sessions` 子查询中添加 `COLLATE utf8mb4_unicode_ci`：

```sql
-- 修复前
WHERE us.tenant_id = t.id

-- 修复后
WHERE us.tenant_id COLLATE utf8mb4_unicode_ci = t.id
```

共修复两处（`onlineSeatsSubquery` 和 `onlineSeatsSubquery2`）。

### 2. Schema 源文件修复（防止新部署复现）
**文件**: `database/schema.sql`

`user_sessions` 表的 CREATE TABLE 语句从：
```sql
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户登录会话记录（在线席位追踪）';
```
修正为：
```sql
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='用户登录会话记录（在线席位追踪）';
```

### 3. ensureUserSessionsTable 函数修复
同步更新了 `customer-management.ts` 中 `ensureUserSessionsTable()` 函数的建表语句，添加 COLLATE 子句。

## 修改文件清单

| 文件 | 修改内容 |
|------|----------|
| `backend/src/routes/admin/customer-management.ts` | 在线席位子查询添加 COLLATE（2处）+ ensureUserSessionsTable 建表语句修复 |
| `database/schema.sql` | user_sessions 表添加 COLLATE=utf8mb4_unicode_ci |

## 验证结果

修复后直接连接本地数据库 `crm_local` 执行查询：

```
all-customers COUNT: 25 ✅ OK
all-customers DATA: 10 rows ✅ OK
renewal-reminders: 10 rows ✅ OK
```

## 经验总结

1. MySQL 8.0 默认排序规则为 `utf8mb4_0900_ai_ci`，所有表定义必须显式指定 `COLLATE=utf8mb4_unicode_ci` 与项目保持一致
2. 涉及跨表 JOIN 或子查询比较时，如果表排序规则不同，需在 SQL 中显式添加 `COLLATE` 子句
3. 此类问题的特征是：简单查询正常，复杂关联查询报 500 错误
