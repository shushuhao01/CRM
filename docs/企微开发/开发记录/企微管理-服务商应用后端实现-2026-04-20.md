## 二、新增路由 - Suite回调处理

**文件**: `backend/src/routes/wecom/suite-callback.ts`

### 2.1 回调验证 (GET /suite/callback)
- 使用SuiteConfig的Token和EncodingAESKey解密echostr
- 验证签名后返回解密结果

### 2.2 回调事件接收 (POST /suite/callback)
解密XML消息，根据InfoType分发处理：

| 事件 | 处理逻辑 |
|------|---------|
| `suite_ticket` | 更新WecomSuiteConfig.suiteTicket，清除token缓存 |
| `create_auth` | 调用get_permanent_code API → 创建/更新WecomConfig记录(authMode=third_party) |
| `cancel_auth` | 禁用对应WecomConfig，清除permanentCode |
| `change_auth` | 重新获取auth_info更新WecomConfig |
| `reset_permanent_code` | 重新获取永久授权码 |

### 2.3 授权完成重定向 (GET /suite/auth-callback)
企业扫码授权完成后，企微重定向到此地址，处理auth_code获取永久授权码，然后重定向到管理后台。

### 2.4 导出函数
- `getSuiteAccessToken()` - 获取suite_access_token（带缓存）
- `getPreAuthCode()` - 获取预授权码

## 三、管理后台API实现

**文件**: `backend/src/routes/admin/wecom-management.ts`

| 接口 | 实现逻辑 |
|------|---------|
| `GET /suite/config` | 从WecomSuiteConfig读取，密钥掩码返回 |
| `PUT /suite/config` | Upsert到WecomSuiteConfig，密钥非掩码值才更新 |
| `POST /suite/test-connection` | 读取config → 调用企微API获取suite_access_token → 返回延迟 |
| `POST /suite/auth-link` | 获取pre_auth_code → 拼接企微授权安装URL |
| `GET /suite/auths` | 查询WecomConfig WHERE authMode=third_party，分页+关联租户名 |
| `GET /suite/auths/:id` | 查询单条详情 |
| `POST /suite/auths/:id/bind-tenant` | 更新WecomConfig.tenantId |
| `DELETE /suite/auths/:id` | 软取消: isEnabled=false + 清除permanentCode |
| `GET /suite/callback-logs` | 分页查询WecomSuiteCallbackLog |

## 五、完整授权流程

```
管理员配置Suite → 配置回调URL → 企微推送suite_ticket → 
管理员生成授权链接 → 企业管理员扫码 → 
企微回调create_auth → 我们获取永久授权码 → 创建WecomConfig →
管理员关联租户 → 租户CRM激活企微功能 → 
调用企微API获取通讯录/客户/群/会话存档
```
