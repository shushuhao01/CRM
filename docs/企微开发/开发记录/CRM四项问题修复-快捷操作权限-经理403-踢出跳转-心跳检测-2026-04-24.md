# CRM 四项问题修复记录

**日期**: 2026-04-24
**范围**: Dashboard快捷操作权限越权、经理角色403报错、被踢下线跳转失败、License心跳崩溃

---

## 问题1：Dashboard快捷操作卡片未按侧边栏菜单权限过滤

**现象**: 经理、销售员、客服等角色能看到并点击"订单审核""回收站"等仅管理员可见的快捷入口。

**根因**: `permittedQuickActions` 有一个后备匹配机制，通过权限码父级匹配（如用户有 `data` 权限 → 匹配 `data:recycle`）让不该显示的快捷操作通过了过滤。

**修复**:
- `src/views/Dashboard.vue` — 移除权限码后备匹配逻辑，严格仅通过侧边栏菜单可见路径过滤快捷操作。只有 `hasMenuPermission` 判定可见的二级菜单路径才会显示对应快捷按钮。
- `src/utils/menu.ts` — `hasMenuPermission` 增加角色名规范化：将 `manager` 角色映射为 `department_manager`，确保 manager 角色与 department_manager 共享完全相同的菜单可见性。

## 问题2：经理角色访问业绩分析/业绩分享/资料列表/绩效数据时弹出"权限不足"

**现象**: 经理角色进入这4个页面时，弹出"权限不足"错误提示。控制台显示 `GET /system/departments/stats` 返回 403。

**根因**: `departmentStore.initData()` 在多个页面 `onMounted` 时被调用，其中的 `fetchDepartmentStats()` 请求 `/system/departments/stats` 接口，该接口后端使用 `requireAdmin` 中间件，只允许 `super_admin` 和 `admin` 访问。经理角色调用时返回 403，Axios 拦截器将错误信息显示为"权限不足"弹窗。

**修复**:
- `src/stores/department.ts` — `fetchDepartmentStats()` 在调用 API 前检查用户角色，仅 `super_admin` 和 `admin` 才发送请求，非管理员直接跳过（使用已有的计算属性回退数据）。

## 问题3：被踢下线后点"重新登录"不跳转到登录页

**现象**: 管理员在在线成员管理中将用户踢出后，被踢用户收到"已被下线"提示，点击"重新登录"后仍停留在当前页面，需手动刷新才能跳转到登录页。

**根因**: `request.ts` 的 `handleUnauthorized` 函数在清除 token 后使用 `safeNavigator.push('/login')`（Vue Router 软导航），但此时应用内存状态（store、组件等）已经不一致，Vue Router 的导航可能被路由守卫拦截或取消，导致跳转无效。

**修复**:
- `src/utils/request.ts` — `handleUnauthorized` 改用 `window.location.href = '/login'` 硬跳转，确保页面完全重载、清除所有内存状态。同时设置弹窗 `showClose: false` 和 `closeOnPressEscape: false`，强制用户通过"重新登录"按钮操作。

## 问题4：License心跳检测 TypeError: Cannot read properties of undefined (reading 'valid')

**现象**: 控制台报错 `[License] 心跳检测失败: TypeError: Cannot read properties of undefined (reading 'valid')`

**根因**: `checkLicenseHeartbeat()` 在某些情况下（如网络异常、响应格式异常）返回 `undefined`，`licenseHeartbeatService.ts` 第71行直接访问 `result.valid` 导致 TypeError。虽然 `checkLicenseHeartbeat` 内有 catch 返回默认值，但 Axios 拦截器可能在 catch 之前已经处理了错误并返回了 undefined。

**修复**:
- `src/services/licenseHeartbeatService.ts` — `check()` 方法增加 `!result` null 检查，并使用可选链 `result?.message` 安全访问属性。
