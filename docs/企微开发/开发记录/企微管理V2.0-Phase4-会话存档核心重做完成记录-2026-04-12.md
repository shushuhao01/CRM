## 完成内容

### 1. 后端：质检规则 + 质检记录路由 (新建)

**文件**: `backend/src/routes/wecom/qualityInspection.ts` (~305行)

| API | 方法 | 说明 |
|-----|------|------|
| `/wecom/quality-rules` | GET | 获取质检规则列表 |
| `/wecom/quality-rules/:id` | GET | 获取单个规则 |
| `/wecom/quality-rules` | POST | 创建质检规则(4种类型) |
| `/wecom/quality-rules/:id` | PUT | 更新质检规则 |
| `/wecom/quality-rules/:id` | DELETE | 删除质检规则 |
| `/wecom/quality-inspections` | GET | 获取质检记录列表(分页) |
| `/wecom/quality-inspections/run` | POST | 执行质检(手动触发) |
| `/wecom/quality-inspections/:id/review` | PUT | 人工复核 |
| `/wecom/quality-inspections/:id` | DELETE | 删除质检记录 |
| `/wecom/quality-inspections/stats` | GET | 质检统计 |

支持4种质检规则类型：
- `response_time` — 响应时长检测
- `msg_count` — 消息数量检测
- `keyword` — 关键词检测
- `emotion` — 情绪分析(预留)
