## 三、获客助手渠道活码数配额控制

**文件**: `backend/src/routes/wecom/acquisition.ts`

在 `POST /acquisition-links` 创建活码时新增配额校验：
1. 读取租户最新购买的获客助手套餐账单记录
2. 从定价配置中找到对应套餐的 `maxChannels`
3. 若 `maxChannels > 0`（非无限制），统计当前活码数
4. 超出时返回 400 + 提示"请升级套餐后继续创建"

## 四、AI额度消耗控制

**文件**: `backend/src/routes/wecom/aiAssistant.ts`

新增工具函数 `checkAndDeductAiQuota(tenantId, deduct)`:
- 读取 `TenantSettings.ai_quota`（购买套餐后累加）
- 从 `wecom_ai_logs` 统计已调用次数
- 返回 `{ allowed, remaining, quota, used }`

新增接口 `GET /ai/quota-status`：返回当前租户AI额度状态（前端用于显示剩余额度）

在 `POST /ai/scripts/:id/ai-rewrite` 接口加入配额前置检查，超限返回：
```json
{ "success": false, "code": "QUOTA_EXCEEDED", "message": "AI调用额度已用完（已用X/共Y次）..." }
```

## 五、会话存档确认函上传

**文件**: `backend/src/routes/wecom/pricing.ts`

新增两个接口：

| 接口 | 说明 |
|------|------|
| `POST /wecom/upload-archive-confirmation` | 上传已签确认函（multipart，支持PDF/图片/Word，最大10MB） |
| `GET /wecom/archive-confirmation-status` | 获取当前租户的确认函上传状态 |

上传文件存储路径：`uploads/archive-confirmations/confirmation_{tenantId}_{timestamp}.ext`

存储状态到：`system_config.tenant_archive_confirmation_{tenantId}`

**文件**: `src/views/Wecom/components/BillingRecordSection.vue`

会话存档说明横幅内新增确认函上传区：
- 显示当前上传状态（文件名、时间、"审核中"/"已激活"标签）
- "上传已签确认函"按钮（直接调用后端 multipart 接口）
- 重新上传替换旧文件

## 六、会话存档自动代购说明

当前实现：
- 用户支付后记录账单（status=pending_payment）
- 后端写入账单记录但**不自动调用企微官方代购API**
- 原因：代购API需要服务商套件 `suite_access_token` + 特定权限，待服务商套件配置完成后实现

企微官方代购相关API（待接入）：
- `POST https://api.work.weixin.qq.com/cgi-bin/externalpay/create_order` - 服务商代客户下单
- 需要：`provider_access_token`、授权企业 `auth_corp_id` 和 `permanent_code`
