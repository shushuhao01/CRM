## 一、总体目标

### 1.1 核心理念

**将所有企微模块的示例数据从前端硬编码迁移到数据库**，通过后端 API 统一管理。每个企微二级菜单的每个按钮、每个详情、每个操作都做好贴近真实的示例数据，给用户真实感受。

**切换逻辑**：
- 用户未配置企微时 → 数据库中的示例数据作为展示（带`_demo`标记的数据行）
- 用户绑定真实企微配置后 → 示例数据自动隐藏不可见，显示真实企微数据
- 用户删除企微配置后 → 示例数据自动恢复可见

## 十、数据库示例数据设计（通用方案）

### 10.1 设计方案

**方案A（推荐）：保持前端示例 + 后端智能切换**

由于项目体量和进度，建议采用更务实的方案：
1. 保持前端 `useWecomDemo.ts` 中的示例数据（已经很完整）
2. 通过 `isDemoMode` 自动判断：无真实配置时显示示例，有真实配置时隐藏
3. 所有示例数据标记 `_demo: true`，真实数据不含此标记
4. **重点优化**：丰富每个模块的示例数据、完善每个操作的交互

**方案B（完整数据库化，可后期实施）：**
1. 创建 `wecom_demo_seeds` 表
