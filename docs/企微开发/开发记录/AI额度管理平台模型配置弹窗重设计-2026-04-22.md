# AI额度管理 - 平台模型配置弹窗重设计

**日期**：2026-04-22  
**涉及模块**：管理后台 → 企微管理 → AI额度管理 → 平台模型 Tab

---

## 一、问题描述

原"添加AI模型"弹窗存在以下问题：
1. 提供商列表简陋（只有 8 个，缺少 Anthropic、Google、豆包、Kimi 等主流模型）
2. 选择提供商后 API 地址不自动填充
3. 模型 ID 是纯文本输入，没有常用模型下拉选择
4. Token 成本需手动查询各家官网，没有参考价预填
5. 弹窗内没有测试连接功能

---

## 二、本次变更

### 文件：`admin/src/views/wecom/AiManagement.vue`

#### 1. 模型目录常量 `MODEL_CATALOG`

新增完整模型目录，覆盖 **13 个提供商，共 50+ 个模型**：

| 分类 | 提供商 |
|------|--------|
| 国际顶级 | OpenAI、Anthropic (Claude)、Google (Gemini)、Meta (Llama)、Mistral AI |
| 国内顶级 | DeepSeek、阿里云通义千问、智谱AI (GLM)、字节豆包、百度文心、Kimi (Moonshot)、讯飞星火、MiniMax |
| 本地/私有 | Ollama、自定义 OpenAI 兼容接口 |

每个模型包含：
- `id`：模型 ID（直接用于 API 调用）
- `name`：友好展示名
- `inputCost`/`outputCost`：元/1K tokens（USD 模型按 7.2 汇率换算，来自各官网公开定价）
- `note`：原始官方标价备注

#### 2. 弹窗交互流程重设计

**选择提供商 → 自动填充 API 地址 + 初始化模型下拉列表**
```
onProviderChange(provider) → apiUrl 自动填入官方端点 → 自动选中该提供商第一个模型
```

**选择模型 → 自动填充展示名称和 Token 成本参考价**
```
onModelChange(modelId) → modelName 自动填 → inputCostPer1k/outputCostPer1k 自动填参考价
```

**字段说明**：
- API 地址：自动填入但可手动覆盖（方便转发代理地址）
- 模型下拉支持 `filterable` + `allow-create`（可搜索，也可自定义输入新模型 ID）
- Token 成本：自动填入参考价，可手动修改（精度 6 位）
- 展示换算：实时显示 `/1M tokens` 对应价格，方便核对

#### 3. 弹窗内测试连接

- 新增「测试连接」按钮
- **编辑模式**：直接调用后端 `POST /ai/models/:id/test` 测试已保存模型
- **新增模式**：提示先保存再从列表测试（因后端 API 需要模型 ID）
- 显示测试结果 tag（成功/失败）和延迟时间

#### 4. 新增图标引入

```ts
import { Connection } from '@element-plus/icons-vue'
```

## 四、注意事项

1. **Ollama 本地模型**：成本为 0，API 地址默认 `http://localhost:11434/v1`，生产环境需填写实际地址
2. **豆包模型 ID**：字节的模型 ID 是具体版本号（如 `doubao-1-5-pro-32k-250115`），未来版本更新后需手动维护目录
3. **弹窗测试功能**：当前仅支持编辑已保存模型时在弹窗内测试，新模型需保存后从列表测试
4. **价格来源**：均为各官网公开定价，部分模型有阶梯定价，此处取标准定价（非缓存命中价）
