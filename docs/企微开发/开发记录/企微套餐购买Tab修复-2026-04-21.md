# 企微套餐购买Tab修复记录

**日期**: 2026-04-21  
**版本**: V4.1  
**任务类型**: Bug修复

---

## 问题描述

CRM端企微管理中：
- 企微授权页（Config.vue）的「企微套餐」Tab
- 会话存档页（ChatArchive.vue）的「套餐购买」Tab
- 获客助手页（Acquisition.vue）的「套餐购买」Tab

三个购买Tab均已在前端组件中存在，但加载后显示空白，无任何套餐数据。

---

## 根因分析

`backend/src/routes/wecom/pricing.ts` 路由文件包含以下5个接口，均已完整实现：

| 接口 | 说明 |
|------|------|
| `GET /wecom/pricing-config` | 获取公开定价配置 |
| `GET /wecom/tenant-package` | 获取当前租户已购套餐 |
| `POST /wecom/claim-package` | 领取/购买企微套餐 |
| `POST /wecom/purchase-ai` | 购买AI额度套餐（含免费一次性限制） |
| `POST /wecom/purchase-archive` | 购买会话存档套餐 |
| `POST /wecom/purchase-acquisition` | 购买获客助手套餐 |

但 **`backend/src/routes/wecom/index.ts` 中漏注册了该路由**，导致所有接口返回404，前端 `PackagePurchaseTab.vue` 加载失败静默显示空白。

---

## 修复方案

**文件**: `backend/src/routes/wecom/index.ts`

### 修改内容

1. 新增导入：
```typescript
import pricingRouter from './pricing';
```

2. 新增路由注册：
```typescript
router.use('/', pricingRouter);
```

| 页面 | Tab名称 | 组件 | 类型参数 |
|------|---------|------|---------|
| `Config.vue`（企微授权） | 企微套餐 | `PackagePurchaseTab` | `type="wecom"` |
| `ChatArchive.vue`（会话存档） | 套餐购买 | `PackagePurchaseTab` | `type="archive"` |
| `Acquisition.vue`（获客助手） | 套餐购买 | `PackagePurchaseTab` | `type="acquisition"` |

---

## 数据流说明

```
管理后台「套餐与定价」保存配置
  ↓ 存入 system_config.wecom_pricing_config
  
CRM租户端 PackagePurchaseTab 加载
  ↓ GET /wecom/pricing-config 读取套餐列表
  
租户点击「免费领取」
  ↓ POST /wecom/claim-package → 存入 tenant_wecom_package_{tenantId}
  
租户点击「立即购买」
  ↓ POST /wecom/purchase-archive 等 → 弹出支付弹窗（扫码收款）
```

## V4.2 补丁（2026-04-21 后续修复）

### 问题
1. 会话存档套餐显示"免费" - 因模板用 `pkg.yearlyPrice > 0` 判断，但存档类型用 `salePrice` 字段
2. 价格显示"官方参考价" - 不应对用户展示
3. 存档套餐按钮点击后 `payAmount` 计算错误（只用 salePrice 单价，未乘以人数）
4. 支付弹窗无确认按钮，用户不知道如何完成流程

### 修复内容
**文件**: `src/views/Wecom/components/PackagePurchaseTab.vue` - 全面重写

- 新增 `getRawPrice(pkg)` 函数：按套餐类型返回正确价格字段
  - wecom → `yearlyPrice`
  - archive → `maxMembers × salePrice`（总价）
  - ai → `price`
  - acquisition → `price`
- 新增 `totalArchivePrice(pkg)` 计算存档总价
- 新增 `getPriceUnit(pkg)` 获取单位文本
- 模板中"免费"判断改为 `getRawPrice(pkg) === 0`
- 移除"官方参考价"显示
- 支付弹窗新增"我已完成支付"确认按钮，点击调用对应后端接口
- 新增 `pendingPkg` / `pendingIdx` 状态追踪待支付套餐

**文件**: `backend/src/routes/wecom/pricing.ts`

- `GET /wecom/pricing-config` 返回 `archivePricing` 时移除 `officialPrice`，只返回 `tierLabel`、`maxMembers`、`salePrice`、`description`

## 注意事项

- 管理后台必须先在「套餐与定价」页面保存套餐配置（写入 `wecom_pricing_config`），CRM端才能读取到套餐
- AI免费体验包设置了 `freeTrialOnce: true` 时，后端会记录 `tenant_ai_free_claimed_{tenantId}`，同一租户不可二次领取
- 折扣设置在 `trialConfig.yearlyDiscount` 字段，对所有套餐类型均通过前端 `computePrice()` 生效
