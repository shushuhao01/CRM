## 二、修复详情

### 2.1 修复1：Admin侧边栏企微管理菜单补全

**问题根因**：`admin/src/layouts/MainLayout.vue` 第98-107行的企微管理菜单区域，路由（router/index.ts）和权限码（permissions.ts）已在Phase 9正确配置了9个菜单项，但侧边栏模板中只写了3个旧的`<el-menu-item>`。

**修复文件**：`admin/src/layouts/MainLayout.vue`

**修复内容**：将企微管理菜单从3项扩展为9项：

| 菜单项 | 权限码 | 路由路径 | 状态 |
|--------|--------|---------|------|
| 企微概览 | wecom-management:overview:view | /wecom-management/overview | 原有 |
| 租户授权管理 | wecom-management:tenant-auth:view | /wecom-management/tenant-auth | **新增** |
| 套餐模板管理 | wecom-management:package-templates:view | /wecom-management/package-templates | **新增** |
| 租户套餐管理 | wecom-management:tenant-packages:view | /wecom-management/tenant-packages | **新增** |
| 资源配额监控 | wecom-management:quota-monitor:view | /wecom-management/quota-monitor | **新增** |
| 会话存档管理 | wecom-management:chat-archive:view | /wecom-management/chat-archive | 原有(改名) |
| 增值服务配置 | wecom-management:vas-config:view | /wecom-management/vas-config | 原有 |
| 增值服务订单 | wecom-management:vas-orders:view | /wecom-management/vas-orders | **新增** |
| 系统配置 | wecom-management:system-config:view | /wecom-management/system-config | **新增** |

**注意事项**：
- 管理员角色需要拥有对应的权限码才能在侧边栏看到菜单
- 如果登录后仍看不到新菜单，需检查后端角色权限表（`admin_roles`）中是否分配了 `wecom-management:tenant-auth:view` 等新权限码
- 超级管理员（super_admin）默认拥有全部权限，可直接看到

### 2.2 修复2：Config.vue 增加SaaS第三方应用扫码授权

**问题根因**：Config.vue 仅实现了私有部署（自建应用）模式的配置表单，缺少SaaS模式下的第三方应用扫码授权UI。

**修复文件**：`src/views/Wecom/Config.vue`

**修复内容**：

1. **通过 `getDeployMode()` 判断部署模式**：
   - `saas` 模式 → 显示扫码授权UI + 授权状态卡片 + 手动添加自建应用入口
   - `private` 模式 → 保持原有的自建应用配置表格

2. **SaaS模式新增功能**：
   - 已授权企业以卡片形式展示（含连接状态、授权类型、操作按钮）
   - "扫码授权接入"按钮 → 打开扫码授权引导弹窗
   - 扫码授权三步流程：生成二维码 → 等待确认 → 授权完成
   - 授权完成后自动刷新企业列表
   - 底部保留"手动添加自建应用"入口（混合模式支持）

3. **授权二维码生成**：
   - 调用 `GET /api/v1/wecom/callback/auth-url` 获取企微授权链接
   - 使用 QR API 生成二维码图片展示
   - 未配置服务商应用时降级为演示模式

4. **新增导入**：
   - `Connection`, `Picture` 图标
   - `getDeployMode` 部署模式判断函数

## 四、经验教训

1. **侧边栏菜单需与路由同步更新**：Phase 9中路由（router/index.ts）和权限码（permissions.ts）都正确新增了，但遗漏了MainLayout.vue的侧边栏模板。后续开发中，每增加路由必须同步检查侧边栏菜单。

2. **双模式渲染不能遗漏**：Phase 2的设计文档明确要求Config.vue需要双模式（SaaS扫码 + 私有自建），但实现时只做了私有模式。后续需严格对照设计文档逐项核对。

3. **UI需求需确认**：独立窗口的Tab vs 左侧菜单是一个UX偏好，应提前与用户确认。
