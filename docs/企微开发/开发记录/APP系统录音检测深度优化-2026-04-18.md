# APP系统录音检测深度优化

**日期**: 2026-04-18  
**模块**: crmAPP - 录音设置  
**文件**: `crmAPP/src/services/recordingService.ts`

---

## 深度分析结论

对04-17的修复进行全面复审，发现以下问题：

### 问题1（关键Bug）：子目录递归扫描缺失

**现象**：小米手机开启录音后，检测仍显示"未开启"。

**根因**：小米将通话录音按年月存储在子目录中：
```
/MIUI/sound_recorder/call_rec/2026/04/录音文件.mp3
```
但 `listAudioFiles()` 只扫描一级目录，无法发现子目录中的文件。

**影响品牌**：小米/红米/POCO（确认），其他品牌也可能存在类似子目录结构。

### 问题2：死代码残留

- `safeStartActivity()` 方法：已定义但从未被调用
- `checkMediaStoreForCallRecordings()`：旧的宽泛MediaStore查询，已被 `checkMediaStoreForCallRecordingsStrict()` 替代但未删除

### 问题3：品牌覆盖不足

缺少以下品牌的专属处理：
- Google Pixel（全球重要品牌，使用Google Dialer）
- Nothing Phone（新兴品牌，使用Google Dialer）
- 传音系 TECNO/Infinix/itel（非洲/东南亚市场主流）

### 问题4：系统Settings检测键值不全

缺少三星、Google、OPPO扩展键名、华为扩展键名。

### 问题5：文件命名规则识别不够精确

- `call` 关键词匹配未排除 `recall` 等词
- 未覆盖 Google Dialer 的 `Recording_电话号码` 格式

---

## 修复内容

### 1. listAudioFiles 递归扫描（核心修复）

**修复前**：只扫描目标目录下的直接文件
**修复后**：递归扫描最多2层子目录

```typescript
// 修复前
private async listAudioFiles(dirPath: string): Promise<RecordingFile[]> {
  const allFiles = await this.listFiles(dirPath)
  return allFiles.filter(f => AUDIO_EXTENSIONS.includes(ext))
}

// 修复后
private async listAudioFiles(dirPath: string, maxDepth: number = 2): Promise<RecordingFile[]> {
  // 扫描当前目录的音频文件
  // + 递归扫描子目录（最多2层深度）
  // 解决小米 call_rec/2026/04/ 等按日期组织的目录结构
}
```

### 2. 删除死代码

- 删除 `safeStartActivity()` 方法
- 删除旧的 `checkMediaStoreForCallRecordings()` 方法（保留严格版 `checkMediaStoreForCallRecordingsStrict()`）

### 3. 新增品牌支持

| 品牌 | 跳转方式 | 专属路径 |
|------|---------|---------|
| Google Pixel | GoogleDialerSettingsActivity / DialerSettingsActivity | `/Recordings/Call recordings/` |
| Nothing Phone | 同Google Dialer | `/Recordings/Call Recordings/` |
| TECNO/Infinix/itel | 通用跳转 | `/Record/Call/`, `/CallRecorder/` |

新增 `enableGoogleDialerRecording()` 方法，包含Google Dialer专属Intent路径。

### 4. 系统Settings检测键值扩充

新增键值：
- `hw_callrecord_auto`（华为扩展）
- `miui_call_recording`（小米扩展）
- `oppo_auto_call_recording`, `oplus_call_recording`（OPPO/一加扩展）
- `vivo_auto_record`（VIVO扩展）
- `samsung_call_recording`, `call_recording_mode`（三星）
- `google_call_recording`, `dialer_call_recording_enabled`（Google）
- `persist.sys.call.recording`（通用Android属性）

### 5. 文件命名规则增强

- 增加 `call_rec` 精确匹配（正则 `\bcall[_\s-]?rec`）
- 排除 `recall` 误匹配
- 新增 Google Dialer `Recording_` 前缀 + 数字的匹配规则

### 6. 专属路径扩充

新增路径：
```
/storage/emulated/0/Recordings/Call recordings/    ← Google Pixel
/storage/emulated/0/Recorder/Call/                 ← 魅族
/storage/emulated/0/CallRecorder/                  ← 传音系
```
