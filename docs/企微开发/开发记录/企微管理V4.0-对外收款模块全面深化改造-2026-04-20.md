# 企微管理V4.0 - 对外收款模块全面深化改造记录

**日期**: 2026-04-20  
**版本**: V4.0  
**开发人员**: AI Assistant  

---

## 一、任务概述

对企微管理的「对外收款」模块进行全面深化改造，涵盖4个Tab页（收款记录、收款统计、退款统计、收款设置），从硬编码演示模式升级为真实API数据驱动，对接企业微信官方对外收款API。

> **重要决策**：
> - 企微官方API不支持发起退款，因此「退款管理」改为「退款统计」，仅展示退款记录，不提供发起退款操作
> - 企微收款码由成员在客户端生成，API不支持创建收款码，因此移除「收款码」Tab

## 二、企微对外收款API分析

### 2.1 企微官方API接口

| API | 说明 | 对接状态 |
|-----|------|----------|
| `externalpay/get_bill_list` | 获取对外收款账单列表 | ✅ 已对接 |
| `externalpay/get_bill_detail` | 获取账单详情 | 🔲 可扩展 |
| 退款API | 企微官方暂未开放退款API | ⚠️ 系统内记录管理 |
| 收款码生成 | 收款码由成员在企微客户端生成 | ⚠️ 系统管理统计 |

### 2.2 权限说明
- 企业需在企微管理后台开通「对外收款」应用
- 第三方应用需要获得「对外收款」权限授权
- Secret需在企微管理后台「应用管理 → 对外收款」中获取
- **注意**：对外收款是企微内置应用，非第三方应用自动拥有，需企业管理员手动授权

## 三、改造内容详情

### 3.1 Tab 1: 收款记录

#### 筛选器优化
- **放大筛选栏**：独立filter-bar卡片，padding加大，更醒目
- **搜索支持多字段**：单号（paymentNo/tradeNo）、客户昵称、备注关键词
- **日期选择器独立分开**：开始日期和结束日期分为两个独立DatePicker，不再使用daterange占据过长空间
- **收款人支持级联选择**：下拉Cascader支持按部门→成员两级筛选
- **收款状态**：支持待支付/已支付/已退款/已取消四种状态筛选
- **重置按钮**：一键清空所有筛选条件

#### 表格字段丰富
| 新增字段 | 说明 |
|---------|------|
| tradeNo | 微信交易单号 |
| departmentName | 收款人所属部门 |
| payMethod | 支付方式 |
| refundAmount | 已退款金额 |
| remark | 备注 |
| currency | 币种 |

#### 汇总卡片
- 改为后端返回真实统计数据（summary字段）
- 新增第5张卡片：退款金额
- 数据跟随筛选条件动态变化

#### 分页
- 后端分页加载（page/pageSize）
- 支持20/50/100条切换
- 显示总记录数 + 跳页

### 3.2 Tab 2: 收款统计

#### 快捷日期
- 改为：**今日、昨日、本周、本月、上月、全部**
- 支持自定义起止日期筛选
- 筛选器尺寸放大（size="default"）

#### 汇总卡片
- 对接后端 `/payments/stats` 真实API数据
- 包含：收款总额（含环比变化）、收款笔数、退款金额、平均客单价
- 环比计算逻辑：对比前一个同等时间段

#### 收款趋势图
- 数据来源：后端按日聚合的trend数组
- 支持切换：收款额/笔数/退款三种视图
- 使用ECharts TrendLineChart组件渲染

#### 成员收款排行
- 默认显示5人，支持翻页加载剩余
- 前端分页（rankPage/rankPageSize）
- 显示：排名、成员名、笔数、收款金额、退款笔数、净收款

#### 状态分布
- 真实数据：已支付/待支付/已退款/已取消百分比
- 显示具体数量

### 3.3 Tab 3: 退款统计

#### 说明
- 企微官方API**不支持**通过接口发起退款，退款需在企微管理后台操作
- 本Tab仅展示从企微同步的退款记录数据，不提供发起退款和审批操作

#### 筛选器
- 放大独立filter-bar卡片
- 支持：退款单号/原单号/付款人搜索、退款状态、起止日期

#### 列表
- 后端分页，默认10条/页
- 字段：退款单号、原单号、付款人、原金额、退款金额、原因、申请时间、完成时间、状态
- 翻页控件
- 仅提供「详情」查看操作

### 3.4 Tab 4: 收款设置（原Tab 5）

> **已移除**：收款码Tab — 企微收款码由成员在客户端生成，API不支持创建，无法通过系统管理

#### 配置项
| 配置项 | 说明 |
|-------|------|
| 对外收款Secret | 连接企微API的凭证 |
| 自动同步 | 定时同步收款记录 |
| 同步频率 | 15分钟~每天 |
| 自动关联CRM订单 | 按金额+付款人匹配 |
| 自动更新订单状态 | 收款后更新为已支付 |
| 退款需要审批 | 退款申请管理员审批 |
| 退款最长天数 | 超期不允许退款 |
| 收款成功通知 | 通知收款员工 |
| 退款通知 | 通知管理员 |
| 待支付超时提醒 | 可配置超时小时数 |

#### API对接
- 设置数据通过 `/payments/settings` GET/PUT 接口读写
- 存储在WecomConfig表的paymentSettings JSON字段中

#### 权限说明
- 新增API权限说明Alert组件
- 说明企微对外收款的权限要求和限制

## 五、部署注意事项

1. **执行数据库迁移**：`database/migration-wecom-payment-enhancement.sql`
2. **确认qrcode包已安装**：项目package.json已包含 `qrcode@^1.5.4`
3. **后端实体注册**：确保 `WecomPaymentRefund` 和 `WecomPaymentQrcode` 实体已在TypeORM配置中注册
4. **企微权限**：企业管理员需在企微管理后台为应用授权「对外收款」权限

## 六、已知限制

1. **退款API**：企微官方暂未开放退款API接口，系统内退款为记录管理 + 审批流程
2. **收款码生成**：企微不提供API创建收款码，需成员在客户端生成后录入系统管理
3. **实时推送**：收款状态变更的实时推送需要配置企微回调URL（后续可扩展）
4. **金额单位**：系统统一使用「分」作为金额单位，显示时转换为元
