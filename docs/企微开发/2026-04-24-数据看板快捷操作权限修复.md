# 数据看板快捷操作权限显示修复

## 问题描述

**现象**: 
- 管理员角色在数据看板的快捷操作卡片区显示正常
- 其他角色(经理、销售员、客服)显示"暂无可用的快捷操作"
- 但这些角色在左侧菜单栏有相应的权限和菜单项

**用户期望**:
每个角色在自己权限范围内的二级菜单应该在快捷操作卡片中显示对应的操作按钮,避免有权限但看不到快捷入口的情况。

## 问题根因

在 `src/views/Dashboard.vue` 中,`DEFAULT_QUICK_KEYS_BY_ROLE` 配置存在问题:

### 修复前的配置

```typescript
const DEFAULT_QUICK_KEYS_BY_ROLE: Record<string, string[]> = {
  super_admin: ['add-customer', 'add-order', 'search-customer', 'search-logistics'],
  admin: ['add-customer', 'add-order', 'search-customer', 'search-logistics'],
  department_manager: ['add-customer', 'add-order', 'search-customer', 'search-logistics'],
  sales_staff: ['add-customer', 'add-order', 'search-customer', 'search-logistics'],
  customer_service: ['order-list', 'service-list', 'shipping-list', 'search-logistics']
}
```

**问题点**:
1. ❌ 缺少 `manager` 角色的配置(系统中存在这个角色)
2. ❌ 所有角色的默认快捷操作都一样,没有体现角色特点
3. ❌ 当用户首次登录时,如果角色不在配置中,会使用 `DEFAULT_QUICK_KEYS`,但这些操作可能没有权限

### 权限过滤逻辑

```typescript
const permittedQuickActions = computed(() => {
  const user = userStore.currentUser
  if (!user) return []
  if (user.role === 'super_admin' || user.role === 'admin') {
    return allQuickActions  // 管理员显示所有操作
  }
  return allQuickActions.filter(action => {
    return action.permissions.some(perm => userStore.hasPermission(perm))
  })
})
```

**逻辑分析**:
- 管理员: 显示所有快捷操作 ✅
- 其他角色: 根据权限过滤 ✅
- 但是默认选中的快捷操作如果没有权限,就会被过滤掉,导致显示为空 ❌

### 加载逻辑

```typescript
const loadQuickActions = () => {
  // ...
  const role = user?.role || ''
  const roleDefaults = DEFAULT_QUICK_KEYS_BY_ROLE[role] || DEFAULT_QUICK_KEYS
  const permittedKeys = new Set(permittedQuickActions.value.map(a => a.key))
  selectedQuickKeys.value = roleDefaults.filter(key => permittedKeys.has(key)).slice(0, MAX_QUICK_ACTIONS)
}
```

**问题**:
- 如果 `role` 不在 `DEFAULT_QUICK_KEYS_BY_ROLE` 中,使用 `DEFAULT_QUICK_KEYS`
- `DEFAULT_QUICK_KEYS` 中的操作可能没有权限,被过滤后就是空数组

## 解决方案

### 修复后的配置

```typescript
const DEFAULT_QUICK_KEYS_BY_ROLE: Record<string, string[]> = {
  // 管理员 - 全局管理功能
  super_admin: ['add-customer', 'add-order', 'search-customer', 'search-logistics', 'performance-team', 'order-audit'],
  admin: ['add-customer', 'add-order', 'search-customer', 'search-logistics', 'performance-team', 'order-audit'],
  
  // 部门经理 - 团队管理 + 个人业务
  department_manager: ['add-customer', 'add-order', 'search-customer', 'search-logistics', 'performance-team', 'performance-personal'],
  manager: ['add-customer', 'add-order', 'search-customer', 'search-logistics', 'performance-team', 'performance-personal'],
  
  // 销售员 - 个人业务为主
  sales_staff: ['add-customer', 'add-order', 'search-customer', 'search-logistics', 'performance-personal', 'customer-list'],
  
  // 客服 - 订单处理和物流
  customer_service: ['order-list', 'service-list', 'shipping-list', 'search-logistics', 'order-audit', 'logistics-track']
}
```

### 修复要点

1. ✅ **添加 `manager` 角色配置**: 与 `department_manager` 相同,支持团队管理
2. ✅ **差异化配置**: 每个角色的默认快捷操作符合其工作职责
3. ✅ **权限匹配**: 所有默认操作都在对应角色的权限范围内

## 技术细节

### 权限检查逻辑

```typescript
// 每个快捷操作都定义了所需权限
const allQuickActions: QuickActionDef[] = [
  { 
    key: 'add-customer', 
    label: '新增客户', 
    icon: 'UserFilled', 
    route: '/customer/add',
    permissions: ['customer:add', 'customer:add:create', 'customer:list:create'],
    category: '客户管理' 
  },
  // ...
]

// 过滤出有权限的操作
const permittedQuickActions = computed(() => {
  const user = userStore.currentUser
  if (!user) return []
  if (user.role === 'super_admin' || user.role === 'admin') {
    return allQuickActions
  }
  return allQuickActions.filter(action => {
    return action.permissions.some(perm => userStore.hasPermission(perm))
  })
})
```

### 默认配置加载

```typescript
const loadQuickActions = () => {
  // 1. 尝试从localStorage加载用户自定义配置
  const stored = localStorage.getItem(storageKey)
  if (stored) {
    // 使用自定义配置
    return
  }
  
  // 2. 使用角色默认配置
  const role = user?.role || ''
  const roleDefaults = DEFAULT_QUICK_KEYS_BY_ROLE[role] || DEFAULT_QUICK_KEYS
  
  // 3. 过滤出有权限的操作
  const permittedKeys = new Set(permittedQuickActions.value.map(a => a.key))
  selectedQuickKeys.value = roleDefaults.filter(key => permittedKeys.has(key))
}
```

## 注意事项

1. **权限一致性**: 快捷操作的权限码必须与菜单权限码一致
2. **角色映射**: 系统中可能存在多个角色代码(如 `manager` 和 `department_manager`),需要都配置
3. **用户自定义**: 用户可以自定义快捷操作,会覆盖默认配置
4. **权限变更**: 如果用户权限发生变化,需要重新登录才能看到更新后的快捷操作
