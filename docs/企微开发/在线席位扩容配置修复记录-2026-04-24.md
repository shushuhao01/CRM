# 在线席位扩容配置修复记录

> 日期：2026-04-24 | 版本：v1.8.0 | 状态：已修复

---

## 一、问题描述

管理后台 → 套餐管理 → 扩容管理页面中，新增了"在线席位扩容"标签页，但保存/编辑在线席位扩容配置时出现 **"保存失败"** 和 **"操作失败"** 双重错误提示。

### 错误现象
- 选择"在线席位"扩容类型 → 填写配置 → 点击保存 → 红色提示"保存失败" + "操作失败"
- 用户数扩容和存储空间扩容可正常保存
- 编辑已有配置也失败

---

## 二、根因分析

### 2.1 数据库 ENUM 定义过旧

`database/schema.sql` 中的 `capacity_price_configs` 和 `capacity_orders` 表创建时 ENUM 定义不包含新增的在线席位类型：

| 表 | 字段 | 原始ENUM | 需要的ENUM |
|---|---|---|---|
| `capacity_price_configs` | `type` | `('user', 'storage')` | `('user', 'storage', 'online_seat')` |
| `capacity_price_configs` | `billing_cycle` | `('monthly', 'yearly', 'follow_package')` | `('monthly', 'yearly', 'permanent', 'follow_package')` |
| `capacity_orders` | `type` | `('user', 'storage')` | `('user', 'storage', 'online_seat')` |
| `capacity_orders` | `status` | `('pending', 'paid', 'closed', 'refunded')` | `('pending', 'paid', 'closed', 'refunded', 'expired')` |

### 2.2 缺失字段

| 表 | 缺失字段 | 用途 |
|---|---|---|
| `capacity_price_configs` | `discount_rules` (JSON) | 折扣梯度配置 |
| `capacity_orders` | `discount_percent` (DECIMAL) | 折扣百分比 |
| `capacity_orders` | `expire_date` (DATETIME) | 扩容到期日 |

### 2.3 自动迁移未启用

`backend/src/routes/admin/capacity.ts` 中的 `ensureTables()` 调用被注释掉，导致后端启动时不会自动修复数据库结构：

```typescript
// 被注释掉了！
// setTimeout(() => {
//   capacityService.ensureTables().catch(() => {});
// }, 3000);
```

### 2.4 前端双重错误提示

`admin/src/views/capacity/PriceConfig.vue` 的 `handleSave` 方法在 catch 块中重复弹出错误：
- 请求拦截器检测到 `success: false` 已弹出 "保存失败"
- catch 块又弹出 "操作失败"

---

## 三、修复内容

### 3.1 更新 `database/schema.sql`

**文件**: `database/schema.sql`

- `capacity_price_configs.type` ENUM 添加 `'online_seat'`
- `capacity_price_configs.billing_cycle` ENUM 添加 `'permanent'`
- `capacity_price_configs` 新增 `discount_rules` JSON 字段
- `capacity_orders.type` ENUM 添加 `'online_seat'`
- `capacity_orders.status` ENUM 添加 `'expired'`
- `capacity_orders` 新增 `discount_percent`、`expire_date` 字段
- 新增在线席位默认价格数据（按月50元、按年300元、永久800元）

### 3.2 启用自动迁移

**文件**: `backend/src/routes/admin/capacity.ts`

取消注释 `ensureTables()` 调用，后端启动时自动：
- MODIFY COLUMN 扩展 ENUM 枚举值
- ADD COLUMN 补充缺失字段
- 为 tenants 表添加 `extra_online_seats`、`user_limit_mode`、`max_online_seats` 字段

### 3.3 补充字段迁移

**文件**: `backend/src/services/CapacityService.ts`

`ensureTables()` 方法补充了以下自动迁移：
- `tenants.extra_online_seats` — 额外增购的在线席位数
- `tenants.user_limit_mode` — 用户限制模式（total/online）
- `tenants.max_online_seats` — 最大在线席位数

### 3.4 修复前端双重错误提示

**文件**: `admin/src/views/capacity/PriceConfig.vue`

`handleSave` 的 catch 块改为 `console.error`，避免与响应拦截器重复弹出 ElMessage 错误。

> **总结**：保存失败的根因是数据库 ENUM 定义不包含 `online_seat` 类型，MySQL 拒绝了 INSERT/UPDATE 操作。通过更新 schema.sql 和启用自动迁移，问题已彻底修复。同时验证了扩容购买全流程（配置→购买→支付→生效→到期回退）以及多端同步和订单隔离逻辑均正确无误。
