# CRM地址数据补全 - 区县乡镇级行政区划完善

**任务日期**: 2026-04-23  
**涉及文件**: `public/data/address.json`  
**参考数据源**: china-division npm 包（基于国家统计局最新行政区划数据）

---

## 一、任务背景

CRM系统客户管理的新增客户表单中，收货地址使用 `public/data/address.json` 提供省-市-区县-乡镇四级联动下拉选择。经全面检查发现，部分省份存在区县级和乡镇级行政区划数据缺失，需要进行补充完善。

## 二、补全前数据概况

| 指标 | 补全前 | 补全后 |
|------|--------|--------|
| 省份数 | 34 | 34（不变） |
| 区县数 | ~2,993 | 3,314 |
| 乡镇数 | ~36,900 | 47,361 |
| 文件大小 | ~1.6 MB | ~2.3 MB |

## 三、补全统计

### 总体数据

- **新增区县**: 321 个
- **新增乡镇（为新增区县）**: ~4,200 个
- **新增乡镇（为已有空区县补充）**: ~1,465 个
- **新增乡镇（为已有区县补充缺失部分）**: ~4,800 个
- **新增乡镇总计**: ~10,500 个

### 补全原则

1. **只增不删** - 不修改、不删除任何已有数据
2. **不引入新错误** - 仅补充缺失项
3. **参考权威数据源** - 使用 china-division 包（基于国家统计局数据）
4. **到乡镇级（街道）** - 不涉及村级数据
5. **value字段** - 使用 pinyin-pro 生成拼音作为唯一标识，与已有数据风格一致

## 五、特殊说明

1. **港澳台地区**: 香港（18区）、澳门（7堂区）、台湾（158区）保持原有结构不变，不添加下级行政区划（行政体制不同）

2. **海南省特殊结构**: 海南省直辖的县级市（五指山、琼海、文昌、万宁、东方等）和县（定安、屯昌等）的镇级单位已在区县层级展示，符合其行政管理实际

3. **东莞市/中山市**: 广东省东莞市和中山市不设区县级，其镇/街道直接在区县层级展示，保持原有结构

4. **农场/牧场等**: 部分省份（特别是黑龙江、新疆、内蒙古）包含国营农场、牧场等作为乡镇级单位，因其有实际居民和地址需求，均已补充

## 六、技术实现

### 使用的工具和脚本

1. **china-division** (npm包) - 提供参考行政区划数据（31省、342市、2978区县、41352乡镇）
2. **pinyin-pro** (npm包) - 生成拼音value值
3. **scripts/patch-address.cjs** - 主补全脚本（第一轮）
4. **scripts/patch-address-2.cjs** - 补充脚本（第二轮，农场等条目及湖北省直辖区）
5. **scripts/address-patch-report.txt** - 详细变更报告

### 数据验证

- JSON格式验证通过
- 所有新增条目均包含 label 和 value 字段
- value 字段全局唯一（通过 pinyin + 自增后缀确保）
- 不影响已有数据的任何字段

## 七、影响范围

- **CRM端**: 客户管理 > 新增/编辑客户 > 收货地址选择
