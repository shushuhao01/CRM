# 管理后台AI额度管理全面优化记录

> 日期：2026-04-21
> 涉及模块：管理后台(Admin)、CRM、会员中心(Website)、后端(Backend)

---

## 一、任务背景

管理后台的「企微管理 > AI额度管理」包含5个子Tab：平台模型、租户额度、计费配置、使用监控、全局设置。经全面审计发现以下问题：

1. **模型目录过旧** — Claude最高只到Opus 4.5，缺少Opus 4.6/4.7、GPT-5.x、Gemini 3.1、豆包2.0、GLM-5等
2. **租户额度完全硬编码** — 后端直接返回 `{ totalQuota: 1000, usedQuota: 0 }`，不读真实数据
3. **计费配置与套餐重复** — 计费配置Tab内有独立套餐列表，与「套餐与定价 > AI额度定价」重复
4. **使用监控缺少日期筛选** — 只有今日/7天/30天，无自定义日期范围和更多快捷日期
5. **全局设置存而不用** — 设置保存到数据库但未在AI调用流程中读取和应用
6. **会员中心AI组件全硬编码** — `WecomAiAssistant.vue` 所有数据为静态假数据，无API调用
7. **会话存档权限未解锁** — 会员中心支付后未解锁 `menuPermissions.chatArchive`
8. **AI额度数据源不一致** — `checkAndDeductAiQuota` 读取 `TenantSettings.ai_quota`，但套餐领取存到 `system_config.tenant_wecom_package`

## 二、修复方案与实施

更新 `MODEL_CATALOG` 常量：

移除过旧模型：GPT-4 Turbo、GPT-3.5 Turbo、o1-mini、Claude 3 Opus

### 2.2 租户额度真实数据映射

**后端文件**：`backend/src/routes/admin/wecom-management.ts`

**根本原因**：`GET /ai/tenant-quotas` 直接返回硬编码值 `totalQuota: 1000, usedQuota: 0`

**修复**：
1. 从 `system_config.tenant_wecom_package_${tenantId}` 读取 `aiPackage.calls` 作为真实总额度
2. 从 `wecom_ai_logs` 表 `COUNT(*)` 获取真实已用次数
3. 支持分页（`page`, `pageSize` 参数，默认15条/页）
4. 返回 `{ data, total, page, pageSize }` 结构

**实现"调整额度"功能**：
- `PUT /ai/tenant-quotas/:tenantId` — 支持 `add/reduce/set` 三种操作
- 直接修改 `system_config.tenant_wecom_package_${tenantId}.aiPackage.calls`
- 额度>0时自动解锁 `menuPermissions.aiAssistant`

**实现"批量分配"功能**：
- `POST /ai/tenant-quotas/batch` — 支持 `target: 'all' | 'no_quota'`
- 遍历所有活跃租户，累加额度到 `aiPackage.calls`

**前端变更**：
- 新增 `quotaPage`, `quotaPageSize`, `quotaTotal` 状态
- 添加 `el-pagination` 分页组件
- 使用率以 `el-progress` 进度条可视化展示
- 未开通租户显示 `el-tag type="info"` 标记

### 2.3 计费配置去重

**文件**：`admin/src/views/wecom/AiManagement.vue`

**方案**：计费配置Tab取消套餐列表（原5个硬编码套餐），仅保留：
- **计费模式选择**：按次数 / 按Token / 包月 / 包年（原只有3个，增加"包年"）
- **计费说明**：根据选择的模式动态显示说明文字
- **超额处理策略**：停止服务 / 发送提醒 / 自动续费
- **Token换算比例**：仅在按Token模式下显示
- 底部提示「套餐管理请前往"套餐与定价 → AI额度定价"」

**套餐与定价Tab联动**：在AI额度定价Tab顶部增加当前计费模式提示，引导用户去计费配置统一管理。

### 2.4 使用监控UI升级

**文件**：`admin/src/views/wecom/AiManagement.vue`（前端）、`backend/src/routes/admin/wecom-management.ts`（后端）

**前端优化**：
1. 筛选栏重新设计 — 背景卡片式，包含租户筛选 + 模型筛选
2. 快捷日期增加「昨天」「近90天」「自定义」
3. 自定义日期使用 `el-date-picker type="daterange"`
4. 总览统计改为6列卡片（今日调用、本月调用、累计调用、累计Token、输入Token、预估成本），带图标和悬浮效果
5. 模型使用统计和功能类型统计各用 `el-card` 包裹
6. 租户TOP10 表格高亮前3名
7. 功能类型统计与租户TOP10并排显示（各占50%宽度）

**后端增强**：
- `GET /ai/usage-stats` 支持 `yesterday`, `90d`, `custom`（`startDate`, `endDate`）、`modelName` 筛选
- `GET /ai/usage-stats/top` 同步支持新日期范围

### 2.5 全局设置生效

**文件**：`backend/src/routes/wecom/aiAssistant.ts`

**根本原因**：`checkAndDeductAiQuota()` 不读取全局设置，也不读取 `tenant_wecom_package` 的额度

**修复**：
1. 额度来源优先级：`system_config.tenant_wecom_package` > `TenantSettings.ai_quota`
2. 读取 `wecom_ai_global_settings.stopOnQuotaExhaust` 决定额度耗尽后是否停止服务
3. 修复后，后台配置的「额度耗尽自动停止」开关真正生效

### 2.6 会员中心AI组件修复

**文件**：`website/src/views/member/wecom/WecomAiAssistant.vue`

**根本原因**：整个组件没有任何API调用，所有数据为硬编码静态值

**修复**：
- `fetchAiData()` — 调用 `/api/v1/public/member/wecom/usage` 获取真实AI额度/使用量
- `fetchPackages()` — 调用 `/api/v1/wecom/ai/packages` 获取可购买套餐
- `fetchOrders()` — 调用 `/api/v1/public/member/wecom/orders` 获取真实订单
- `fetchModelUsage()` — 调用 `/api/v1/wecom/ai/model-usage` 获取模型使用明细
- `fetchUsageBreakdown()` — 调用 `/api/v1/wecom/ai/usage-trend` 获取功能类型分布
- `handleActivate()` — 调用 `/api/v1/wecom/ai/orders` POST创建购买订单

### 2.7 会员中心Usage端点补充AI数据

**文件**：`backend/src/routes/public/member-wecom.ts`

**修复**：`GET /usage` 返回值新增 `aiStats` 字段：

### 2.8 会话存档权限解锁补全

**文件**：`backend/src/routes/public/member-wecom.ts`

在 `GET /order/:orderNo` 端点中，当 `status === 'paid'` 时补充解锁 `menuPermissions.chatArchive = true` 到 `tenant_wecom_package`。

## 三、数据流说明

### AI额度数据流
```
管理后台套餐与定价 → wecom_pricing_config.aiPackages（配置）
       ↓
CRM购买 → wecom/ai/orders → system_config.tenant_wecom_package_${tenantId}.aiPackage（额度写入）
       ↓
管理后台AI额度管理.租户额度 → 读取 tenant_wecom_package + wecom_ai_logs 统计
       ↓
CRM/会员中心AI调用 → checkAndDeductAiQuota() 读取 tenant_wecom_package 额度 + 全局设置
```

### 计费模式数据流
```
管理后台AI额度管理.计费配置 → system_config.wecom_ai_billing（保存mode/overQuotaAction）
       ↓
套餐与定价.AI额度定价 → 引用计费模式，套餐额度单位随之变化
       ↓
CRM/会员中心AI使用 → 根据计费模式扣费/展示
```
