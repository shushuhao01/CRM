# CRM 套餐领取与权限解锁修复记录

> 日期：2026-04-21
> 问题：获客助手/企微授权/AI助手的免费套餐领取全部失败（提示"登录已过期"），领取后未解锁对应功能权限

---

## 一、问题现象

1. **获客助手**：点击"免费领取"按钮，弹出"登录已过期，请重新登录"
2. **企微授权**：套餐购买/领取同样失败，报401
3. **AI助手**：订单与使用量页面的免费体验包领取走了独立API路径，虽不受影响但缺少防重复和权限解锁
4. **权限不联动**：即使领取成功，侧栏菜单的功能锁（🔒）不会解除

---

## 二、根因分析

### 2.1 核心Bug：`pricing.ts` 缺失认证中间件

**根因**：`backend/src/routes/wecom/pricing.ts` 的 **所有11个路由** 均未添加 `authenticateToken` 中间件。

对比其他模块（如 `config.ts`、`acquisition.ts`）均正确添加了认证中间件，但 `pricing.ts` 完全遗漏。

导致链路：
```
前端 POST /wecom/claim-package
  → 后端无 authenticateToken → req.user = undefined
  → tenantId = req.user?.tenantId = undefined
  → 返回 401 { success: false, message: '未登录' }
  → 前端拦截器捕获 HTTP 401
  → 触发 handleUnauthorized() → "登录已过期" 弹窗
```

### 2.2 免费套餐无防重复

- **企微套餐**（`/claim-package`）：无防重复检查，可反复领取
- **获客助手**（`/purchase-acquisition`）：无防重复检查
- **AI套餐**（`/purchase-ai`）：已有防重复（`tenant_ai_free_claimed_*`）✓
- **AI订单**（`/ai/orders` in aiAssistant.ts）：走独立路径，无防重复检查

### 2.3 权限不联动

套餐领取只写了账单记录，没有更新 `tenant_wecom_package_*` 的 `menuPermissions`，导致：
- 获客助手领取后 `acquisition` 权限仍为 false
- AI领取后 `aiAssistant` 权限仍为 false
- 会话存档购买后 `chatArchive` 权限仍为 false
- 前端 `WecomStandalone.vue` 的 `hasMenuPermission()` 读到 false → 显示🔒

---

## 三、修复方案

### 3.1 添加认证中间件（11个路由全量修复）

**文件**：`backend/src/routes/wecom/pricing.ts`

所有路由均添加 `authenticateToken`：

| 路由 | 方法 | 修复前 | 修复后 |
|------|------|--------|--------|
| `/pricing-config` | GET | 无中间件 | `authenticateToken` |
| `/tenant-package` | GET | 无中间件 | `authenticateToken` |
| `/claim-package` | POST | 无中间件 | `authenticateToken` |
| `/purchase-ai` | POST | 无中间件 | `authenticateToken` |
| `/purchase-archive` | POST | 无中间件 | `authenticateToken` |
| `/upload-archive-confirmation` | POST | 无中间件 | `authenticateToken` |
| `/archive-confirmation-status` | GET | 无中间件 | `authenticateToken` |
| `/confirm-payment` | POST | 无中间件 | `authenticateToken` |
| `/purchase-acquisition` | POST | 无中间件 | `authenticateToken` |
| `/billing-records` | GET | 无中间件 | `authenticateToken` |
| `/cancel-order` | POST | 无中间件 | `authenticateToken` |

### 3.2 免费套餐防重复

| 类型 | config key | 文件 |
|------|-----------|------|
| 企微套餐 | `tenant_wecom_free_claimed_{tenantId}` | pricing.ts `/claim-package` |
| 获客助手 | `tenant_acquisition_free_claimed_{tenantId}` | pricing.ts `/purchase-acquisition` |
| AI额度(pricing) | `tenant_ai_free_claimed_{tenantId}` | pricing.ts `/purchase-ai`（已有） |
| AI额度(aiAssistant) | `tenant_ai_free_claimed_{tenantId}` | aiAssistant.ts `/ai/orders`（新增） |

### 3.3 权限解锁机制

新增 `unlockTenantPermissions()` 工具函数，在以下时机调用：

| 时机 | 解锁的权限key |
|------|--------------|
| 获客助手免费领取 | `acquisition`, `contactWay` |
| AI免费领取 | `aiAssistant` |
| 会话存档购买 | `chatArchive` |
| 任意类型付费确认 | 对应类型的权限 |

权限存储在 `system_config` 表的 `tenant_wecom_package_{tenantId}` → `menuPermissions` 对象中。

### 3.4 前端权限刷新

- `PackagePurchaseTab.vue`：领取/购买成功后触发 `window.dispatchEvent(new CustomEvent('wecom-package-changed'))`
- `AiAssistant.vue`：免费领取成功后同样触发
- `WecomStandalone.vue`：`onMounted` 注册 `wecom-package-changed` 事件监听器，收到事件后重新调用 `getTenantPackage()` 刷新 `menuPermissions`

| 文件 | 修改类型 | 说明 |
|------|---------|------|
| `backend/src/routes/wecom/pricing.ts` | 核心修复 | 添加 authenticateToken × 11、防重复 × 2、unlockTenantPermissions 函数、confirm-payment 权限解锁 |
