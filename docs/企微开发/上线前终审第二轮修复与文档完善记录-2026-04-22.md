## 一、schema.sql 兼容性修复

### 修复1：call_lines 表INSERT数据类型不匹配
- **问题**：`call_lines` 表 `id` 字段定义为 `INT AUTO_INCREMENT`，但 INSERT 语句插入了字符串值（`'line_default_001'`等），在MySQL严格模式下会报错
- **修复**：移除 INSERT 中的 `id` 列，让数据库自动分配自增ID
- **影响范围**：仅影响初始化默认数据，不影响业务逻辑

### 修复2：末尾SELECT语句移除
- **问题**：schema.sql 末尾有7条 `SELECT` 语句，在 phpMyAdmin/宝塔面板批量执行时会返回多个结果集，可能导致执行体验异常
- **修复**：将 SELECT 语句改为 SQL 注释（`--` 开头），保留信息但不执行

## 二、官网功能特性页修复

### 修复：Features.vue 图片引用错误
- **问题**：Features.vue 中 `<img src="/images/feature-wecom.png">` 等引用了不存在的图片文件，Vite 编译时报错 `[plugin:vite:import-analysis] Failed to resolve import`
- **根因**：`website/public/images/` 目录不存在，所有图片引用都是无效的
- **修复方案**：彻底重写 Features.vue，移除所有图片依赖，改为纯 CSS 渐变+SVG 图标的图示卡片设计
- **改进**：使用 `v-for` 数据驱动渲染，代码量从 461 行优化为约 200 行

### 完善：首页核心功能区域
- **修改文件**：`website/src/views/Home.vue`
- **新增内容**：在"强大的核心功能"区域添加4个新功能卡片：
