### 1.2 CRM侧边栏H5与服务商应用的关系

```
┌─────────────────────────────────────────────────────┐
│ 当前CRM内的侧边栏H5 (已完成，不需要改动)              │
├─────────────────────────────────────────────────────┤
│ 位置: CRM系统内部 -> 企微管理 -> 侧边栏             │
│ 入口: CRM系统登录后，在企微管理菜单中访问             │
│ 技术: Vue 3 + Element Plus，嵌入CRM主应用            │
│ 鉴权: CRM JWT Token                                 │
│ 用途: CRM用户在PC端管理侧边栏应用配置                │
│ 目标: 管理员配置侧边栏的网页链接和授权码              │
└─────────────────────────────────────────────────────┘
                    ↕ 完全独立，互不影响
┌─────────────────────────────────────────────────────┐
│ 服务商应用H5 (需要新开发)                             │
├─────────────────────────────────────────────────────┤
│ 位置: 企业微信客户端内部                              │
│ 入口: 企微工作台 / 企微聊天侧边栏 (企微客户端中)     │
│ 技术: Vue 3 + Vant 4 (移动端UI库)，独立H5项目        │
│ 鉴权: 企微JS-SDK -> 获取企微userId -> 绑定CRM用户     │
│ 用途: 企微用户在手机/桌面端使用CRM能力                │
│ 目标: 工作台应用(首页/客户/统计/我的)                 │
│       聊天侧边栏(客户画像/快捷话术/快捷下单)          │
└─────────────────────────────────────────────────────┘
```

**核心区别:**
- **CRM侧边栏H5**: 在CRM系统PC端配置管理，使用Element Plus，已完成✅
- **服务商应用H5**: 在企业微信客户端中运行，使用Vant(移动端)，需要新开发🔨
- 两者共享同一套后端API，但前端完全独立
- CRM侧边栏的"配置"功能，管理的就是服务商应用中侧边栏页面的链接地址

## 二、企微服务商后台操作步骤

> 以下操作在 https://open.work.weixin.qq.com 进行

### 2.1 步骤一：注册服务商账号

```
前提: 需要一个已认证的企业微信
操作:
  1. 访问 https://open.work.weixin.qq.com
  2. 使用企业微信管理员扫码登录
  3. 完成服务商注册(填写企业信息、联系人等)
  4. 记录 provider_corp_id (服务商CorpID)

产出:
  ✅ 获得服务商CorpID (provider_corp_id)
  ✅ 获得服务商Secret (provider_secret)
```

### 2.2 步骤二：创建第三方应用(应用套件)

```
位置: 服务商后台 -> 应用管理 -> 创建应用

填写内容:
  1. 应用名称: 云客CRM (与CRM系统名称一致)
  2. 应用Logo: 上传CRM Logo
  3. 应用简介: 私域客户管理，深度集成企业微信
  4. 应用详细描述: 提供客户管理/订单管理/数据分析等能力
  5. 应用分类: 客户管理 / CRM
  6. 适用范围: 选择所需的可见范围

产出:
  ✅ 获得 Suite ID (应用套件ID)
  ✅ 获得 Suite Secret (应用套件Secret)
```

### 2.3 步骤三：配置回调URL

```
位置: 应用管理 -> 应用详情 -> 开发配置

需要配置:
  1. 数据回调URL:
     https://你的域名/api/v1/wecom/suite/callback
     (对应后端 suite-callback.ts 已实现的路由)

  2. 回调Token: 自定义生成(32位随机字符串)
  3. 回调EncodingAESKey: 自定义生成(43位随机字符串)

  4. 指令回调URL: (同上，或单独配置)
     https://你的域名/api/v1/wecom/suite/callback

验证:
  企微会立即发送验证请求(GET)
  后端 suite-callback.ts 已实现签名验证 → 返回解密的echostr
  验证通过后保存配置

⚠️ 重要: 回调URL必须是HTTPS，且公网可访问
```

### 2.4 步骤四：配置应用功能入口

```
位置: 应用管理 -> 应用详情 -> 功能配置

配置项:
  1. 工作台应用主页:
     https://你的域名/h5/index.html
     (服务商H5应用首页)

  2. 聊天侧边栏:
     添加侧边栏页面:
     a. 客户画像: https://你的域名/h5/index.html#/sidebar/portrait
     b. 快捷话术: https://你的域名/h5/index.html#/sidebar/scripts
     c. 快捷下单: https://你的域名/h5/index.html#/sidebar/quick-order

  3. 消息推送URL(可选):
     https://你的域名/api/v1/wecom/suite/callback
```

### 2.5 步骤五：配置权限(API权限)

```
位置: 应用管理 -> 应用详情 -> API权限

必须申请的权限:
  ✅ 通讯录读取 (读取成员/部门信息)
  ✅ 外部联系人读取 (获取客户信息)
  ✅ 外部联系人管理 (管理客户标签等)
  ✅ 客户群管理 (群列表/群成员)
  ✅ 获客助手 (活码/获客链接)
  ✅ 微信客服 (客服消息)
  ✅ 会话存档 (如需)
  ✅ JS-SDK相关权限 (wx.agentConfig等)

可选权限:
  □ 审批 (如需集成审批流程)
  □ 打卡/日程 (如需)
```

### 2.6 步骤六：配置可信域名

```
位置: 应用管理 -> 应用详情 -> 开发配置

配置:
  1. 可信域名: 你的域名 (如 crm.example.com)
     - 需要在域名根目录放置验证文件
     - 文件名和内容由企微提供

  2. 授权完成回调域名: 你的域名
     回调URL: https://你的域名/api/v1/wecom/suite/auth-callback
     (对应 suite-callback.ts 中的 /auth-callback 路由，已实现)

验证文件放置:
  将企微提供的验证文件放到前端项目的 public/ 目录
  确保 https://你的域名/WW_verify_xxxxx.txt 可访问
```

### 2.7 步骤七：绑定对公账户(收益相关)

```
位置: 服务商后台 -> 财务管理

操作:
  1. 绑定企业对公银行账户
  2. 企微会打小额验证金(如0.01元)
  3. 确认金额完成绑定
  4. (可选) 缴纳3万保证金(提现返利时需要)
  5. (可选) 首次充值服务商钱包(代购模式需要)

⚠️ 此步骤非开发必需，可在业务跑通后再操作
```

## 三、开发策略分析：独立开发 vs 融入现有项目

### 3.1 结论：H5前端独立开发 + 后端融入现有项目

```
┌─────────────────────────────────────────────────────┐
│                  推荐架构方案                         │
├─────────────────────────────────────────────────────┤
│                                                      │
│  H5前端(新建独立项目):                               │
│    d:\kaifa\CRM - 1.8.0\h5\                         │
│    ├── package.json      (Vue3 + Vant4 + Vite)      │
│    ├── src/                                          │
│    │   ├── views/                                    │
│    │   │   ├── app/        (工作台页面)               │
│    │   │   │   ├── Home.vue                          │
│    │   │   │   ├── CustomerList.vue                  │
│    │   │   │   ├── CustomerDetail.vue                │
│    │   │   │   ├── Stats.vue                         │
│    │   │   │   └── Profile.vue                       │
│    │   │   └── sidebar/    (侧边栏页面)               │
│    │   │       ├── Portrait.vue                      │

│  后端(融入现有backend):                               │

│    ├── h5-app.ts           🔨 新增(H5工作台API)      │

│    https://domain/h5/    -> 服务商H5                  │
│    https://domain/api/   -> 后端API                   │
│                                                      │
└─────────────────────────────────────────────────────┘
```

### 3.2 为什么H5前端要独立？

| 维度 | 独立H5项目 | 融入CRM前端 |
|------|-----------|------------|
| UI框架 | Vant 4 (移动端) | Element Plus (PC端) |
| 运行环境 | 企微客户端内嵌浏览器 | PC浏览器 |
| 包体积 | 小巧精简(~200KB) | 庞大(CRM完整包>5MB) |
| 加载速度 | 快速(企微内网络敏感) | 过重，不适合移动端 |
| 鉴权方式 | 企微JS-SDK鉴权 | CRM JWT登录 |
| 路由 | Hash模式(企微兼容) | History模式 |
| 构建部署 | 独立构建、独立部署 | 与CRM耦合 |

**结论: H5前端必须独立开发，不可能融入CRM前端项目。**

### 3.3 为什么后端要融入？

| 维度 | 融入现有backend | 独立后端 |
|------|----------------|---------|
| 数据复用 | ✅ 直接访问所有CRM数据表 | ❌ 需跨库或API调用 |
| 认证复用 | ✅ 复用现有JWT/权限/租户体系 | ❌ 需重建 |
| 代码复用 | ✅ 复用WecomApiService等服务 | ❌ 需重写 |
| 部署成本 | ✅ 单一进程 | ❌ 多进程维护 |
| suite回调 | ✅ 已在backend实现 | ❌ 需迁移 |

**结论: 后端必须融入现有backend项目，新增路由文件即可。**

```
目标: 创建独立H5前端项目，跑通开发环境

任务:
  1. 初始化 h5/ 目录下的 Vue3 + Vite + Vant4 项目

     /app/customers     -> 客户列表
     /app/customer/:id  -> 客户详情
     /app/stats         -> 数据统计
     /app/profile       -> 我的
     /sidebar/portrait  -> 侧边栏客户画像

  3. 封装企微JS-SDK工具

### Phase 2: 后端H5 API开发 (3天)

```
目标: 为H5前端提供专用API，复用现有服务层

新增后端文件:

  1. routes/wecom/h5-auth.ts (H5鉴权)
     POST /wecom/h5/login
       接收企微userId → 查WecomUserBinding → 返回JWT token
     GET  /wecom/h5/current-user
       返回当前用户信息(角色/权限/数据范围)
     GET  /wecom/h5/jssdk-config
       返回JS-SDK签名配置(corpId/timestamp/nonce/signature)
     GET  /wecom/h5/agent-config
       返回agentConfig签名(应用级别签名)

  2. routes/wecom/h5-app.ts (工作台API)
     GET  /wecom/h5/app/home
       首页数据(客户数/新增/动态)，按数据范围过滤
     GET  /wecom/h5/app/customers
       客户列表，按数据范围过滤，支持搜索/分页
     GET  /wecom/h5/app/customer/:id
       客户详情(融合CRM+企微数据)
     GET  /wecom/h5/app/stats
       数据统计(按角色展示不同维度)

  3. routes/wecom/h5-sidebar.ts (侧边栏API)
     GET  /wecom/h5/sidebar/portrait
       客户画像(当前聊天客户的融合数据)
     GET  /wecom/h5/sidebar/scripts
       话术列表(分类+搜索)
     POST /wecom/h5/sidebar/send-script
       发送话术(调用企微sendChatMessage)
     POST /wecom/h5/sidebar/quick-order
       快捷下单(创建CRM订单)
     GET  /wecom/h5/sidebar/customer-bindinfo
       通过企微externalUserId查询CRM客户

  4. 在 routes/wecom/index.ts 中注册新路由
     router.use('/h5', h5AuthRouter);
     router.use('/h5', h5AppRouter);
     router.use('/h5', h5SidebarRouter);
```

### Phase 5: JS-SDK鉴权与企微绑定流程 (2天)

```
目标: 打通企微JS-SDK鉴权全链路

鉴权流程:
  1. H5页面加载 → 引入企微JS-SDK
  2. 调用后端 GET /wecom/h5/jssdk-config 获取签名
     后端: 使用 corp_access_token 调用企微 get_jsapi_ticket
     计算签名: SHA1(jsapi_ticket + noncestr + timestamp + url)
  3. 前端: wx.config({...}) 初始化
  4. 调用后端 GET /wecom/h5/agent-config 获取应用签名
     后端: 使用 suite_access_token 调用企微 get_jsapi_ticket
     计算签名(应用级别)
  5. 前端: wx.agentConfig({...}) 初始化应用级别API
  6. 鉴权成功后:
     - 工作台: wx.invoke('getCurExternalContact') 获取用户身份
     - 侧边栏: wx.invoke('getCurExternalContact') 获取当前聊天客户

H5登录绑定流程:
  1. JS-SDK鉴权成功 → 获取企微userId
  2. 调用 POST /wecom/h5/login { wecomUserId }
  3. 后端查询 WecomUserBinding 表
     - 已绑定: 返回CRM用户JWT Token + 权限信息
     - 未绑定: 返回绑定页面，引导用户绑定CRM账号
  4. 后续请求携带JWT Token访问API
```

```

  1. 后端部署到可HTTPS访问的服务器
  2. H5构建部署到 /h5/ 路径
  3. Admin后台配置suite信息
  4. 企微后台配置好回调URL和可信域名

测试清单:
  □ 回调验证: 企微后台验证回调URL通过

### 6.2 corp_access_token获取(第三方应用)

```
第三方应用模式下获取企业Token:
  POST https://qyapi.weixin.qq.com/cgi-bin/service/get_corp_token
    ?suite_access_token={SUITE_ACCESS_TOKEN}
  Body: {
    auth_corpid: "企业CorpID",
    permanent_code: "永久授权码"
  }
  返回: { access_token, expires_in }

  此token用于:
  - 调用该企业的通讯录API
  - 调用该企业的外部联系人API
  - 获取JS-SDK ticket
  等
```

### 6.3 JS-SDK双重签名

```
企微H5应用需要两层签名:

第一层: wx.config (企业级)
  使用 corp_jsapi_ticket 签名
  权限: 基础JS-SDK能力(分享/地理位置等)

第二层: wx.agentConfig (应用级)
  使用 agent_jsapi_ticket 签名
  权限: 应用级能力(getCurExternalContact/sendChatMessage等)

  获取 agent_jsapi_ticket:
  GET https://qyapi.weixin.qq.com/cgi-bin/ticket/get
    ?access_token={CORP_ACCESS_TOKEN}
    &type=agent_config
```
