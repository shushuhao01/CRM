### P0 - 最高优先级拆分建议

│   ├── CustomerInfoCard.vue      # 客户基本信息卡片（~500行）

## 六、拆分注意事项

### 通用原则
1. **先写测试再拆分**：拆分前确保核心功能有测试覆盖
2. **逐步进行**：一次只拆一个文件，验证无误后再拆下一个
3. **保持路由不变**：拆分后保持 URL 路由路径不变，对用户透明
4. **共享状态处理**：使用 Vue 的 provide/inject 或 composables 传递共享状态

### Vue 组件拆分步骤
1. 识别组件内独立的功能区块（通常对应 UI 的不同区域）
2. 提取数据获取逻辑到 composables 文件
3. 将 template 中的功能区块提取为子组件
4. 通过 props 传入数据，emit 传出事件
5. 将 scoped style 跟随组件迁移
6. 主组件只保留布局和子组件协调

### 后端路由拆分步骤
1. 创建子目录和 index.ts 聚合文件
2. 按功能域将路由处理函数移到子文件
3. index.ts 中 import 并 use 各子路由
4. 更新上级路由注册（从文件改为目录）
5. 公共中间件和辅助函数提取到 helpers.ts

### 风险控制
- 每次拆分后运行完整的编译检查（`npm run build`）
- 拆分后对比原功能确保一致性
- 注意循环依赖问题
- 注意 CSS scoped 作用域变化

## 七、总结

| 统计项 | 数量 |
|--------|------|
| 特大文件（3000+行） | 17个 |
| 超大文件（2000-3000行） | 10个 |
| 大文件（1000-2000行） | 35+个 |
| **建议优先拆分** | **P0级 6个文件** |

项目当前的巨型文件主要集中在 **CRM前端视图层**，原因是 Vue 单文件组件将 template + script + style 集中在一起。建议在不影响现有功能的前提下，**从P0级文件开始逐步拆分**，每次拆分一个文件并充分验证。

**重要提醒**：当前项目即将上线，建议 **上线后的第一个迭代周期** 专门用于代码重构和拆分，避免上线前的大范围改动引入风险。
