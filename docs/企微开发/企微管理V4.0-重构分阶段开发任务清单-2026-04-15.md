### 2.2 CRM自动匹配机制（3天）

#### 2.2.1 后端自动匹配服务（Day2下午-Day3）

**文件：** `backend/src/services/WecomAutoMatchService.ts`

**核心逻辑：**
```
同步企微客户完成 → 触发自动匹配流程:
1. 遍历所有有手机号(phone字段)的 WecomCustomer
2. 精确匹配 CRM Customer 表的 phone 字段
3. 检查：
   - 已关联（crmCustomerId有值）→ 跳过
   - 匹配到且未关联 → 创建 WecomAutoMatchSuggestion(status=pending)
   - 未匹配到 → 不处理
4. 返回新增待确认数量 → 前端徽标展示
```

**新增API端点：**
```
GET    /wecom/customers/auto-match/pending       获取待确认匹配列表
  参数: configId, page, pageSize
  返回: { list: [{ suggestion, wecomCustomer, crmCustomer }], total, pendingCount }

POST   /wecom/customers/auto-match/:id/confirm   确认匹配
  动作: 更新suggestion.status='confirmed' + 写入wecomCustomer.crmCustomerId

POST   /wecom/customers/auto-match/:id/reject    拒绝匹配
  动作: 更新suggestion.status='rejected'

GET    /wecom/customers/auto-match/count          获取待匹配数量（徽标用）
  返回: { pendingCount: number }
```

```
  [====二维码====]
  等待扫码授权...
  二维码有效期: 10分钟 [刷新]

  -- 或者 --
  自建应用接入(需手动配置) [展开指引 v]
```

**技术实现：**
```
1. 前端调用: GET /api/v1/public/member/wecom/auth-qrcode
2. 后端: 读取Admin配置的suite信息
   -> 获取pre_auth_code(企微API)
   -> 生成授权URL
3. 生成二维码展示给用户
4. 轮询检测授权结果: GET /api/v1/public/member/wecom/auth-status
5. 授权成功 -> 自动创建WecomConfig记录 -> 页面刷新

注意: 需要阶段七的服务商应用配置作为前置条件
      如果Admin未配置suite信息，显示"自建应用接入"模式
```

**后端API(公开，会员中心专用)：**
```
GET  /public/member/wecom/auth-qrcode    获取授权二维码
GET  /public/member/wecom/auth-status    轮询授权状态
POST /public/member/wecom/unbind/:id     解绑企微
```
