## 📝 文档规范

### 文件命名规范

- 问题修复: `YYYY-MM-DD-问题简述.md`
- 功能开发: `功能名称-开发文档.md`
- 测试文档: `功能名称-测试指南.md`

### 文档结构

每个问题修复文档应包含:

1. **问题描述** - 现象和影响
2. **问题根因** - 技术分析
3. **解决方案** - 实施方案和代码修改
4. **测试验证** - 测试用例
5. **相关文件** - 涉及的代码文件
6. **修复记录** - 时间和人员
