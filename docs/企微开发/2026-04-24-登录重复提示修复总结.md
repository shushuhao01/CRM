# 登录密码错误重复提示问题 - 修复总结

## 问题

用户在CRM登录页输入错误密码后,点击一次登录按钮,会连续弹出3次错误提示,每次剩余尝试次数递减(4→3→2),导致用户很快被锁定账户。

## 根本原因

前端 `loginWithRetry` 方法默认重试3次,即使密码错误也会继续重试,导致:
- 点击一次登录 → 实际调用3次后端API
- 后端失败计数+3
- 用户看到3次错误提示

## 解决方案

修改 `src/stores/user/authActions.ts` 中的 `loginWithRetry` 方法:

**核心修改**:
```typescript
// 🔥 认证失败(密码错误、账户锁定等)不重试,直接抛出错误
const authErrors = ['INVALID_CREDENTIALS', 'ACCOUNT_LOCKED', 'ACCOUNT_DISABLED', 'IP_NOT_AUTHORIZED', 'USER_STATUS_INVALID']
if (authErrors.includes(errorCode)) {
  console.log('[Auth] 认证失败,不重试:', errorCode)
  throw lastError
}

// 🔥 只有网络错误、服务器错误、频繁请求才重试
const shouldRetry = (
  errorMessage.includes('频繁') || 
  errorMessage.includes('429') || 
  errorMessage === 'RATE_LIMITED' ||
  errorMessage.includes('网络') ||
  errorMessage.includes('超时') ||
  errorMessage.includes('服务器')
)
```

## 修复效果

✅ 密码错误时只调用一次后端API  
✅ 用户只看到一次错误提示  
✅ 失败计数准确,点一次登录只消耗一次机会  
✅ 保留了网络错误的重试机制  

## 测试要点

1. **密码错误**: 输入错误密码 → 只弹出1次提示 → 失败计数+1
2. **账户锁定**: 连续5次错误 → 第5次提示锁定 → 无重复提示
3. **网络错误**: 断网登录 → 自动重试3次 → 提示网络错误
4. **正常登录**: 正确密码 → 一次成功 → 跳转首页
