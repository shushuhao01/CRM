## 一、系统架构概述

### 1.1 整体架构

```
                          ┌──────────────────────┐
                          │     Nginx 反向代理     │
                          └───┬──────┬──────┬────┘
                              │      │      │
              ┌───────────────┤      │      ├───────────────┐
              ▼               ▼      │      ▼               ▼
        ┌──────────┐  ┌──────────┐  │  ┌──────────┐  ┌──────────┐
        │ CRM前端  │  │ 管理后台  │  │  │  官 网   │  │ 移动APP  │
        │ :5173    │  │ :5174    │  │  │ :5175    │  │ :5176    │
        └────┬─────┘  └────┬─────┘  │  └────┬─────┘  └────┬─────┘
             │              │       │       │              │
             └──────────────┴───┬───┘───────┴──────────────┘
                                ▼
                        ┌──────────────┐
                        │ Backend API  │
                        │ :3000        │
                        └───────┬──────┘
                                │
                    ┌───────────┼───────────┐
                    ▼           ▼           ▼
              ┌──────────┐ ┌────────┐ ┌──────────┐
              │ MySQL 8  │ │ Redis  │ │ 文件存储  │
              │ :3306    │ │ :6379  │ │ (可选)    │
              └──────────┘ └────────┘ └──────────┘
```

### 1.2 子系统说明

| 子系统 | 目录 | 端口 | 说明 |
|--------|------|------|------|
| Backend API | `backend/` | 3000 | Node.js + Express + TypeORM |
| CRM前端 | `src/` | 5173 | Vue 3 + Element Plus |
| 管理后台 | `admin/` | 5174 | Vue 3 + Element Plus |
| 官方网站 | `website/` | 5175 | Vue 3 + SCSS |
| 移动APP | `mobile/` | 5176 | Vue 3 + Vant |

### 1.3 多租户架构

系统采用共享数据库、租户ID隔离的多租户模式：
- 每个租户拥有唯一的 `tenant_id`
- 所有业务表均包含 `tenant_id` 字段
- 中间件 `tenantAuth.ts` 自动注入租户上下文
- API查询自动添加租户条件过滤

## 四、后端子系统（Backend）

### 4.1 应用启动流程

```
backend/src/app.ts
  ├── 1. 加载环境变量（dotenv）
  ├── 2. 初始化Express应用
  ├── 3. 注册全局中间件
  │   ├── CORS
  │   ├── JSON解析 / URL编码
  │   ├── 请求日志
  │   ├── 静态文件服务
  │   └── 速率限制（登录接口）
  ├── 4. 初始化数据库连接（TypeORM）
  ├── 5. 注册API路由（50+顶级路由）
  ├── 6. 注册404和错误处理中间件
  ├── 7. 启动WebSocket服务
  ├── 8. 启动定时任务调度器
  └── 9. 监听端口启动HTTP服务
```

### 4.2 中间件说明

| 中间件 | 文件 | 用途 |

| auth | `middleware/auth.ts` | JWT Token验证，解析用户信息 |

| checkLicenseWrite | `middleware/checkLicenseWrite.ts` | 授权检查（写操作） |
| errorHandler | `middleware/errorHandler.ts` | 全局错误处理 |
| loginLimiter | `middleware/auth.ts` | 登录接口频率限制 |

## 九、数据库维护

### 9.1 数据库概览

- **数据库类型**：MySQL 8.0+
- **字符集**：utf8mb4 + utf8mb4_unicode_ci
- **初始化脚本**：`database/schema.sql`（v4.1.0，约5267行）
- **总表数量**：约110+张表

### 9.2 初始化步骤

1. 创建数据库：
```sql
CREATE DATABASE crm_local CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

2. 导入schema.sql：
```bash
mysql -u root -p crm_local < database/schema.sql
```

3. 或在phpMyAdmin中导入：
   - 选择数据库 `crm_local`
   - 点击「导入」→ 选择 `schema.sql` → 执行

### 9.3 预设账号

| 账号 | 密码 | 角色 |
|------|------|------|
| superadmin | super123456 | 超级管理员 |
| admin | admin123 | 管理员 |
| manager | manager123 | 部门经理 |
| sales | sales123 | 销售员 |
| service | service123 | 客服 |

### 9.4 重要注意事项

1. **外键检查**：schema.sql开头关闭了外键检查（`SET FOREIGN_KEY_CHECKS = 0`），末尾恢复
2. **DROP TABLE**：脚本包含 `DROP TABLE IF EXISTS`，在已有数据的库上执行会**删除数据**
3. **字符集**：确保MySQL服务器支持 utf8mb4
4. **时区**：脚本设置了 `+08:00` 时区
5. **JSON列**：部分表使用了JSON类型，需MySQL 5.7.8+
6. **函数索引**：`api_configs` 表使用了 `JSON_LENGTH` 函数索引，需MySQL 8.0.13+

### 9.5 数据库备份建议

```bash
# 完整备份
mysqldump -u root -p --single-transaction --routines --triggers crm_local > backup_$(date +%Y%m%d).sql

# 仅结构
mysqldump -u root -p --no-data crm_local > structure_$(date +%Y%m%d).sql

# 恢复
mysql -u root -p crm_local < backup_20260422.sql
```

## 十一、常见问题与排查

### 11.1 后端启动失败

**问题1：数据库连接失败**
```
Error: connect ECONNREFUSED 127.0.0.1:3306
```
排查步骤：
1. 检查MySQL服务是否运行：`systemctl status mysql`
2. 检查 `.env` 中数据库配置是否正确
3. 检查用户名密码是否正确
4. 检查数据库是否已创建

**问题2：端口被占用**
```
Error: listen EADDRINUSE: address already in use :::3000
```
排查步骤：
1. 查看占用进程：`lsof -i :3000`
2. 终止进程：`kill -9 <PID>`
3. 或修改 `.env` 中的 PORT

**问题3：TypeORM 实体不匹配**
```
QueryFailedError: Unknown column 'xxx' in 'field list'
```
排查步骤：
1. 检查 schema.sql 是否已包含该字段
2. 重新导入 schema.sql（注意备份数据）
3. 或手动执行 ALTER TABLE 添加缺失字段

### 11.2 前端构建失败

**问题1：依赖安装失败**
```bash
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

**问题2：TypeScript 编译错误**
```bash
npm run type-check     # 检查类型错误
```

**问题3：Vite 构建内存不足**
```bash
export NODE_OPTIONS="--max-old-space-size=4096"
npm run build
```

### 11.3 企微相关问题

**问题1：回调验证失败**
- 检查回调URL是否可从外网访问
- 检查Token和EncodingAESKey是否一致
- 检查域名SSL证书是否正常

**问题2：通讯录同步失败**
- 检查应用可见范围是否包含目标部门
- 检查通讯录读取权限是否已申请
- 查看 `wecom_configs` 表中的token是否过期

**问题3：会话存档无数据**
- 确认已购买会话存档许可
- 确认存档密钥配置正确
- 检查 `wecom_archive_settings` 和 `wecom_archive_members` 表

### 11.4 短信发送失败

- 检查短信额度是否充足
- 检查模板是否已通过审核（状态为 active）
- 检查阿里云短信配置（AccessKey、SecretKey、签名）
- 查看 `sms_records` 表中的 `status` 和 `error_message` 字段

### 11.5 支付相关问题

- 检查支付宝/微信支付证书是否正确
- 确认回调URL已配置且可访问
- 查看 `PaymentService.ts` 和 `payment` 路由的日志
- 对账时检查 `capacity_orders` 或 `sms_quota_orders` 表

### 11.6 WebSocket 连接失败

- 检查Nginx是否配置了WebSocket代理
- 确认 `proxy_http_version 1.1` 和 `Upgrade` 头已设置
- 检查防火墙是否放行了WebSocket端口
- 查看浏览器控制台是否有跨域错误

## 十二、安全注意事项

### 12.1 敏感信息保护
- `.env` 文件不要提交到版本库
- JWT密钥使用强随机字符串
- 数据库密码定期更换
- 支付证书严格保密

### 12.2 API安全
- 所有业务API都经过JWT认证
- 登录接口有频率限制（防暴力破解）
- 敏感信息（手机号、身份证等）脱敏显示
- 文件上传有类型和大小限制

### 12.3 数据安全
- 用户密码使用 bcrypt 加密存储
- 定期备份数据库
- 操作日志完整记录
- 多租户数据严格隔离

### 12.4 防护建议
- Nginx配置HTTPS强制跳转
- 启用CORS白名单
- 配置WAF防火墙
- 定期更新依赖包修复安全漏洞
