> 日期: 2026-04-14
> 核心原则: 企微模块独立做精做透，不重构CRM主体

---

## 一、纠正上版方案的错误理解

上一版方案犯了几个严重错误，现在纠正：

### 错误1: 要把企微粉丝灌入CRM客户列表
- 现实: CRM客户列表只存放成交客户，企微那么多粉丝进来是噪音
- 纠正: WecomCustomer和Customer保持独立，各管各的

### 错误2: 要把聊天记录自动转为跟进记录
- 现实: 消息记录太多太杂，跟进记录应该简明扼要
- 纠正: 跟进记录由销售手动填写，想看详细聊天去会话存档

### 错误3: 要大规模重构客户管理和菜单结构
- 现实: CRM主体已经很完善了，没必要翻一遍
- 纠正: 企微模块自己做好自己的事，通过侧边栏等轻量方式和CRM连接

## 三、获客助手+活码 - 全面智能增强设计

这是你最核心的需求，需要从当前的"基础CRUD"升级到"全面智能多维度控制和分析平台"。

### 3.1 当前已有能力

已完成的功能：
- 获客链接CRUD（创建/编辑/删除/列表）
- 权重配置（1-10滑块，预估分配比例）
- 智能调度（工作时段/工作日/备用成员）
- 渠道分析（点击/添加/流失/转化率/留存率）
- 每日趋势图（ECharts折线）
- 标签管理（标签组CRUD，同步企微）
- 使用量监控（配额/告警）
- 二维码展示

### 3.2 需要增强的能力（活码+智能控制+深度分析）

#### A. 活码引擎（当前缺失，需新建）

活码和普通获客链接的本质区别：活码是一个永久不变的二维码，背后的接待人员可以动态切换。

功能设计：

```
活码管理
+-- 创建活码
|   +-- 活码名称
|   +-- 活码类型: 单人码 / 多人轮询码 / 分时段码 / 地域码
|   +-- 分配规则配置（见下方详细设计）
|   +-- 欢迎语配置（支持按时段/渠道/标签差异化）
|   +-- 自动打标签规则
|   +-- 渠道参数(state)配置
|
+-- 活码列表
|   +-- 活码二维码（永久有效，可下载多尺寸）
|   +-- 实时状态（当前接待人/今日添加数/在线状态）
|   +-- 快捷操作（暂停/恢复/复制链接/下载码）
|
+-- 分配规则引擎
    +-- 轮询模式: 按顺序/按权重轮询
    +-- 上限控制: 每人每日/每周/每月上限
    +-- 时段路由: 不同时段分配不同成员
    +-- 在线优先: 优先分配在线状态成员
    +-- 饱和度控制: 当前客户数接近上限时降低权重
    +-- 地域路由: 按客户IP归属地分配对应区域销售
    +-- 标签路由: 按客户已有标签分配专属销售
```

数据模型（新增Entity）：

```
WecomLiveCode（活码表）
- id: number
- tenantId: string
- wecomConfigId: number
- codeName: string (活码名称)
- codeType: 'single' | 'round_robin' | 'time_based' | 'geo_based'

- linkId: string (关联获客链接ID)
- state: string (渠道标识)
- welcomeConfig: JSON {
    default: string,
    timeRules: [{ start: '09:00', end: '18:00', msg: '...' }],
    channelRules: [{ state: 'douyin', msg: '...' }]
  }
- tagRules: JSON [{ condition: 'always' | 'channel' | 'time', tagIds: [] }]
- assignRule: JSON {
    mode: 'round_robin' | 'weighted' | 'time_based' | 'geo_based',
    dailyLimit: number,
    weeklyLimit: number,
    monthlyLimit: number,
    onlinePriority: boolean,
    saturationThreshold: number (0-100%),
    members: [{ userId, weight, geoAreas, timeSlots }]
  }
- status: 'active' | 'paused' | 'expired'
- totalScanCount: number
- totalAddCount: number
- createdBy: string
- createdAt, updatedAt
```

## 五、侧边栏 - 融入企微快速查看CRM客户资料

### 5.1 当前已有能力

- 侧边栏应用管理（CRUD）
- H5绑定账号接口（含安全加固/限流/Referer白名单）
- 客户详情展示（基于authenticateSidebarToken鉴权）
- JS-SDK配置接口
- 已有展示内容：客户基本信息、购买统计、订单列表、物流信息、售后记录、收货信息

### 5.2 侧边栏的定位（关键理解）

侧边栏的核心场景：销售员在企微和客户聊天时，右侧自动弹出这个客户在CRM系统里的完整资料。

不需要在侧边栏里做复杂操作，核心是"快速查看"：
- 这个客户买过什么？花了多少钱？
- 最近订单状态是什么？
- 有没有未处理的售后？
- 上次跟进说了什么？

### 5.3 需要增强的能力

#### A. 客户匹配优化

当前流程：销售员需要手动在侧边栏绑定账号。

增强方案：
```
自动匹配流程:
1. 打开侧边栏 -> JS-SDK获取当前聊天对象的externalUserId
2. 用externalUserId在WecomCustomer表查找
3. 如果WecomCustomer已关联crmCustomerId -> 直接展示CRM客户资料
4. 如果未关联 -> 用WecomCustomer的name/phone模糊匹配Customer表
5. 如果匹配到 -> 提示"是否关联此CRM客户？"一键确认
6. 如果匹配不到 -> 显示"该客户暂未在CRM系统中"
```

```
├── 通讯录          <- 低频
├── 企微联动        <- 低频（成员绑定）
└── 企微授权        <- 低频，排最后
```

CRM主体菜单完全不动，客户管理、订单管理等保持现状。

## 八、SaaS与私有部署差异

| 功能 | SaaS模式 | 私有部署模式 |
|------|---------|-------------|
| 获客助手 | 按套餐限制链接数/活码数 | 无限制 |
| 活码 | 基础版5个/标准版20个/企业版无限 | 无限制 |
| 会话存档 | 增值服务(VAS)，按席位计费 | 客户自行开通 |
| 质检规则 | 按套餐限制规则数 | 无限制 |
| 渠道分析 | 基础版仅7天/标准版30天/企业版365天 | 无限制 |
| 侧边栏 | 全部套餐可用 | 可用 |
| 对外收款 | 全部套餐可用 | 可用 |
| 数据导出 | 标准版以上 | 可用 |

## 十、总结

这版方案的核心思路：

1. **不动CRM主体** - 客户管理、订单管理、数据看板等完全不动
2. **企微模块自己做精** - 在企微管理模块内部把功能做深做透
3. **轻量连接** - 通过侧边栏（查看CRM数据）和客户关联（WecomCustomer.crmCustomerId）实现企微和CRM的数据打通，不需要重构
4. **核心聚焦** - 获客助手+活码做成智能获客平台，会话存档做成全面质检平台

> 等你确认这个方向，我们就可以开始逐个模块写代码了。
