# 登录密码错误重复提示问题修复

## 问题描述

**现象**: 用户在CRM登录页输入错误密码后,点击一次登录按钮,会连续弹出多次错误提示,并且每次提示的剩余尝试次数都在递减。例如:
- 第1次提示: "用户名或密码错误,还可尝试4次"
- 第2次提示: "用户名或密码错误,还可尝试3次"  
- 第3次提示: "用户名或密码错误,还可尝试2次"

**影响**: 用户点击一次登录,实际消耗了3次登录机会,导致账户很快被锁定(5次失败后锁定)。

## 问题根因

### 1. 前端重试机制

在 `src/stores/user/authActions.ts` 中的 `loginWithRetry` 方法:

```typescript
const loginWithRetry = async (username: string, password: string, rememberMe = false, maxRetries = 3, tenantId?: string) => {
  let lastError: unknown

  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      console.log(`[Auth] 登录尝试 ${attempt}/${maxRetries}`)
      const result = await loginWithApi(username, password, rememberMe, tenantId)
      // ...
    } catch (error: unknown) {
      lastError = error
      const errorMessage = error instanceof Error ? error.message : String(error)
      console.error(`[Auth] 登录尝试 ${attempt}/${maxRetries} 失败:`, errorMessage)

      // 🔥 问题所在: 只有在频繁请求(429错误)时才会重试,但密码错误也会重试
      if ((errorMessage.includes('频繁') || errorMessage.includes('429') || errorMessage === 'RATE_LIMITED') && attempt < maxRetries) {
        const delay = Math.min(1000 * Math.pow(2, attempt - 1), 10000)
        console.log(`[Auth] ${delay}ms后重试...`)
        await new Promise(resolve => setTimeout(resolve, delay))
        continue
      }

      if (attempt === maxRetries) {
        throw lastError
      }
    }
  }
}
```

**问题**: 
- 循环会执行3次 (`maxRetries = 3`)
- 每次循环都会调用 `loginWithApi`,即使密码错误也会继续尝试
- 只有在最后一次尝试时才会抛出错误

### 2. 后端失败计数

在 `backend/src/controllers/UserController.ts` 的 `login` 方法中:

```typescript
// 验证密码
const isPasswordValid = await bcrypt.compare(password, user.password);

if (!isPasswordValid) {
  // 🔥 每次请求都会递增失败次数
  user.loginFailCount = (user.loginFailCount || 0) + 1;
  const maxFails = 5;
  const remaining = maxFails - user.loginFailCount;

  // 如果失败次数超过5次，锁定账户
  if (user.loginFailCount >= maxFails) {
    user.status = 'locked';
    user.lockedAt = new Date();
  }

  try {
    await this.userRepository.save(user);
  } catch (_saveError) {
    logger.warn('保存登录失败次数异常:', _saveError);
  }

  // 返回错误
  throw new BusinessError(
    remaining > 0
      ? `用户名或密码错误，还可尝试${remaining}次，超过将锁定账户`
      : '用户名或密码错误',
    'INVALID_CREDENTIALS'
  );
}
```

**结果**: 
- 用户点击一次登录 → 前端重试3次 → 后端失败计数+3
- 用户看到3次错误提示,剩余次数从4→3→2

## 解决方案

### 方案1: 修改前端重试逻辑(推荐)

**修改文件**: `src/stores/user/authActions.ts`

**修改内容**: 密码错误等认证失败不应该重试,只有网络错误或服务器错误才重试

```typescript
const loginWithRetry = async (username: string, password: string, rememberMe = false, maxRetries = 3, tenantId?: string) => {
  let lastError: unknown

  for (let attempt = 1; attempt <= maxRetries; attempt++) {
    try {
      console.log(`[Auth] 登录尝试 ${attempt}/${maxRetries}`)
      const result = await loginWithApi(username, password, rememberMe, tenantId)

      if (result === true) {
        console.log('[Auth] ✅ 登录成功')
        return true
      }

      if (token.value && isLoggedIn.value) {
        console.log('[Auth] ✅ 登录成功（通过状态确认）')
        return true
      }

      console.warn(`[Auth] 登录尝试 ${attempt}/${maxRetries} 返回值异常:`, result)

    } catch (error: unknown) {
      lastError = error
      const errorMessage = error instanceof Error ? error.message : String(error)
      const errorCode = (error as any)?.response?.data?.code || (error as any)?.code || ''
      
      console.error(`[Auth] 登录尝试 ${attempt}/${maxRetries} 失败:`, errorMessage)

      // 🔥 修复: 认证失败(密码错误、账户锁定等)不重试,直接抛出错误
      const authErrors = ['INVALID_CREDENTIALS', 'ACCOUNT_LOCKED', 'ACCOUNT_DISABLED', 'IP_NOT_AUTHORIZED']
      if (authErrors.includes(errorCode)) {
        console.log('[Auth] 认证失败,不重试:', errorCode)
        throw lastError
      }

      // 🔥 只有网络错误、服务器错误、频繁请求才重试
      const shouldRetry = (
        errorMessage.includes('频繁') || 
        errorMessage.includes('429') || 
        errorMessage === 'RATE_LIMITED' ||
        errorMessage.includes('网络') ||
        errorMessage.includes('超时') ||
        errorMessage.includes('服务器')
      )

      if (shouldRetry && attempt < maxRetries) {
        const delay = Math.min(1000 * Math.pow(2, attempt - 1), 10000)
        console.log(`[Auth] ${delay}ms后重试...`)
        await new Promise(resolve => setTimeout(resolve, delay))
        continue
      }

      // 最后一次尝试或不应重试的错误,直接抛出
      if (attempt === maxRetries || !shouldRetry) {
        throw lastError
      }
    }
  }

  throw lastError || new Error('登录失败，请稍后重试')
}
```

**优点**:
- 密码错误只会调用一次后端API
- 用户体验更好,不会看到多次错误提示
- 失败计数准确,点一次登录只消耗一次机会
- 保留了网络错误的重试机制

### 方案3: 后端优化(可选)

**修改文件**: `backend/src/controllers/UserController.ts`

**优化内容**: 添加请求去重机制,防止短时间内重复请求

```typescript
// 在UserController类中添加请求缓存
private loginAttempts = new Map<string, { count: number; timestamp: number }>()

login = catchAsync(async (req: Request, res: Response) => {
  const { username, password, tenantId } = req.body;
  
  // 🔥 防止短时间内重复请求(5秒内相同用户名的请求视为重复)
  const attemptKey = `${username}_${tenantId || 'default'}`
  const now = Date.now()
  const lastAttempt = this.loginAttempts.get(attemptKey)
  
  if (lastAttempt && now - lastAttempt.timestamp < 5000) {
    // 5秒内的重复请求,返回相同的错误,但不增加失败计数
    throw new BusinessError('请勿频繁尝试登录', 'TOO_MANY_ATTEMPTS')
  }
  
  // 记录本次尝试
  this.loginAttempts.set(attemptKey, { count: 1, timestamp: now })
  
  // 清理过期记录(10秒前的)
  for (const [key, value] of this.loginAttempts.entries()) {
    if (now - value.timestamp > 10000) {
      this.loginAttempts.delete(key)
    }
  }
  
  // ... 原有登录逻辑
})
```
