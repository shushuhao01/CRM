### 1.2 数据流向

```
Admin配置:
  套餐模板 -> 会员中心展示价格
  AI API Key -> CRM调用大模型
  服务商suite_id -> 会员中心扫码授权
  会话存档采购成本 -> 定价利润空间

会员购买:
  选择套餐+支付 -> 创建VAS订单 -> Admin审核/自动开通
  -> 写入tenant_settings(席位/额度/到期时间)
  -> CRM实时读取权限状态

CRM使用:
  启动时检查: GET /system/tenant-features
  -> 返回: 企微配额/席位/AI额度/功能开关
  -> 前端按权限渲染菜单和功能
```

### 2.2 问题分析

```
问题1: 缺少服务商应用管理
  - 当前无法在Admin中配置suite_id/suite_secret
  - 无法生成/管理第三方应用授权链接
  - 租户的扫码授权流程无后台支撑

问题2: AI助手管理缺失
  - 无AI模型API Key的统一配置
  - 无AI调用次数的额度管理
  - 无AI计费/收费标准配置
  - 无AI使用量监控

问题3: 会话存档代购流程缺失
  - 客户在我们这里购买,但需要我们去企微服务商下单
  - 采购成本(服务商折扣) vs 售价(我们定价)的管理
  - 订单履约状态跟踪

问题4: 菜单层级不够清晰
  - "增值服务配置"和"套餐模板管理"功能重叠
  - "系统配置"内容杂糅(全局+存档+OSS+通知)
  - 缺少获客助手的使用量管理

## 三、服务商应用管理(核心新增)

> 这是SaaS模式的关键: 我们作为企微服务商,提供第三方应用给租户安装
> 租户只需扫码授权,无需手动配置Secret

### 3.1 服务商资质申请流程

```
=== 前置条件 ===

1. 企业资质:
   - 已注册企业微信(作为服务商)
   - 完成企业实名认证
   - 有有效的营业执照

2. 开发者资质:
   - 企微开放平台注册: https://open.work.weixin.qq.com
   - 完成服务商认证(需缴纳保证金)
   - 获取服务商CorpID

3. 创建第三方应用:
   - 开放平台 -> 应用管理 -> 创建应用
   - 填写应用信息(名称/Logo/简介/类目)
   - 配置应用权限(通讯录/客户联系/会话存档等)
   - 设置授权回调URL
   - 提交审核 -> 审核通过后获得 suite_id

4. 获取的关键凭证:
   - suite_id: 应用套件ID
   - suite_secret: 应用套件Secret
   - suite_ticket: 企微每10分钟推送(需接收回调)
   - 回调URL: 接收企微推送的事件通知

=== 企微API申请权限清单 ===

  基础权限(第三方应用必需):
  [v] 获取部门列表/成员信息 (通讯录读取)
  [v] 外部联系人管理 (客户/客户群)
  [v] 客户联系-联系我(活码)
  [v] 客户联系-获客助手
  [v] 聊天工具栏(侧边栏)

  增值权限(需额外申请):
  [v] 会话存档 (需企业已开通+单独申请)
  [v] 微信客服 (需单独申请)
  [v] 对外收款 (需单独申请)

  注意: 每个权限需在应用配置中勾选
        部分权限需要企微审核通过才能使用
```

### 3.3 服务商授权流程(后端实现要点)

```
=== 后端需新增的API和服务 ===

1. Suite回调接收路由(公开,无需鉴权):
   POST /api/v1/wecom/suite/callback
     - 验证签名(Token+EncodingAESKey)
     - 处理 suite_ticket -> 更新缓存
     - 处理 create_auth -> 保存授权信息
     - 处理 cancel_auth -> 标记取消

2. Suite Token管理服务:
   - getSuiteAccessToken(): 用suite_id+suite_secret+suite_ticket获取
   - 缓存管理(2小时有效期,提前10分钟刷新)

3. 授权链接生成:
   POST /api/v1/admin/wecom-management/suite/auth-link
     - 调用企微API获取 pre_auth_code
     - 拼接授权URL
     - 可选:设置预授权范围(session_info)

4. 永久授权码获取:
   - create_auth回调时,用 auth_code 换取 permanent_code
   - 保存到 wecom_suite_auths 表

5. 企业Token获取(日常API调用):
   getCorpToken(corpId):
     - 用suite_access_token + permanent_code获取
     - 替代原来的 corpId+corpSecret 方式

数据模型:

  WecomSuiteAuth (服务商授权记录表) - 新增
  - id: number (PK)
  - suiteId: string
  - corpId: string (授权企业CorpID)
  - corpName: string (授权企业名称)
  - permanentCode: string (加密存储)
  - authInfo: JSON (授权信息:agentId/权限等)
  - authUserInfo: JSON (授权管理员信息)
  - tenantId: string | null (关联租户,可为空待关联)
  - wecomConfigId: number | null (关联的WecomConfig记录)
  - status: 'active' | 'cancelled' | 'pending'
  - authTime: datetime
  - cancelTime: datetime | null
  - createdAt, updatedAt
```

## 四、会话存档代购管理(核心新增)

> 客户在我们平台购买会话存档 -> 我们向企微服务商API下单 -> 企微开通 -> 客户使用
> 我们可以自己定价(高于采购成本),服务商有折扣

### 4.1 商业模型

```
成本与定价:

  企微官方会话存档定价(参考):
    5人以下:   ¥1,500/年
    5-20人:    ¥300/人/年
    20-50人:   ¥250/人/年
    50-200人:  ¥200/人/年
    200人以上: ¥150/人/年

  服务商折扣(我们的采购成本):
    通常为官方价的 6-8折(具体看服务商等级)
    假设7折: 实际采购成本 = 官方价 x 0.7

  我们的售价(自定义):
    策略: 采购成本 + 服务费(平台维护/技术支持/增值功能)
    建议: 在官方价基础上加 10-30% (因为我们提供了额外价值)
    或: 固定套餐定价(如基础版1000/年/10人)

  利润空间:
    售价(官方价 x 1.1~1.3) - 采购成本(官方价 x 0.7)
    = 官方价 x 0.4~0.6 (约40-60%利润率)

  示例:
    客户买50人/年会话存档
    官方价: 50 x ¥250 = ¥12,500
    采购成本(7折): ¥8,750
    我们售价(1.2倍): ¥15,000
    利润: ¥15,000 - ¥8,750 = ¥6,250 (利润率41.7%)
```

## 十一、服务商收益归属与provider_id绑定

> 关键: 所有扫码授权/应用安装必须携带服务商provider_id
> 只有归属到我们服务商的企业客户,我们才能拿到返利收益
> 会员中心/CRM/推广渠道等所有入口都必须确保provider绑定

### 11.1 企微服务商收益机制

```
=== 企微服务商返利模式(官方规则) ===

核心概念:
  服务商(我们) 推广企微应用/服务给企业客户
  -> 企业客户通过我们的链接/二维码购买企微增值服务
  -> 企微官方给我们返利(约10%)
  -> 返利提现需缴纳3万元保证金

详细说明:
  1. 保证金:
     - 3万元保证金是「提现返利」时才需要缴纳
     - 开发应用、客户授权安装、日常使用都不需要保证金
     - 保证金可退(退出服务商时)
     - 属于合规经营的押金,不是手续费

  2. 返利比例:
     - 接口许可(会话存档等): 约10%返利
     - 获客助手使用量: 约10%返利
     - 具体比例以企微服务商后台为准,可能随政策调整

  3. 客户归属条件(关键!):
     - 客户通过我们的服务商链接授权安装应用
     - 授权时URL必须携带 provider 参数
     - 客户在企微管理后台从"我们的应用"入口购买服务
     - 满足以上条件,该企业才归属于我们,我们才有返利

  4. 归属绑定:
     - 一旦企业通过我们授权安装,归属关系确立
     - 企业后续在企微购买增值服务,我们持续获得返利
     - 如果企业取消授权,归属关系可能解除
```

## 十二、购买模式配置 - 内购 vs 企微直购

> 客户购买会话存档/接口许可有两种方式:
> 方式A: 在我们平台内购(我们代购+加价)
> 方式B: 客户授权后自行去企微购买(我们拿返利)
> 管理后台需要配置开关控制使用哪种模式

### 12.1 两种购买模式对比

```
=== 模式A: 平台内购(代购模式) ===

流程:
  客户在会员中心/CRM中选择套餐 -> 支付给我们
  -> 我们向企微服务商API下单(或手动下单)
  -> 企微开通服务 -> 客户使用

优势:
  - 利润空间大(采购折扣+自定义加价)
  - 客户体验好(一站式购买)
  - 我们掌控定价权
  - 可以捆绑我们的增值服务(AI/质检等)

劣势:
  - 需要垫资(先收客户钱,再向企微下单)
  - 履约环节复杂
  - 如果企微暂未开放服务商代购API,需手动操作

=== 模式B: 企微直购(返利模式) ===

流程:
  客户通过我们的链接授权安装应用(provider绑定)
  -> 客户自行在企业微信管理后台购买会话存档等服务
  -> 企微收款 -> 我们获得约10%返利

优势:
  - 无需垫资
  - 无履约风险
  - 操作简单(引导客户去企微买即可)

劣势:
  - 利润低(仅10%返利)
  - 客户可能跳过我们直接去企微买(如果provider没绑上)
  - 定价由企微控制,我们无法加价

=== 混合模式(推荐) ===

  - 会话存档等高价值服务: 推荐内购(利润高)
  - 获客助手使用量: 可以走返利(单价低,量大)
  - 接口许可费: 按策略选择

  最终由Admin后台的开关决定对外展示哪种购买方式
```

### 13.2 服务商成本与收益计算

```
=== 返利模式(企微直购) ===

  客户购买接口许可: 50人 x ¥50 = ¥2,500
  我们返利(10%): ¥250

  客户购买会话存档(服务版): 50人 x ¥900 = ¥45,000
  我们返利(10%): ¥4,500

  客户购买获客助手: 5000个 x ¥2 = ¥10,000
  我们返利(10%): ¥1,000

  单客户年收益(返利模式): ≈ ¥5,750

=== 平台内购模式(代购+加价) ===

  我们采购成本(假设企微给服务商无额外折扣,仅有返利):
    实际成本 = 官方价  (先全额付企微,后拿10%返利)
    等效成本 = 官方价 x 0.9

  我们的售价(自定义,可以高于或低于官方):
    策略A: 低于官方价(吸引客户走我们内购)
      售价 = 官方价 x 0.95 (客户省5%,我们赚5%+10%返利)
    策略B: 等于官方价(含增值服务)
      售价 = 官方价 (客户同价,但享受我们的AI/质检等增值)
    策略C: 高于官方价(纯增值溢价)
      售价 = 官方价 x 1.2 (客户付溢价,享受全部增值服务)

  建议: 策略B(等价含增值)最合理
    售价 = 官方价, 含我们的AI质检/数据分析/技术支持
    实际利润 = 10%返利 + 增值服务费
    客户体验: 同样价格但得到更多功能,性价比高

=== 示例: 50人企业(会话存档服务版) ===

  返利模式:
    客户去企微买: 50 x ¥900 = ¥45,000
    我们返利: ¥4,500
    利润: ¥4,500

  平台内购(策略B):
    我们售价: ¥45,000(含AI质检+数据分析)
    采购成本: ¥45,000(先付) - ¥4,500(返利) = ¥40,500
    增值服务费: ¥0(含在套餐中,吸引客户)
    利润: ¥4,500 + 增值服务的附加价值

  平台内购(策略A,让利吸引):
    我们售价: ¥42,750(官方95折)
    采购成本: ¥40,500(含返利)
    利润: ¥2,250 + 客户黏性(因为比官方便宜)
```

### 14.1 授权状态枚举

```
授权生命周期:

  [待授权] --> [已授权] --> [正常使用]
                  |              |
                  |              +---> [我方暂停] (Admin操作)
                  |              |       |

                  |              |
                  |              +---> [到期未续费]
                  |
                  +---> [授权失败]

状态定义:
  pending:       待授权(链接已生成,等待扫码)
  authorized:    已授权(正常使用中)
  suspended:     我方暂停(Admin主动操作,如违规)
  cancelled:     客户取消授权(客户在企微后台取消)
  deleted:       客户删除应用(客户在企微后台删除)

  failed:        授权失败(授权过程中出错)

   如果连续多次未响应,可能被暂停推送

2. 接口许可5个起购:
   企业安装我们应用时,至少需要购买5个接口许可
   我们的套餐起售人数不应低于5人

3. 授权安装时的权限确认:
   企业管理员可以在授权时缩小权限范围
   我们需要处理"部分授权"的情况
   比如企业只授权通讯录,不授权会话存档

4. permanent_code 需安全存储:
   永久授权码等同于企业的API密钥
   必须加密存储,不可泄露
   泄露可能导致企业数据被非法访问

5. 接口调用频率限制:
   第三方应用比自建应用限制更严格
   通常: 20次/秒/应用(全局)
   需要做好限流和队列处理

6. 会话存档拉取限制:
   每次最多拉取1000条
   需要游标分页
   建议使用定时任务持续增量拉取

7. 数据合规:
   会话存档数据属于企业机密
   我们作为服务商不得擅自使用/分析企业聊天数据
   需要在隐私协议中明确说明数据处理方式
```

      }

### 18.2 接口许可的收费与管理

```
=== 方案: 含在我们套餐中(推荐) ===

  我们的套餐价格 = 增值服务费 + 接口许可费(代购)

  例:
    云客CRM企微基础套餐: ¥3,000/年/10人
    其中:
      接口许可: 10人 x ¥50 = ¥500(采购成本,返利后¥450)
      平台服务费: ¥2,500(我们的收入)

  客户感知: 只付一笔钱,不用操心接口许可

=== 接口许可超量处理 ===

  场景: 客户买了10人,但企微后台设了15人可见范围

  处理:
  1. CRM系统检测到超量 -> 提示客户增购
  2. Admin监控到超量 -> 通知客户增购
