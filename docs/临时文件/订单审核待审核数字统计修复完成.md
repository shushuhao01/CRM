# 订单审核待审核数字统计修复完成

## 问题描述
CRM系统的订单管理页面中，"取消订单审核"按钮的徽章显示的待审核数量不正确，只记录当前页面数量，而不是按后端API统计返回的总数。

## 根本原因
前端调用了不存在的 `getPendingCancelCount` API (`/orders/pending-cancel-count`)，该API在后端不存在。后端实际只有 `/orders/pending-cancel` API，该API返回分页数据，包含 `pagination.total` 字段表示总数。

## 修复方案

### 1. 前端修改 (src/views/Order/List.vue)
修改 `loadPendingCancelCount` 函数，改为调用 `getPendingCancelOrders` API 并从 `response.pagination.total` 获取总数：

```typescript
const loadPendingCancelCount = async () => {
  try {
    const { orderApi } = await import('@/api/order')
    // 使用 pending-cancel API 获取总数，只请求第一页1条数据即可
    const response = await orderApi.getPendingCancelOrders({
      page: 1,
      pageSize: 1
    })

    if (response && response.pagination) {
      pendingCancelCount.value = response.pagination.total || 0
    }
  } catch (error) {
    console.error('获取待审核取消订单数量失败:', error)
    // 静默失败，不影响用户体验
  }
}
```

### 2. API清理 (src/api/order.ts)
删除不存在的 `getPendingCancelCount` 方法定义。

### 3. 端点清理 (src/api/config.ts)
删除 `PENDING_CANCEL_COUNT` 端点配置。
