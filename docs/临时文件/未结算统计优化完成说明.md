# 未结算统计优化完成说明

## 修改概述

优化了财务管理模块中增值管理和结算报表的"未结算"统计卡片，现在只统计有效和待处理的订单，排除了无效和补单状态的订单。同时添加了鼠标悬浮提示，显示排除的订单数量和金额信息。

## 修改内容

### 1. 后端API修改

#### 1.1 增值管理统计API (`backend/src/routes/valueAdded/valueAddedOrders.ts`)

**修改位置：** `/api/v1/value-added/stats` 接口

**修改内容：**
- 未结算统计查询添加状态过滤条件：`order.status IN ('valid', 'pending')`
- 新增 `unsettledAll` 统计：包含所有未结算订单（含无效和补单），用于tooltip显示

```typescript
// 未结算（排除无效和补单状态，只统计有效和待处理）
queryBuilder.clone()
  .andWhere('order.settlement_status = :settlementStatus', { settlementStatus: 'unsettled' })
  .andWhere('order.status IN (:...validStatuses)', { validStatuses: ['valid', 'pending'] })
  .select([
    'COUNT(*) as count',
    'SUM(order.unit_price) as amount'
  ]).getRawOne(),

// 全部未结算（包含无效和补单，用于tooltip显示）
queryBuilder.clone()
  .andWhere('order.settlement_status = :settlementStatus', { settlementStatus: 'unsettled' })
  .select([
    'COUNT(*) as count',
    'SUM(order.unit_price) as amount'
  ]).getRawOne()
```

**返回数据结构：**
```json
{
  "unsettled": {
    "count": 100,  // 仅有效和待处理的未结算订单数
    "amount": 5000.00
  },
  "unsettledAll": {
    "count": 150,  // 全部未结算订单数（含无效和补单）
    "amount": 7500.00
  }
}
```

## 业务逻辑说明

### 修改前
- 未结算统计包含所有 `settlement_status = 'unsettled'` 的订单
- 包括：有效、待处理、无效、补单等所有状态

### 修改后
- 未结算统计只包含 `settlement_status = 'unsettled'` 且 `status IN ('valid', 'pending')` 的订单
- 排除：无效（invalid）和补单（supplemented）状态的订单
- 原因：无效和补单订单无需结算，不应计入未结算统计

### Tooltip提示信息
用户鼠标悬浮在未结算卡片的问号图标上时，会看到：
1. 当前显示的是有效和待处理的订单
2. 全部未结算订单数（含无效和补单）
3. 已排除的订单数量和金额

## 影响范围

### 直接影响
1. 增值管理页面的未结算统计卡片
2. 结算报表页面的未结算统计卡片

### 不影响
1. 已结算统计（保持不变）
2. 有效/无效/待处理统计（保持不变）
3. 订单列表数据（保持不变）
4. 其他模块功能（保持不变）
