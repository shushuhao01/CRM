# 服务管理菜单显示修复说明

## 问题描述

销售员和部门经理角色在系统管理的角色权限设置中，权限树显示有"服务管理"模块的权限，但左侧菜单栏没有显示"服务管理"菜单项。

## 问题原因

权限标识不一致导致菜单权限匹配失败：

1. **权限树定义**（`src/services/permissionService.js`）：
   - 模块ID: `communication`
   - 权限代码: `communication.call`, `communication.call.view` 等

2. **菜单配置**（`src/config/menu.ts`）：
   - 原来使用: `service`, `service:call`, `service:call:view` 等
   - 与权限树不匹配

3. **菜单渲染逻辑**（`src/utils/menu.ts`）：
   - 通过 `hasMenuPermission` 函数检查用户权限
   - 权限标识不匹配导致权限检查失败
   - 菜单被过滤掉，不显示

## 修复方案

修改 `src/config/menu.ts` 文件，将"服务管理"模块的权限标识统一为 `communication`：

### 1. 菜单配置修改

```typescript
{
  id: 'service-management',
  title: '服务管理',
  icon: 'Phone',
  roles: ['super_admin', 'admin', 'department_manager', 'sales_staff'],
  permissions: ['communication'], // 修改：service -> communication
  children: [
    {
      id: 'service-call',
      title: '通话管理',
      path: '/service-management/call',
      roles: ['super_admin', 'admin', 'department_manager', 'sales_staff', 'customer_service'],
      permissions: ['communication.call'], // 修改：service:call -> communication.call
      dataScope: 'self'
    },
    {
      id: 'service-sms',
      title: '短信管理',
      path: '/service-management/sms',
      roles: ['super_admin', 'admin', 'customer_service'],
      permissions: ['communication.sms'], // 修改：service:sms -> communication.sms
      dataScope: 'all'
    }
  ]
}
```

### 2. 角色权限配置修改

#### 管理员权限

```typescript
  'communication.sms', 'communication.sms.view', 'communication.sms.send', 'communication.sms.template',

```

#### 部门经理权限
```typescript

  // 服务管理权限 - 一级菜单和二级菜单（修复：使用communication）

  // ...
```

#### 销售员权限

```typescript
  // ...

```

## 修复效果

修复后：
1. 销售员和部门经理登录系统后，左侧菜单栏会显示"服务管理"菜单项
2. 展开"服务管理"可以看到"通话管理"子菜单
3. 点击"通话管理"可以正常访问通话管理页面
4. 权限树中的权限设置与菜单显示完全一致

## 技术说明

### 权限匹配机制

菜单工具函数 `hasMenuPermission` 支持两种权限格式的匹配：
- 冒号格式：`customer:list`
- 点号格式：`customer.list`

匹配逻辑：
```typescript
const matchPermission = (requiredPerm: string, userPerms: string[]): boolean => {
  // 直接匹配
  if (userPerms.includes(requiredPerm)) return true
  // 转换冒号为点号后匹配
  const dotFormat = requiredPerm.replace(/:/g, '.')
  if (userPerms.includes(dotFormat)) return true
  // 转换点号为冒号后匹配
  const colonFormat = requiredPerm.replace(/\./g, ':')
  if (userPerms.includes(colonFormat)) return true
  // 检查父级权限
  const parentPerm = requiredPerm.split(/[:.]/)[0]
  if (userPerms.includes(parentPerm)) return true
  return false
}
```

### 为什么使用 communication 而不是 service

1. **避免命名冲突**：`service` 容易与"售后管理"（aftersale）混淆
2. **语义更清晰**：`communication` 明确表示通讯管理（通话+短信）
3. **与权限树一致**：权限树中已经定义为 `communication`

## 注意事项

1. **不要修改权限树**：权限树是权限系统的核心定义，已经在数据库中使用
2. **只修改菜单配置**：通过修改菜单配置来适配权限树
3. **保持一致性**：确保菜单配置、路由配置、权限树三者的权限标识一致
