# 翻页控件显示修复

## 问题描述
业绩分享页面的翻页控件不显示

## 修复时间
2026-02-27

## 问题排查

### 1. 检查模板代码
- ✅ 翻页控件的 HTML 代码存在
- ✅ 没有 `v-if` 条件隐藏
- ✅ 位置在表格下方

### 2. 检查样式问题
发现的问题:
- `:deep(.el-pagination)` 有 `margin-top: 24px`,可能导致布局问题
- 缺少 `display: flex` 属性

### 3. 修复措施

#### 修复 1: 优化分页容器样式
```css
.pagination-wrapper {
  display: flex;
  justify-content: center;
  margin-top: 16px;
  margin-bottom: 20px;  /* 新增底部间距 */
  padding: 16px 20px;
  background: white;
  border-radius: 8px;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.06);
  position: relative;  /* 新增定位 */
  z-index: 1;  /* 新增层级 */
  min-height: 60px;  /* 新增最小高度,确保可见 */
}
```

#### 修复 2: 优化 el-pagination 样式
```css
:deep(.el-pagination) {
  display: flex;  /* 新增 */
  justify-content: center;
  align-items: center;  /* 新增 */
  /* 移除 margin-top: 24px */
}
```

## 可能的原因

### 1. 样式冲突
- `.pagination-wrapper` 的 `margin-top` 和 `:deep(.el-pagination)` 的 `margin-top` 冲突
- 缺少 `display: flex` 导致布局异常

### 2. 层级问题
- 可能被其他元素遮挡
- 添加 `z-index: 1` 确保在最上层

### 3. 高度问题
- 容器高度为 0 导致不可见
- 添加 `min-height: 60px` 确保有足够高度

## 验证步骤

1. 刷新页面,检查翻页控件是否显示
2. 检查浏览器开发者工具:
   - Elements 面板查看 `.pagination-wrapper` 元素是否存在
   - 查看元素的计算样式(Computed)
   - 检查是否有 `display: none` 或 `visibility: hidden`
   - 查看元素的尺寸(width, height)

3. 检查控制台是否有错误信息

4. 测试翻页功能:
   - 切换每页条数
   - 点击页码
   - 使用跳转输入框

## 临时调试方法

如果仍然不显示,可以添加调试样式:
```vue
<div class="pagination-wrapper" style="border: 2px solid red; min-height: 100px;">
  <el-pagination ... />
</div>
```

这样可以看到容器是否存在,以及位置在哪里。
