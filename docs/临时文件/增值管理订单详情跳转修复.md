# 增值管理订单详情跳转修复

## 问题描述

用户反馈：在增值管理列表中点击订单号超链接，提示"订单不存在"，但在订单管理模块中该订单是存在的。

## 根本原因

**错误代码**：
```vue
<el-link type="primary" @click="goToOrderDetail(row.id)">
  {{ row.orderNumber || '-' }}
</el-link>
```

**问题分析**：
1. `row.id` 是增值订单表（`value_added_orders`）的主键 ID
2. 订单详情页需要的是原始订单表（`orders`）的 ID
3. 增值订单表中有 `orderId` 字段，存储的是原始订单的 ID
4. 使用错误的 ID 导致订单详情页找不到订单

## 数据表关系

```
orders 表（原始订单）
├─ id: 订单主键
├─ order_number: 订单号
└─ ...

value_added_orders 表（增值订单）
├─ id: 增值订单主键（UUID）
├─ order_id: 关联 orders.id
├─ order_number: 订单号（冗余字段）
└─ ...
```

## 修复方案

**文件**：`src/views/Finance/ValueAddedManage.vue`

```vue
<!-- ❌ 错误：使用增值订单ID -->
<el-link type="primary" @click="goToOrderDetail(row.id)">
  {{ row.orderNumber || '-' }}
</el-link>

<!-- ✅ 正确：使用原始订单ID -->
<el-link type="primary" @click="goToOrderDetail(row.orderId)">
  {{ row.orderNumber || '-' }}
</el-link>
```

**goToOrderDetail 函数**（无需修改）：
```javascript
const goToOrderDetail = (id: string) => {
  if (id) router.push(`/order/detail/${id}`)
}
```

- ✅ 订单详情页显示完整的订单信息

## 相关文件

- `src/views/Finance/ValueAddedManage.vue` - 增值管理主页面
- `src/views/Order/Detail.vue` - 订单详情页

## 总结

修复很简单，只需将 `row.id` 改为 `row.orderId`，使用正确的订单 ID 进行跳转。这是一个典型的关联表 ID 使用错误问题。
