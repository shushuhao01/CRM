# 生产环境用户管理500错误修复

**问题时间**: 2026-03-05  
**问题描述**: 生产环境用户管理页面报500错误,开发环境正常  
**错误信息**: GET /api/v1/roles 500 (Internal Server Error)

---

## 🔍 问题分析

### 错误日志

```
GET https://abc789.cn/api/v1/roles 500 (Internal Server Error)
[RoleAPI] API添加失败，尝试从localStorage获取
[RoleAPI] 错误详情: Request failed with status code 500
```

### 根本原因

**生产环境和开发环境的 `roles` 表结构不一致!**

#### 开发环境roles表字段

```sql
- id
- name
- code              ✅ 有
- display_name
- description
- status            ✅ 有
- level             ✅ 有
- color             ✅ 有
- permissions       ✅ 有
- roleType          ✅ 有
- data_scope        ✅ 有
- is_system
- created_at
- updated_at
```

#### 生产环境roles表字段

```sql
- id
- name
- display_name
- description
- is_system
- created_at
- updated_at
```

**缺失字段**:
- ❌ `code` - 角色代码
- ❌ `status` - 状态
- ❌ `level` - 角色级别
- ❌ `color` - 角色颜色
- ❌ `permissions` - 权限列表
- ❌ `roleType` - 角色类型
- ❌ `data_scope` - 数据范围

### 为什么会500错误?

后端代码 `backend/src/controllers/RoleController.ts` 的 `getRoles` 方法中:

```typescript
const roles = await dataSource.query(
  `SELECT id, name, code, description, status, level, color, permissions, roleType, data_scope as dataScope, created_at as createdAt, updated_at as updatedAt
   FROM roles ${whereClause} ORDER BY level ASC, created_at DESC LIMIT ? OFFSET ?`,
  [...params, Number(limit), offset]
);
```

查询语句包含了 `code`, `status`, `level`, `color`, `permissions`, `roleType`, `data_scope` 这些字段,但生产环境的表中没有这些字段,导致SQL执行失败,返回500错误。

---

## ✅ 修复方案

### 方案: 添加缺失字段到生产环境数据库

在生产环境的 `roles` 表中添加缺失的字段。

---

## 📝 修复步骤

### 步骤1: 诊断生产环境表结构

```bash
cd backend
node diagnose-roles-table-production.js
```

这个脚本会:
1. 连接生产环境数据库
2. 查询roles表结构
3. 检查缺失的字段
4. 生成修复SQL

### 步骤2: 在宝塔执行修复SQL

**文件**: `backend/database-migrations/fix-roles-table-production.sql`

**执行方式**:
1. 登录宝塔面板
2. 进入"数据库"
3. 选择生产环境数据库
4. 点击"管理" > "SQL"
5. 复制并执行以下SQL:

```sql
-- 添加缺失字段到roles表
ALTER TABLE `roles`
  ADD COLUMN `code` varchar(50) NOT NULL DEFAULT '' COMMENT '角色代码' AFTER `name`,
  ADD COLUMN `status` varchar(20) NOT NULL DEFAULT 'active' COMMENT '状态' AFTER `description`,
  ADD COLUMN `level` int(11) NOT NULL DEFAULT 0 COMMENT '角色级别' AFTER `status`,
  ADD COLUMN `color` varchar(20) DEFAULT NULL COMMENT '角色颜色' AFTER `level`,
  ADD COLUMN `permissions` json DEFAULT NULL COMMENT '权限列表' AFTER `color`,
  ADD COLUMN `roleType` varchar(20) DEFAULT NULL COMMENT '角色类型' AFTER `permissions`,
  ADD COLUMN `data_scope` varchar(20) DEFAULT NULL COMMENT '数据范围' AFTER `roleType`;

-- 添加索引
ALTER TABLE `roles`
  ADD UNIQUE KEY `uk_code` (`code`),
  ADD KEY `idx_status` (`status`);

-- 更新现有数据
UPDATE `roles` SET `code` = `name` WHERE `code` = '';
UPDATE `roles` SET `roleType` = IF(`is_system` = 1, 'system', 'custom');
UPDATE `roles` SET `data_scope` = 'all';
```
