# 客户状态字段说明与优化建议

## 当前状态

### 客户状态字段值
系统目前支持以下客户状态：
1. **活跃** (active) - 绿色标签
2. **非活跃** (inactive) - 灰色标签
3. **潜在客户** (potential) - 橙色标签
4. **流失客户** (lost) - 红色标签
5. **黑名单** (blacklist) - 红色标签

## 问题分析

### 1. "活跃"和"非活跃"判断标准不明确
目前系统中没有自动判断客户是否活跃的逻辑，这两个状态需要手动设置，但缺乏明确的判断标准：
- 多久没有下单算非活跃？
- 多久没有联系算非活跃？
- 是否需要自动更新状态？

### 2. 状态分类不够清晰
- "活跃/非活跃"是基于行为的状态
- "潜在客户"是基于转化阶段的状态
- "流失客户"是基于结果的状态
- "黑名单"是基于信用的状态

这些状态维度不统一，容易混淆。

## 优化建议

### 方案一：保留并完善（推荐）
保留客户状态字段，但需要：

1. **明确判断标准**
   - 活跃：最近30天内有下单或联系记录
   - 非活跃：30-90天内无下单或联系记录
   - 流失：90天以上无任何互动
   - 潜在客户：有意向但未下单
   - 黑名单：恶意客户或信用不良

2. **自动更新机制**
   - 系统每天自动根据最后下单时间和最后联系时间更新状态
   - 也支持手动修改状态

3. **增加状态说明**
   - 在客户详情页显示状态判断依据
   - 例如："活跃（最后下单：3天前）"

### 方案二：简化状态
将状态简化为更明确的分类：

1. **正常客户** (normal) - 可以正常交易
2. **潜在客户** (potential) - 有意向但未成交
3. **黑名单** (blacklist) - 禁止交易

去掉"活跃/非活跃/流失"这些需要复杂判断的状态。

### 方案三：分离维度
将不同维度的状态分开：

1. **客户类型**：正常客户、潜在客户、黑名单
2. **活跃度**：活跃、一般、沉睡（自动计算）
3. **风险等级**：正常、关注、高风险

## 实施建议

### 短期（推荐）
1. 保留现有字段，不做大改动
2. 在客户详情页添加状态说明
3. 提供状态修改功能，让销售人员手动维护

### 长期
1. 实现自动状态更新机制
2. 根据实际业务需求调整状态分类
3. 增加客户活跃度分析报表

**建议保留客户状态字段**，因为它对客户管理有实际价值，但需要：
1. 明确各状态的定义和判断标准
2. 提供清晰的状态说明
3. 考虑后续添加自动更新机制

如果觉得当前状态分类过于复杂，可以先简化为"正常、潜在、黑名单"三种状态，后续再根据需求扩展。
