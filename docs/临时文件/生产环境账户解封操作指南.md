# 生产环境账户解封操作指南

## 问题描述
CRM系统成员多次登录密码错误后，账户被自动锁定（status = 'locked'），目前管理后台用户管理页面还没有开发解封按钮。

## 账户锁定机制
根据代码分析，账户锁定涉及以下字段：
- `status`: 账户状态，值为 'locked' 表示已锁定
- `login_fail_count`: 登录失败次数
- `locked_at`: 锁定时间

## 生产环境解封方法

### 方法一：通过宝塔面板 phpMyAdmin（推荐）

1. 登录宝塔面板
2. 进入数据库管理 → phpMyAdmin
3. 选择你的CRM数据库
4. 执行以下SQL语句：

```sql
-- 查询被锁定的用户
SELECT id, username, name, status, login_fail_count, locked_at, tenant_id
FROM users
WHERE status = 'locked'
ORDER BY locked_at DESC;
```

5. 找到需要解封的用户后，执行解封SQL：

```sql
-- 解封指定用户（替换 'user_id_here' 为实际用户ID）
UPDATE users
SET 
  status = 'active',
  login_fail_count = 0,
  locked_at = NULL
WHERE id = 'user_id_here';
```

6. 如果需要批量解封某个租户的所有锁定用户：

```sql
-- 批量解封指定租户的所有锁定用户（替换 'tenant_id_here' 为实际租户ID）
UPDATE users
SET 
  status = 'active',
  login_fail_count = 0,
  locked_at = NULL
WHERE status = 'locked' 
  AND tenant_id = 'tenant_id_here';
```

7. 验证解封结果：

```sql
-- 再次查询确认状态已更新
SELECT id, username, name, status, login_fail_count, locked_at
FROM users
WHERE id = 'user_id_here';
```

### 方法二：通过SSH连接MySQL（适用于有SSH权限）

1. SSH连接到服务器
2. 登录MySQL：

```bash
mysql -u root -p
```

3. 选择数据库：

```sql
USE your_database_name;
```

4. 执行上述相同的SQL语句进行解封

### 方法三：通过Navicat等数据库管理工具

1. 使用Navicat连接到生产环境数据库
2. 打开查询窗口
3. 执行上述SQL语句

## 完整解封SQL模板

```sql
-- ============================================
-- CRM账户解封SQL脚本
-- 使用前请先查询确认用户信息
-- ============================================

-- 第一步：查询被锁定的用户列表
SELECT 
  id,
  username,
  name,
  real_name,
  phone,
  status,
  login_fail_count,
  locked_at,
  tenant_id,
  department_name
FROM users
WHERE status = 'locked'
ORDER BY locked_at DESC;

-- 第二步：解封单个用户（根据用户名）
UPDATE users
SET 
  status = 'active',
  login_fail_count = 0,
  locked_at = NULL
WHERE username = '被锁定的用户名';

-- 或者：解封单个用户（根据用户ID）
UPDATE users
SET 
  status = 'active',
  login_fail_count = 0,
  locked_at = NULL
WHERE id = '用户ID';

-- 第三步：验证解封结果
SELECT 
  id,
  username,
  name,
  status,
  login_fail_count,
  locked_at
FROM users
WHERE username = '被锁定的用户名';

-- 可选：批量解封所有锁定用户（谨慎使用）
-- UPDATE users
-- SET 
--   status = 'active',
--   login_fail_count = 0,
--   locked_at = NULL
-- WHERE status = 'locked';
```

## 注意事项

1. **执行前务必备份**：在执行UPDATE语句前，建议先备份users表
2. **确认用户身份**：先用SELECT查询确认是正确的用户
3. **租户隔离**：如果是多租户环境，注意添加 `tenant_id` 条件
4. **记录操作**：建议记录解封操作的时间和原因
5. **通知用户**：解封后通知用户可以重新登录

## 预防措施

建议后续开发以下功能：
1. 在用户管理页面添加"解封"按钮
2. 添加自动解封机制（如15分钟后自动解封）
3. 添加管理员手动重置密码功能
4. 记录账户锁定和解封的操作日志

## 快速操作示例

假设用户名为 `zhangsan` 的账户被锁定：

```sql
-- 1. 查询确认
SELECT id, username, name, status FROM users WHERE username = 'zhangsan';

-- 2. 解封
UPDATE users SET status = 'active', login_fail_count = 0, locked_at = NULL WHERE username = 'zhangsan';

-- 3. 验证
SELECT id, username, name, status FROM users WHERE username = 'zhangsan';
```

执行完成后，用户即可正常登录。
