# 个人业绩客户明细权限修复说明

**修复时间**: 2026-03-05  
**问题描述**: 生产环境个人业绩页面的客户明细列表没有按角色过滤,所有人都能看到所有客户  
**修复目标**: 不管是谁(包括管理员),在个人业绩页面都只能看到自己的客户

---

## 🐛 问题分析

### 原始问题

在"业绩统计 > 个人业绩"页面中,客户明细列表调用的是通用的客户列表API `/api/v1/customers`,该API的权限过滤逻辑是:

- **管理员/超级管理员**: 可以看到所有客户
- **部门经理**: 可以看到本部门的客户
- **普通成员**: 只能看到自己的客户

但在"个人业绩"页面中,应该**不管是谁,都只能看到自己的客户**,即使是管理员也应该只看到自己创建或负责的客户。

### 根本原因

后端API没有提供"强制只查询当前用户客户"的参数,导致管理员在个人业绩页面也能看到所有客户。

---

## ✅ 修复方案

### 方案概述

在后端客户列表API中添加 `onlyMine` 参数,当传入 `onlyMine=true` 时,强制只返回当前用户的客户,不管用户角色是什么。

### 修改文件

#### 1. 后端API修改

**文件**: `backend/src/routes/customers.ts`

**修改内容**:

```typescript
// 添加 onlyMine 参数
const {
  page = 1,
  pageSize = 10,
  name,
  phone,
  keyword,
  level,
  status,
  startDate,
  endDate,
  onlyMine  // 🔥 新增：强制只查询当前用户的客户（用于个人业绩页面）
} = req.query;

// 权限过滤逻辑
if (onlyMine === 'true' || onlyMine === true) {
  // 🔥 强制只查询当前用户的客户（用于个人业绩页面）
  console.log('[客户列表] 强制只查询当前用户的客户, userId:', userId);
  
  // 获取分享给当前用户的客户
  const shareRepository = AppDataSource.getRepository(CustomerShare);
  const sharedCustomers = await shareRepository.find({
    where: {
      sharedTo: userId,
      status: 'active'
    },
    select: ['customerId']
  });
  const sharedCustomerIds = sharedCustomers.map(s => s.customerId);

  if (sharedCustomerIds.length > 0) {
    queryBuilder.where(
      '(customer.createdBy = :userId OR customer.salesPersonId = :userId OR customer.id IN (:...sharedIds))',
      { userId, sharedIds: sharedCustomerIds }
    );
  } else {
    queryBuilder.where(
      '(customer.createdBy = :userId OR customer.salesPersonId = :userId)',
      { userId }
    );
  }
} else if (userRole !== 'admin' && userRole !== 'super_admin') {
  // 原有的角色权限过滤逻辑
  // ...
}
```
