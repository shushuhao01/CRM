# 客户重复验证提示优化

## 问题描述

在客户管理模块的新增客户页面，当验证客户手机号已存在时：
- 如果是其他人创建的客户，会提示"该客户由其他成员创建，您没有权限保存修改"
- 但如果是自己创建的客户，也会显示同样的提示，这不够友好

## 优化方案

区分两种情况，显示不同的提示信息：

### 1. 自己创建的客户
提示：**您已经创建过该客户资料，无需重复创建。可以直接前往客户列表查看客户信息。**

### 2. 其他人创建的客户
提示：**该客户由其他成员创建，您没有权限保存修改。如需操作，请联系管理员或客户创建者。**

## 实现细节

### 前端修改 (src/views/Customer/Add.vue)

#### 1. 更新验证结果类型定义
```typescript
const customerVerifyResult = ref<{
  type: 'success' | 'warning' | 'error'
  message: string
  owner?: string
  createTime?: string
  customerId?: string
  isOwnCustomer?: boolean // 新增：标记是否是自己创建的客户
} | null>(null)
```

#### 2. 验证客户时判断归属
```typescript
if (response.data) {
  const existingCustomer = response.data
  const isOwnCustomer = existingCustomer.creatorName === userStore.currentUser?.name
  
  customerVerifyResult.value = {
    type: 'warning',
    message: isOwnCustomer ? '该手机号已存在客户记录' : '该手机号已存在客户记录',
    owner: existingCustomer.creatorName || existingCustomer.name,
    createTime: existingCustomer.createTime,
    customerId: existingCustomer.id,
    isOwnCustomer // 标记是否是自己创建的
  }
}
```

#### 3. 模板中根据归属显示不同提示
```vue
<div v-if="shouldDisableSave" class="permission-warning">
  <el-divider />
  <p v-if="customerVerifyResult.isOwnCustomer" style="color: #e6a23c; font-weight: 500;">
    <el-icon><Warning /></el-icon>
    提示：您已经创建过该客户资料，无需重复创建。可以直接前往客户列表查看客户信息。
  </p>
  <p v-else style="color: #e6a23c; font-weight: 500;">
    <el-icon><Warning /></el-icon>
    权限提示：该客户由其他成员创建，您没有权限保存修改。如需操作，请联系管理员或客户创建者。
  </p>
</div>
```

### 后端接口 (backend/src/routes/customers/core.ts)

后端 `/check-exists` 接口已经返回了 `creatorName` 字段，无需修改：

```typescript
return res.json({
  success: true,
  code: 200,
  message: '该手机号已存在客户记录',
  data: {
    id: existingCustomer.id,
    name: existingCustomer.name,
    phone: existingCustomer.phone,
    creatorName: ownerName, // 创建者姓名
    createTime: existingCustomer.createdAt?.toISOString() || ''
  }
});
```

## 用户体验提升

### 修改前
无论是谁创建的客户，都显示：
> ⚠️ 权限提示：该客户由其他成员创建，您没有权限保存修改。如需操作，请联系管理员或客户创建者。

### 修改后

**场景1：自己创建的客户**
> ⚠️ 提示：您已经创建过该客户资料，无需重复创建。可以直接前往客户列表查看客户信息。

**场景2：其他人创建的客户**
> ⚠️ 权限提示：该客户由其他成员创建，您没有权限保存修改。如需操作，请联系管理员或客户创建者。

## 测试建议

1. **测试自己创建的客户**
   - 创建一个新客户
   - 再次尝试用相同手机号创建
   - 验证提示信息是否显示"您已经创建过该客户资料"

2. **测试其他人创建的客户**
   - 用另一个账号创建客户
   - 切换回当前账号，尝试用相同手机号创建
   - 验证提示信息是否显示"该客户由其他成员创建"

3. **验证查看详情按钮**
   - 点击"查看客户详情"按钮
   - 确认能正确跳转到客户详情页
