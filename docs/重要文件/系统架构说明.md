## 项目概述

这是一个基于 Vue 3 + TypeScript + Element Plus 的现代化 CRM 客户关系管理系统。项目采用 **四系统联合架构**，包含 CRM 主系统、管理后台（Admin）、官网（Website）和移动端APP（crmAPP），支持 **SaaS 多租户** 和 **私有部署** 两种模式。

## 四系统架构总览

```
┌───────────────────────────────────────────────────────────────────┐
│                    CRM 平台总体架构                                │
├─────────────┬─────────────┬──────────────┬────────────────────────┤
│  CRM 主系统  │  管理后台    │    官网      │   移动端 APP            │
│  (Vue 3)    │  (Admin)    │  (Website)   │   (uni-app)            │
│  端口:5173  │  端口:5174  │  端口:5175   │   H5/微信小程序/App    │
├─────────────┴─────────────┴──────────────┴────────────────────────┤
│                    共用后端 API (Express)                          │
│                    端口:3000  API前缀:/api/v1                     │
├───────────────────────────────────────────────────────────────────┤
│                    MySQL 数据库 (127张表)                          │
│                    支持多租户 tenant_id 数据隔离                    │
└───────────────────────────────────────────────────────────────────┘
```

| 系统 | 目录 | 技术栈 | 作用 |
|------|------|--------|------|
| CRM 主系统 | `/src` | Vue 3 + Element Plus | 销售团队日常使用的核心业务系统 |
| 管理后台 | `/admin` | Vue 3 + Element Plus | 平台运营方管理租户、授权、套餐、版本 |
| 官网 | `/website` | Vue 3 | 产品展示、注册、套餐购买 |
| 移动端 APP | `/crmAPP` | uni-app (Vue 3) | 销售人员外出使用的移动端 |
| 后端 | `/backend` | Express + TypeORM + MySQL | 共用API服务，支持WebSocket实时推送 |

## 部署模式

系统支持两种部署模式，通过环境变量 + RSA 签名许可证控制：

| 模式 | 环境变量 | 说明 |
|------|----------|------|
| 私有部署 | `DEPLOY_MODE=private` | 卖给客户独立部署，单租户 |
| SaaS 多租户 | `DEPLOY_MODE=saas` + `SAAS_LICENSE_TOKEN` | 平台运营方自己运营，多租户数据隔离 |

> ⚠️ SaaS 模式受 RSA 签名守卫保护（`SaaSGuardService`），私有客户无法擅自启用。

### 3. 数据存储模块
**位置**: `backend/src/config/database.ts`, `backend/src/entities/`

**存储策略**:
- **生产环境**: MySQL 数据库（127张表，TypeORM实体映射）
- **多租户隔离**: 所有业务表含 `tenant_id` 字段，通过中间件自动过滤
- **智能数据库切换**: 根据 NODE_ENV 自动选择 .env.local（开发）或 .env（生产）

**自动切换**:
```typescript
// 系统自动检测环境并切换存储方式
const isProduction = import.meta.env.VITE_USE_REAL_API === 'true'

if (isProduction) {
  // 使用后端 API
  await axios.post('/api/customers', data)
} else {
  // 使用 localStorage
  localStorage.setItem('crm_customers', JSON.stringify(data))
}
```

**数据键名规范**:

## 安全机制

### 1. 认证安全
- JWT Token 认证
- Token 过期自动刷新
- 登录失败次数限制

### 2. 权限安全
- 前端路由守卫
- 后端接口权限验证
- **多租户数据隔离**（tenant_id 全链路过滤）

### 3. 数据安全
- 敏感数据加密存储
- 电话号码脱敏显示
- 操作日志记录
- **敏感信息权限管理**（按角色控制可见字段）

### 4. SaaS 部署安全
- **RSA 签名守卫**：SaaS 模式需要有效的 SAAS_LICENSE_TOKEN
- **RSA 非对称加密**：公钥验签硬编码，私钥不随代码交付
- **自动降级**：无有效许可证时强制运行为私有模式

## 常见问题

### Q: 如何添加新的权限？
A: 在 `src/config/defaultRolePermissions.ts` 中添加权限代码，然后在对应角色的 `permissions` 数组中添加。

### Q: 如何添加新的菜单？
A: 在 `src/router/index.ts` 中添加路由，并在路由 meta 中配置权限。

### Q: 如何切换开发/生产环境？
A: 修改 `.env.development` 或 `.env.production` 中的 `VITE_USE_REAL_API` 变量。

### Q: 如何对接新的第三方服务？
A: 在 `backend/src/services/` 中创建新的服务类，在 `backend/src/routes/` 中添加路由。
