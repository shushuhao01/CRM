## 🔄 完整支付流程

```
官网注册 → 选择套餐 → 创建订单 → 支付 → 回调处理 → 生成授权码 → 发送短信 → CRM激活
```

### 流程详解

1. **用户在官网注册**
   - 填写公司信息、联系方式
   - 选择套餐类型

2. **创建支付订单**
   - 生成订单号
   - 返回支付二维码

3. **用户完成支付**
   - 扫码支付（微信/支付宝）
   - 支付平台回调通知

4. **系统处理回调**
   - 验证签名
   - 更新订单状态
   - 生成授权码（TENANT-XXXX-XXXX-XXXX-XXXX）
   - 激活租户
   - 发送短信通知

5. **CRM登录验证**
   - 用户输入授权码
   - 验证租户身份
   - 显示租户信息
   - 允许登录

## 🔐 部署模式

### SaaS模式（多租户）
- 需要授权码验证
- 支持多租户隔离
- 心跳检测授权状态
- 后端配置：`DEPLOY_MODE=saas` + `SAAS_LICENSE_TOKEN=<RSA签名JWT>`
- 前端配置：`VITE_DEPLOY_MODE=saas`（或由授权码类型自动识别）
- ⚠️ SaaS 模式受 RSA 签名守卫保护，无有效 Token 自动降级为私有模式

### 私有部署模式
- 使用私有授权码激活（`PRIV-` 前缀）
- 单租户使用
- 后端配置：`DEPLOY_MODE=private`
- 前端配置：`VITE_DEPLOY_MODE=private`（或由授权码类型自动识别）

## 🧪 测试方法

### 1. 开发环境快速测试

```bash
# 1. 注册租户
curl -X POST http://localhost:3000/api/v1/public/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "测试公司",
    "phone": "13800138000",
    "email": "test@example.com",
    "packageCode": "SAAS_PRO"
  }'

# 2. 创建支付订单
curl -X POST http://localhost:3000/api/v1/public/payment/create \
  -H "Content-Type: application/json" \
  -d '{
    "tenantId": "租户ID",
    "packageId": 2,
    "payType": "wechat",
    "billingCycle": "monthly"
  }'

# 3. 模拟支付成功
curl -X POST http://localhost:3000/api/v1/public/payment/mock-pay/订单号

# 4. 验证授权码
curl -X POST http://localhost:3000/api/v1/tenant-license/verify \
  -H "Content-Type: application/json" \
  -d '{"licenseKey": "TENANT-XXXX-XXXX-XXXX-XXXX"}'
```

### 2. CRM登录测试

1. 设置环境变量 `VITE_DEPLOY_MODE=saas`
2. 访问 CRM 登录页面
3. 系统会弹出授权码验证弹窗
4. 输入授权码验证
5. 验证成功后显示租户信息
6. 输入用户名密码登录
