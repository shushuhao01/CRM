## 推送方式说明

### 使用的方法
采用了 `git subtree push` 方式推送，这种方式会：
- ✅ 保留完整的Git提交历史
- ✅ 只推送指定目录的内容
- ✅ 自动过滤其他目录
- ⚠️ 需要处理所有历史提交（994个），耗时较长

### 推送过程
```bash
# Admin后台推送
git subtree push --prefix=admin crm-admin main
# 结果: 63个对象，成功推送

# 官网推送  
git subtree push --prefix=website crm-website main
# 结果: 成功推送（后台进程完成）
```

## 常见问题

### Q: 为什么推送这么慢？
A: git subtree需要处理所有历史提交(994个)，第一次推送会比较慢，后续更新会快很多。

### Q: 如何更新已推送的仓库？
A: 继续使用 `git subtree push` 命令，Git会自动计算增量更新。

### Q: 本地开发会受影响吗？
A: 完全不会！本地继续在同一目录开发，只是GitHub上分离了仓库。

### Q: 客户能看到完整历史吗？
A: 使用git subtree推送的仓库包含完整历史。如果不想包含历史，可以使用快速推送脚本。
