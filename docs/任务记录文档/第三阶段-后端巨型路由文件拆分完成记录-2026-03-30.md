## 🔧 拆分方法

每个子路由文件导出一个 `registerXxxRoutes(router: Router): void` 函数，由 `index.ts` 创建 Router 实例并调用各注册函数：

```typescript
// index.ts 模式
import { Router } from 'express';
import { registerSettingsRoutes } from './systemSettings';
import { registerConfigRoutes } from './systemConfig';

const router = Router();
registerSettingsRoutes(router);
registerConfigRoutes(router);
export default router;
```

## 🔍 修复的关键问题

1. **动态import路径修复**：拆分后文件多了一级目录，`import('../entities/xxx')` 需改为 `import('../../entities/xxx')`
2. **函数内import提升**：原始文件中部的 `import` 语句在包裹进函数后不合法（TS1232），需提升到文件顶部
3. **跨文件函数引用**：`valueAdded` 中的 `updateCompanyStats`、`findPriceByOrderDate`、`syncOrdersToValueAdded` 等函数被两个子文件共用，提取到 `valueAddedHelpers.ts`
4. **缺失import补全**：`systemSettings.ts` 缺少 `AppDataSource`，`logisticsCompany.ts` 缺少 `logisticsTraceService` 等
5. **UTF-8编码处理**：PowerShell处理大文件时编码问题，改用Node.js脚本处理
