## 任务一：列宽与操作按钮布局优化

### 问题描述

1. **套餐列宽度过大**：套餐列使用了 `1fr` 占据全部剩余空间，但实际内容（如"企业版"、"专业版"）很短，导致该列浪费大量空间，而订单号列反而因 `130px` 固定宽度被截断。
2. **操作列按钮文字纵向堆叠**：操作列宽度仅 `90px`，"支付"和"取消"两个按钮被挤压成纵向排列，导致有操作按钮的行高度明显大于其他行，页面行高不一致，视觉效果差。

### 修复方案

#### 1. 调整 grid 列宽分配

**修改前：**
```css
grid-template-columns: 90px 130px 80px 1fr 70px 80px 72px 90px;
/*                      日期  订单号  类型  套餐  金额  支付方式 状态  操作 */
```

**修改后：**
```css
grid-template-columns: 90px 1fr 80px auto 70px 80px 72px 110px;
/*                      日期  订单号  类型  套餐  金额  支付方式 状态  操作 */
```

- **订单号列**：`130px` → `1fr`，长订单号获得充足空间展示
- **套餐列**：`1fr` → `auto`，按内容自适应，不再浪费空间
- **操作列**：`90px` → `110px`，给按钮横向排列留出空间

#### 2. 操作列按钮强制横向排列

**`.col-action` 样式增加：**
```css
white-space: nowrap;
flex-wrap: nowrap;
```

**`.btn-action` 样式增加：**
```css
white-space: nowrap;
flex-shrink: 0;
```

## 任务二：添加搜索栏功能

### 需求
在账单记录标题栏下方添加搜索栏，支持：
- **订单号搜索**：模糊匹配订单号
- **日期范围筛选**：按开始/结束日期筛选

### 实现方案（全链路）

#### 1. 后端 Service 层 (`backend/src/services/MemberService.ts`)
- `getBills` 方法新增可选 `filters` 参数：`{ orderNo?, startDate?, endDate? }`
- 动态拼接 SQL WHERE 条件：`order_no LIKE ?`、`created_at >= ?`、`created_at <= ?`
- COUNT 查询同步应用筛选条件，确保分页总数准确

#### 2. 后端路由层 (`backend/src/routes/public/member.ts`)
- 从 `req.query` 提取 `orderNo`、`startDate`、`endDate` 参数
- 封装为 `filters` 对象传递给 `memberService.getBills()`

#### 3. 前端 API 层 (`website/src/api/member.ts`)
- `getMemberBills` 新增可选 `filters` 参数
- 使用 `URLSearchParams` 构建查询字符串，仅在有值时添加参数

#### 4. 前端页面 (`website/src/views/member/Bills.vue`)

**模板部分：**
- 标题右侧按钮组：搜索 / 重置 / 刷新
- 搜索栏：订单号输入框（支持回车搜索）+ 开始日期 + 结束日期

**脚本部分：**
- 新增 `searchOrderNo`、`searchStartDate`、`searchEndDate` 响应式变量
- `loadBills()` 调用时自动携带搜索参数
- `handleSearch()` 重置到第1页并加载
- `handleReset()` 清空搜索条件并重新加载

**样式部分：**
- `.search-bar`：flex 布局、浅灰背景、圆角容器、支持 flex-wrap 响应式
- `.search-field`：纵向排列 label + input，输入框聚焦时高亮边框

## 任务三：取消订单成功提示改为顶部 Toast 通知

### 问题描述
取消订单成功后，页面弹出一个模态弹窗（需点击"确定"才能关闭），交互过重，用户体验不佳。

### 修复方案
- 新增顶部 Toast 通知组件（`showToast` 方法），支持 `success / error / info` 三种类型
- 从页面顶部下滑出现，3 秒后自动消失，无需用户操作
- 取消订单成功/失败均改用 Toast 通知，不再使用 `customAlert` 弹窗
- 使用 Vue `<transition>` 实现丝滑的滑入滑出动画

### 修改文件
| 文件 | 修改内容 |
|------|---------|
| `website/src/views/member/Bills.vue` | 新增 Toast 组件 + 取消订单提示改用 showToast |

## 任务四：修改密码页面支持短信验证码重置密码

### 问题描述
修改密码页面仅支持"原密码 + 新密码"的方式，当用户忘记密码时无法操作。

### 修复方案
页面改为**双模式切换**设计：

#### 模式一：原密码修改（默认）
- 保持原有流程不变：原密码 → 新密码 → 确认新密码
- 成功后改用 Toast 顶部提示

#### 模式二：短信验证码重置（忘记密码）
- **手机号输入**：自动从 localStorage 读取当前登录租户的绑定手机号并预填
- **获取验证码按钮**：调用已有的 `POST /api/v1/public/member/send-code` 接口
  - 60 秒倒计时防重复发送
  - 验证手机号格式
- **验证码输入**：6 位验证码
- **新密码 + 确认密码**
- **重置按钮**：调用已有的 `POST /api/v1/public/member/set-password` 接口
  - 传递 phone + code + password + tenantId
  - 后端验证验证码后设置新密码

#### UI 设计
- 顶部 Tab 切换栏：「🔧 原密码修改」|「📱 短信验证码重置」
- Tab 激活态使用品牌主色（#6366f1）
- 短信模式下方有灰色提示文案说明用途
- 手机号输入框与获取验证码按钮同行排列
- 成功/失败统一使用顶部 Toast 通知

### 复用的后端接口（无需修改后端）
| 接口 | 用途 |
|------|------|
| `POST /api/v1/public/member/send-code` | 发送短信验证码 |
| `POST /api/v1/public/member/set-password` | 验证码验证后设置新密码 |

### 修改文件
| 文件 | 修改内容 |
|------|---------|
| `website/src/views/member/Password.vue` | 完全重写，双模式切换 + 短信验证码流程 + Toast 通知 |
