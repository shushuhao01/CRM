# SaaS 私钥工具使用指南与优化增强

> 📅 完成日期：2026-04-06  
> 📋 类型：工具增强 + 使用指南  
> 🏷️ 标签：SaaS部署、私钥工具、RSA签名、许可证管理

## 📌 任务背景

### 问题描述
项目中已经实现了完整的 SaaS 部署 RSA 签名保护机制（2026-04-02 完成），但存在以下使用问题：
1. **工具位置不明确** — 运营方不知道私钥工具在哪里、怎么用
2. **缺少快捷入口** — `package.json` 中没有对应的 npm scripts
3. **缺少验证功能** — 无法快速验证现有 Token 是否有效/即将过期
4. **缺少过期预警** — 服务运行中不会主动提醒许可证即将过期
5. **域名校验缺失** — Token 中有 `allowedDomains` 字段但后端没有实际校验

### 涉及工具位置
| 文件 | 说明 | 安全性 |
|------|------|--------|
| `backend/scripts/generate-saas-license.js` | 🔑 SaaS 许可证签发工具（含 RSA 私钥） | ⛔ 绝不交付客户 |
| `backend/src/services/SaaSGuardService.ts` | 🛡️ SaaS 守卫服务（含 RSA 公钥） | ✅ 可交付 |
| `backend/src/middleware/saasGuard.ts` | 🚧 路由级守卫中间件 | ✅ 可交付 |
| `backend/src/config/deploy.ts` | ⚙️ 部署模式配置 | ✅ 可交付 |

## 🔧 本次优化内容

### 优化 1：脚本新增 `--verify` 参数
**问题**：运营方无法快速验证一个已有的 Token 是否有效、什么时候过期。  
**修复**：在 `generate-saas-license.js` 中新增 `--verify <token>` 功能。

```bash
# 验证指定 Token
cd backend
node scripts/generate-saas-license.js --verify eyJhbGciOiJSUzI1NiI...

# 输出示例：
# ✅ 签名验证通过！Token 有效！
# 📋 许可证详情：
#    类型: saas_platform_license
#    许可证ID: dcb259ce-...
#    绑定域名: localhost
#    剩余有效期: 3646 天
```

### 优化 2：脚本新增 `--verify-env` 参数
**问题**：验证 Token 需要手动复制很长的字符串，容易出错。  
**修复**：新增 `--verify-env` 参数，自动从 `.env.local`/`.env` 文件读取 Token 验证。

```bash
cd backend
node scripts/generate-saas-license.js --verify-env

# 自动搜索顺序: .env.local → .env → .env.production → .env.production.server
```

### 优化 3：`package.json` 新增 npm scripts 快捷命令
**问题**：运营方不知道工具在哪里。  
**修复**：在 `backend/package.json` 中添加 5 个快捷命令。

```bash
cd backend

# 查看帮助
npm run saas:license:help

# 生成新许可证
npm run saas:license -- --domain crm.yunkes.com --expires 365

# 验证 .env 中的 Token
npm run saas:license:verify-env

# 验证指定 Token
npm run saas:license:verify -- eyJhbGciOiJSUzI1NiI...

# 重新生成 RSA 密钥对（私钥泄露时使用）
npm run saas:keygen
```

### 优化 4：SaaSGuardService 新增过期预警
**问题**：Token 快过期时没有任何提醒，直到过期才发现 SaaS 模式不可用。  
**修复**：在 `SaaSGuardService.ts` 中添加过期预警逻辑：
- 启动时检查：剩余不足 30 天时输出 `⚠️ WARN` 日志
- 定期验证时检查：每小时巡检时也会检查过期预警
- 日志内容包含续期命令提示

```
[SaaSGuard] ⚠️ SaaS 许可证将在 25 天后过期，请尽快续期！
[SaaSGuard] 续期方式: cd backend && node scripts/generate-saas-license.js --expires 365
```

### 优化 5：SaaSGuardService 新增域名校验警告
**问题**：Token 中 `allowedDomains` 字段存在但没有校验，形同虚设。  
**修复**：在 `SaaSGuardService.ts` 验证通过后，增加域名匹配检查：
- 从 `API_BASE_URL` 或 `CRM_URL` 环境变量提取当前服务器域名
- 与 Token 中的 `allowedDomains` 比对
- 不匹配时输出警告日志（**仅警告，不阻断**，避免影响迁移场景）
- 通配符 `*` 跳过检查

### 优化 6：帮助信息增强
**修复**：`--help` 输出中增加了完整的使用示例和参数说明。

## 📋 完整使用指南（快速参考）

### 🟢 场景 1：首次部署 SaaS 服务器

```bash
# 1. 在你的开发电脑上生成许可证
cd backend
node scripts/generate-saas-license.js --domain crm.yunkes.com --expires 365

# 2. 复制输出的 SAAS_LICENSE_TOKEN=eyJ... 整行

# 3. SSH 到服务器，编辑 .env
DEPLOY_MODE=saas
SAAS_LICENSE_TOKEN=eyJhbGciOiJSUzI1NiI...（粘贴完整 Token）

# 4. 重启后端
pm2 restart crm-backend

# 5. 查看日志确认
pm2 logs crm-backend --lines 30 | grep SaaSGuard
# 应看到：✅ SaaS 平台许可验证通过！
```

### 🔄 场景 2：许可证续期（Token 过期续期）

> ⚠️ **核心概念：续期 ≠ 换私钥！**
>
> | 概念 | 类比 | 会过期吗 | 续期时需要换吗 |
> |------|------|----------|----------------|
> | **RSA 密钥对**（私钥+公钥） | 公章本身 | **永不过期** | ❌ 不需要（除非私钥泄露） |
> | **SAAS_LICENSE_TOKEN** | 用公章盖出的证书 | ✅ 按设定天数过期 | ✅ 重新签发一张新的 |
>
> 续期只需用**同一把私钥**重新签发新 Token，私钥和公钥都不用动。
> 只有私钥泄露时才需要换密钥对（见场景4）。

```bash
# 1. 用现有私钥重新签发新 Token（私钥不变！）
cd backend
node scripts/generate-saas-license.js --domain crm.yunkes.com --expires 365

# 2. 复制新 Token，替换服务器 .env 中的旧 SAAS_LICENSE_TOKEN
# 3. 重启后端: pm2 restart crm-backend
# ✅ 完成！无需改任何代码，无需重新部署
```

### 🔍 场景 3：验证现有 Token 状态

```bash
cd backend

# 方式1：自动从 .env 读取验证（推荐）
node scripts/generate-saas-license.js --verify-env

# 方式2：手动指定 Token 验证
node scripts/generate-saas-license.js --verify eyJhbGciOiJSUzI1NiI...
```

### 🚨 场景 4：私钥泄露应急处理

```bash
# 1. 重新生成密钥对
cd backend
node scripts/generate-saas-license.js --generate-keys

# 2. 更新公钥 → backend/src/services/SaaSGuardService.ts 的 SAAS_PUBLIC_KEY
# 3. 更新私钥 → backend/scripts/generate-saas-license.js 的 PRIVATE_KEY
# 4. 重新签发许可证
node scripts/generate-saas-license.js --domain crm.yunkes.com --expires 365
# 5. 更新服务器 .env 中的 Token
# 6. 重新部署后端代码（公钥改了需要重新部署）
# 7. 重启服务
```
