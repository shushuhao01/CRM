## 一、后端新增文件（2个）

### 1.1 `backend/src/services/SubscriptionService.ts` — 订阅核心服务（~790行）

**签约相关：**
- `createSubscription()` — 创建订阅记录 + 调用第三方签约接口
- `createWechatContract()` — 微信委托代扣签约 (PAPPAY V3 API)
- `createAlipayAgreement()` — 支付宝周期扣款签约 (CYCLE_PAY_AUTH)
- `handleSignSuccess()` — 处理签约成功回调 → 更新状态 → 执行首次扣款

**扣款相关：**
- `executeDeduction()` — 执行扣款（创建扣款日志 → 调用渠道扣款 → 更新订阅/租户）
- `wechatDeduct()` — 微信委托代扣扣款 (`/v3/papay/pay/transactions/apply`)
- `alipayDeduct()` — 支付宝协议扣款 (`alipay.trade.pay`)

**取消/查询：**
- `cancelSubscription()` — 取消订阅（调用第三方解约 + 更新本地状态）
- `cancelWechatContract()` — 微信解约 (`/v3/papay/sign/contracts/{id}/terminate`)
- `cancelAlipayAgreement()` — 支付宝解约 (`alipay.user.agreement.unsign`)
- `getSubscription()` — 查询订阅状态（含扣款历史）

**定时任务：**
- `runAutoDeduct()` — 自动扣款（每天01:00，查询到期订阅执行扣款）
- `runDeductRetry()` — 扣款重试（每天14:00，宽限期策略：1天/3天/5天重试，第7天过期）

**开发辅助：**
- `mockSignSuccess()` — 开发环境模拟签约成功

### 1.2 `backend/src/routes/public/subscription.ts` — 订阅管理API路由

## 四、扣款失败宽限期策略

```
Day 0: 扣款失败（第1次）→ 记录失败，设置1天后重试
Day 1: 第1次重试 → 失败 → 设置2天后重试
Day 3: 第2次重试 → 失败 → 设置2天后重试
Day 5: 第3次重试 → 失败 → 订阅状态改为 expired，停止服务
Day 7: 宽限期结束（期间服务仍可用，因为expire_date还未到）

每次扣款失败都会发送提醒（通过已有的通知服务）
```
