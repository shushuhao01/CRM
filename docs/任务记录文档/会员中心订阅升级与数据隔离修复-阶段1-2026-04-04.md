## 📋 发现的问题清单

### 🔴 问题1：会员登录多租户数据隔离漏洞（已修复）

**严重度**：🔴 高（数据安全）

**问题描述**：
`MemberService.ts` 中 `loginByPassword`、`loginByCode`、`setPassword` 三个方法都通过 `WHERE phone = ?` 查询，**未做多租户隔离**。

- 如果同一手机号注册了多个租户（不同公司），`loginByPassword` 和 `loginByCode` 只取 `tenants[0]`，其他租户永远无法登录
- `setPassword` 会批量修改所有同手机号租户的密码，跨租户泄露

**修复方案**：
1. 登录接口增加可选 `tenantId` 参数，精确匹配指定租户
2. 同手机号存在多个活跃租户时，返回租户列表让前端选择
3. 验证码登录场景下，验证码验证后被消费，通过临时 `selectToken`（5分钟有效）安全传递选择结果
4. 密码登录场景下，先对所有租户验证密码，筛选匹配的再返回列表
5. `setPassword` 支持 tenantId 参数精确修改

### 🟢 问题2：密码一致性分析（无需修复，设计正确）

**结论**：会员中心密码与CRM登录密码**不一致是正确的设计**。

| 系统 | 密码存储位置 | 默认值 | 设置时机 |
|------|------------|--------|---------|
| 会员中心 | `tenants.password_hash` | 无（可选设置） | 注册时可选填 / 登录后设置 |
| CRM系统 | `users.password` | `Aa123456` | 注册自动创建admin账号时 |

**原因**：
- 会员中心是**企业主/负责人**使用的管理面板（管订阅、看账单、看授权）
- CRM系统是**企业内部员工**使用的工作台（管客户、跟订单、打电话）
- 两套密码独立是**正确的安全隔离设计**

**建议**：在注册成功页面和会员中心密码页面增加说明提示，告知用户两套密码独立。

### 🟡 问题3：订阅管理页功能不足（阶段2修复）

免费试用用户登录会员中心后，订阅管理页显示"当前套餐暂不支持订阅付费模式"。
应改为：展示可选的付费套餐列表，选中后直接在会员中心完成签约。

### 后端 MemberService 改动详情

#### 新增：selectToken临时缓存机制
```
selectTokenStore: Map<token, { phone, tenantIds[], expiresAt }>
generateSelectToken(phone, tenantIds) → token (5分钟有效)
verifySelectToken(token, phone, tenantId) → boolean (一次性消费)
```

#### loginByPassword 改动
- 增加可选 `tenantId` 参数
- 有tenantId时：`WHERE phone = ? AND id = ?` 精确匹配
- 无tenantId且多租户时：对所有活跃租户验证密码，收集匹配的
  - 匹配1个 → 直接登录
  - 匹配多个 → 返回 `needSelectTenant: true` + 租户列表
- 抽取 `completeLogin()` 公共方法消除重复代码

#### loginByCode 改动
- 增加可选 `tenantId` 参数
- 验证码验证后查询租户列表
- 多租户时返回 `selectToken`（因验证码已被消费，不能重复使用）

#### 新增：loginBySelectToken
- 验证 selectToken 有效性（手机号+tenantId+过期时间）
- 一次性消费token，防止重放
- 完成登录

#### setPassword 改动
- 增加可选 `tenantId` 参数
- 有tenantId时精确修改指定租户密码
- 无tenantId时修改该手机号下所有租户密码（兼容旧逻辑）

### 前端 Login.vue 改动详情

#### 新增多租户选择弹窗
- 条件显示：登录接口返回 `data.needSelectTenant === true` 时
- 展示租户列表（企业名称、租户编码、联系人）
- 点击选择后：
  - 密码模式 → 重新调用 memberLogin + tenantId
  - 验证码模式 → 调用 selectTenantLogin + selectToken

#### UI设计
- 弹窗居中显示，毛玻璃背景遮罩
- 租户卡片列表，每项含头像+企业名+编码
- hover紫色边框，点击后自动登录
