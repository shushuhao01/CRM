# 短信自动发送规则修复 — 数据库缺表+前端BUG修复

> **执行日期**: 2026-04-11  
> **任务类型**: BUG修复  
> **涉及模块**: CRM短信管理 - 自动发送规则、短信发送记录  
> **前置依赖**: 短信管理模块增强v1.8.1（2026-04-11已完成）

---

## 一、问题描述

1. **自动发送规则获取失败**：CRM端短信管理「自动发送」Tab加载规则时报500错误
2. **新建规则选择模板加载不上真实模板**：模板下拉框为空或数据不足
3. **表单验证规则绑定错误**：新建/编辑规则表单的验证规则绑定了数据数组而非验证对象

---

## 二、根因分析

### 2.1 数据库层面（核心原因）

| 问题 | 原因 | 影响 |
|------|------|------|
| `sms_auto_send_rules` 表不存在 | schema.sql 遗漏了该表定义，TypeORM `synchronize: false` 不会自动建表 | 所有自动发送规则 CRUD 接口报500 |
| `sms_records` 表缺少4个字段 | `sender_user_id`、`sender_department_id`、`trigger_source`、`auto_rule_id` 未加入schema | 自动发送触发后写入记录失败、角色数据范围过滤失效 |

### 2.2 前端层面

| 问题 | 原因 | 影响 |
|------|------|------|
| 表单 `:rules="autoSendRules"` 绑定错误 | `autoSendRules` 是 `ref<any[]>([])` 数据数组，不是验证规则对象 | 表单验证不生效（必填校验失效） |
| 模板下拉框加载不完整 | `loadApprovedTemplates()` 使用 `getTemplates({ status: 'active' })` 默认分页仅10条 | 超过10个模板时部分模板不可选 |
| 未使用 `getAvailableTemplates` API | 该API返回租户active模板+全局预设模板，更适合模板选择场景 | 预设模板可能缺失 |

---

## 三、修复内容

### 3.1 数据库修复

| 文件 | 操作 | 说明 |
|------|------|------|
| `database/schema.sql` | 修改 | ① sms_records 表新增4个字段定义+3个索引；② 新增 sms_auto_send_rules 完整表定义；③ 版本号 1.9.6→1.9.7 |
| `database/fix-sms-auto-send.sql` | 新增 | 本地数据库修复脚本（已执行） |
| `database/migration-v1.8.1-sms-auto-send.sql` | 新增 | 生产环境增量迁移SQL，兼容phpMyAdmin，幂等安全 |

#### 新增表：`sms_auto_send_rules`

```
字段: id, tenant_id, name, template_id, template_name, trigger_event,
      effective_departments(JSON), time_range_config(JSON), enabled,
      created_by, created_by_name, stats_sent_count, stats_fail_count,
      last_triggered_at, description, created_at, updated_at
索引: tenant_id, trigger_event, enabled
```

#### `sms_records` 新增字段

| 字段名 | 类型 | 说明 |
|--------|------|------|
| `sender_user_id` | VARCHAR(50) | 发送人用户ID，用于员工角色数据范围过滤 |
| `sender_department_id` | VARCHAR(100) | 发送人部门ID，用于经理角色数据范围过滤 |
| `trigger_source` | VARCHAR(20) DEFAULT 'manual' | 触发来源：manual=手动发送，auto=自动触发 |
| `auto_rule_id` | VARCHAR(50) | 自动发送规则ID，关联 sms_auto_send_rules.id |

### 3.2 前端修复

| 文件 | 操作 | 说明 |
|------|------|------|
| `src/views/ServiceManagement/SmsManagement.vue` | 修改 | ① `:rules="autoSendRules"` 改为 `:rules="autoSendRulesValidation"`；② `loadApprovedTemplates()` 改用 `getAvailableTemplates()` API 加载完整模板列表；③ 新增 `getAvailableTemplates` 导入 |
