# Phase D 完成记录：官网注册流程改造

## 一、Register.vue 改造内容

### 1.1 步骤2（填写信息）新增内容

**密码设置字段：**
- 新增 `form.password` 输入框，推荐填写但非必填
- 设置密码后可登录会员中心管理订阅、查看账单和授权信息
- 注册提交时传递 `password` 字段到后端（后端 bcrypt 哈希存储）

**免费试用自动续费选项：**
- 仅当选择 `FREE_TRIAL` 时显示
- `form.autoRenew` 勾选框 — 默认勾选，文案提示"推荐，试用期内可随时取消"
- 勾选后展开套餐选择列表（`renewablePlans`），默认选基础版
- 支持用户自选续费套餐（显示套餐名、价格、是否有订阅优惠）
- 不勾选则保持原有的7天免费试用流程，无变化
- 提交时传递 `autoRenew` 和 `autoRenewPackage` 到后端

### 1.2 步骤3（确认支付）新增内容

**订阅/正常支付切换：**
- 仅当套餐 `subscription_enabled=true` 且为 SaaS 非免费套餐时显示
- 两个选项：
  - 🔄 订阅付费（推荐）— 显示订阅折扣价、自动续费省心
  - 💳 正常支付 — 显示原价、一次性支付
- 默认选中订阅付费
- 应付金额随支付模式动态变化

**支付方式根据模式筛选：**
- 订阅模式：只显示套餐配置的渠道（微信委托代扣/支付宝周期扣款）
- 正常支付：显示全部方式（微信/支付宝/对公转账）

**确认按钮文案：**
- 订阅模式：「确认签约并支付」
- 正常模式：「确认支付」

### 1.3 顶部提示横幅增强

- 付费套餐步骤3时显示 `🔄 订阅¥XX/月` 标签
- 免费试用勾选自动续费时显示 `🔄 到期自动续费` 标签

### 1.4 其他改造

- 「已有账号」链接增加会员中心入口
- 引入 `computed` 用于计算订阅价格等派生数据

## 二、新增计算属性

| 属性 | 类型 | 说明 |
|------|------|------|
| `currentPkg` | Package | 当前选中套餐对象 |
| `currentPkgSubscriptionEnabled` | boolean | 当前套餐是否支持订阅 |
| `currentPkgSubscriptionChannels` | string | 支持的订阅渠道 |
| `currentPkgSubscriptionDiscountRate` | number | 订阅折扣率 |
| `subscriptionPrice` | string | 计算后的订阅月价 |
| `renewablePlans` | Package[] | 免费试用可续费的套餐列表 |

## 三、新增样式

- `.auto-renew-section` — 自动续费配置区块（绿色背景）
- `.renew-plan-options` — 续费套餐选择列表

## 四、用户体验流程对照

### 免费试用注册
```
步骤1: 选FREE_TRIAL → 步骤2: 填信息+设密码+[勾选到期自动续费+选套餐] → 直接注册成功
                                                          ↓
                                               默认勾选(可取消)
                                               默认基础版(可换)
```

### 付费套餐注册（套餐启用订阅时）
```
步骤1: 选套餐 → 步骤2: 填信息+设密码 → 步骤3: [订阅付费(推荐)/正常支付] → 选渠道 → 支付
                                                    ↓
                                          订阅: 只显示微信代扣/支付宝扣款
                                          正常: 显示微信/支付宝/对公转账
```

### 付费套餐注册（套餐未启用订阅时）
```
步骤1: 选套餐 → 步骤2: 填信息+设密码 → 步骤3: 选支付方式 → 支付（与原流程一致）
```
