# 短信管理模块增强 — 角色权限 & 自动发送 & 客户详情集成

> **执行日期**: 2026-04-11  
> **任务类型**: 功能增强 + 优化改造  
> **涉及模块**: CRM短信管理、客户详情、订单管理、自动发送服务  
> **前置依赖**: 短信管理模块审查与改造v3（2026-04-10已完成）

## 一、任务概述

基于短信管理模块v3方案的基础，本次任务聚焦5个核心需求：

1. **短信管理角色权限细化**：所有角色可用短信管理，但数据范围不同（员工→个人，经理→部门，管理员→全租户）
2. **自动发送功能增强**：补全更多触发事件（订单确认/付款/新客户创建），添加统计详情查看
3. **客户详情页短信发送改造**：替换简易表单为三步式SendSmsDialog（选模板→填变量→预览确认）
4. **变量自动匹配增强**：前后端变量映射表扩展，覆盖更多场景
5. **所有场景自动匹配变量**：预设模板配置自动发送时变量正确匹配

## 三、功能详情

### 3.1 短信管理角色权限（已完成）

**后端数据范围过滤**（`/sms/records` 和 `/sms/statistics` 和 `/sms/customers/search`）：
- `admin/super_admin`：查看本租户所有记录
- `department_manager`：查看本部门记录（通过 `senderDepartmentId` 或 `applicantDept` 匹配）
- `sales_staff/customer_service`等：仅查看个人记录（通过 `applicant` 或 `senderUserId` 匹配）

**前端界面差异**：
- **管理员**：看到"待审核模板"/"待审核短信"统计卡片 + "短信审核"Tab + "自动发送"Tab
- **非管理员**：看到"我的待审核"/"已通过申请"统计卡片 + "审核记录"Tab + "发送记录"Tab + "发送短信"按钮

**客户选择数据范围**：`/sms/customers/search` 接口已按角色过滤：
- 员工只能选择自己负责的客户
- 经理能选择本部门所有客户
- 管理员能选择全部客户

### 3.2 客户详情页短信发送改造

**Before**：简易表单（选号码→选模板→填变量→发送），变量匹配有限

**After**：复用 `SendSmsDialog` 三步式流程：
1. **步骤1 - 选择模板**：网格卡片展示所有可用模板，支持搜索和分类
2. **步骤2 - 填写信息**：
   - 自动填充接收人（`presetCustomer` prop 传入当前客户）
   - 选择模板后自动匹配变量（调用 `autoMatchVariables` 工具）
   - 所有变量支持手动修改
   - 支持选择更多客户、定时发送等
3. **步骤3 - 预览确认**：最终内容预览、费用估算、确认发送

## 四、待后续实现的任务
