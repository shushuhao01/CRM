### 任务 2.1：修复客户列表 N+1 查询（最关键） ✅ 已确认完成（之前已优化）
- **严重等级**：🔴 严重
- **文件**：`backend/src/routes/customers.ts` 第222-270行
- **当前问题**：每个客户记录执行3个额外查询（订单数、分享状态、销售人员），分页20条=61次数据库查询
- **修复方案**：
  1. 使用 `IN` 查询批量获取所有客户的订单统计：
     ```typescript
     const orderCounts = await orderRepository.createQueryBuilder('order')
       .select('order.customerId', 'customerId')
       .addSelect('COUNT(*)', 'count')
       .where('order.customerId IN (:...ids)', { ids: customerIds })
       .groupBy('order.customerId')
       .getRawMany();
     ```
  2. 批量获取分享状态：`await shareRepository.find({ where: { customerId: In(customerIds), status: 'active' } })`
  3. 批量获取销售人员：`await userRepository.find({ where: { id: In(salesPersonIds) } })`
  4. 用Map做内存映射，避免循环查询
- **预估耗时**：3-4小时
- **收益**：查询次数从 61次 → 4次，响应时间下降90%

### 任务 2.2：修复客户统计全量查询 ✅ 2026-03-30
- **严重等级**：🔴 严重
- **文件**：`backend/src/routes/customers.ts` 第193-213行
- **当前问题**：为统计"未下单客户数"，先查所有有订单的客户ID，再在JS中做差集
- **修复方案**：改用 `LEFT JOIN + IS NULL` 一条SQL完成
  ```sql
  SELECT COUNT(*) FROM customers c
  LEFT JOIN orders o ON c.id = o.customer_id AND o.tenant_id = ?
  WHERE c.tenant_id = ? AND o.id IS NULL
  ```
- **预估耗时**：1-2小时

### 任务 2.3：Dashboard SQL聚合替代内存过滤 ✅ 2026-03-30
