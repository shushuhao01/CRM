### 3.3 订阅签约流程

```
用户点击"确认并签约授权"
        │
        v
  后端创建订阅记录 (status=signing)
  并调用微信/支付宝签约接口
        │
        v
  ┌─── 微信渠道 ───┐  ┌─── 支付宝渠道 ───┐
  │ 弹出签约页面    │  │ 跳转签约页面      │
  │ 或扫码签约      │  │                  │
  └───────┬────────┘  └────────┬─────────┘
          │                    │
     用户授权签约          用户授权签约
          │                    │
          v                    v
  后端收到签约回调 → 更新 subscription.status = 'active'
        │
        v
  首次扣款（立即扣第一期）
        │
        v
  扣款成功 → 激活租户 → 跳转支付成功页
```

## 七、风险与注意事项

### 7.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 微信委托代扣商户权限未开通 | 无法对接微信订阅 | 提前申请，审核周期约5-10工作日 |
| 支付宝周期扣款权限未签约 | 无法对接支付宝订阅 | 提前签约，需提供业务资质 |
| 扣款失败率较高 | 用户体验差，收入损失 | 完善重试机制，多渠道通知 |
| 会员Token安全性 | 信息泄露风险 | HTTPS强制，Token定期轮换 |

### 7.2 业务注意事项

| 事项 | 说明 |
|------|------|
| 订阅优惠定价策略 | 折扣不宜过大(建议5-15%)，避免影响正常收入 |
| 退款政策 | 订阅扣款后是否支持退款，需明确在协议中 |
| 取消订阅后的服务 | 建议当前计费周期内继续服务，到期停止 |
| 试用期自动续费提醒 | 试用到期前3天发送短信/邮件提醒 |
| 隐私合规 | 委托代扣需要用户明确授权，签约页面需有协议说明 |

### 7.3 兼容性保证

```
重要原则：
1. 所有改动不影响现有支付流程
2. 套餐不开启订阅时，行为与当前完全一致
3. 会员中心是增量功能，不影响CRM系统和Admin后台
4. 数据库变更全部使用 ALTER TABLE ADD COLUMN，不修改现有字段
5. 新增API独立路由，不修改现有公开API签名
```
