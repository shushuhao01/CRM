## 一、项目现状分析

### 1.1 官网 (Website) 现状

| 项目 | 现状 | 说明 |
|------|------|------|
| 技术栈 | Vue 3 + TypeScript + 纯手写SCSS | 端口8080 |
| 现有页面 | 首页/功能/价格/注册/支付成功/帮助文档/协议 | 共10个页面 |
| 注册流程 | 步骤1选套餐 → 步骤2填信息+验证码 → 步骤3支付 | 三步骤流程 |
| 导航栏"登录" | 直接跳转 https://app.yunke-crm.com | 指向CRM系统 |
| 会员中心 | **不存在** | 注册后无法回官网管理账号 |
| 支付方式 | 微信扫码/支付宝扫码/对公转账 | 仅一次性支付 |
| 订阅付费 | **不支持** | 无委托代扣功能 |

### 1.2 后端 (Backend) 支付体系现状

| 服务 | 文件 | 功能 | 状态 |
|------|------|------|------|
| PaymentService | `services/PaymentService.ts` (659行) | 通用支付（创建订单/回调/激活） | 已完成 |
| WechatPayService | `services/WechatPayService.ts` (536行) | 微信Native扫码支付 | 已完成 |
| AlipayService | `services/AlipayService.ts` (482行) | 支付宝当面付扫码支付 | 已完成 |
| 公开支付路由 | `routes/public/payment.ts` (288行) | 创建订单/查询/回调 | 已完成 |

**关键发现：**
- 仅支持 `NATIVE` 扫码一次性支付，不支持订阅/委托代扣
- 没有 `subscriptions` 相关表或字段
- `payment_orders` 表没有订阅关联字段

### 1.3 套餐管理 (Package) 现状

**数据库表 `tenant_packages` 字段：**

```
id, name, code, type(saas/private), description, price, original_price,
billing_cycle(monthly/yearly/once), yearly_discount_rate, yearly_bonus_months,
yearly_price, duration_days, max_users, max_storage_gb, features, modules,
is_trial, is_recommended, is_visible, sort_order, status, created_at, updated_at
```

**缺少的订阅相关字段：**
- ❌ `subscription_enabled` — 是否支持订阅模式
- ❌ `subscription_channels` — 支持的订阅渠道（微信/支付宝/全部）
- ❌ `subscription_discount_rate` — 订阅优惠折扣率

### 1.4 租户 (Tenant) 现状

**数据库表 `tenants` 关键字段：**

```
id, name, code, package_id, contact, phone, email,
max_users, max_storage_gb, expire_date, license_key, license_status,
status, features, created_at, updated_at
```

**缺少的会员登录相关字段：**
- ❌ `password_hash` — 会员中心登录密码
- ❌ `last_login_at` — 最近登录时间

### 1.5 管理后台 (Admin) 套餐编辑现状

**Packages.vue 编辑弹窗现有配置区块：**
1. ✅ 基本信息（名称/代码/类型/描述）
2. ✅ SaaS定价配置（月付价格/计费周期）
3. ✅ 年付优惠配置（赠送月数/折扣/固定年价）
4. ✅ 资源配置（有效期/用户数/存储空间）
5. ✅ 私有部署定价（永久买断价/年度授权价）
6. ✅ 功能特性列表
7. ❌ **订阅付费配置** — 不存在

## 三、技术约束与前提条件

### 3.1 微信委托代扣 (PAPPAY) 要求

| 项目 | 要求 | 说明 |
|------|------|------|
| 商户权限 | 需在微信商户平台申请"委托代扣"产品 | 需单独签约 |
| API版本 | 推荐使用V3 API | `/v3/papay/sign/contracts` |
| 签约方式 | H5签约 / APP签约 / 扫码签约 | 官网建议H5签约 |
| 扣款限额 | 单笔上限取决于商户签约协议 | 通常500-2000元 |
| 扣款通知 | 扣款前需提前通知用户 | 微信要求至少提前24小时 |

### 3.2 支付宝周期扣款要求

| 项目 | 要求 | 说明 |
|------|------|------|
| 产品权限 | 需签约"周期扣款"产品 | 需单独申请 |
| API接口 | `alipay.user.agreement.page.sign` | 签约 |
| 扣款接口 | `alipay.trade.pay` (协议扣款) | 扣款 |
| 签约方式 | 页面跳转签约 / 扫码签约 | 官网建议页面跳转 |
| 扣款限额 | 受协议约束 | 通常每笔不超过协议金额 |

### 3.3 需要确认的事项

> **⚠️ 开始实施前需确认：**
>
> 1. 微信商户号是否已开通"委托代扣"产品权限？
> 2. 支付宝应用是否已签约"周期扣款"产品？
> 3. 会员中心登录方式偏好：A.手机号+密码 / B.纯验证码 / C.两者都支持？（推荐C）
> 4. 订阅扣款失败后的降级策略：立即停用 or 宽限期？（推荐3次重试 + 7天宽限期）
> 5. 免费试用"到期自动续费"默认勾选的版本：基础版？还是用户自选？
