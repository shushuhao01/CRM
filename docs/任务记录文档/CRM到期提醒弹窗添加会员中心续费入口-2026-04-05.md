# CRM到期提醒弹窗添加会员中心续费入口

**日期**: 2026-04-05  
**状态**: ✅ 已完成  
**涉及模块**: CRM前端（主应用） / 后端租户心跳接口 / 授权设置页面  
**类型**: 功能增强（到期提醒续费入口 + 联系客服弹窗 + SaaS即将到期预警）

---

## 一、任务概述

在CRM主应用的到期提醒弹窗和写入限制弹窗中，添加"去会员中心续费"和"联系客服续费"两个操作入口，支持用户在授权过期或即将到期时快速完成续费操作。同时为SaaS模式补齐即将到期预警功能（剩余有效期 ≤ 总有效期20%时通知）。

### 核心需求
1. 到期弹窗中添加 **"去会员中心续费"** 按钮 → 点击后打开会员中心登录页面
2. 到期弹窗中添加 **"联系客服续费"** 按钮 → 点击后打开客服弹窗（二维码/微信客服/电话/邮箱）
3. SaaS模式也需有即将到期预警（剩余有效期 ≤ 总有效期20%）
4. 确保数据隔离安全，不同租户互不影响
5. 授权设置页面（LicenseSettings.vue）也添加续费入口

---

## 二、数据隔离安全说明

| 检查项 | 结果 | 说明 |
|--------|------|------|
| 心跳接口数据隔离 | ✅ 安全 | `WHERE id = ?` 仅查当前租户，tenantId从JWT获取 |
| 不同租户同手机号 | ✅ 隔离 | 心跳以tenantId为准，不涉及手机号查询 |
| 不同租户同用户名 | ✅ 隔离 | 同上，到期预警逻辑不涉及用户表 |
| 到期预警只通知当前租户 | ✅ 正确 | nearExpiry基于当前租户自身的expire_date计算 |
| 客服弹窗数据 | ✅ 安全 | 展示的是系统全局配置，不涉及租户私有数据 |
| 会员中心跳转 | ✅ 安全 | 仅打开外部链接，不携带租户敏感数据 |

## 四、功能详细说明

### 4.1 到期弹窗（licenseDialog.ts）

**弹窗内容包含：**
- 过期说明文字
- 🔑 **去会员中心续费** 按钮（蓝色，打开 `{websiteUrl}/member/login` 新标签页）
- 💬 **联系客服续费** 按钮（绿色，触发 ContactServiceDialog 弹窗）
- 底部操作按钮：
  - SaaS模式：「我知道了」
  - 私有部署：「已续费，同步授权信息」+「我知道了」
- 提示文字：说明续费后如何恢复系统

**触发场景：**
1. 心跳检测发现授权过期（SaaS + 私有部署）
2. 写入操作被403拦截（LICENSE_EXPIRED_WRITE_BLOCKED）

### 4.2 联系客服弹窗（ContactServiceDialog.vue）

**展示内容：**
- 微信客服二维码（来自 serviceQRCode / contactQRCode 配置）
- 微信在线客服链接（来自 customerServiceUrl 配置，点击直达微信对话）
- 客服热线电话（一键拨打）
- 客服邮箱（一键发送邮件）
- 工作时间

**数据来源：**
1. 优先从 `localStorage('crm_config_system')` 读取基本联系信息
2. 再从公开API `GET /api/v1/public/website-config` 获取更完整的客服配置（含微信客服URL等）

**触发方式：**
- 通过 `window.dispatchEvent(new CustomEvent('open-contact-service-dialog'))` 事件
- App.vue 监听该事件并打开弹窗

### 4.3 SaaS即将到期预警（后端心跳增强）

**后端改动（`/tenant-license/heartbeat`）：**
- SQL增加 `activated_at, created_at` 字段查询
- 计算总有效期：`过期日 - 激活日/创建日`
- 计算预警阈值：`max(总有效期 * 20%, 7天)`
- 当 `剩余天数 ≤ 阈值` 时设置 `nearExpiry = true`
- 返回字段：`daysUntilExpiry, nearExpiry, expireDate`

**前端改动（licenseHeartbeatService.ts）：**
- SaaS模式心跳成功后检查 `result.nearExpiry`
- 弹出 ElNotification 通知，包含：
  - 到期天数和日期
  - "去会员中心续费"链接
  - "联系客服续费"链接
- `nearExpiryNotified` 标志防止重复弹出

### 4.4 授权设置页面（LicenseSettings.vue）

- 过期 el-alert 和即将到期 el-alert 中新增按钮区域
- 🔑 去会员中心续费（蓝色按钮，跳转会员中心）
- 💬 联系客服续费（绿色按钮，打开客服弹窗）
- 使用 `configStore.systemConfig.websiteUrl` 获取网站地址

## 五、20%有效期预警算法

```
总有效期天数 = expire_date - activated_at（或created_at）
预警阈值 = max(总有效期 × 20%, 7天)
触发条件 = 剩余天数 > 0 && 剩余天数 ≤ 预警阈值

示例：
- 年度套餐(365天)：阈值 = max(73天, 7天) = 73天，即到期前73天开始预警
- 月度套餐(30天)：阈值 = max(6天, 7天) = 7天，即到期前7天开始预警
- 试用7天：阈值 = max(1.4天, 7天) = 7天，即第1天就开始预警
```

## 七、注意事项

1. **localStorage key**: CRM主应用系统配置存储在 `crm_config_system`（不是 `crm_system_config`），所有读取配置的地方需使用正确的key
2. **websiteUrl为空时**: "去会员中心续费"按钮会隐藏（v-if/条件判断），只显示"联系客服续费"
3. **客服信息为空时**: ContactServiceDialog 会显示空状态提示"暂无客服联系方式，请联系系统管理员"
4. **SaaS模式到期**: 不再直接跳转登录页，改为显示带续费入口的弹窗（非到期的暂停/禁用仍跳转登录页）
5. **事件监听清理**: App.vue 在 onUnmounted 中移除事件监听，防止内存泄漏
6. **防重复弹窗**: `isShowingLicenseDialog` 标志和 `nearExpiryNotified` 标志分别防止到期弹窗和预警通知重复弹出
