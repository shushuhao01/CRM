# 清理不活跃过期客户数据功能

**日期**: 2026-04-05  
**状态**: ✅ 已完成  
**涉及模块**: 管理后台(admin) - 租户客户详情页 / 私有客户详情页 + 后端 tenants.ts

---

## 一、需求概述

在管理后台的 **租户客户详情页** 和 **私有客户详情页** 添加"清理过期客户数据"按钮，用于清理不活跃过期客户的数据库记录和物理文件（图片等），腾出服务器空间。

### 核心规则
1. **活跃有效期内**：清理按钮禁用 + 提示"仍在有效期内"
2. **过期不足30天**：清理按钮禁用 + 提示"30天保护期"
3. **永久有效客户**：清理按钮禁用 + 提示"永久有效不允许清理"
4. **已清理过**：显示清理时间标识，按钮禁用，不可重复清理
5. **过期≥30天**：按钮可用，需两步确认（危险警告 + 输入租户编码）
6. **私有部署客户**：数据在其自有数据库，按钮禁用，显示说明提示

### 数据隔离安全要求（最高优先级）
- 严格按 `tenant_id` 隔离，绝不能误删其他租户数据
- 即使不同租户有相同手机号/用户名，也必须隔离
- 清理操作全程在数据库事务内执行，失败自动回滚

## 二、后端API（已完成）

| 接口 | 路径 | 功能 |
|------|------|------|
| GET | `/tenants/:id/cleanup-status` | 查询清理状态（canCleanup/reason/expiredDays/dataCleaned/lastCleanup） |
| POST | `/tenants/:id/cleanup-data` | 执行数据清理（需传 confirmCode=租户编码） |

### 后端清理流程
1. 查询租户信息，验证存在性
2. 二次确认校验（confirmCode === tenant.code）
3. 检查是否已清理过（data_cleaned_at）
4. 过期时间校验（永久有效拒绝/有效期内拒绝/不足30天拒绝）
5. 收集待清理的文件路径（customer_files/order_attachments/after_sale_attachments）
6. 事务内按 `tenant_id` 删除 30+ 张业务表的数据
7. 清理物理文件（uploads目录下的图片、附件等）
8. 更新 tenants 表的 `data_cleaned_at` 标记
9. 写入 `tenant_cleanup_logs` 清理日志
10. 写入 `tenant_license_logs` 授权操作日志

### 清理涉及的数据表（30+张，按依赖顺序）
文件记录 → 操作日志 → 通知/消息 → 短信模板 → 增值服务 → 外包 → 业绩/佣金 → 物流 → 售后 → 订单 → 客户 → 商品 → 组织架构 → 基础配置 → 支付/订阅 → 授权日志

## 三、后端API安全审查

### 3.1 数据隔离检查

| 检查项 | 结果 | 说明 |
|--------|------|------|
| 清理SQL使用参数化查询 `WHERE tenant_id = ?` | ✅ 安全 | 绝不拼接用户输入到SQL |
| 表名来自硬编码白名单（非用户输入） | ✅ 安全 | `tablesToClean` 数组写死在代码中 |
| 先检查表是否有 tenant_id 列再删除 | ✅ 安全 | INFORMATION_SCHEMA 查询确认 |
| 文件查询也限定 tenant_id | ✅ 安全 | `WHERE tenant_id = ?` |
| 事务内执行，失败回滚 | ✅ 安全 | queryRunner 事务 + SET FOREIGN_KEY_CHECKS |
| 二次确认需输入租户编码 | ✅ 安全 | `confirmCode !== tenant.code` 校验 |
| 过期时间校验 | ✅ 安全 | 有效期内/不足30天/永久有效 均拒绝 |
| 已清理过拒绝重复清理 | ✅ 安全 | `data_cleaned_at` 非空则拒绝 |

**结论：后端API数据隔离安全，无跨租户泄露风险。**

## 四、前端实现

### 4.1 租户客户详情页（tenant-customers/Detail.vue）

**新增"数据清理"卡片**，位于"数据导出/导入"卡片下方，包含：

| 功能 | 说明 |
|------|------|
| 清理状态加载 | 页面加载时自动调用 `GET /tenants/:id/cleanup-status` |
| 已清理提示 | 绿色 el-alert，显示清理时间和清理详情 |
| 不可清理提示 | 黄色 el-alert，显示禁止原因 |
| 可清理提示 | 红色 el-alert，警告此操作不可恢复 |
| 清理状态信息 | el-descriptions 展示清理状态和过期天数 |
| 清理按钮 | el-button danger，不可清理时自动 disabled |
| 两步确认流程 | 第1步：危险警告；第2步：输入租户编码二次确认 |
| 清理结果展示 | 清理完成后显示删除记录数、文件数、文件大小 |

**交互流程：**
```
点击「清理过期数据」
  → 弹出危险警告确认框（列出将清理的内容）
  → 确认后弹出输入框（需输入租户编码）
  → 编码匹配 → 调用 POST /cleanup-data
  → 成功 → 显示清理结果 + 刷新页面数据
  → 失败 → 显示错误信息
```

### 4.2 私有客户详情页（private-customers/Detail.vue）

**新增"数据清理"卡片**，位于"账单记录"卡片下方，包含：

| 功能 | 说明 |
|------|------|
| 信息提示 | el-alert(info) 说明私有部署数据存储在客户自有服务器 |
| 清理建议 | 列出3种替代方案（联系运维/使用维护工具/吊销授权） |
| 禁用按钮 | 始终 disabled 的清理按钮 + 文字说明"不支持远程清理" |
