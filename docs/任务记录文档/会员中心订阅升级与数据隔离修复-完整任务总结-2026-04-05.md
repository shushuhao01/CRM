## 三、发现并修复的问题清单

### 🔴 P0：PaymentService.activateTenant 不更新 package_id

**严重度**：P0 致命（套餐升级完全不生效）

**问题**：支付成功后 `activateTenant` 仅更新 `license_key`、`expire_date`、`max_users`，不更新 `package_id`、`max_storage_gb`、`features`。导致：
- 升级支付成功 → 租户仍绑定旧套餐
- CRM系统通过 `package_id` JOIN 获取权限 → 权限不变
- 管理后台显示旧套餐
- 会员中心显示旧套餐

**修复**：
- 重写 `activateTenant` 方法，完整更新 `package_id`、`max_users`、`max_storage_gb`、`features`
- 根据 `billing_cycle`（monthly/yearly/annual/perpetual）正确计算服务时长
- 续费场景从当前到期日延长，而非从当天算起
- 区分新开通/续费/升级三种场景的日志记录

### 🔴 P0：SubscriptionService.executeDeduction 不同步套餐

**严重度**：P0 致命（订阅扣款后套餐不升级）

**问题**：订阅扣款成功后仅更新 `expire_date` 和 `license_status`，不更新 `package_id` 等套餐信息。导致：
- 免费试用用户开通付费订阅 → 扣款成功但仍是试用套餐
- 切换订阅套餐 → 扣款成功但套餐不变

**修复**：
- 扣款成功后从 `tenant_packages` 查询订阅套餐的完整信息
- 更新租户表的 `package_id`、`max_users`、`max_storage_gb`、`features`、`status`、`license_status`
- 套餐不存在时降级为仅延长到期日

### 🔴 P0：MemberService 多租户数据隔离漏洞（阶段1已修复）

**严重度**：P0 致命（数据安全）

**问题**：`loginByPassword`、`loginByCode`、`setPassword` 通过 `WHERE phone = ?` 查询，无多租户隔离。
- 同手机号多租户只能登录第一个
- `setPassword` 批量修改所有同号租户密码

**修复**：
- 登录接口增加可选 `tenantId` 参数
- 同手机号多租户返回选择列表
- 验证码登录使用 `selectToken` 机制安全传递选择
- 前端新增多租户选择弹窗

### 🟢 密码一致性：设计正确无需修复

| 系统 | 密码存储 | 默认值 | 用途 |
|------|---------|--------|------|
| 会员中心 | `tenants.password_hash` | 无（可选设置） | 企业主管理面板 |
| CRM系统 | `users.password` | `Aa123456` | 企业员工工作台 |

两套密码独立是正确的安全隔离设计。

## 五、核心修复技术细节

### 5.1 PaymentService.activateTenant 重写

**参数变更**：
```typescript
// 改前
private async activateTenant(tenantId: string, packageId: string)

// 改后
private async activateTenant(tenantId: string, packageId: string, billingCycle?: string, bonusMonths?: number)
```

**计费周期→服务时长映射**：
| billing_cycle | 计算逻辑 | 示例 |
|---------------|---------|------|
| `monthly` | 使用套餐默认 `duration_days` | 30天 |
| `yearly` / `annual` | 365 + bonusMonths × 30 | 365+60=425天(送2月) |
| `perpetual` | 36500天（100年） | 永久 |

**续费延长逻辑**：
```
if 同套餐续费 AND 当前未过期:
  新到期日 = 当前到期日 + durationDays  (顺延)
else:
  新到期日 = 今天 + durationDays  (从今天算)
```

**租户表更新字段**：
```sql
UPDATE tenants SET
  package_id = ?,          -- 🔑 关键：更新到新套餐
  license_key = ?,         -- 新授权码
  license_status = 'active',
  status = 'active',
  expire_date = ?,         -- 正确计算的到期日
  max_users = ?,           -- 新套餐用户数
  max_storage_gb = ?,      -- 新套餐存储配额
  features = COALESCE(?, features),  -- 新套餐功能特性
  updated_at = NOW()
WHERE id = ?
```

## 六、全链路同步验证

### 链路1：续费升级页 → 一次性支付

```
Renew.vue → handlePay(packageId, billingCycle)
→ POST /payment/create (存储 billing_cycle, bonus_months)
→ 支付成功回调 → updateOrderStatus('paid')
→ 读取订单 billing_cycle, bonus_months
→ activateTenant(tenantId, packageId, billingCycle, bonusMonths)
→ UPDATE tenants SET package_id=新套餐, expire_date=新日期, max_users=新配额...
→ 所有下游系统 JOIN tenant_packages ON package_id → 自动获取新套餐 ✅
```

### 链路2：订阅管理页 → 自动续费

```
Subscription.vue → createSubscription(packageId, channel)
→ 创建订阅记录(package_id) → 签约 → handleSignSuccess()
→ executeDeduction() → 扣款成功
→ 查询套餐信息 → UPDATE tenants SET package_id=订阅套餐, expire_date=下期...
→ 所有下游系统 JOIN tenant_packages ON package_id → 自动获取新套餐 ✅
```

### 下游系统自动同步（无需额外修改）

| 系统 | SQL JOIN | 同步时机 |
|------|---------|---------|
| CRM系统权限 | `UserController.ts` → `LEFT JOIN tenant_packages p ON t.package_id = p.id` | 用户登录时 |
| CRM授权验证 | `tenantLicense.ts` → `LEFT JOIN tenant_packages p ON t.package_id = p.id` | license check 时 |
| 管理后台列表 | `admin/tenants.ts` → `LEFT JOIN tenant_packages p ON t.package_id = p.id` | 查看时 |
| 管理后台详情 | `admin/tenants.ts` → 同上 | 查看时 |
| 客户管理 | `admin/customer-management.ts` → 同上 | 查看时 |
| 会员中心 | `MemberService.getProfile()` → `LEFT JOIN tenant_packages tp ON t.package_id = tp.id` | 刷新页面时 |
