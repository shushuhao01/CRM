## 1. APP 服务器地址隐私保护（额外任务）

### 问题描述
APP 多处直接暴露了真实的服务器地址信息：
- `server-config/index.vue` 格式提示中包含真实域名 `abc789.cn`
- 登录页底部直接展示完整的服务器 host:port
- 设置页直接展示完整的服务器 host:port

### 修复方案

#### 1.1 新增 `maskedDisplayUrl` getter
**文件**：`crmAPP/src/stores/server.ts`

新增脱敏显示的 getter，规则如下：
- **IP 地址脱敏**：`192.168.1.100` → `192.***.***. 100`
- **域名脱敏**：`abc789.cn` → `ab****.cn`
- **保留原始 `displayUrl`**：仅供服务器配置页使用（管理员主动配置场景）

#### 1.2 修改格式提示
**文件**：`crmAPP/src/pages/server-config/index.vue`

```
修改前：支持格式：abc789.cn、192.168.1.100:3000
修改后：支持格式：crm.yourcompany.com、192.168.1.100:3000
```

#### 1.3 登录页 / 设置页使用脱敏地址
**文件**：
- `crmAPP/src/pages/login/index.vue`：`{{ serverStore.displayUrl }}` → `{{ serverStore.maskedDisplayUrl }}`
- `crmAPP/src/pages/settings/index.vue`：同上

## 3. 任务 5.3：APP 静音/免提真实实现

### 问题描述
`calling/index.vue` 的 `toggleMute()` 和 `toggleSpeaker()` 仅修改 UI 状态变量，没有调用系统 API 真实控制音频。

### 修复方案
**文件**：`crmAPP/src/pages/calling/index.vue`

#### 修改前（纯 UI 模拟）
```typescript
const toggleMute = () => {
  isMuted.value = !isMuted.value
  uni.showToast({ title: isMuted.value ? '已静音' : '已取消静音', icon: 'none' })
}
```

#### 修改后（真实 AudioManager 调用）
```typescript
const toggleMute = () => {
  const newState = !isMuted.value
  // #ifdef APP-PLUS
  try {
    plus.android.importClass('android.content.Context')
    plus.android.importClass('android.media.AudioManager')
    const activity: any = plus.android.runtimeMainActivity()
    const audioManager: any = activity.getSystemService('audio')
    audioManager.setMicrophoneMute(newState)
    isMuted.value = newState
  } catch (e) {
    isMuted.value = newState  // 失败也更新 UI
  }
  // #endif
  // #ifndef APP-PLUS
  isMuted.value = newState
  // #endif
  uni.showToast({ ... })
}
```

**免提同理**，调用 `audioManager.setSpeakerphoneOn(newState)`。

#### 安全性保障
- **平台条件编译**：`#ifdef APP-PLUS` 确保仅在 APP 原生环境调用
- **异常捕获**：`try/catch` 包裹，原生调用失败仍更新 UI 状态
- **H5 降级**：非 APP 平台保持原有 UI 模拟行为

## 4. 任务 5.1：官网 SEO 基础优化

### 问题描述
官网为纯 SPA 架构，缺少 SEO 基础设施：无 `robots.txt`、无 `sitemap.xml`、缺少 Open Graph 标签和结构化数据、页面切换不更新 `document.title`。

### 修复方案

#### 4.1 增强 `index.html`
**文件**：`website/index.html`

新增内容：
- `<meta name="author">` — 公司名称
- `<meta name="robots" content="index, follow">`
- `<link rel="canonical">`
- `<meta property="og:*">` — Open Graph 社交分享标签（type/title/description/url/site_name/locale）
- `<script type="application/ld+json">` — JSON-LD 结构化数据（SoftwareApplication 类型）

#### 4.2 新建 `robots.txt`
**文件**：`website/public/robots.txt`（新建）

```
User-agent: *
Allow: /
Disallow: /api/
Disallow: /admin/
Sitemap: https://www.yunke-crm.com/sitemap.xml
```

#### 4.3 新建 `sitemap.xml`
**文件**：`website/public/sitemap.xml`（新建）

包含 7 个核心页面 URL，设置合理的 `changefreq` 和 `priority`。

#### 4.4 路由级页面标题与描述
**文件**：`website/src/router/index.ts`

- 每条路由添加 `meta: { title, description }`
- 新增 `router.afterEach()` 守卫：
  - 动态设置 `document.title`
  - 动态更新 `<meta name="description">` 的 `content` 属性
