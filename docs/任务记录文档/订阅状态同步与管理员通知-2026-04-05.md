# 订阅状态同步与管理员通知

**日期**: 2026-04-05  
**状态**: ✅ 已完成  
**涉及模块**: 订阅服务(SubscriptionService)、管理后台(admin)、Webhook回调

---

## 一、问题概述

用户提出三个关键问题：
1. 客户在微信/支付宝平台取消订阅后，系统是否同步？
2. 客户详情页是否完整显示订阅状态（未订阅/订阅中/已取消等）？
3. 订阅状态变更时是否通知管理员？

---

## 二、现状分析

### ✅ 已有功能
- 客户详情页（Detail.vue）已有订阅状态字段：`订阅中`/`签约中`/`订阅暂停`/`一次性自付`
- 会员中心取消订阅 → 本地DB更新为 `cancelled` + 调用第三方解约API ✅
- 后端有 AdminNotificationService 通知框架，支持系统消息/钉钉/企微/邮件/公众号

### ❌ 三个缺失

| 缺失项 | 详情 |
|-------|------|
| 平台侧取消不同步 | 微信签约回调只处理 `change_type=ADD`，不处理 `DELETE`（用户在微信取消代扣）；支付宝只处理 `status=NORMAL`，不处理 `UNSIGN` |
| 状态显示不完整 | 客户详情缺少 `已取消订阅` 和 `订阅已过期` 两种状态；后端查询只查 `active/signing/paused`，忽略了 `cancelled/expired` |
| 无管理员通知 | 订阅创建/激活/取消/暂停/过期/扣款失败 均无通知；EVENT_TYPES 中无任何订阅相关事件 |

## 三、修复方案与实施

### 3.1 微信/支付宝平台侧取消同步

**修改文件**: 
- `backend/src/routes/public/subscription.ts`
- `backend/src/services/SubscriptionService.ts`

**实现**:
- 微信签约回调 `/sign-notify/wechat`：新增 `change_type === 'DELETE'` 分支 → 调用 `handlePlatformUnsign()`
- 支付宝签约回调 `/sign-notify/alipay`：新增 `status === 'UNSIGN'` 分支 → 调用 `handlePlatformUnsign()`
- `handlePlatformUnsign()` 方法：
  - 支持通过 subscriptionId / wechat_contract_id / alipay_agreement_no 三种方式查找订阅
  - 更新状态为 `cancelled`，记录取消原因为"平台侧取消"
  - 发送管理员通知

### 3.2 扣款回调异步对账

**修改文件**: 
- `backend/src/routes/public/subscription.ts`
- `backend/src/services/SubscriptionService.ts`

**实现**:
- 微信扣款回调 `/deduct-notify/wechat`：解析 `resource.transaction_id` 和 `trade_state` → 调用 `handleDeductCallback()`
- 支付宝扣款回调 `/deduct-notify/alipay`：解析 `trade_no` 和 `trade_status` → 调用 `handleDeductCallback()`
- `handleDeductCallback()` 方法：幂等处理扣款结果确认，失败时更新失败计数

### 3.3 管理员通知

**修改文件**: `backend/src/services/AdminNotificationService.ts`

**新增事件类型**:

| 事件类型 | 标签 | 级别 | 分类 |
|---------|------|-----|------|
| `subscription_created` | 新订阅签约 | info | 订阅动态 |
| `subscription_activated` | 订阅已激活 | success | 订阅动态 |
| `subscription_cancelled` | 订阅已取消 | warning | 订阅动态 |
| `subscription_paused` | 订阅已暂停 | warning | 订阅动态 |
| `subscription_expired` | 订阅已过期 | error | 订阅动态 |
| `subscription_deduct_failed` | 订阅扣款失败 | error | 订阅动态 |
| `subscription_platform_cancelled` | 平台侧取消订阅 | error | 订阅动态 |

**通知触发点**（SubscriptionService.ts）:

| 触发时机 | 事件类型 | 通知内容 |
|---------|---------|---------|
| 用户发起签约 | subscription_created | 租户名+套餐+金额+渠道 |
| 签约成功回调 | subscription_activated | 租户名+渠道+金额 |
| 用户主动取消 | subscription_cancelled | 租户名+联系人+生效日期 |
| 连续3次扣款失败 | subscription_paused | 租户名+渠道+金额 |
| 每次扣款失败 | subscription_deduct_failed | 租户名+金额+失败原因 |
| 3次重试全失败 | subscription_expired | 租户名+金额+渠道 |
| 平台侧解约 | subscription_platform_cancelled | 租户名+联系人+渠道+剩余服务截止日 |

### 3.4 客户详情订阅状态完善

**修改文件**:
- `admin/src/views/tenant-customers/Detail.vue`
- `backend/src/routes/admin/tenants.ts`

**前端新增状态显示**:

| 状态 | 标签样式 | 显示文本 |
|-----|---------|---------|
| active | success/dark | 订阅中 |
| signing | warning | 签约中 |
| paused | danger | 订阅暂停 |
| cancelled | info | 已取消订阅 |
| expired | danger/dark | 订阅已过期 |
| null/其他 | default | 一次性自付 |

**后端查询优化**:
- 移除 `status IN ('active','signing','paused')` 限制
- 使用 `ORDER BY FIELD(status, 'active', 'signing', 'paused', 'cancelled', 'expired')` 优先返回活跃状态
- 新增返回 `cancel_date` 和 `cancel_reason` 字段

## 五、订阅状态完整流转图

```
创建签约 → signing (签约中)
    ↓ 签约成功回调
  active (订阅中) ←→ 自动扣款成功(续期)
    ↓ 扣款失败
  active (fail_count+1)
    ↓ 连续3次失败
  paused (订阅暂停)
    ↓ 重试3次仍失败
  expired (订阅已过期)

取消路径:
  active/paused → cancelled (已取消)
  ├─ 用户在会员中心取消
  ├─ 用户在微信支付设置中取消 (平台DELETE回调)
  └─ 用户在支付宝设置中取消 (平台UNSIGN回调)
```

## 六、测试要点

1. **会员中心取消订阅** → 管理员收到 `subscription_cancelled` 通知 → 客户详情显示"已取消订阅"
2. **模拟微信 DELETE 回调** → 系统更新订阅状态为 cancelled → 管理员收到 `subscription_platform_cancelled` 通知
3. **模拟支付宝 UNSIGN 回调** → 同上
4. **扣款失败3次** → 管理员收到3次 `subscription_deduct_failed` + 1次 `subscription_paused`
5. **重试3次全失败** → 管理员收到 `subscription_expired` 通知
6. **客户详情页** → 正确显示所有6种状态标签
7. **通知规则** → 管理员可在通知设置中为新的订阅事件配置钉钉/企微/邮件推送
