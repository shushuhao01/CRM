## 任务 4.3：Admin后台登录添加验证码 ✅

### 问题
- Admin登录无验证码，生产环境有暴力破解风险

### 实现方案
1. **后端**：安装 `svg-captcha` 包，在 `admin/auth.ts` 添加 `GET /auth/captcha` 路由
2. **验证码特性**：4位彩色字符、排除易混淆字符(0oO1ilI)、3条干扰线、2分钟有效
3. **存储**：内存Map缓存（定时清理过期验证码），生产可替换Redis
4. **登录校验**：验证码用后即弃，一次性使用
5. **向下兼容**：不传验证码参数也能登录（兼容已有客户端）

### 修改文件
- `backend/src/routes/admin/auth.ts` — 添加captcha生成路由+登录验证码校验
- `backend/src/middleware/adminAuth.ts` — `/auth/captcha` 添加为公开路径
- `admin/src/api/admin.ts` — 新增 `getCaptcha` API方法
- `admin/src/stores/user.ts` — login方法添加 `captchaId/captchaCode` 参数
- `admin/src/views/Login.vue` — 添加验证码输入框、SVG显示、点击刷新、登录失败自动刷新

### 新增依赖
```
backend: svg-captcha (npm install svg-captcha --save)
```

## 任务 4.7：ESLint规则统一 ✅

### 问题
- 主前端使用ESLint 9 flat config，Admin使用ESLint 8 `.eslintrc.cjs`，规则不一致

### 修改方案
因ESLint版本差异，保持各自配置格式但统一核心规则：
- `vue/vue3-recommended` → `vue/vue3-essential`（与主前端 flat/essential 对齐）
- `@typescript-eslint/no-unused-vars` 从 `'warn'` 改为与主前端一致的 `['error', { argsIgnorePattern: '^_', ... }]`
- 新增 `@typescript-eslint/no-unsafe-argument: 'off'`
- 添加注释说明两端规则对齐关系

### 修改文件
- `admin/.eslintrc.cjs` — 统一规则
