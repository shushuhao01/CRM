## 一、功能概述

用户限制模式分为两种：
- **限制注册用户数（total）**：限制租户可注册的总用户数，超过上限不可新增用户。
- **限制同时在线席位（online）**：限制同时在线人数（席位），注册用户总数不受限制。

套餐可配置为 `total`、`online` 或 `both`（可选模式），激活后模式跟随租户，管理员可在后台切换。

## 四、关键逻辑说明

### 4.1 用户限制模式与授权码绑定

- **激活时**：用户限制模式 `user_limit_mode` 随套餐配置写入租户记录（tenants表），授权码激活后模式固定在租户级别。
- **变更方式**：仅管理员可在管理后台的租户详情页点击"切换"按钮修改。
- **优先级**：`user_limit_mode` 存储在租户表，独立于套餐。升级套餐不会自动改变已有模式，除非管理员手动同步。

### 4.2 在线席位制逻辑

- **max_online_seats**：基础席位数（来自套餐/管理员配置）
- **extra_online_seats**：扩容额外席位数（通过扩容购买增加）
- **总席位** = max_online_seats + extra_online_seats
- **注册用户**：online模式下不受限制，可无限注册
- **在线控制**：CRM后端在用户登录时检查当前在线数是否超过总席位

### 4.3 扩容购买

- 扩容类型 `type: 'user'` 在API层统一发送
- 后端根据租户的 `user_limit_mode` 自动决定是增加 `max_users` 还是 `extra_online_seats`
- 前端仅需根据模式切换显示文案（席位/人）

## 五、已知遗留项

1. **ESLint parsing error**：admin和website项目中的 `Parsing error: '>' expected` 是eslint配置与Vue SFC `<script setup lang="ts">` 的兼容性问题，非代码缺陷。
2. **组件名称警告**：`Dashboard`、`Profile`、`Renew`、`Capacity` 组件名为单词，触发eslint `multi-word` 规则警告，是预存问题。

## 六、总结

本次开发完成了用户限制模式（online/total）在三个系统中的全链路支持：

- **管理后台**：租户/私有客户的创建、编辑、详情、列表全面支持两种模式的配置和展示。
- **官网**：会员中心的概览、企业信息、续费升级、扩容管理全部根据模式动态显示。
- **CRM端**：限制逻辑由后端控制，前端管理员可管理在线成员和强制踢出。

所有修改保持最小化原则，仅针对相关功能做调整，未引入无关改动。
