## 阶段 2：邮件设置功能审查与错误提示优化 ✅

### 问题描述

用户在基础配置的邮件设置中填写163邮箱SMTP配置后，发送测试邮件报错：
```
Invalid login: 535 Error: authentication failed
```

### 原因分析

1. **配置问题（非代码Bug）**：163邮箱需要使用**授权码**而非登录密码
   - 用户填写的 `WANGYI@hf4638` 是登录密码，不是163邮箱授权码
   - 163邮箱需先在网页版设置中开启SMTP/POP3服务，然后获取授权码
   
2. **代码不足**：后端错误提示直接返回原始英文错误，用户无法理解如何修复

### 修复内容

#### 修复1：后端 Admin 邮件测试 - 友好错误提示

**修改文件**: `backend/src/routes/admin/systemConfig.ts`

新增智能错误识别，针对以下常见SMTP错误返回中文指引：

| 错误类型 | 错误特征 | 友好提示 |
|---------|---------|---------|
| 认证失败 | 535/authentication failed | 提示使用授权码而非登录密码 |
| 连接失败 | connect/econnrefused/timeout | 提示检查服务器地址和端口 |
| SSL/TLS错误 | certificate/ssl/tls | 提示检查SSL/TLS设置与端口匹配 |
| DNS解析失败 | ENOTFOUND | 提示检查SMTP服务器地址拼写 |
| 发件人错误 | envelope/sender/from | 提示检查发件人邮箱格式 |
| 收件人错误 | recipient/rcpt | 提示测试邮箱地址无效 |

同时优化了测试邮件的HTML内容，增加了配置信息展示。

#### 修复2：后端 CRM 租户端邮件测试 - 同步友好错误提示

**修改文件**: `backend/src/routes/system/systemSettings.ts`

与 Admin 端保持一致的错误识别逻辑。

#### 修复3：前端邮件配置 - 增加授权码说明

- 将"邮箱密码"标签改为"邮箱密码/授权码"

### 163邮箱配置指南（供用户参考）

1. 登录 https://mail.163.com → 设置 → POP3/SMTP/IMAP
2. 开启 POP3/SMTP 服务（需要手机验证）
3. 系统会生成一个**授权码**（16位字母，如 `ABCDEFGHIJKLMNOP`）
4. 在CRM邮件配置中填写：
   - SMTP服务器: `smtp.163.com`
   - SMTP端口: `465`
   - 启用SSL: ✅
   - 邮箱密码: 填写**授权码**（不是登录密码！）
