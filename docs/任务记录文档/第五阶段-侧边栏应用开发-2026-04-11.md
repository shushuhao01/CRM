## 一、Bug修复 - 会话存档支付弹窗支付方式互斥

### 问题描述
会话存档增值服务购买弹窗中，支付方式（微信支付/支付宝/对公转账）可以同时选中，缺少互斥关系。

### 根因分析
Element Plus `el-radio-button` 组件在 2.3.x 版本使用 `label` 属性绑定值，而代码中错误使用了 `value` 属性（`value` 是 2.6+ 版本才引入的新写法），导致 radio-group 的 v-model 无法正确绑定，表现为可同时选中多个选项。

### 修复方案
将 `el-radio-button` 的 `value` 属性改为 `label` 属性。

### 修改文件
| 文件 | 修改内容 |
|------|---------|
| src/views/Wecom/ChatArchive.vue | `el-radio-button value="wechat"` → `el-radio-button label="wechat"` (3处) |

## 二、Phase 5 - 企微侧边栏应用开发

### 2.1 功能概述

在企业微信聊天窗口右侧嵌入侧边栏应用，员工与客户聊天时可实时查看：
- 客户基本信息（来自企微同步 + CRM关联）
- 购买统计（订单数、累计消费、最后购买时间）
- 最近订单列表（最近5笔）
- 客户标签和等级
- 跳转CRM查看完整详情

### 2.2 技术架构

```
[企微聊天窗口] → [侧边栏iframe] → /wecom-sidebar?corpId=xxx
                      |
                      v
              [JS-SDK获取当前客户]
                      |
                      v
              [验证绑定/登录绑定]
                      |
                      v
              [加载客户详情API]
                      |
                      v
              [展示客户信息+订单]
```

### 2.4 后端API设计

#### POST /wecom/sidebar/bind-account
- 用途: 侧边栏用户绑定CRM账号（登录）
- 认证: 无需authenticateToken，用CRM用户名密码验证
- 请求: `{ wecomUserId, corpId, username, password }`
- 响应: `{ token, user, binding }`
- 逻辑: 验证密码 → 创建/更新WecomUserBinding → 签发侧边栏JWT

#### GET /wecom/sidebar/verify-binding
- 用途: 检查企微成员是否已绑定CRM账号
- 认证: 无
- 参数: `?wecomUserId=xxx&corpId=xxx`
- 响应: `{ bound, token?, user?, binding? }`

#### GET /wecom/sidebar/customer-detail
- 用途: 获取当前聊天客户的CRM详情+订单
- 认证: authenticateSidebarToken（侧边栏JWT）
- 参数: `?externalUserId=xxx`
- 响应: `{ found, wecomCustomer, crmCustomer, orders[], stats }`

#### POST /wecom/sidebar/js-sdk-config
- 用途: 获取企微JS-SDK签名配置
- 认证: 无
- 请求: `{ url, corpId }`
- 响应: `{ corpId, agentId, timestamp, nonceStr, corpSignature, agentSignature }`

### 2.5 侧边栏认证机制

采用侧边栏专用JWT token：
- 类型标识: `type: 'sidebar'`
- 包含字段: wecomUserId, crmUserId, crmUserName, tenantId, corpId
- 使用同一JWT密钥签发，通过 type 字段区分
- 存储在 localStorage('wecom_sidebar_token')
- 由 authenticateSidebarToken 中间件验证

### 2.6 JS-SDK集成

WecomApiService新增方法：
- `getJsSdkTicket(accessToken)` - 获取企业级jsapi_ticket（带缓存）
- `getAgentJsSdkTicket(accessToken)` - 获取应用级agent_ticket（带缓存）
- `generateJsSdkSignature(ticket, nonceStr, timestamp, url)` - SHA1签名生成

前端页面流程：
1. 动态加载 jWeixin-1.2.0.js
2. 调用后端获取签名配置
3. wx.config() 注入企业权限
4. wx.agentConfig() 注入应用权限
5. wx.invoke('getCurExternalContact') 获取当前外部联系人ID

### 2.7 侧边栏管理页面增强

Sidebar.vue 新增内置侧边栏客户详情URL提示：
- 显示当前系统的侧边栏地址
- 说明如何在企微后台配置
- 提示替换 corpId 参数

### 2.8 页面状态流转

```
loading → 检测企微环境
  ├─ 非企微环境 → no-sdk（开发模式可手动输入调试）
  └─ 企微环境 → JS-SDK初始化
       ├─ 失败 → no-sdk
       └─ 成功 → getCurExternalContact
            ├─ 失败 → error
            └─ 成功 → checkBinding
                 ├─ 未绑定 → login（登录绑定CRM账号）
                 └─ 已绑定 → detail（加载客户详情）
```

## 三、会话存档UI重构 - 微信风格三栏布局

### 3.1 需求说明
用户要求会话存档的聊天会话Tab要像微信那样的UI：
- 左侧：客户/会话列表
- 右侧：聊天内容（气泡式）
- 顶部：筛选搜索工具栏
- 右面板：客户信息面板

### 3.2 布局重构

**原布局**: 2栏（左=会话列表 + 右=聊天区）
**新布局**: 顶部搜索栏 + 3栏（左=会话列表 + 中=消息区 + 右=客户信息面板）

```
┌─────────────────────────────────────────────────┐
│  🔍 搜索    消息类型 ▼   日期范围      [搜索]   │
├──────────┬─────────────────────┬────────────────┤
│ 会话列表   │  消息区 (微信风格)   │ 客户信息面板    │
│ 👤 张三   │  ┌────────────────┐│ 👤 张三       │
│ 👤 李四   │  │ 今天           ││ ID: xxx       │
│ 👤 王五   │  │ 👤: 你好       ││               │
│           │  │     好的 :🧑‍💼  ││ 📊 会话概况    │
│           │  │ 👤: 产品价格？  ││ 消息数: 42    │
│           │  │   报价单 :🧑‍💼  ││               │
│           │  └────────────────┘│ 👥 对话成员    │
│           │                    │ 📋 快捷操作    │
├──────────┴─────────────────────┴────────────────┤
│  « 1 2 3 »                                      │
└─────────────────────────────────────────────────┘
```

### 3.3 新增功能

| 功能 | 说明 |
|------|------|
| **顶部筛选栏** | 搜索联系人、消息类型筛选、日期范围筛选、会话总数 |
| **日期分割线** | 同一天的消息不重复显示日期，跨天显示分割线（今天/昨天/x月x日） |
| **消息内搜索** | 聊天区顶部可折叠的消息内搜索栏 |
| **客户信息面板** | 可折叠的右侧面板，显示客户资料、会话统计、对话成员 |
| **快捷操作** | 质检标记、导出聊天记录（TXT格式） |
| **短时间格式** | 气泡旁显示 HH:MM 短时间，不再显示完整日期时间 |
| **头像改方形** | 会话列表和气泡头像改为微信风格的圆角方形 |
| **背景色优化** | 消息区背景改为 #ebebeb（微信灰色），更贴合微信风格 |
