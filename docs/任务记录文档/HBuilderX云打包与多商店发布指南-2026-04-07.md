### 1.2 逐项详解

| 序号 | 配置项 | 填什么 | 为什么 |
|------|--------|--------|--------|
| **①** | Android 包名 | `com.xianhu.crm.dialer` | 应用唯一标识，各商店保持一致，一旦确定不能改 |
| **②** | 证书选择 | **☑ 使用自有证书** | 上架商店必须用自有证书，公共测试证书只能用于开发调试 |
| **③** | 证书别名 | `crm_dialer` | 生成 keystore 时设置的 alias |
| **④** | 证书私钥密码 | `XianHu@CRM2026!` | 生成 keystore 时设置的密码（密钥库密码和别名密码相同） |
| **⑤** | 证书文件 | 点「浏览」选择 `D:\kaifa\CRM - 1.8.0\crmAPP\crm-dialer-release.keystore` | 之前生成的签名证书文件 |
| **⑥** | 打包类型 | **☑ 正式包(release)** | 上架商店必须是正式包，测试包无法提交 |
| **⑦** | 渠道名称 | **留空** | 你要用DCloud多渠道发布的话，渠道会自动处理；或者填 `default` |
| **⑧** | 广告联盟 | **☐ 不勾选** | 你的APP是企业工具，不需要接入广告。勾了会植入广告SDK，商店审核可能会问 |
| **⑨** | 打包方式 | **☑ 安心打包** | 见下方详解 |

### 1.3 广告联盟、传统打包、安心打包的区别

#### 📢 广告联盟（DCloud AD）
| 项目 | 说明 |
|------|------|
| **是什么** | DCloud 提供的广告变现服务，接入后APP内会展示广告 |
| **你要不要勾** | ❌ **不勾！** 你的APP是企业内部工具，不是面向C端用户的免费APP |
| **勾了会怎样** | APP内会出现广告位（开屏、插屏、信息流等），影响用户体验 |
| **什么时候该勾** | 只有做免费APP靠广告变现时才需要 |

#### 📦 传统打包 vs 安心打包
| 对比项 | 传统打包 | ⭐ 安心打包（推荐） |
|--------|---------|-------------------|
| **代码上传** | 完整源码上传到DCloud云端编译 | 前端代码在本地加密后再上传 |
| **安全性** | 源码完整经过DCloud服务器 | ✅ 代码加密传输，DCloud看不到源码 |
| **编译速度** | 一样 | 一样 |
| **费用** | 免费 | 免费 |
| **兼容性** | 100%兼容 | 99.9%兼容（极少数特殊原生插件可能不支持） |
| **你选哪个** | — | ✅ **选安心打包** |

> **结论：选安心打包。** 你的APP涉及企业CRM数据和服务器地址，安心打包能保护代码安全。只有在安心打包编译失败时，才退回传统打包。

## 二、DCloud 多商店一键发布（uni 多渠道上传）

### 2.1 功能入口

打包成功后，HBuilderX 会弹出下载链接。下载APK后：

**菜单 → 发行 → 原生APP-上传到应用商店**

或者直接在 [DCloud 开发者中心](https://dev.dcloud.net.cn/) → 我的应用 → 选择你的应用 → **「多渠道发布」**

### 2.2 首次配置：绑定各商店开发者账号

在 DCloud 开发者中心的「多渠道发布」页面，需要先绑定各商店的开发者账号：

| 商店 | 需要填写的信息 | 你的状态 |
|------|---------------|---------|
| **华为** | 华为开发者账号 + AppID + Client ID + Client Secret | ✅ 已审核通过 |
| **小米** | 小米开发者账号 + AppID | ✅ 已审核通过 |
| **OPPO** | OPPO开发者账号 + AppKey + AppSecret | ✅ 已审核通过 |
| **vivo** | vivo开发者账号 + Access Key + Access Secret | ✅ 已审核通过 |
| **荣耀** | 荣耀开发者账号 + AppID + Client ID + Client Secret | ✅ 已审核通过 |

#### 获取各商店的 API 密钥（用于自动上传）

**华为 AppGallery Connect：**
1. 登录 https://developer.huawei.com/consumer/cn/service/josp/agc/index.html
2. 「用户与访问」→「Connect API」→ 创建 API 客户端
3. 记录 **Client ID** 和 **Client Secret**
4. 「我的应用」中找到 **AppID**

**小米开放平台：**
1. 登录 https://dev.mi.com/distribute/app/list
2. 创建应用后，获取小米 **AppID**
3. DCloud只需要账号密码登录即可

**OPPO 开放平台：**
1. 登录 https://open.oppomobile.com/
2. 「管理中心」→「账号设置」→「API配置」
3. 获取 **AppKey** 和 **AppSecret**

**vivo 开放平台：**
1. 登录 https://dev.vivo.com.cn/
2. 「管理中心」→ 找到应用
3. 获取 **Access Key** 和 **Access Secret**

**荣耀 开发者服务平台：**
1. 登录 https://developer.honor.com/
2. 类似华为的配置方式
3. 获取 **Client ID** 和 **Client Secret**

### 2.3 一键发布操作步骤

```
第1步：在 DCloud 开发者中心绑定完所有商店账号

第2步：上传 APK
       → 点「上传新版本」
       → 选择打好的 APK 文件
       → 系统会自动读取版本号、包名等信息

第3步：填写应用信息
       → 应用名称：云客CRM外呼助手
       → 一句话简介：企业电销外呼管理工具，高效外呼智能管理
       → 详细描述：[粘贴之前准备好的应用描述]
       → 应用截图：上传 5-8 张截图
       → 应用分类：商务办公 / 效率工具
       → 隐私政策URL：你的隐私政策网址

第4步：勾选要发布的商店
       ☑ 华为应用市场
       ☑ 小米应用商店
       ☑ OPPO 软件商店
       ☑ vivo 应用商店
       ☑ 荣耀应用市场

第5步：点击「提交发布」→ 系统自动向5个商店提交审核
```

### 2.4 各商店特殊注意事项

| 商店 | ⚠️ 特殊要求 | 怎么处理 |
|------|------------|---------|
| **华为** | 需要**软件著作权证书**才能上架 | 如果还没有，先跳过华为，其他商店先上 |
| **华为** | 隐私政策要求非常严格 | 确保隐私弹窗+隐私政策URL都有 |
| **小米** | 需要提供**测试账号** | 在审核备注中填写测试服务器+账号密码 |
| **OPPO** | 需要**软件著作权**或**ICP备案** | 同华为 |
| **vivo** | 需要**软件著作权** | 同上 |
| **荣耀** | 类似华为的要求 | 同华为 |

> 💡 **如果暂时没有软著**：先提交**小米**（一般不强制要求软著），其他商店同步申请软著（正常1-3个月下证，加急7-15天）。

## 四、打包后流程

### 4.1 打包成功
```
HBuilderX 控制台输出：
[info] 项目 'crmAPP' 打包成功
[info] 下载地址: https://ide.dcloud.net.cn/build/download/xxx-xxx-xxx

→ 点击链接下载 APK
→ 重命名为: 云客CRM外呼助手_v1.0.0_release.apk
→ 保存到: D:\kaifa\CRM - 1.8.0\crmAPP\release\
```

### 4.2 打包失败常见原因

| 错误提示 | 原因 | 解决 |
|---------|------|------|
| `AppID is invalid` | AppID不正确 | 检查manifest.json的appid字段 |
| `证书密码错误` | 密码输错了 | 密码是 `XianHu@CRM2026!` 注意大小写和特殊字符 |
| `证书别名不存在` | alias输错了 | 别名是 `crm_dialer` 注意下划线 |
| `打包队列排队中` | 排队正常现象 | 耐心等待，不要重复提交 |
| `编译失败` | 代码有语法错误 | 查看控制台详细报错，修改后重新打包 |
| `安心打包失败` | 极少数兼容问题 | 改为传统打包重试 |

### 4.3 真机测试
```
打包完成 → 手机安装APK → 按测试清单逐项验证 → 全部通过 → 提交商店
```
