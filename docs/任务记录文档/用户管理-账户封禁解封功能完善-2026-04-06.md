# 用户管理 - 账户封禁/解封功能完善

## 任务信息
- **任务日期**: 2026-04-06
- **任务类型**: 功能完善
- **涉及模块**: 系统管理 > 用户管理
- **优先级**: 高

## 任务背景

CRM系统的用户管理模块已有多次登录失败自动锁定账户的机制（后端已实现 `loginFailCount >= 5` 时自动锁定），但前端用户管理界面缺少：
1. 显示用户账号是否被封禁/锁定的状态列
2. 管理员手动封禁和解除封禁的操作入口
3. 搜索筛选中缺少"锁定"状态选项

## 现有基础设施

### 数据库/实体（已有，无需修改）
- `User` 实体 `status` 字段：`'active' | 'inactive' | 'resigned' | 'locked'`
- `loginFailCount` 字段：记录登录失败次数
- `lockedAt` 字段：记录账户锁定时间

### 后端API（已有，无需修改路由）
- `PATCH /api/v1/users/:id/status` - 更新用户状态，支持 `active/inactive/locked`
- `POST /api/v1/users/:id/unlock` - 解锁用户账户（重置失败次数和锁定时间）

### 前端API服务（已有）
- `userApiService.updateUserStatus(id, status)` - 更新用户状态
- `userApiService.unlockAccount(userId)` - 解锁账户

## 功能流程

### 自动封禁流程（已有）
```
用户连续登录失败 5 次
  → 后端自动将 status 设为 'locked'
  → 记录 lockedAt 时间
  → 用户无法再登录，提示"账户已被锁定，请联系管理员"
```

### 管理员手动封禁流程（新增）
```
管理员在用户列表 → 点击"更多" → 选择"封禁账户"
  → 确认对话框
  → 调用 PATCH /api/v1/users/:id/status { status: 'locked' }
  → 后端更新状态并记录 lockedAt
  → 刷新用户列表
  → 账号状态列显示"已封禁"红色标签
```

### 管理员解除封禁流程（新增）
```
管理员在用户列表 → 点击"更多" → 选择"解除封禁"
  → 确认对话框
  → 调用 POST /api/v1/users/:id/unlock
  → 后端重置 status='active'、lockedAt=null、loginFailCount=0
  → 刷新用户列表
  → 账号状态列显示"正常"绿色标签
```

### 密码重置自动解锁（已有）
```
管理员重置用户密码时
  → 如果用户 status === 'locked'
  → 自动解锁：status='active'、lockedAt=null、loginFailCount=0
```

## 权限控制
- 封禁/解封操作需要 `canEditUser` 权限（管理员或经理）
- 系统预设用户（superadmin、admin）不可被封禁
- 操作日志记录在后端 `logOperation` 中
