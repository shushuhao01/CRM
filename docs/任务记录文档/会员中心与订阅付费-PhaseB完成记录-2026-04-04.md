## 一、后端新增文件（3个）

### 1.1 `backend/src/middleware/memberAuth.ts` — 会员认证中间件
- `generateMemberToken()` — 生成会员JWT Token（7天有效期）
- `verifyMemberToken()` — 验证会员JWT Token
- `memberAuth` — Express中间件，验证Bearer Token → 查询租户 → 注入 `req.memberTenant`
- 独立 JWT Secret（`MEMBER_JWT_SECRET`），与CRM用户和Admin认证完全隔离

### 1.2 `backend/src/services/MemberService.ts` — 会员中心服务（~550行）
- `loginByPassword()` — 手机号+密码登录
- `loginByCode()` — 手机号+验证码登录
- `setPassword()` — 首次设置/重置密码（需验证码）
- `changePassword()` — 修改密码（验证旧密码）
- `getProfile()` — 获取完整会员资料（租户+套餐+授权+订阅+使用情况）
- `updateProfile()` — 修改企业名称/联系人/邮箱
- `getBills()` — 查询账单（合并 payment_orders + subscription_deduct_logs）
- `getLicense()` — 获取授权信息和操作日志
- `logLogin()` — 记录登录日志到 member_login_logs

### 1.3 `backend/src/routes/public/member.ts` — 会员中心API路由（~210行）
| 方法 | 路径 | 说明 | 鉴权 |
|------|------|------|------|
| POST | `/login` | 登录（支持密码/验证码两种方式） | 无 |
| POST | `/send-code` | 发送验证码 | 无 |
| POST | `/set-password` | 设置/重置密码 | 无 |
| GET | `/profile` | 获取会员资料 | Member JWT |
| PUT | `/profile` | 修改会员资料 | Member JWT |
| GET | `/bills` | 查询账单 | Member JWT |

| POST | `/change-password` | 修改密码 | Member JWT |
| POST | `/logout` | 退出登录 | 无 |

## 五、认证体系总结

```
三套完全独立的认证体系：

1. CRM用户   → authenticateToken 中间件 → req.user
   Secret: JWT_SECRET (crm-secret-key)

2. Admin管理员 → adminAuthMiddleware 中间件 → req.adminUser  
   Secret: JWT_SECRET (admin-secret-key)

3. 会员中心   → memberAuth 中间件 → req.memberTenant  (新增)
   Secret: MEMBER_JWT_SECRET (member-center-secret-key-2026)
```
