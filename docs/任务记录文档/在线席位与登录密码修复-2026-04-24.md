# 在线席位与登录密码相关修复 - 2026-04-24

## 修复内容概述

### 1. 登录页"请先验证企业身份"与企业信息同时显示 [高优先级]

**文件**: `src/views/Login.vue`

**根因**: Vue 模板中 `v-else` 配对错误。过期提醒横幅使用了独立的 `v-if="licenseVerified && tenantExpired"`，导致其后的 `v-else` 与过期横幅配对，而非与第一个已验证状态的 `v-if` 配对。当 license 已验证但未过期时，"请先验证企业身份"引导入口会与企业信息同时显示。

**修复方案**: 将过期提醒横幅移入已验证状态区块内部，使 `v-else` 正确对应未验证状态。

### 2. 密码被重置后提示"首次登录"改为"密码已被重置" [高优先级]

**文件**:
- `backend/src/controllers/UserController.ts` — 后端密码重置逻辑
- `src/App.vue` — 前端密码状态判断
- `src/components/PasswordChangeModal.vue` — 密码修改弹窗

**根因**: 后端管理员重置密码时将 `passwordLastChanged` 设为 `null`，前端以 `!user.passwordLastChanged` 判断"首次登录"，导致密码重置场景被误判为首次登录。同时 `isFirstLogin` 的计算逻辑 `neverChangedPassword || forcePasswordChange` 在 `forcePasswordChange=true` 时始终为 `true`。

**修复方案**:
- 后端: 管理员重置密码时不再清空 `passwordLastChanged`，保留原值以区分首次登录和密码重置
- 前端 App.vue: `isFirstLogin = neverChangedPassword`（仅"从未改过密码"才算首次登录）
- 前端 PasswordChangeModal.vue: 新增"密码已被重置 - 请设置新密码"标题和提示文案

### 3. 管理后台客户列表"在线/席位/用户"列优化 [中优先级]

**文件**:
- `admin/src/views/private-customers/List.vue`
- `admin/src/views/tenant-customers/List.vue`
- `admin/src/views/customer-management/AllCustomers.vue`
- `admin/src/views/customer-management/RenewalReminders.vue`

**修改内容**:
- 数字字体从 12px 放大到 14px，分隔符从 10px 放大到 12px
- 移除整个单元格的 `el-tooltip` 包裹
- 将 tooltip 仅添加到"限在"/"限注"标签上，悬浮时显示限制模式说明
- 标签字体从 9px 调整为 10px，增加内边距提升可读性

### 4. syncAllTenantOnlineCount 遗漏 'both' 模式 [高优先级]

**文件**: `backend/src/services/OnlineSeatService.ts`

**根因**: `syncAllTenantOnlineCount()` 的 SQL 条件 `WHERE t.user_limit_mode = 'online'` 只同步 `online` 模式的租户在线人数，遗漏了 `both` 模式。

**修复**: 改为 `WHERE t.user_limit_mode IN ('online', 'both')`

### 5. 清理与注释修正

- **移除 DEBUG 代码**: `backend/src/routes/onlineSeat.ts` 中 `/stats` 路由的调试查询
- **注释修正**: 心跳路由注释从"每5分钟"修正为"每30秒"
