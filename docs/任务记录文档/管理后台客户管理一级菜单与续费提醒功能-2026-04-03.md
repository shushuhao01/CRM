# 管理后台客户管理一级菜单与续费提醒功能 - 2026-04-03

## 任务概述

在管理后台（Admin）新增一级菜单「**客户管理**」，包含两个二级菜单：
1. **所有客户** — 统一展示私有客户和租户客户的合并数据，含统计概览、筛选器、完整字段列表（默认10条/页，后端翻页）
2. **续费提醒** — 统计即将到期的客户，按 20% 预警和 10% 重点提醒分级，支持跟进操作和最新跟进记录展示

## 需求分析

### 业务背景

原系统将私有客户和租户客户分开管理，缺少一个统一视角来查看所有客户的整体情况。同时缺少续费提醒机制，开发团队的销售人员无法及时跟进即将到期的客户，影响续费率和销售业绩。

## 技术实现说明

### 1. 所有客户列表 — UNION ALL 合并查询

后端使用 `UNION ALL` 将 `licenses` 表（私有客户）和 `tenants` 表（租户客户）合并为统一数据结构：

```sql
(SELECT ... FROM licenses WHERE deleted_at IS NULL ...)
UNION ALL
(SELECT ... FROM tenants LEFT JOIN tenant_packages ... WHERE deleted_at IS NULL ...)
ORDER BY createdAt DESC LIMIT ? OFFSET ?
```

统一字段映射：
- `licenses.customer_name` → `customerName`
- `tenants.name` → `customerName`
- `licenses.customer_contact` → `contact`
- `tenants.contact` → `contact`
- `licenses.expires_at` → `expireDate`
- `tenants.expire_date` → `expireDate`
- 新增 `customerType` 字段标识来源（'private' / 'tenant'）

### 2. 续费提醒 — 剩余百分比计算

```sql
-- 剩余百分比 = 剩余天数 / 总订阅天数 * 100
CASE
  WHEN DATEDIFF(expires_at, COALESCE(activated_at, created_at)) > 0
  THEN ROUND(DATEDIFF(expires_at, NOW()) * 100.0 / DATEDIFF(expires_at, COALESCE(activated_at, created_at)), 1)
  ELSE 0
END as remainingPercent
```

预警级别：
- 🔴 **重点提醒**：`remainingPercent ≤ 10%`
- 🟡 **预警提醒**：`10% < remainingPercent ≤ 20%`
- 🟢 **关注客户**：`20% < remainingPercent ≤ 50%`

排除条件：永久授权客户（`license_type = 'perpetual'`）和已过期客户不在列表中。

### 3. 跟进记录 — 共享数据

跟进记录存储在 `customer_follow_ups` 表中，通过 `customer_id + customer_type` 关联。
- 「所有客户」页面和「续费提醒」页面共用同一套跟进数据
- 列表中通过子查询获取最新跟进记录（`ORDER BY created_at DESC LIMIT 1`）
- 跟进对话框支持查看历史记录和添加新记录

## 注意事项

1. **不影响现有功能**：私有客户和租户客户的原有列表页面、操作功能完全不变
2. **数据只读合并**：所有客户列表仅做查询展示，不修改原始数据
3. **跟进记录独立存储**：新建 `customer_follow_ups` 表，不修改任何现有表结构
4. **续期操作复用**：续期跳转到原有的私有/租户客户列表页面，使用已有的续期对话框
5. **自动建表**：`customer_follow_ups` 表由后端启动时自动创建，无需手动执行 SQL
6. **响应式适配**：两个页面均支持移动端自适应布局
