## 补充修复（2026-04-05 第二轮）

### JWT_REFRESH_SECRET 环境变量补全

**问题**：所有 `.env` 文件均缺少 `JWT_REFRESH_SECRET` 配置。生产环境每次服务重启会自动生成新的随机RefreshToken密钥，导致所有用户的RefreshToken失效（需重新登录）。

**修复**：
- `.env.local` — 添加独立的 `JWT_REFRESH_SECRET` 开发密钥
- `.env.production.server` — 添加独立的 `JWT_REFRESH_SECRET` 生产密钥
- `.env` / `.env.production` / `.env.example` — 添加占位符和说明
- `backend/src/config/jwt.ts` — 弱密钥黑名单新增3个模板占位符密钥

**安全说明**：`JWT_SECRET` 和 `JWT_REFRESH_SECRET` 使用不同的密钥，即使AccessToken密钥泄露也无法伪造RefreshToken。
