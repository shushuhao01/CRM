## 二、⚠️ 最急需解决的问题分析

### 2.1 企微会话存档SDK(C++) 与 Node.js 集成 — 分析

**当前状态：**
- ✅ `WecomChatArchiveService.ts`（620行）已实现 **HTTP API模式**
- ✅ 能获取：开通成员列表、同意状态、生成会话元数据
- ❌ **无法拉取实际聊天消息内容**（需要Finance SDK C++库）
- ❌ **无法解密消息体**（RSA解密函数已写好，但没有密文数据来源）

**技术难点：**
企微会话存档的消息拉取 API（`GetChatData`）是 **C++ SDK 接口**，不是 HTTP API。必须：
1. 下载企微提供的 `libWeWorkFinanceSdk_C.so`（Linux）或 `.dll`（Windows）
2. 通过 Node.js 的 `ffi-napi` 或 `node-addon-api` 调用 C++ 动态链接库
3. 处理内存管理、回调、加密解密

**评估：**
- **紧急程度**：🟡 中等 — 当前HTTP API模式已能展示会话关系和元数据，用户体验可接受
- **替代方案**：企微现已提供 **HTTP方式的内容存档**（高级版），可考虑申请
- **如果必须对接SDK**：需要 2-3 天开发 Node.js C++ Addon

### 2.2 侧边栏认证分析 — 深度核实

**实际安全架构（已实现）：**

```
前端路由: /wecom-sidebar → requiresAuth: false ← 因为嵌入企微，无法用CRM登录态
    ↓
WecomSidebar/CustomerDetail.vue
    ↓ 加载时
    ├─ 1. 加载企微JS-SDK (jWeixin-1.2.0.js)
    ├─ 2. 请求后端 /wecom/sidebar/js-sdk-config 获取签名
    ├─ 3. wx.config() + wx.agentConfig() 验证企微环境
    ├─ 4. wx.invoke('getCurExternalContact') 获取当前聊天对象
    ├─ 5. 检查 localStorage 缓存的 sidebar_token
    ├─ 6. 或调用 /wecom/sidebar/verify-binding 验证绑定
    ├─ 7. 未绑定 → 显示登录表单 → /wecom/sidebar/bind-account
    └─ 8. 已绑定 → 获取JWT(type=sidebar) → 调用 /sidebar/customer-detail
              ↓
    后端 authenticateSidebarToken 中间件验证JWT
```

**结论：`requiresAuth: false` 是正确设计，不是安全漏洞！**

| 层级 | 保护措施 | 状态 |
|------|---------|------|
| 前端路由 | `requiresAuth: false` | ✅ 正确（侧边栏无法使用CRM登录态） |
| 企微JS-SDK | `wx.config` + `wx.agentConfig` 双重验证 | ✅ 已实现 |
| 绑定验证 | `sidebarVerifyBinding` → 后端检查绑定状态 | ✅ 已实现 |
| 数据接口 | `authenticateSidebarToken` 中间件验证JWT | ✅ 已实现 |
| JWT载荷 | `type: 'sidebar'` 标记，含 tenantId/crmUserId | ✅ 已实现 |
| 输入校验 | 用户名≤100、密码≤128，防bcrypt慢哈希攻击 | ✅ 已实现 |

**可优化点（非紧急）：**
1. ⚠️ `bind-account` 和 `verify-binding` 接口无速率限制（可加IP限流）
2. ⚠️ `js-sdk-config` 接口无鉴权（可加Referer白名单）
3. ⚠️ JWT无过期刷新机制（token过期后需重新登录）
4. ⚠️ 缓存的token存在localStorage，XSS风险

## 四、会话存档SDK集成方案（详细）

### 4.1 方案A：Node.js FFI调用C++ SDK（推荐）

```
架构：
Node.js Backend
    ↓ ffi-napi / node-addon-api
C++ WeWorkFinanceSdk
    ↓ HTTP
企微会话存档服务器
```

**步骤：**

1. **安装依赖**
```bash
cd backend
npm install ffi-napi ref-napi ref-struct-napi
```

2. **下载SDK**
- 从企微开发者文档下载 `libWeWorkFinanceSdk_C.so`（Linux）
- Windows版需 `.dll`

3. **创建Wrapper** `backend/src/services/WecomFinanceSDK.ts`
```typescript
// 封装C++函数调用：
// NewSdk() → 初始化
// GetChatData() → 拉取密文消息
// DecryptData() → RSA解密
// DestroySdk() → 释放
```

4. **集成到ChatArchiveService**
```typescript
// 在 syncChatRecords 中添加 finance_sdk 模式分支
if (config.chatArchivePrivateKey && sdkAvailable) {
  mode = 'finance_sdk'
  // 调用SDK拉取实际消息 → RSA解密 → 入库
}
```

### 4.2 方案B：纯HTTP替代方案

如果暂时无法部署C++库，可采用"HTTP API增强+模拟体验"：
- 当前元数据 + 外部联系人信息 = 展示会话关系图
- 提示用户"消息内容需配置企微高级存档服务"
- 已实现的`ConversationView.vue`在有真实消息数据后可直接使用

## 五、客服实时消息方案

### 5.1 当前架构
```
客户发消息 → 企微客服 → callback.ts 接收事件 → 仅入库
前端 Service.vue → 手动点"同步"按钮 → 轮询API
```

### 5.2 目标架构
```
客户发消息 → 企微客服 → callback.ts 接收事件 → 入库 + WebSocket广播
前端 Service.vue → WebSocket监听 → 实时更新会话列表和消息
```

### 5.3 实现步骤
1. callback.ts 中客服事件处理器添加 WebSocket 推送
2. Service.vue 建立 WS 连接，监听 `kf_session_update` 事件
3. 收到事件后自动刷新会话列表或追加消息
