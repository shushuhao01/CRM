## 一、环境配置文件说明

项目 `backend/` 目录下有7个环境配置文件，各自用途如下：

| 文件名 | 用途说明 |
|--------|---------|
| `.env` | **生产环境主配置**（部署到服务器时使用）。包含数据库连接、JWT密钥、CORS、邮件、短信等完整配置。占位符需改为实际值。 |
| `.env.database` | **数据库配置模板**。详细注释的数据库配置参考文件，适合首次部署时复制为 `.env` 使用。 |
| `.env.development` | **开发环境配置**。使用 SQLite 作为本地开发数据库（`./data/crm_dev.db`），无需 MySQL。日志级别为 debug。 |
| `.env.example` | **配置模板示例**。完整的环境变量清单和说明文档，作为所有配置项的参考模板。新开发者应复制此文件。 |
| `.env.local` | **本地开发环境配置**（当前使用）。连接本地 MySQL 数据库（`crm_local`），SaaS 模式已启用，含 SAAS_LICENSE_TOKEN。系统优先加载此文件。 |
| `.env.production` | **生产环境简化配置**。精简版生产配置，仅含核心必要项。适合简单部署场景。 |
| `.env.production.server` | **宝塔面板生产配置**。针对宝塔面板服务器的专用配置，含实际数据库连接信息和域名CORS配置。部署时复制为服务器上的 `.env`。 |

### 配置加载优先级
```
.env.local > .env.development > .env.production > .env
```
系统根据 `NODE_ENV` 自动选择对应环境文件，`.env.local` 始终最高优先级。

## 二、schema.sql 文件说明

**文件路径**：`database/schema.sql`  
**文件作用**：CRM系统的**数据库完整注册表/初始化脚本**，包含：
- 所有表结构定义（CREATE TABLE）
- 默认初始数据（INSERT INTO）
- 索引和约束定义
- 数据修复脚本

**使用方式**：
1. 在宝塔面板 phpMyAdmin 中直接导入执行
2. 在本地 MySQL 客户端执行 `source schema.sql`
3. 在 Navicat / DBeaver 等工具中导入执行

## 六、注意事项

1. **执行兼容性**：schema.sql 已确保兼容 phpMyAdmin、MySQL CLI、Navicat 等工具直接执行
2. **幂等安全**：平台管理类表使用 `CREATE TABLE IF NOT EXISTS`，可重复执行不报错
3. **CRM核心表**：使用 `DROP TABLE IF EXISTS` + `CREATE TABLE`，重新执行会清空数据，生产环境请谨慎
4. **外键约束**：文件开头 `SET FOREIGN_KEY_CHECKS = 0` 禁用外键检查，结尾恢复
5. **字符集**：所有表统一使用 `utf8mb4` + `utf8mb4_unicode_ci`，完整支持中文和emoji
