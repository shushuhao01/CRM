## 第1步：上架前代码修补（⚠️ 必做）

### 1.1 添加首次启动隐私弹窗（🔴 最重要！不做100%被拒）

**为什么？**  
国内所有安卓应用商店（包括小米）都强制要求：  
- APP 首次启动时必须弹出隐私政策弹窗
- 用户**主动点击"同意"**后才能继续使用
- 必须有"不同意"按钮，点击后退出APP
- 不能在同意前收集任何用户信息

**我已经帮你添加好了这个功能（见下面的代码修改），这里说明原理：**

在 `splash` 启动页中：
1. 检查本地是否已同意过 → `uni.getStorageSync('privacy_agreed')`
2. 如果没同意过 → 弹出隐私弹窗
3. 用户点"同意" → 标记已同意 → 继续正常启动流程
4. 用户点"不同意" → 直接退出APP

### 1.2 生产包去除 console.log

在 `vite.config.ts` 中配置生产打包时移除 console：

```typescript
// vite.config.ts 需要添加 esbuild 配置
esbuild: {
  drop: process.env.NODE_ENV === 'production' ? ['console', 'debugger'] : []
}
```

## 第3步：生成 Android 正式签名证书

### 3.1 前置条件
确保电脑已安装 JDK（Java Development Kit）。  
检查方法：打开终端输入 `java -version`，如果报错需要先装 JDK。

### 3.2 生成 Keystore

打开 **PowerShell**，执行以下命令：

```powershell
# 进入项目目录
cd "D:\kaifa\CRM - 1.8.0\crmAPP"

# 生成签名证书（有效期100年）
keytool -genkey -alias crm_dialer -keyalg RSA -keysize 2048 -validity 36500 -keystore crm-dialer-release.keystore
```

按提示输入以下信息：

```
密钥库口令：          [输入一个复杂密码，例如: XianHu@CRM2026!]
重新输入口令：        [重复上面的密码]
您的名字与姓氏：      广州仙狐网络科技有限公司
您的组织单位名称：    技术部
您的组织名称：        广州仙狐网络科技有限公司
您所在城市或地区：    广州
您所在的省/自治区：   广东
该单位的双字母国家代码：CN
以上正确吗？          yes
```

### 3.3 记录签名信息（⚠️ 极其重要！必须永久保存！）

```powershell
# 查看证书指纹信息
keytool -list -v -keystore crm-dialer-release.keystore -alias crm_dialer
```

**请用笔记本记录以下信息并安全保存：**

```

🔑 别名(Alias)：crm_dialer
🔒 密钥库密码：[你设置的密码]
🔒 别名密码：  [你设置的密码]（通常和密钥库密码一样）
📋 SHA1指纹：  [从上面命令的输出中复制]
📋 SHA256指纹：[从上面命令的输出中复制]
```

> 🚨 **重要警告：** 
> - Keystore 文件丢失 = 永远无法更新APP（只能重新上架新应用）
> - 建议备份到U盘、云盘、公司保险柜等多个地方
> - 密码忘记也无法恢复

## 第4步：使用 HBuilderX 云端打包生成 APK

### 4.1 安装 HBuilderX

1. 下载地址：https://www.dcloud.io/hbuilderx.html
2. 选择 **App开发版**（不是标准版！）
3. 下载后解压即可使用（绿色免安装）

### 4.2 导入项目

1. 打开 HBuilderX
2. 菜单 → **文件** → **导入** → **从本地目录导入**
3. 选择 `D:\kaifa\CRM - 1.8.0\crmAPP` 目录
4. 登录 DCloud 账号（左下角登录按钮，或菜单→帐户→登录）

### 4.3 关联 AppID

1. 双击打开 `src/manifest.json`
2. HBuilderX 会显示可视化编辑器
3. 在 **基础配置** → **uni-app 应用标识(AppID)** 处
4. 点击 **「重新获取」** 或直接填入第2步获取的正式 AppID
5. 确认 **应用名称** 为 `云客CRM外呼助手`
6. 确认 **应用版本名称** 为 `1.0.0`
7. 确认 **应用版本号** 为 `100`

### 4.4 配置 App 图标

1. 在 manifest.json 可视化编辑器中选择 **「App图标配置」** 标签
2. 点击 **「浏览」** 选择一张 1024x1024 的图标（你已有 `static/icons/logo-1024.png`）
3. 点击 **「自动生成所有图标并替换」**

### 4.5 执行云端打包 🎯

1. 菜单 → **发行** → **原生App-云打包**
2. 在弹出的对话框中配置：

| 配置项 | 选择/填写 |
|---|---|
| **Android 包名** | `com.xianhu.crm.dialer` |
| **使用自有证书** | ✅ 选中 |
| **证书别名** | `crm_dialer` |
| **证书私钥密码** | [你设置的密码] |
| **证书文件** | 点击浏览，选择 `crm-dialer-release.keystore` |
| **打包类型** | ✅ 正式包（release） |
| **Android 版本** | 保持默认（传统打包） |
| **渠道包** | 留空 |

3. 点击 **「打包」**
4. 等待打包完成（通常 5-20 分钟，排队时可能更久）
5. 打包完成后，HBuilderX 控制台会显示下载链接
6. **下载 APK 文件**，保存到：`D:\kaifa\CRM - 1.8.0\crmAPP\release\`

> 💡 **Tips：**  
> - 如果显示排队，耐心等待，不要重复提交
> - 如果打包失败，查看控制台错误信息，常见原因：AppID无效、证书密码错误
> - 打包成功后 APK 文件名类似：`__UNI__XXXXXX_0407XXXX.apk`

### 4.6 重命名 APK

将下载的 APK 重命名为规范名称：

```
云客CRM外呼助手_v1.0.0_release.apk
```

| **无法正常体验功能** | 提供可用的测试账号和服务器地址 |
| **隐私政策链接无法访问** | 确保隐私政策URL是公开可访问的 |
| **应用描述与实际不符** | 修改描述使之与APP实际功能一致 |
| **应用崩溃/ANR** | 修复Bug后重新提交 |
| **图标/截图不规范** | 更换符合规范的图片 |
| **缺少软著证书** | 小米一般不强制要求软著，但如被要求需补充 |

### 8.3 驳回后重新提交

2. 针对性修改
3. 如果需要改代码：修改 → 重新打包（重复第4步）→ 重新上传APK
4. 重新提交审核

## 📎 附录C：版本更新流程（以后每次更新APP时参考）

```
1. 修改版本号
   - crmAPP/src/config/app.ts     → APP_VERSION = '1.1.0', APP_VERSION_CODE = 110
   - crmAPP/src/manifest.json     → versionName: '1.1.0', versionCode: '110'
   - crmAPP/package.json          → version: '1.1.0'

2. 开发修改代码

3. HBuilderX 云打包（使用同一个 Keystore！）

4. 真机测试

5. 小米后台 → 应用管理 → 版本更新 → 上传新APK → 填写更新日志 → 提交审核
```

> ⚠️ **版本号规则：**
> - `versionName`：展示给用户看的版本，如 `1.0.0` → `1.1.0` → `1.2.0`
> - `versionCode`：每次必须递增的整数，如 `100` → `110` → `120`
> - 商店不允许 versionCode 不变或减小
