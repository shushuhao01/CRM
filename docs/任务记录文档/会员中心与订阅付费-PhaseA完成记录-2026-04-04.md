## 一、完成内容清单

### 1.1 数据库自动迁移（后端启动时自动执行）

| 变更项 | 类型 | 说明 |
|--------|------|------|
| `tenant_packages.subscription_enabled` | 新增字段 | TINYINT(1) DEFAULT 0，是否支持订阅 |
| `tenant_packages.subscription_channels` | 新增字段 | VARCHAR(50) DEFAULT 'all'，订阅渠道 |
| `tenant_packages.subscription_discount_rate` | 新增字段 | DECIMAL(5,2) DEFAULT 0.00，订阅折扣率 |
| `tenants.password_hash` | 新增字段 | VARCHAR(255)，会员中心登录密码 |
| `tenants.last_login_at` | 新增字段 | DATETIME，最近登录时间 |
| `subscriptions` | 新建表 | 订阅记录主表（14个业务字段+索引） |
| `subscription_deduct_logs` | 新建表 | 扣款明细表（17个业务字段+索引） |
| `member_login_logs` | 新建表 | 会员登录日志表 |

迁移方式：采用与现有 `ensureYearlyColumns()` 相同的模式，在首次请求时自动检测并迁移，无需手动执行SQL。

### 1.2 后端API改造

| 文件 | 改动 |
|------|------|
| `backend/src/routes/admin/packages.ts` | 新增 `ensureSubscriptionColumns()` + `ensureSubscriptionTables()` 函数；GET/POST/PUT 路由增加3个订阅字段的读写 |
| `backend/src/routes/public/packages.ts` | GET 响应增加 `subscription_enabled`、`subscription_channels`、`subscription_discount_rate`、`subscription_price`（计算值）字段 |

### 1.3 管理后台前端改造

| 文件 | 改动 |
|------|------|

### 1.4 官网类型同步

| 文件 | 改动 |

## 二、管理后台新增UI说明

### 套餐卡片新增标签
- 当套餐开启订阅时，卡片上显示绿色 `🔄 支持订阅(X折)` 标签
- 下方显示支持的渠道文字（微信代扣/支付宝扣款/微信+支付宝）

### 编辑弹窗新增区块（仅SaaS非试用套餐且价格>0时显示）
1. **支持订阅** — Switch开关
2. **订阅渠道** — RadioButton（全部支持/仅微信/仅支付宝）
3. **订阅优惠** — InputNumber（百分比，如10=9折）
4. **价格预览** — 显示正常月付价、订阅价格、月省/年省金额、官网展示文案

## 三、兼容性验证

- 所有数据库变更为 ADD COLUMN / CREATE TABLE，不影响现有数据

- 套餐不开启订阅（subscription_enabled=0）时，官网和管理后台行为完全不变
