### ✅ 阶段 2：概览支付不再创建重复账单
**问题**: 每次点击支付都调用 `/payment/create` 创建新订单，导致重复账单。应该为已有订单重新生成二维码。  
**修改文件**:
- `backend/src/routes/public/payment.ts` — 新增 `POST /public/payment/repay/:orderNo` 接口
- `backend/src/services/PaymentService.ts` — 新增 `createWechatOrderForExisting` 和 `createAlipayOrderForExisting` 公开方法
- `website/src/views/member/Dashboard.vue` — `handleGenQuickPay` 改用 `/repay/:orderNo`
- `website/src/views/member/Bills.vue` — `handleCreatePayOrder` 改用 `/repay/:orderNo`

**核心逻辑**: `/repay/:orderNo` 接口检查已有订单状态为pending，若支付方式未变且已有二维码则直接返回，否则重新生成二维码并更新订单记录，不创建新订单。

### ✅ 阶段 7：概览卡片真实数据映射
**问题**: 用户数显示0（应至少为1，管理员），存储空间可能不准确。  
**修改文件**: `backend/src/services/MemberService.ts`  
**修改内容**:
- `getProfile` 方法：新增从 `users` 表实时查询真实用户数（`SELECT COUNT(*) FROM users WHERE tenant_id = ?`）
- 用户数使用 `Math.max(1, dbCount)` 保证至少1个（管理员/创建者）
- 新增从 `uploads` 表查询真实存储使用量（`SELECT SUM(file_size) FROM uploads WHERE tenant_id = ?`）
- 套餐最大用户数和存储优先使用套餐表字段（`pkg_max_users`/`pkg_max_storage`），回退使用租户表字段

## ⚠️ 注意事项
- 项目中无真实微信/支付宝图标资源（png/svg），所有emoji图标均已移除，仅保留文字
- `repay` 接口：同一支付方式且已有二维码时直接返回缓存，切换支付方式时重新生成
- 用户数至少为1的逻辑：即使users表查询为0（边界情况），也返回1，因为开通账户即有管理员
- 订阅降级限制基于 `profile.package.price` 比较，确保用户不会意外降级

### ✅ 追加 2：升级/订购后管理后台和CRM信息同步验证与修复
**问题**: 支付成功后 `license_status` 被设为 `'paid'`，但CRM的 verify 端点只处理 `'pending'` → `'active'`，导致 `'paid'` 状态永远不会变成 `'active'`。  
**修改文件**: `backend/src/routes/tenantLicense.ts`  
**修改内容**:
- `POST /verify` 端点：`license_status === 'paid'` 现在和 `'pending'` 一样被激活为 `'active'`
- `GET /check-private` 端点：查询条件从 `license_status = 'active'` 改为 `IN ('active', 'paid')`
- `POST /verify-code` 端点：`paid` 状态允许通过编码登录并自动激活为 `'active'`

**同步确认清单**:
| 同步项目 | 状态 | 说明 |
|---------|------|------|
| 套餐ID (`package_id`) | ✅ 已同步 | `activateTenant` 中 UPDATE tenants SET package_id |

| 存储空间 (`max_storage_gb`) | ✅ 已同步 | 从套餐表读取并写入租户表 |
| 功能特性 (`features`) | ✅ 已同步 | 套餐features同步到租户 |

| 授权码 (`license_key`) | ✅ 已同步 | 支付后生成新授权码 |
| 授权状态 (`license_status`) | 🔧 已修复 | paid→active 激活流程修复 |
| 管理员账号 | ✅ 已同步 | `createDefaultAdmin` 自动创建 |
| 授权日志 | ✅ 已记录 | activate/renew/upgrade |
| 管理后台通知 | ✅ 已发送 | payment_success 通知 |
| 短信/邮件通知 | ✅ 已发送 | sendActivationNotification |
