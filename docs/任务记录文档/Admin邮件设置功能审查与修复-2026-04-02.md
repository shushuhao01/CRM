## 二、分析发现的问题

### 问题1（严重）：CRM租户端测试邮件是空壳实现
- **文件**: `src/views/System/Settings/EmailSettings.vue` 第142-156行
- **现象**: `handleTestEmail()` 只执行了 `setTimeout(2000)` 然后显示虚假的成功提示
- **影响**: 管理员以为测试成功但实际上没有发送任何邮件，无法验证配置是否正确

### 问题2（严重）：CRM租户端邮件设置无初始数据加载
- **文件**: `src/views/System/Settings/EmailSettings.vue`
- **现象**: 组件没有 `onMounted` 从后端加载已保存的配置，每次打开都是硬编码默认值 `smtp.qq.com`
- **影响**: 保存后的配置无法回显，用户体验差

### 问题3（中等）：Admin基础邮件配置未与通知服务联动
- **文件**: `backend/src/services/AdminNotificationService.ts`
- **现象**: Admin基础配置的SMTP信息存在 `system_config` 表，但通知服务的邮件渠道从 `admin_notification_channels` 表读取SMTP，两套独立配置互不关联
- **影响**: 管理员在基础配置中配好了邮件，但通知服务的邮件渠道不自动使用

### 问题4（低）：Admin邮件测试按钮缺少loading状态
- **文件**: `admin/src/views/settings/Basic.vue`
- **现象**: 测试邮件按钮没有loading状态，可能被重复点击

### 问题5（记录）：注册成功后的邮件通知被注释掉了
- **文件**: `backend/src/controllers/public/RegisterController.ts` 第272-275行
- **现象**: 代码注释了 `sendByTemplate('account_activation', ...)` 的邮件发送
- **说明**: 注册成功后只发短信通知，邮件通知从未实际发出。此问题记录但本次不修改（需要单独评估是否开启）

## 五、数据隔离审查结论

### 邮件配置存储隔离

| 存储位置 | 表名 | 是否有 tenant_id | 说明 |
|---------|------|-----------------|------|
| Admin全局邮件配置 | `system_config` | ❌ 无 | 全局唯一配置，合理 |
| CRM租户邮件配置 | `system_configs` | ✅ 有 | 通过TypeORM Entity + getTenantRepo自动隔离 |
| Admin通知渠道 | `admin_notification_channels` | ❌ 无 | 管理员全局通知，合理 |
| CRM通知渠道 | `notification_channels` | ✅ 有 | 通过getTenantRepo自动隔离 |
| 通知模板 | `notification_templates` | ✅ 有 | 支持租户级模板 |

### 隔离确认

- ✅ 不同租户保存的邮件SMTP配置互不可见（各自的 `tenant_id` 下）
- ✅ 即使两个租户有相同手机号的客户，邮件配置也在各自租户空间下
- ✅ 即使两个租户有相同用户名的管理员，各自的邮件设置独立
- ✅ Admin全局配置不会泄露给任何租户前端

## 六、邮件通知体系架构说明

### 三层邮件体系

```
┌─────────────────────────────────────────────────────┐
│  层级1: Admin基础邮件配置 (Basic.vue → 邮件配置Tab)    │
│  存储: system_config.email_settings                   │
│  用途: 全局SMTP配置，测试邮件发送                       │
│  场景: 管理员测试邮件是否配置正确                        │
├─────────────────────────────────────────────────────┤
│  层级2: Admin通知服务 (NotificationService.vue)        │
│  存储: admin_notification_channels                    │
│  用途: 管理员通知渠道（邮件/钉钉/企微/微信公众号）        │
│  场景: 新租户注册、支付成功、到期告警 → 通知管理员         │
│  改进: 现在支持从层级1 fallback SMTP配置                │
├─────────────────────────────────────────────────────┤
│  层级3: 通知模板 (NotificationTemplates.vue)           │
│  存储: notification_templates + notification_channels  │
│  用途: 配置带变量的邮件/短信模板，发送给租户              │
│  场景: 支付成功、到期提醒、过期通知 → 通知租户            │
│  流程: sendByTemplate → 渲染变量 → 渠道服务发送          │
└─────────────────────────────────────────────────────┘
```

### 已接入模板发送的业务场景

| 场景 | 模板代码 | 触发方 | 发送对象 | 状态 |
|------|---------|-------|---------|------|
| 支付成功 | `payment_success` | 支付宝/微信回调 | 租户 | ✅ 已接入 |
| 授权即将到期 | `license_expire_soon` | 定时任务 | 租户 | ✅ 已接入 |
| 授权已过期 | `license_expired` | 定时任务 | 租户 | ✅ 已接入 |
| 待支付提醒 | `payment_pending` | 定时任务 | 租户 | ✅ 已接入 |
| 新版本发布 | `version_published` | 版本管理 | 租户 | ✅ 已接入 |
| 注册成功邮件 | `account_activation` | 注册流程 | 租户 | ❌ 注释中 |

### 模板变量示例

```
邮件主题: 支付成功 - 订单 {{orderNo}}
邮件内容:
  尊敬的 {{contactPerson}}，您好！
  您的订单已支付成功，详情如下：
    订单编号：{{orderNo}}
    套餐名称：{{packageName}}
    支付金额：{{amount}}
    支付方式：{{paymentMethod}}
```
