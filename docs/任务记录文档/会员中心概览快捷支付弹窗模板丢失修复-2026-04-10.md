# 会员中心概览页快捷支付弹窗模板丢失修复

**日期**: 2026-04-10  
**类型**: Bug修复  
**涉及模块**: Website官网 — 会员中心概览页  
**严重级别**: 🔴 高（核心支付功能完全不可用）

---

## 📋 问题描述

**现象**：在会员中心概览页 (`/member/dashboard`) 的「最近账单」区域，点击待支付账单的「支付」按钮后，页面全屏出现模糊遮罩层，但支付二维码弹窗内容（金额、支付方式选择、二维码）没有显示。

**影响范围**：所有会员用户在概览页无法使用快捷支付功能，需跳转到账单列表页才能支付。

---

## 🔍 根因分析

**文件**: `website/src/views/member/Dashboard.vue`

弹窗模板（第134-137行）的 HTML 内容丢失，仅保留了外层遮罩容器和一个注释占位符：

```html
<!-- 修复前（损坏状态） -->
<div v-if="showQuickPay" class="pay-overlay" @click.self="closeQuickPay">
  <!-- ...existing code... -->
</div>
```

此时：
- `showQuickPay = true` → 遮罩层（`.pay-overlay`）正常渲染 → 页面出现全屏模糊
- 但弹窗主体 `.quick-pay-dialog` 完全不存在 → 看不到任何弹窗内容
- JS逻辑（`handleQuickPay`、`handleGenQuickPay`、`switchQuickChannel` 等）完好无损
- CSS样式（`.quick-pay-dialog`、`.quick-qr-area` 等）完好无损
- 仅 **模板HTML内容** 被意外清空

**推测原因**：之前某次编辑时，`<!-- ...existing code... -->` 注释作为占位符被写入但未被实际内容替换。

---

## ✅ 修复内容

恢复 `.quick-pay-dialog` 弹窗完整模板，包含以下子区域：

| 区域 | 内容 | 绑定变量/方法 |
|------|------|--------------|
| 弹窗头部 | 标题「💳 快捷支付」+ 关闭按钮 | `closeQuickPay()` |
| 金额信息 | 账单金额 + 套餐名称 | `quickPayBill.amount/packageName` |
| 支付渠道选择 | 微信支付 / 支付宝 双选 | `quickPayChannel`、`switchQuickChannel()` |
| 加载状态 | 「正在生成收款码...」带旋转动画 | `quickPayGenerating` |
| 二维码展示 | 扫码提示 + 二维码图片 + 「我已支付」按钮 | `quickPayQr`、`generateQRCodeUrl()`、`handleCheckQuickPay()` |
| 兜底状态 | 「正在加载支付信息...」 | 无QR且非生成中时显示 |

```html
<!-- 修复后（完整弹窗） -->
<div v-if="showQuickPay" class="pay-overlay" @click.self="closeQuickPay">
  <div class="quick-pay-dialog">
    <div class="quick-pay-header">
      <h3>💳 快捷支付</h3>
      <button class="pay-close" @click="closeQuickPay">&times;</button>
    </div>
    <div class="quick-pay-body">
      <!-- 金额信息 -->
      <div class="quick-pay-info">
        <div class="quick-pay-amount">¥{{ quickPayBill?.amount }}</div>
        <div class="quick-pay-name">{{ quickPayBill?.packageName }}</div>
      </div>
      <!-- 渠道选择 -->
      <div class="quick-channel-opts">
        <label class="qc-opt" :class="{ selected: quickPayChannel === 'wechat' }" @click="switchQuickChannel('wechat')">
          <input type="radio" value="wechat" v-model="quickPayChannel" /> 💚 微信支付
        </label>
        <label class="qc-opt" :class="{ selected: quickPayChannel === 'alipay' }" @click="switchQuickChannel('alipay')">
          <input type="radio" value="alipay" v-model="quickPayChannel" /> 🔵 支付宝
        </label>
      </div>
      <!-- 二维码区域 -->
      <div v-if="quickPayGenerating" class="quick-loading-area">...</div>
      <div v-else-if="quickPayQr" class="quick-qr-area">...</div>
      <div v-else class="quick-loading-area">正在加载支付信息...</div>
    </div>
  </div>
</div>
```
