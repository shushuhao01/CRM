## 二、修复清单

### 🔧 管理后台退款增强

#### 2.1 新增退款预览API

**文件**: `backend/src/routes/admin/sms-quota.ts`

新增 `GET /sms-quota/refund-preview/:orderId` 接口，返回退款预览信息：

```json
{
  "success": true,
  "data": {
    "orderNo": "SQ20260410123456",
    "packageName": "标准包",
    "tenantName": "测试企业",
    "originalAmount": 25.00,
    "originalSmsCount": 500,
    "totalQuota": 1500,
    "usedQuota": 300,
    "remainingQuota": 1200,
    "refundableSmsCount": 500,
    "refundAmount": 25.00,
    "payType": "wechat",
    "canRefund": true
  }
}
```

#### 2.2 退款弹窗UI改造

**文件**: `admin/src/views/sms-management/QuotaPackages.vue`

- **移除**: 原来的 `el-popconfirm` 简单确认
- **新增**: 独立退款预览弹窗，展示：
  - 订单基本信息（订单号、租户、套餐、原始金额/条数）
  - 租户额度信息（总额度、已使用、当前剩余）
  - 可退条数和退款金额（醒目大字展示）
  - 退款原因输入框（选填）
  - 额度已全部使用时显示"无法退款"提示
  - 确认退款按钮显示退款金额

### 🔧 管理后台日期范围筛选

**文件**: `admin/src/views/sms-management/QuotaPackages.vue`

- 新增 `el-date-picker` 日期范围选择器（type="daterange"）
- loadOrders 函数增加 `startDate` / `endDate` 参数传递
- 后端已在阶段1支持 `startDate` / `endDate` 查询参数

### 🔧 CRM端自动轮询支付状态

**文件**: `src/views/ServiceManagement/SmsManagement.vue`

- **创建订单后自动启动轮询**: `handleBuyQuota` 成功后调用 `startQuotaPolling()`
- **轮询间隔**: 每3秒调用 `querySmsQuotaOrder` 查询支付状态
- **支付成功自动处理**: 检测到 `status === 'paid'` 后自动停止轮询、提示成功、刷新额度数据
- **关闭弹窗停止轮询**: `closeQuotaDialog` 中调用 `stopQuotaPolling()`
- **组件卸载清理**: `onUnmounted` 中调用 `stopQuotaPolling()` 防止内存泄漏
