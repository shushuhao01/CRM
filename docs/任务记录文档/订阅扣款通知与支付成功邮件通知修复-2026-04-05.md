## 一、任务概述

### 任务15：自动续费订阅扣款成功和失败都提醒运营管理员（短信和邮件）

**问题描述**：自动续费订阅扣款成功时，没有任何管理员通知；扣款失败虽有系统内通知，但缺少短信渠道推送。管理员无法及时掌握订阅扣款动态。

### 任务16：注册时填了邮箱，但支付成功没有邮件通知客户

**问题描述**：用户注册时填写了邮箱字段（存储在 `tenants.email`），但支付成功后 `sendActivationNotification()` 中邮件发送部分仅为 TODO 注释，未实际实现。导致客户只能通过短信收到开通通知，无法通过邮件收到完整的账号信息。

## 七、使用说明

### 启用短信通知渠道
1. 在管理后台 → 设置 → 基础配置 → 短信设置 中配置阿里云短信 AccessKey
2. 在阿里云短信控制台申请管理员通知模板（模板参数：`title`、`content`）
3. 在管理后台 → 设置 → 通知服务 → 通知渠道 中启用「短信通知」渠道
4. 填写接收手机号（支持多个），填写短信模板CODE
5. 在「通知规则」页签中，为需要的事件启用 sms 渠道

### 支付成功邮件通知
- 前提条件：用户注册时填写了邮箱 + 管理后台基础配置中邮件设置已启用
- 自动触发：支付成功 → 激活租户 → 自动发送邮件
- 邮件内容：订单信息 + 账号信息 + CRM 登录链接

## 八、注意事项

1. **短信模板需在阿里云控制台审核通过**后才能使用，开发环境下未配置短信时会模拟发送（打印到控制台）
2. **邮件发送依赖 SMTP 配置**，如果管理后台未配置邮件设置，会跳过邮件发送并记录警告日志
3. **所有通知发送失败不影响主业务流程**，均使用 try-catch 包裹
4. 短信渠道的 `phone_numbers` 字段存储在 `admin_notification_channels.config_data` JSON 中
5. 如需新增 sms 渠道默认记录，可执行：
   ```sql
   INSERT INTO admin_notification_channels (id, channel_type, name, is_enabled, config_data)
   VALUES (UUID(), 'sms', '短信通知', 0, '{"phone_numbers":[],"template_code":""}')
   ON DUPLICATE KEY UPDATE updated_at = CURRENT_TIMESTAMP;
   ```
