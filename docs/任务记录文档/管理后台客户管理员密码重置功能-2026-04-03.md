# 管理后台客户管理员密码重置功能 - 2026-04-03

## 任务概述

为管理后台（Admin）的**私有客户列表**和**租户客户列表**新增「**重置管理员密码**」功能。当客户管理员忘记密码时，平台管理员可在后台一键重置密码，生成随机临时密码并复制发送给客户。同时修复租户客户列表「解锁账号」功能缺失后端路由的Bug。

## 问题分析

### 原有问题

| # | 问题描述 | 影响范围 | 严重程度 |
|---|---------|---------|---------|
| 1 | **租户客户缺少 `unlock-admin` 后端路由** — 前端 `tenant-customers/List.vue` 调用 `POST /tenants/:id/unlock-admin`，但 `tenants.ts` 路由文件中没有该端点，导致解锁操作报 404 | 租户客户列表 | 🔴 高 |
| 2 | **私有客户缺少"重置密码"功能** — 列表页面没有重置管理员密码的操作项，管理员忘记密码只能手动操作数据库 | 私有客户管理 | 🔴 高 |
| 3 | **租户客户缺少"重置密码"功能** — 同上，租户管理员忘记密码无法通过后台重置 | 租户客户管理 | 🔴 高 |
| 4 | **缺少密码重置后端API** — 两种客户类型都没有专用的密码重置后端路由 | 后端API | 🔴 高 |
| 5 | **缺少密码重置成功后的信息复制/发送功能** — 重置密码后应展示新密码并支持一键复制 | 前端交互 | 🟡 中 |
| 6 | **前端 `admin.ts` 缺少重置密码API方法** — 没有对应的前端API封装 | 前端API层 | 🔴 高 |

### 修复方案

- **后端**：为私有客户和租户客户分别添加 `reset-admin-password` 路由，补充租户客户缺失的 `unlock-admin` 路由
- **前端**：在两个列表页面的"更多"下拉菜单中添加"重置密码"选项，添加密码重置成功对话框（含一键复制功能）
- **API层**：在 `admin.ts` 中添加对应的API方法

## API 接口清单

### 新增接口

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/api/v1/admin/private-customers/:id/reset-admin-password` | 重置私有客户管理员密码 |
| POST | `/api/v1/admin/tenants/:id/unlock-admin` | 解锁租户管理员账号（修复缺失） |
| POST | `/api/v1/admin/tenants/:id/reset-admin-password` | 重置租户管理员密码 |

### 接口返回格式

**重置密码接口统一返回格式：**

```json
{
  "success": true,
  "message": "密码重置成功",
  "data": {
    "username": "13800138000",
    "customerName": "XXX公司",
    "tempPassword": "AhKm5nP3",
    "resetCount": 1
  }
}
```

**解锁账号接口返回格式：**

```json
{

  "message": "已解锁管理员账号 13800138000",
  "data": {
    "unlockedCount": 1,
    "usernames": "13800138000",
    "fixedPasswordCount": 0
  }
}
```

## 密码重置逻辑说明

### 密码生成规则
- 8位随机字符，字符集为：大写字母（排除 I、O）+ 小写字母（排除 l、o）+ 数字（排除 0、1）
- 确保包含大写字母、小写字母、数字三种类型
- 使用 `crypto.randomBytes()` 生成密码学安全的随机数

### 密码更新策略
- 使用 `bcryptjs` 12轮哈希加密存储
- 重置密码时同时解锁账号（`status → active`, `login_fail_count → 0`, `locked_at → NULL`）
- 记录完整操作日志（`license_logs` / `tenant_license_logs` 表）

### 数据隔离
- **私有客户**：查找条件 `WHERE username = ? AND role = 'admin' AND tenant_id IS NULL`
- **租户客户**：查找条件 `WHERE tenant_id = ? AND role = 'admin'`

## 功能交互流程

```
管理员在列表页 → 点击"更多" → 选择"重置密码"
    ↓
弹出确认对话框（警告：原密码将立即失效）
    ↓ 确认
调用后端API → 生成随机临时密码 → bcrypt加密存储 → 同时解锁账号
    ↓
展示密码重置成功对话框
    ├── 显示：客户名称、管理员账号、新临时密码
    ├── 「一键复制账号密码」按钮 → 格式化文本到剪贴板
    └── 安全提示：临时密码，请登录后立即修改
    ↓
管理员将复制的信息通过微信/短信/邮件发送给客户管理员
```

### 复制文本格式示例

```
【云客CRM - 密码重置通知】

客户名称：XXX公司
管理员账号：13800138000
新密码（临时）：AhKm5nP3

⚠️ 请登录后立即修改密码！
如有问题请联系技术支持。
```

## 安全考量

1. **临时密码不存储明文**：生成后仅在API响应中返回一次，数据库只存储bcrypt哈希
2. **操作日志记录**：每次密码重置操作都记录到对应的日志表，包含操作人、IP、时间
3. **确认机制**：重置前有二次确认对话框，防止误操作
4. **同时解锁**：重置密码时自动解锁被锁定的账号，避免客户无法登录

## 未来优化建议

1. **短信通知集成**：目前依赖管理员手动复制发送，后续可对接阿里云短信直接发送临时密码
2. **自助重置链接**：可开发邮箱/手机验证码自助重置密码功能
3. **密码有效期**：可设置临时密码有效期（如24小时），过期后必须重新申请
4. **强制修改标记**：可在用户表添加 `force_password_change` 字段，首次登录后强制修改密码
