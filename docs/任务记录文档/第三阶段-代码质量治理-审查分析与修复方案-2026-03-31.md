## ✅ 任务3.8 console.log → Winston logger（2026-03-31 完成）

### 实现方案

**1. logger.ts 增强：添加 console 兼容包装器**

在 `backend/src/config/logger.ts` 中新增 `log` 导出对象：
```typescript
export const log = {
  info: (...args: unknown[]) => logger.info(formatArgs(args)),
  error: (...args: unknown[]) => logger.error(msg, { stack }),
  warn: (...args: unknown[]) => logger.warn(formatArgs(args)),
  debug: (...args: unknown[]) => logger.debug(formatArgs(args)),
}
```
- 接受与 `console.log` 相同的可变参数（自动拼接字符串，Error对象保留stack）
- 所有日志输出统一路由到 Winston（文件轮转+级别管理+时间戳）

**后端增强**：`/api/v1/data/search-customer` 端点
- 新增销售员信息批量查询（IN查询 + Map映射）
- 新增匹配类型标记（客户姓名/手机号/客户编码）
- 新增分页限制（最大50条/页）
- 响应格式对齐前端SearchNew.vue的字段需求

**前端改造**：`src/views/Data/SearchNew.vue`

| 指标 | 改造前 | 改造后 |
|------|--------|--------|
| 搜索方式 | 加载全部客户+订单到内存做过滤 | 调用后端API `/data/search-customer` |
| 内存占用 | 全量客户+用户+订单数据 | 仅搜索结果 |
| 依赖Store | customerStore, userStore, orderStore | 无（直接API调用） |
| 搜索范围 | 客户+订单号+物流单号（全量遍历） | 客户模糊搜索 + 精确搜索兜底 |

**移除的依赖**：
- `useCustomerStore` → 不再需要
- `useUserStore` → 不再需要
- `useOrderStore` → 不再需要
- `customerStore.loadCustomers()` → 不再调用

## ✅ 任务3.11 客户列表统计数据独立接口（2026-03-31 完成）

### 后端新增端点

**`GET /api/v1/customers/stats`**

```json
{
  "success": true,
  "data": {
    "totalCustomers": 1234,
    "monthCustomers": 56,
    "newCustomers": 8,
    "noOrderCustomers": 789
  }
}
```

**特性**：
- 30秒 CacheService 租户感知缓存（`t:{tenantId}:customers:list-stats`）
- 使用 LEFT JOIN + IS NULL 高性能方案统计未下单客户
- 与列表接口共享租户隔离逻辑

### 前端改造

**`src/api/customer.ts`**：
- 新增 `CustomerStatsResponse` 接口
- 新增 `customerApi.getStats()` 方法
- `CustomerListResponse` 添加可选 `statistics` 字段（向后兼容）

**`src/views/Customer/List.vue`**：
- 新增 `loadCustomerStats()` 独立函数
- `onMounted` 中的 `Promise.all` 添加并行统计加载
- 统计卡片先显示0，后异步填充（不阻塞列表渲染）
