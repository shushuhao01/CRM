# 售后管理模块权限显示修复

> **任务日期**: 2026-04-06
> **问题描述**: 销售员(sales_staff)和部门经理(department_manager)角色登录后，侧边栏不显示"售后管理"模块，但默认预设权限中包含该模块权限
> **影响范围**: 售后管理模块菜单可见性
> **修复类型**: 权限过滤代码修复

---

## 一、问题分析

### 1.1 权限加载链路

用户登录时，权限按以下优先级加载：

```
1. 后端API返回的 rolePermissions（数据库 roles 表 permissions 字段）
2. localStorage 中 crm_roles 动态配置
3. defaultRolePermissions.ts 默认权限配置（兜底）
4. 角色名称映射后再查 defaultRolePermissions.ts
```

### 1.2 根本原因

**发现了两个关键问题：**

#### 问题1：`authApiService.ts` 中 `getDefaultPermissions()` 方法过时

文件路径: `src/services/authApiService.ts`

该方法作为 mock/预设账号登录路径的权限源，其 `department_manager` 和 `sales_staff` 角色的权限列表**严重过时**，缺少以下模块权限：

| 缺失模块 | 权限前缀 | 说明 |
|---------|---------|------|
| 售后管理 | `aftersale` / `aftersale.*` | 完全缺失 |
| 通话管理 | `communication` / `communication.*` | 完全缺失 |
| 财务管理 | `finance` / `finance.*` | 完全缺失 |

而 `src/config/defaultRolePermissions.ts`（权威配置文件）中这些权限是完整的。

#### 问题2：后端数据库权限可能不完整

当真实API登录时，后端从数据库 `roles` 表查询权限：
```sql
SELECT permissions FROM roles WHERE code = ?
```

如果数据库中的角色权限是在售后管理模块添加之前保存的，`rolePermissions` 字段不包含 `aftersale` 相关权限。由于 `loginWithApi` 优先使用后端返回的权限（`userData.rolePermissions`），不会回退到 `defaultRolePermissions.ts`，导致菜单被权限过滤掉。

### 1.3 菜单过滤逻辑确认

菜单配置（`src/config/menu.ts`）本身是正确的：
- "售后管理"菜单 `roles` 包含 `department_manager` 和 `sales_staff` ✅
- `permissions: ['aftersale']` 与权限标识一致 ✅
- 模块ID `'service'` 在默认启用模块列表中 ✅

**结论：这是权限过滤代码问题 + 可能的数据库权限不完整问题，不是菜单配置问题。**

## 二、修复方案

### 2.1 修复文件清单

| 文件 | 修改内容 | 说明 |
|------|---------|------|
| `src/services/authApiService.ts` | 更新 `getDefaultPermissions()` 方法 | 补全 aftersale、communication、finance 权限 |
| `src/stores/user/authActions.ts` | 在 `loginWithApi` 添加 aftersale 权限补全 | 防止数据库权限不完整 |
| `src/stores/user/authActions.ts` | 在 `initUser` 添加 aftersale 权限补全 | 防止会话恢复时权限丢失 |

### 2.2 修复详情

#### 修改1：`src/services/authApiService.ts` — getDefaultPermissions

**修改前（过时配置，缺少售后管理等权限）：**
```typescript
'department_manager': [
  'dashboard', 'dashboard:personal', 'dashboard:department',
  'customer', ...,
  'order', ...,
  'performance', ...,
  'logistics', ...,
  'data', ...
  // ❌ 完全缺失 aftersale、communication、finance
],
'sales_staff': [
  // ❌ 同样缺失
]
```

**修改后（与 defaultRolePermissions.ts 保持同步）：**
```typescript
'department_manager': [
  'dashboard', 'dashboard.view', 'dashboard.export',
  'customer', ...,
  'order', ...,
  'communication', 'communication.call', 'communication.call.view', 'communication.call.make', 'communication.call.record',
  'performance', ...,
  'logistics', ...,
  // ✅ 售后管理权限
  'aftersale', 'aftersale.list', 'aftersale.list.view',
  'aftersale.add', 'aftersale.add.create',
  'aftersale.data', 'aftersale.data.view', 'aftersale.data.analysis',
  'data', ...,
  'finance', 'finance.performance_data', 'finance.performance_data.view',
  'finance.cod_application', 'finance.cod_application.view', 'finance.cod_application.create'
],
// sales_staff 同理
```

#### 修改2：`src/stores/user/authActions.ts` — loginWithApi 权限补全

在权限加载完成后、设置到 Store 之前，增加售后管理权限补全逻辑：

```typescript
// 🔥 补全内置角色可能缺失的售后管理模块权限
const builtInRoles = ['department_manager', 'sales_staff', 'customer_service', 'manager', 'sales']
if (builtInRoles.includes(resolvedRole) && userPermissions.length > 0 && !userPermissions.includes('*')) {
  const defaultPerms = getDefaultRolePermissions(resolvedRole)
  const aftersalePerms = defaultPerms.filter(p => p === 'aftersale' || p.startsWith('aftersale.'))
  for (const p of aftersalePerms) {
    if (!userPermissions.includes(p)) {
      userPermissions.push(p)
    }
  }
}
```

#### 修改3：`src/stores/user/authActions.ts` — initUser 会话恢复补全

在会话恢复（页面刷新）时也进行同样的权限补全，防止从 localStorage 恢复的旧权限缺少 aftersale。
