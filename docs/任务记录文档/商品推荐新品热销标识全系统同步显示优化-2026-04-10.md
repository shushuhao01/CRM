## 一、问题背景

CRM系统商品管理支持为商品设置三种标识：
- **推荐商品** (`isRecommended`)
- **新品** (`isNew`)
- **热销商品** (`isHot`)

这些标识可以在商品编辑页通过Switch开关设置，在商品列表和订单新增/编辑页面也有展示。但经审查发现以下问题：

### 1.1 发现的问题

| # | 问题 | 严重度 | 影响 |
|---|------|--------|------|
| 1 | 后端`Product`实体缺少标识数据库列 | 🔴 致命 | 数据库无法存储标识值，保存后丢失 |
| 2 | 后端创建/更新产品API不处理标识字段 | 🔴 致命 | 前端提交的标识数据被后端忽略 |
| 3 | 后端返回数据不包含标识字段 | 🔴 致命 | 列表/详情/创建/更新响应中无标识 |
| 4 | 前端编辑页保存时不传递标识字段 | 🔴 致命 | 即使后端支持，前端也不发送数据 |
| 5 | 前端编辑页加载时标识硬编码为false | 🔴 高 | 编辑商品时标识总是显示为关闭 |
| 6 | 前端`Product`接口缺少标识字段定义 | 🔴 高 | TypeScript类型安全缺失 |
| 7 | 商品详情页无标识显示 | 🔴 高 | 进入详情看不到任何标识信息 |
| 8 | 商品详情页`productInfo`初始化缺标识字段 | 🔴 高 | 数据绑定缺失 |
| 9 | 订单详情商品清单无标识 | 🟡 中 | 查看订单详情时看不到商品标识 |

### 1.2 已正常工作的页面

| 页面 | 文件 | 状态 |
|------|------|------|
| 商品编辑页 | `Product/Edit.vue` | ✅ Switch开关设置正常 |
| 商品列表页 | `Product/List.vue` | ✅ 名称列下方显示标识 |
| 新增订单-商品选择 | `Order/Add.vue` | ✅ 商品卡片和已选列表均有标识 |
| 编辑订单-商品选择 | `Order/Edit.vue` | ✅ 商品卡片和已选列表均有标识 |

### 2.0 后端Product实体添加数据库列（核心修复）

**文件**: `backend/src/entities/Product.ts`

在 `Product` 实体中添加三个 `tinyint` 类型的数据库列：

```typescript
@Column({ name: 'is_recommended', type: 'tinyint', default: 0, comment: '是否推荐' })
isRecommended: boolean

@Column({ name: 'is_new', type: 'tinyint', default: 0, comment: '是否新品' })
isNew: boolean

@Column({ name: 'is_hot', type: 'tinyint', default: 0, comment: '是否热销' })
isHot: boolean
```

```vue
    <el-tag v-if="isRecommended" type="warning" :size="size" effect="light">⭐ 推荐</el-tag>
    <el-tag v-if="isNew" type="success" :size="size" effect="light">🆕 新品</el-tag>
    <el-tag v-if="isHot" type="danger" :size="size" effect="light">🔥 热销</el-tag>
  </div>
</template>

<script setup lang="ts">
defineProps<{
  isRecommended?: boolean
  isNew?: boolean
  isHot?: boolean
  size?: 'small' | 'default' | 'large'

</script>
```

这样可以在所有需要的地方统一调用，减少重复代码。
