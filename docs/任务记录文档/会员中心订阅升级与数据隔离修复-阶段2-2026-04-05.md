## 📋 阶段2发现的问题清单

### 🔴 问题1：PaymentService.activateTenant 不更新 package_id（严重）

**严重度**：🔴 高（功能缺陷 — 升级后套餐不生效）

**问题描述**：
`PaymentService.activateTenant` 方法在支付成功后：
- ❌ 不更新 `package_id`（租户仍绑定旧套餐）
- ❌ 不更新 `max_storage_gb`（存储配额不变）
- ❌ 不同步 `features`（功能特性不变）
- ❌ 不考虑 `billing_cycle`（按月/按年/永久，duration_days 计算错误）
- ❌ 续费时从当天算起，而不是从当前到期日延长

**影响范围**：
- 会员中心续费升级页（Renew.vue）支付成功后 → 套餐不更新
- CRM系统通过 `package_id` JOIN 获取权限 → 用户看到的仍是旧套餐权限
- 管理后台客户列表/详情 → 显示旧套餐

### 🔴 问题2：SubscriptionService.executeDeduction 不同步套餐（严重）

**严重度**：🔴 高（功能缺陷 — 订阅扣款后套餐不升级）

**问题描述**：
`SubscriptionService.executeDeduction` 在订阅扣款成功后：
- ❌ 只更新 `expire_date` 和 `license_status`
- ❌ 不更新 `package_id`（免费试用用户开通订阅后仍是试用套餐）
- ❌ 不同步 `max_users`、`max_storage_gb`、`features`

**影响范围**：
- 会员中心订阅管理页（Subscription.vue）开通订阅后 → 套餐不升级
- 免费试用用户开通付费订阅 → 仍显示试用版

## 🔧 阶段2修复详情

### 文件1：PaymentService.ts

#### updateOrderStatus 改动
- 从订单中额外读取 `billing_cycle` 和 `bonus_months` 字段
- 将这两个参数传递给 `activateTenant`

```
// 改前
'SELECT tenant_id, package_id FROM payment_orders WHERE order_no = ?'
activateTenant(tenantId, packageId)

// 改后
'SELECT tenant_id, package_id, billing_cycle, bonus_months FROM payment_orders WHERE order_no = ?'
activateTenant(tenantId, packageId, billingCycle, bonusMonths)
```

#### activateTenant 完全重写

**新增参数**：`billingCycle?: string`, `bonusMonths?: number`

**新增功能**：

1. **读取租户当前状态**：查询 `expire_date`, `package_id`, `license_status`，用于判断是新开通、续费还是升级
2. **读取套餐完整信息**：查询 `id, name, code, type, duration_days, max_users, max_storage_gb, features, modules`
3. **根据计费周期计算服务时长**：
   - `monthly` → 使用套餐默认 `duration_days`
   - `yearly` / `annual` → 365 + bonusMonths * 30 天
   - `perpetual` → 36500天（100年）
4. **续费延长逻辑**：同套餐续费时，如果当前未过期，从到期日顺延而非从今天算
5. **完整更新租户字段**：
   - `package_id` → 新套餐ID（🔑 关键修复）
   - `max_users` → 新套餐用户数
   - `max_storage_gb` → 新套餐存储配额
   - `features` → 新套餐功能特性
   - `license_key` → 新生成授权码
   - `license_status` → 'active'
   - `status` → 'active'
   - `expire_date` → 正确计算的到期日
6. **区分日志场景**：授权日志 action 区分 `activate`/`renew`/`upgrade`

### 文件2：SubscriptionService.ts

#### executeDeduction 改动

**修复前**：扣款成功后仅更新 `expire_date` 和 `license_status`

**修复后**：
1. 从 `tenant_packages` 查询订阅套餐的完整信息
2. 更新租户表的：`package_id`, `expire_date`, `license_status`, `status`, `max_users`, `max_storage_gb`, `features`
3. 套餐不存在时降级处理：仅延长到期日

## 🔗 全链路同步验证

### 链路1：续费升级页（Renew.vue）→ 一次性支付

```
用户选择套餐 → handlePay() 发送 packageId + billingCycle
→ POST /payment/create 创建订单（存储 billing_cycle, bonus_months）
→ 支付成功 → updateOrderStatus('paid') 
→ 读取订单 billing_cycle, bonus_months
→ activateTenant() 更新 package_id + 完整字段
→ CRM 下次 license check → JOIN tenant_packages → 获取新套餐权限 ✅
→ 管理后台 → JOIN tenant_packages → 显示新套餐 ✅
→ 会员中心 → getProfile() JOIN tenant_packages → 显示新套餐 ✅
```

### 链路2：订阅管理页（Subscription.vue）→ 自动续费

```
用户选择套餐 → createSubscription() 发送 packageId + channel
→ 创建订阅记录（存储 package_id）→ 返回签约链接
→ 签约成功 → handleSignSuccess() → executeDeduction()
→ 首期扣款成功 → 更新 package_id + 完整字段
→ 后续每期扣款 → executeDeduction() → 延长到期日 + 同步套餐 ✅
→ CRM / 管理后台 / 会员中心 都显示新套餐 ✅
```

### 数据同步关系

所有下游系统都通过 `LEFT JOIN tenant_packages p ON t.package_id = p.id` 获取套餐信息：

| 系统 | SQL位置 | 同步方式 |
|------|---------|---------|
| CRM系统 | `tenantLicense.ts` `WHERE t.license_key = ?` | license check 时 JOIN |
| CRM模块权限 | `UserController.ts` `WHERE t.id = ?` | 用户登录时 JOIN |
| 管理后台列表 | `admin/tenants.ts` 列表/详情查询 | 实时 JOIN |
| 管理后台客户管理 | `admin/customer-management.ts` | 实时 JOIN |
| 会员中心 | `MemberService.getProfile()` | 刷新页面时 JOIN |
