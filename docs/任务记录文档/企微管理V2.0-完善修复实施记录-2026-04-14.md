## 详细修改说明

### 🟡 P1-1: 侧边栏速率限制

**文件**: `backend/src/routes/wecom/sidebar.ts`

- 引入 `express-rate-limit`（已安装依赖）
- `sidebarAuthLimiter`: 每IP每15分钟最多10次请求
  - 应用于 `POST /sidebar/bind-account`
  - 应用于 `GET /sidebar/verify-binding`
- `jsSdkConfigLimiter`: 每IP每分钟最多20次请求
  - 应用于 `POST /sidebar/js-sdk-config`

### 🟡 P1-2: JS-SDK Config Referer白名单

**文件**: `backend/src/routes/wecom/sidebar.ts`

- 新增 `validateJsSdkReferer` 中间件
- 从环境变量 `SIDEBAR_ALLOWED_DOMAINS` 读取白名单（逗号分隔）
- 开发环境自动放行 `localhost` 和 `127.0.0.1`
- 未配置白名单时兼容放行（避免初始部署中断）
- 不匹配时返回 `403`

### 🟡 P1-3: 侧边栏JWT Token刷新

**后端**: `backend/src/routes/wecom/sidebar.ts`
- 新增 `POST /sidebar/refresh-token` 接口
- 受 `authenticateSidebarToken` 中间件保护
- 验证用户仍存在且激活后签发新token

**前端**: `src/api/wecom.ts`
- 新增 `refreshSidebarToken(token)` API函数

**前端**: `src/views/WecomSidebar/CustomerDetail.vue`
- 新增 `isTokenExpiringSoon(token)` 工具函数（解析JWT exp，判断剩余<1天）
- 在 `checkBindingAndLoad` 中，缓存token存在时检查过期，自动刷新

### 🟡 P1-4: sessionStorage替代localStorage

**文件**: `src/views/WecomSidebar/CustomerDetail.vue`

- 所有6处 `localStorage.getItem/setItem/removeItem('wecom_sidebar_token')` 改为 `sessionStorage`
- 降低XSS攻击窃取长期凭据的风险
- 关闭标签页后token自动清除

### 🔴 P0-1: 客服实时消息WebSocket推送

**后端**: `backend/src/routes/wecom/callback.ts`
- `kf_msg_or_event` 处理器增强：接收事件后通过 `webSocketService.broadcast()` 广播
- 广播事件名: `wecom:kf_session_update`
- 包含 configId, configName, openKfId 等信息
- 按租户ID隔离广播

**后端**: `backend/src/services/WecomApiService.ts`
- 新增 `syncKfMessages()` 方法，封装企微 `/kf/sync_msg` API
- 支持游标分页和回调token

**前端**: `src/services/webSocketService.ts`
- 注册 `wecom:kf_session_update` socket事件 → 转发到内部事件系统
- 注册 `wecom:acquisition_update` socket事件 → 转发到内部事件系统

**前端**: `src/views/Wecom/Service.vue`
- `onMounted` 中通过 `webSocketService.on()` 订阅 `wecom:kf_session_update`
- 收到事件时：自动刷新会话列表 + ElNotification右下角提示
- `onUnmounted` 中自动取消订阅，防止内存泄漏

### 🟢 P2-1: 标签管理CRUD

**后端**: `backend/src/services/WecomApiService.ts`
- `addCorpTag(accessToken, groupName, tags, groupId?)` — 创建标签组+标签
- `editCorpTag(accessToken, id, name, order?)` — 编辑标签/标签组名称
- `deleteCorpTag(accessToken, tagIds?, groupIds?)` — 删除标签/标签组

**后端**: `backend/src/routes/wecom/acquisition.ts`
- `GET /acquisition-tags?configId=` — 获取标签组列表
- `POST /acquisition-tags` — 创建标签组（含标签）
- `PUT /acquisition-tags/:id` — 编辑标签组/标签名称
- `DELETE /acquisition-tags` — 删除标签/标签组

**前端**: `src/api/wecom.ts`
- `getAcquisitionTags(configId)` 
- `createAcquisitionTagGroup({ configId, groupName, tags })`
- `editAcquisitionTag(id, { configId, name })`
- `deleteAcquisitionTags({ configId, tagIds?, groupIds? })`

**前端**: `src/views/Wecom/Acquisition.vue`
- 新增 `tagDialogVisible`, `editingTagGroup`, `tagForm` 状态
- 新增 `fetchTagGroups()` 从API拉取真实标签数据
- `handleAddTagGroup` → 打开创建弹窗
- `handleEditTagGroup` → 打开编辑弹窗（可改标签组名称）
- `handleSubmitTagGroup` → 提交创建/编辑
- `handleDeleteTagGroup` → 确认后删除
- Tab切换到"标签管理"时自动拉取数据
- 新增标签组创建/编辑弹窗（支持多标签逗号/顿号分隔输入）
