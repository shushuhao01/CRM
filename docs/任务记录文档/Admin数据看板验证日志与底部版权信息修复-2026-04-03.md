# Admin数据看板验证日志与底部版权信息修复 - 2026-04-03

## 任务概述

修复管理后台（admin）数据看板页面的两个问题：
1. **最近验证日志**：字段缺失客户名称，后端查询未关联 licenses 表
2. **底部版权信息**：硬编码为 `© 2024 CRM Platform Admin v1.0.0`，未映射系统配置的版权备案信息

## 问题分析

### 问题1：最近验证日志字段缺失

**现象**：Dashboard 底部"最近验证日志"表格仅显示授权码、操作、结果、IP地址、消息、时间，缺少客户名称等关键信息。

**根因**：后端 `GET /api/v1/admin/dashboard/recent-logs` 仅对 `license_logs` 表做简单 `find()` 查询，未 JOIN `licenses` 表获取客户名称（`customer_name`）等关联信息。

**涉及实体字段**：
- `license_logs` 表有：`license_id`, `license_key`, `action`, `ip_address`, `result`, `message`, `created_at`
- `licenses` 表有：`customer_name`（客户名称）, `created_by`（创建者/操作人）

### 问题2：底部版权信息硬编码

**现象**：`MainLayout.vue` 第163行 footer 写死为 `© 2024 CRM Platform Admin v1.0.0`，所有Admin页面底部都显示此固定文字，不会随系统配置变化。

**参考对比**：CRM前端（`src/App.vue`）已通过 `configStore.systemConfig` 动态显示版权文字、ICP备案号、公安备案号、技术支持等，但Admin后台未同步此机制。

**数据源**：Admin后台 `GET /api/v1/admin/system-config` 接口返回 `admin_system_config` 配置数据，包含 `copyrightText`、`icpNumber`、`policeNumber`、`techSupport`、`companyName`、`systemVersion` 等字段。

## 修复方案

### 阶段1：修复验证日志字段缺失

| 文件 | 修改类型 | 说明 |
|------|---------|------|
| `backend/src/routes/admin/dashboard.ts` | 修改 | `recent-logs` 接口从 TypeORM `find()` 改为原生 SQL JOIN 查询，LEFT JOIN `licenses` 表获取 `customer_name` 和 `created_by` |
| `admin/src/views/Dashboard.vue` | 修改 | 表格新增"客户名称"列（`customerName`），位于第一列，无数据时显示 `-` |

|------|---------|------|

## 详细修改说明

### 1. 后端 - dashboard.ts 验证日志查询优化

**修改前**（简单 TypeORM 查询）：
```typescript
const logRepo = AppDataSource.getRepository(LicenseLog);
const list = await logRepo.find({
  order: { createdAt: 'DESC' },
  take: 20
});
```

**修改后**（SQL JOIN 查询，补充客户名称）：
```sql
SELECT
  ll.id, ll.license_id AS licenseId, ll.license_key AS licenseKey,
  ll.action, ll.machine_id AS machineId, ll.ip_address AS ipAddress,
  ll.user_agent AS userAgent, ll.result, ll.message,
  ll.created_at AS createdAt,
  l.customer_name AS customerName,
  l.created_by AS createdBy
FROM license_logs ll
LEFT JOIN licenses l ON ll.license_id = l.id
ORDER BY ll.created_at DESC
LIMIT 20
```

### 2. 前端 - Dashboard.vue 验证日志表格

新增"客户名称"列，调整列宽使整体布局更紧凑：

| 列名 | 字段 | 宽度 | 说明 |
|------|------|------|------|
| **客户名称** *(新增)* | `customerName` | 130px | 来自 licenses 表的 customer_name |
| 授权码 | `licenseKey` | 180px *(原200)* | 截断显示+tooltip |
| 操作 | `action` | 80px *(原100)* | tag标签 |
| 结果 | `result` | 80px | 成功/失败 tag |
| IP地址 | `ipAddress` | 140px | 直接显示 |
| 消息 | `message` | min-width 180px *(原200)* | 溢出tooltip |
| 时间 | `createdAt` | 170px *(原180)* | 格式化日期 |

### 3. 前端 - MainLayout.vue 底部版权信息

**修改前**：
```html
<el-footer class="footer">
  <span>© 2024 CRM Platform Admin v1.0.0</span>
</el-footer>
```

**修改后**：
```html
<el-footer class="footer">
  <div class="footer-content">
    <span>{{ footerCopyright }}</span>
    <!-- ICP备案号（条件显示） -->
    <!-- 公安备案号（条件显示） -->
    <!-- 技术支持（条件显示） -->
  </div>
</el-footer>
```

**版权文字逻辑**（`footerCopyright` computed）：
1. 优先使用系统配置的 `copyrightText`
2. 如无，自动拼接：`© 当前年份 公司名称 v版本号`
3. 如连公司名也无，降级显示 `© 当前年份 CRM Platform Admin`

**数据来源**：`GET /api/v1/admin/system-config` → `admin_system_config` 记录，与CRM前端通过 `GET /api/v1/admin/public/system-config` 读取的是**同一份数据源**（`system_config` 表的 `admin_system_config` 行），实现了Admin与CRM的版权/备案信息同步。

## 与CRM前端的同步关系

```
system_config 表 (config_key = 'admin_system_config')
    │
    ├── Admin后台 MainLayout.vue
    │   └── GET /api/v1/admin/system-config（需认证）
    │       → 获取完整配置，渲染 footer 版权/备案信息
    │
    └── CRM前端 App.vue
        └── GET /api/v1/admin/public/system-config（公开接口）
            → 获取过滤后的配置，渲染 footer 版权/备案信息
```

两端读取同一数据源，管理员在 Admin 后台的"基础配置"→"系统配置"中修改版权/备案信息后，CRM前端和Admin后台底部均会同步更新。

## 测试验证

1. **验证日志**：登录Admin后台 → 仪表盘 → 滚动到底部"最近验证日志"区域 → 确认第一列"客户名称"正常显示
2. **版权信息**：登录Admin后台 → 任意页面底部 → 确认显示系统配置中的版权文字、备案号等（而非硬编码的 `© 2024 CRM Platform Admin v1.0.0`）
3. **配置同步**：在Admin后台"系统设置 > 基础配置"中修改版权文字或备案号 → 刷新页面 → 确认底部footer同步更新

## 注意事项

- 后端 `recent-logs` 使用 `LEFT JOIN`，即使 `license_id` 在 `licenses` 表中不存在也不会丢失日志记录
- footer 所有备案/技术支持信息均为条件渲染（`v-if`），如系统未配置则不显示，不会出现空白占位
- 获取系统配置失败时（如网络异常），footer 降级显示 `© 当前年份 CRM Platform Admin`，不影响页面使用
