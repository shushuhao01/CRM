## 二、已发现问题与已执行修复

### 2.1 🔴 严重问题（已修复）

#### ① targetSdkVersion 过低 → 已升级至 33
**文件**: `src/manifest.json`
**问题**: 原来设置为30（Android 11），国内应用商店2024年起普遍要求≥31，Google Play要求≥33。
**修复**: 升级 `targetSdkVersion` 为 33。

#### ② Android 11+ 存储权限适配不完整
**文件**: `src/manifest.json` + `src/services/recordingService.ts`
**问题**: 原来使用 `READ_EXTERNAL_STORAGE`/`WRITE_EXTERNAL_STORAGE` 在 Android 11+ 已废弃，无法正常读取通话录音。
**修复**:
- manifest.json 添加 `READ_MEDIA_AUDIO` + `MANAGE_EXTERNAL_STORAGE` 权限
- recordingService.ts `checkPermissions()` 方法完全重写，按 SDK 版本分级适配

#### ③ 登录密码明文存储 → 已改为加密存储
**文件**: `src/pages/login/index.vue`
**问题**: "记住密码"功能使用 `uni.setStorageSync('savedPassword', password)` 明文保存。
**修复**: 改用 `setEncryptedStorage`/`getEncryptedStorage` 加密读写。

#### ④ 版本号硬编码分散 → 已统一使用 APP_VERSION
**文件**: `src/utils/device.ts` + `src/services/websocket.ts`
**问题**: `device.ts` 和 `websocket.ts` 中硬编码 `'1.0.0'`，更新版本时容易遗漏。
**修复**: 统一 import `APP_VERSION` 常量。

#### ⑤ 无全局错误处理 → 已添加
**文件**: `src/App.vue`
**问题**: 未注册 `onError` 回调，JS异常会导致白屏无提示。
**修复**: 添加 `onError` 全局错误捕获。

#### ⑥ 无Android返回键处理 → 已添加双击退出
**文件**: `src/App.vue`
**问题**: 用户按返回键可能直接退出应用或行为异常。
**修复**: 添加 `plus.key.addEventListener('backbutton')` 双击退出逻辑。

#### ⑦ 权限声明不完整 → 已补充
**文件**: `src/manifest.json`
**新增权限**:
- `ACCESS_WIFI_STATE` - 获取WiFi状态
- `READ_MEDIA_AUDIO` - Android 13+ 读取音频
- `MANAGE_EXTERNAL_STORAGE` - Android 11+ 全文件访问
- `REQUEST_INSTALL_PACKAGES` - 应用更新安装
- `POST_NOTIFICATIONS` - Android 13+ 通知权限

#### ⑧ 应用配置常量不完整 → 已补充
**文件**: `src/config/app.ts`
**新增**: 版本码、包名、公司名、客服电话/邮箱/微信等常量集中管理。

## 四、安卓端打包上架完整流程

### 4.1 准备阶段

#### 步骤1：注册 DCloud 开发者账号
1. 访问 https://dev.dcloud.net.cn/
2. 使用公司邮箱注册账号
3. 创建应用 → 获得正式 AppID（格式类似 `__UNI__XXXXXXX`）
4. 修改 `src/manifest.json` 中的 `appid` 字段

#### 步骤2：生成 Android 签名证书
```bash
# 使用 JDK 的 keytool 生成签名文件
keytool -genkey -alias crm_dialer -keyalg RSA -keysize 2048 -validity 36500 -keystore crm-dialer-release.keystore

# 按提示输入：
# - 密钥库口令（务必牢记并安全保存）
# - 姓名：广州仙狐网络科技有限公司
# - 组织单位：技术部
# - 组织名称：广州仙狐网络科技有限公司
# - 城市：广州
# - 省/自治区：广东
# - 国家代码：CN
```

> ⚠️ **重要**: 签名文件和密码必须永久安全保存！丢失后无法更新应用。

#### 步骤3：获取签名证书信息
```bash
# 获取 SHA1/SHA256 指纹（各应用商店需要）
keytool -list -v -keystore crm-dialer-release.keystore -alias crm_dialer
```

### 4.2 构建阶段

#### 方式A：HBuilderX 云端构建（推荐）
1. 安装 [HBuilderX](https://www.dcloud.io/hbuilderx.html) 最新版
2. 导入 `crmAPP` 项目
3. 菜单 → 发行 → 原生App-云打包
4. 配置项：
   - 选择 Android 平台
   - 上传签名证书 `.keystore` 文件
   - 填写别名和密码
   - 选择 "使用公共测试证书" 或 "使用自有证书"
   - 打包类型选择 "正式包"
5. 提交打包，等待完成后下载 APK

#### 方式B：本地离线构建
1. 下载 [uni-app 原生SDK](https://nativesupport.dcloud.net.cn/AppDocs/download/android)
2. 使用 Android Studio 集成构建
3. 配置 `build.gradle` 签名信息
4. 执行 `./gradlew assembleRelease`
