# 管理后台客户删除回收站功能 - 2026-04-02

## 任务概述

为管理后台（admin）的私有客户和租户客户列表的"更多"按钮中的删除功能实现**软删除**，并在系统设置中新增**回收站**页面，支持查看已删除客户、恢复和彻底删除。

## 问题分析

### 原有问题
1. 私有客户列表和租户客户列表的删除操作为**物理删除**（直接从数据库删除），数据不可恢复
2. 系统设置中没有回收站功能
3. `licenses` 和 `tenants` 数据库表中没有 `deleted_at` 软删除字段

### 修复方案
- 数据库层面：为 `licenses` 和 `tenants` 表添加 `deleted_at`、`deleted_by` 字段
- 后端层面：将删除操作改为软删除（设置 `deleted_at` 时间戳），列表查询排除已删除记录，新增回收站 API
- 前端层面：修改删除确认提示文案，新增回收站页面

## 数据库变更

### licenses 表新增字段
```sql
ALTER TABLE `licenses`
  ADD COLUMN `deleted_at` DATETIME NULL DEFAULT NULL COMMENT '软删除时间（回收站）' AFTER `updated_at`,
  ADD COLUMN `deleted_by` VARCHAR(36) NULL DEFAULT NULL COMMENT '删除操作人' AFTER `deleted_at`,
  ADD INDEX `idx_licenses_deleted_at` (`deleted_at`);
```

### tenants 表新增字段
```sql
ALTER TABLE `tenants`
  ADD COLUMN `deleted_at` DATETIME NULL DEFAULT NULL COMMENT '软删除时间（回收站）' AFTER `updated_at`,
  ADD COLUMN `deleted_by` VARCHAR(36) NULL DEFAULT NULL COMMENT '删除操作人' AFTER `deleted_at`,
  ADD INDEX `idx_tenants_deleted_at` (`deleted_at`);
```

## 功能说明

### 删除流程（软删除）
1. 用户在客户列表点击"更多 → 删除"
2. 弹出确认框，提示"删除后将移入回收站，可在系统设置的回收站中恢复"
3. 确认后，后端设置 `deleted_at = NOW()`，记录不会被物理删除
4. 该客户从列表中消失，统计数据同步更新

### 回收站功能
- **查看**：系统设置 → 回收站，展示所有已删除的私有客户和租户客户
- **筛选**：支持按客户类型（全部/私有客户/租户客户）筛选，支持关键词搜索
- **恢复**：点击"恢复"按钮，清除 `deleted_at`，客户重新出现在客户列表
- **彻底删除**：点击"彻底删除"，物理删除数据库记录及关联日志，不可恢复
- **清空回收站**：一键清空所有回收站记录

## 生产环境部署步骤
1. 在 phpMyAdmin 中执行 `production-add-soft-delete-for-recycle-bin.sql`
2. 重新部署后端服务
3. 重新构建并部署 admin 前端

## 注意事项
- 之前通过物理删除的客户**无法恢复**，软删除仅对迁移后的新删除操作生效
- `deleted_at IS NULL` 条件已添加到列表查询和统计查询，不影响现有正常数据展示
- 回收站数据没有自动过期清理机制，需要管理员手动操作"彻底删除"或"清空回收站"
