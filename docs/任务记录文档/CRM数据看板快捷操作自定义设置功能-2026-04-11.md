**影响范围**: CRM前端 - 数据看板（Dashboard）  
**任务状态**: ✅ 已完成（含二轮优化）

---

## 一、需求描述

CRM系统主视图数据看板中的快捷操作卡片进行功能增强：

1. **添加设置入口** - 在快捷操作卡片右上角添加设置图标按钮
2. **支持自定义选择** - 用户点击设置后弹出配置弹窗，可从系统可用操作中选择最多 **6个** 快捷操作
3. **布局选择** - 设置弹窗中支持选择一行2个或一行3个按钮布局，默认一行2个
4. **角色权限过滤** - 根据用户角色和权限自动过滤，仅显示用户有权限访问的操作项
5. **分类展示** - 操作按分类（客户管理/订单管理/物流管理等）分组展示，超多选项时支持翻页
6. **用户偏好持久化** - 使用 localStorage 按用户ID保存配置（含布局偏好），刷新后保留
7. **自适应布局** - 按钮自适应卡片大小，不同列数下自动调整图标和字体尺寸，保持与左右卡片协调

### 默认4个快捷操作
- 新增客户、新增订单、客户查询、物流查询

## 三、技术实现详解

### 3.1 模板层改造

#### 快捷操作卡片
- 卡片头部：Setting 图标按钮（hover 时渐显，不喧宾夺主）
- 操作网格支持动态列数：`:class="cols-${quickActionsColumns}"`
- 3列模式下图标缩小为18px，文字缩小为12px，自适应卡片空间
- 空状态引导用户去设置

#### 设置弹窗（全新UI设计）
- **顶部**：图标+标题+副标题说明
- **布局设置区**：蓝色渐变背景条，左侧布局切换（一行2个/一行3个 Radio），右侧已选计数徽章
- **操作选择区**：按分类（10个分类）分组展示，每个分类有图标+标题
  - 分类内3列网格展示操作卡片
  - 卡片点击选中（蓝色高亮+弹跳勾选动画），再次点击取消
  - 满6个时未选中的卡片自动灰化禁用
- **翻页**：每页4个分类，超出则显示翻页控制器
- **底部**：左侧"恢复默认"按钮，右侧"取消"+"保存设置"按钮
- 使用临时编辑状态（tempSelectedKeys / tempQuickColumns），取消不影响实际配置

### 3.2 脚本层改造

#### 数据定义 - 26个可选快捷操作（10个分类）

| 分类 | 操作项 |
|------|--------|
| 客户管理 | 新增客户、客户列表、客户查询、客户标签 |
| 订单管理 | 新增订单、订单列表、订单审核 |
| 物流管理 | 物流查询、发货列表、物流跟踪、物流公司 |
| 售后管理 | 新建售后、售后订单、售后数据 |
| 业绩统计 | 个人业绩、团队业绩、业绩分析、业绩分享 |
| 商品管理 | 商品列表、库存管理 |
| 财务管理 | 绩效数据 |
| 资料管理 | 资料列表、回收站 |
| 服务管理 | 短信管理、通话管理 |
| 系统管理 | 用户管理、部门管理 |

#### 角色默认快捷操作

```typescript
super_admin / admin / department_manager / sales_staff:
  → ['add-customer', 'add-order', 'search-customer', 'search-logistics']

customer_service:
  → ['order-list', 'service-list', 'shipping-list', 'search-logistics']
```

#### 布局配置持久化

```
localStorage key:
  crm_dashboard_quick_actions_{userId}  → 选中的key数组
  crm_dashboard_quick_layout_{userId}   → 列数 (2 或 3)
```

#### 核心计算属性

| 属性 | 说明 |
|------|------|
| `permittedQuickActions` | 权限过滤后的全部可选操作 |
| `categorizedActions` | 按分类组织的有权限操作（含图标/颜色元数据） |
| `pagedCategories` | 当前页要显示的分类（每页4个） |
| `totalCategoryPages` | 总分类页数 |
| `visibleQuickActions` | 最终在卡片上显示的操作列表 |

#### 弹窗编辑流程

```
点击设置 → openQuickSettings()
  → 拷贝 selectedQuickKeys → tempSelectedKeys
  → 拷贝 quickActionsColumns → tempQuickColumns
  → 编辑（点选/取消/切换布局）
  → 保存：写入 selectedQuickKeys + quickActionsColumns + localStorage
  → 取消：丢弃临时状态，不影响实际配置
```

### 3.3 样式层（全新设计）

#### 快捷操作卡片样式
- `.quick-actions.cols-3`：三列网格，gap缩小到10px
- `.cols-3 .action-item`：padding缩小，适配更紧凑布局
- `.cols-3 .action-icon`：34px（默认38px）
- `.cols-3 .action-label`：12px字号
- 所有按钮有 hover 浮起效果 + 图标光泽扫过动画

#### 设置弹窗样式
- `.qs-layout-section`：蓝色渐变背景的布局设置条
- `.qs-selected-count`：药丸计数徽章，满额时变橙色警示
- `.qs-category-header`：分类标题（彩色图标+粗体文字）
- `.qs-action-card`：操作选择卡片，选中时蓝色边框+背景
- `.qs-action-check`：蓝色圆形勾选标记，弹跳动画
- `.qs-pagination`：翻页控制器
- `.qs-dialog-footer`：两端对齐的操作按钮区

## 四、权限过滤机制说明

### 4.1 权限检查流程

```
用户登录 → 后端返回角色权限列表 → 存入 userStore.permissions
                                          ↓
Dashboard挂载 → permittedQuickActions computed触发
                                          ↓
遍历 allQuickActions (26项) → 检查每项的 permissions 数组
                                          ↓
      使用 userStore.hasPermission(perm) 检查
                                          ↓
      ┌── admin/super_admin: 直接返回全部 26 项
      └── 其他角色: permissions.some() 匹配
                                          ↓
      过滤后 → 按 category 分组 → 设置弹窗分类展示
                                          ↓
      用户选择 (最多6项) → visibleQuickActions → 卡片渲染
```

### 4.2 各角色可见操作示例

| 角色 | 可见操作数 | 默认显示 |
|-----|-----------|---------|
| 超级管理员 | 全部26个 | 新增客户、新增订单、客户查询、物流查询 |
| 管理员 | 全部26个 | 新增客户、新增订单、客户查询、物流查询 |
| 部门经理 | 视权限配置 | 新增客户、新增订单、客户查询、物流查询 |
| 销售人员 | 视权限配置 | 新增客户、新增订单、客户查询、物流查询 |
| 客服人员 | 视权限配置 | 订单列表、售后订单、发货列表、物流查询 |

## 五、参考实现

本功能参考了 admin 管理后台的 Dashboard 快捷操作设置实现（`admin/src/views/Dashboard.vue` 297-349行），在此基础上增加了：

1. **权限过滤** - CRM端需要基于角色权限动态过滤
2. **分类组织** - 26个操作按10个分类组织，翻页展示
3. **布局选择** - 支持2列/3列布局切换
4. **临时编辑状态** - 弹窗编辑不影响实际展示，保存才生效
5. **用户ID隔离** - localStorage key按用户ID隔离
6. **自适应布局** - 不同列数下图标/字号/间距自动调整
7. **全新设计UI** - 卡片式选择、分类标题、弹跳动画、渐变背景
