### 一、数据隔离安全修复

#### 1.1 wecom.ts 企业微信路由 - 租户隔离修复 ✅
- **文件**: `backend/src/routes/wecom.ts`
- **修改内容**:
  - 添加 `import { getTenantRepo } from '../utils/tenantRepo'` 导入
  - 24处 `AppDataSource.getRepository()` 调用中，23处替换为 `getTenantRepo()`
  - 保留1处（回调接口，无认证上下文，需遍历所有租户配置验证签名）
  - 涉及实体: `WecomConfig`、`WecomUserBinding`、`WecomCustomer`、`WecomAcquisitionLink`、`WecomServiceAccount`、`WecomPaymentRecord`
- **影响**: 防止跨租户读取/修改企业微信配置和客户数据

#### 1.2 license.ts 授权路由 - 用户数统计修复 ✅
- **文件**: `backend/src/routes/license.ts` (第183行)
- **修改内容**: 
  - `SELECT COUNT(*) as count FROM users WHERE status = 'active'`
  - → `SELECT COUNT(*) as count FROM users WHERE status = 'active' AND tenant_id IS NULL`
- **影响**: 私有部署下只统计本租户的用户数

#### 1.3 create-admin-user.js 管理员创建脚本 - 租户过滤修复 ✅
- **文件**: `backend/create-admin-user.js` (第21行)
- **修改内容**:
  - `where: { username: 'admin' }`
  - → `where: { username: 'admin', tenantId: null }`
- **影响**: 防止在SaaS环境下找到其他租户的admin用户

#### 1.4 数据库唯一约束修复SQL ✅
- **文件**: `backend/database-migrations/20260404-private-deploy-unique-constraints.sql`
- **内容**: 
  - `users` 表添加 `(tenant_id, username)` 复合唯一索引
  - `roles` 表修改 `code` 唯一约束为 `(tenant_id, code)` 租户级
  - 包含冲突数据检查和幂等执行支持

### 二、私有部署一键安装体系

#### 2.1 目录结构 ✅
```
private-deploy/
├── install.bat              # Windows一键安装脚本
├── install.sh               # Linux/macOS一键安装脚本
├── init-mysql-database.js   # MySQL数据库自动初始化
├── nginx.conf.template      # Nginx配置模板
├── clean-for-delivery.bat   # 交付包安全清理脚本
└── README.md                # 私有部署完整指南
```

#### 2.2 install.bat - Windows一键安装脚本 ✅
- 7步安装流程: 环境检测 → 数据库初始化 → 安装后端依赖 → 构建后端 → 构建前端 → 安装PM2 → 启动服务
- 自动检测 Node.js、npm、MySQL
- 调用 init-mysql-database.js 完成数据库初始化
- 使用PM2管理后端进程

#### 2.3 install.sh - Linux一键安装脚本 ✅
- 与 install.bat 相同的7步流程
- 支持宝塔面板环境
- 自动设置PM2开机自启（root权限下）

#### 2.4 init-mysql-database.js - MySQL数据库自动初始化 ✅
- 交互式向导收集数据库配置
- 自动创建数据库
- 导入 `database/schema.sql` 表结构
- 插入初始数据（角色、权限、默认管理员admin/admin123、物流公司）
- 自动生成 `backend/.env` 配置文件（含随机JWT_SECRET）
- 支持环境变量传参（用于自动化部署）

#### 2.5 nginx.conf.template - Nginx配置模板 ✅
- Vue Router history模式支持
- API反向代理（/api/ → :3000）
- WebSocket代理（Socket.IO + 移动端）
- 文件上传/录音目录映射
- Gzip压缩配置
- 安全配置（禁止访问 .env、node_modules 等）
- HTTPS配置模板（注释状态，按需启用）

#### 2.6 clean-for-delivery.bat - 交付包安全清理脚本 ✅
- 清除所有 .env 文件（所有目录）
- 清除开发数据库文件
- 清除日志、上传文件、录音文件
- 清除 node_modules 和构建产物
- 清除含敏感信息的开发脚本
- 交互确认防止误操作

#### 2.7 README.md - 私有部署完整指南 ✅
- 系统要求（Node.js、MySQL、硬件推荐）
- 三种部署方案详细步骤：
  - 方案一：Windows本地部署
  - 方案二：Linux服务器部署
  - 方案三：宝塔面板部署
- 首次使用配置指引
- 团队成员分享方式
- 常见问题FAQ
- 维护与升级指南
