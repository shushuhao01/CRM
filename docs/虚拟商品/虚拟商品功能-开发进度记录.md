### 核心流程

1. **审核通过 → 虚拟无需发货型** → 直接 completed（completionSource=audit_auto_complete）
2. **审核通过 → 卡密/资源型** → virtual_pending（虚拟待发货）
3. **审核通过 → 实物商品** → pending_shipment（待发货）
4. **审核拒绝** → 释放所有预占卡密/资源
5. **虚拟发货确认** → 库存标记used + 写入发货记录 + 生成领取令牌 + completed
6. **批量虚拟发货** → 自动使用预占库存发货

## 全流程验收总结

### 虚拟订单完整流程（已签收为最终状态）

```
下单（预占卡密/资源）
  → 提审 → 待审核
  → 审核通过
    ├── 无需发货型 → 直接 signed（已签收）✅
    └── 卡密/资源型 → virtual_pending（虚拟待发货）
          → 物流发货页确认发货（选卡密/资源）
          → signed（已签收）✅ + 生成claimToken
  → 客户通过领取链接 + 手机号 + 密码/验证码 登录
  → 查看并确认领取（status→claimed）✅
  → 业绩统计：signed状态纳入签收业绩计算 ✅
  → 代收管理：不出现 ✅
  → 物流列表/状态更新：不出现 ✅
  → 售后：隐藏退货/换货/维修，保留退款/重发 ✅
```
