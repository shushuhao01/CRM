# 物流状态自动同步 - 开发文档

> 日期: 2026-04-09
> 版本: 1.0

## 一、功能概述

物流状态自动同步服务，根据最新的物流动态信息自动检测物流状态，并安全地将终态物流状态映射为订单状态更新。

### 核心原则
- **订单状态 ≠ 物流状态**：两者是不同维度
- **退回优先于签收**：检测优先级保证"退回签收"不会被误判为"已签收"
- **在途不动订单**：揽收/运输/派送 只更新物流状态字段，不改变订单状态
- **安全保护**：只处理 shipped/rejected/package_exception 状态的订单

## 三、物流状态检测优先级

```
1️⃣ 退回 returned        （最高优先，防止"退回签收"误判）
2️⃣ 拒收 rejected
3️⃣ 签收 delivered       （排在退回/拒收之后）
4️⃣ 异常 exception
5️⃣ 派送中 out_for_delivery
6️⃣ 运输中 in_transit    （含转寄/改派/改地址）
7️⃣ 已揽收 picked_up
8️⃣ 待揽收 pending
```

## 四、物流状态 → 订单状态映射规则

| 物流状态 | 当前订单状态 | → 目标订单状态 |
|---|---|---|
| pending/picked_up/in_transit/out_for_delivery | 任意 | **不更新** |
| delivered | shipped | → delivered |
| rejected | shipped | → rejected |
| exception | shipped | → package_exception |
| returned | rejected | → rejected_returned |
| returned | shipped | → logistics_returned |
| returned | package_exception | → logistics_returned |
| 其他组合 | — | **不更新** |

## 五、API 接口

### 手动触发同步
```
POST /api/v1/logistics/status/auto-sync/trigger
```

### 查看同步状态
```
GET /api/v1/logistics/status/auto-sync/status
```

### 预览检测结果（调试用）
```
POST /api/v1/logistics/status/auto-sync/preview
Body: { "description": "快件已签收", "currentOrderStatus": "shipped" }
```

## 六、同步触发方式

| 方式 | 说明 |
|------|------|
| **定时任务** | 每15分钟自动执行，扫描所有 shipped/rejected/package_exception 订单 |
| **实时触发** | 每次查询物流轨迹时（单个/批量），自动检测并更新 |
| **手动触发** | 通过 API 手动触发一次完整同步 |

## 七、关键BUG修复

### 后端 `detectStatusFromDescription` 优先级错误（已修复）
- **问题**：签收判断在退回/拒收之前，导致"退回签收"被错误识别为"已签收"
- **位置**：`LogisticsTraceService.ts` 第1358行
- **修复**：改为调用统一的 `detectLogisticsStatus` 函数（退回优先于签收）

### 转寄/改派/改地址 归类错误（已修复）
- **问题**：原本归为异常(exception)，但转寄是正常的重新路由
- **修复**：移至运输中(in_transit)，转寄后签收视为正常签收
