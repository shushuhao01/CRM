# 物流面单打印功能优化记录

**日期**: 2026-04-08  
**模块**: 物流管理 → 发货列表 → 打印面单 / 批量打印面单  
**状态**: ✅ 已完成

---

## 一、背景问题

面单打印功能已开发一半，存在以下问题需要处理：

1. **运单号获取时机不对**：原设计为"留空则自动生成"，在用户点击"打印面单"时才调用物流API获取运单号。导致打印等待时间过长（1分钟+），且订单多时新窗口渲染慢。
2. **全屏预览需要打印按钮**：在新窗口右上角需要"打印全部面单"按钮，先预览再打印。
3. **打印后需确认发货**：打印执行后弹出确认对话框，用户确认已打印才更新发货状态。

## 二、修改方案与实施

### 2.1 依赖确认

### 2.2 运单号获取前置化（核心改动）

#### 设计原则
- ❌ 不允许"留空自动生成"
- ✅ 必须先通过按钮调用API获取运单号，或手动输入
- ✅ 运单号就绪后才能点击"打印面单"

#### 单个打印 (`PrintLabelDialog.vue`)

**改动内容：**
- 运单号输入框旁增加 **「获取单号」按钮**
  - 调用 `logisticsApi.checkCreateOrderSupport()` 检查支持
  - 支持则调用 `logisticsApi.createOrder()` 获取运单号
  - 获取成功自动填充到输入框并刷新面单预览
  - 不支持或失败给出明确提示让用户手动输入
- 已有运单号时重新获取会弹出确认提示
- **打印按钮禁用条件**：无物流公司 或 无运单号时灰色不可点
- 移除了 `handlePrint()` 中原有的自动获取运单号逻辑

#### 批量打印 (`BatchPrintLabelDialog.vue`)

**改动内容：**
- 每行运单号输入框旁增加 **单个获取按钮**（圆形按钮）
- 工具栏增加 **「批量获取运单号」按钮**，显示缺运单号的数量
  - 并发检查所有缺运单号订单的物流API支持
  - 逐个调用API获取运单号
  - 完成后汇报成功/失败数量
- 状态标签更新：
  - `就绪` = 有物流公司 + 有运单号（绿色）
  - `缺单号` = 有物流公司但无运单号（黄色）
  - `不支持` = 物流API不支持自动打单（红色）
  - `缺物流` = 未选择物流公司（红色）
- **打印按钮**只计算"就绪"订单数，按钮文本为"打印就绪订单 (N)"
- 移除了 `handleBatchPrint()` 中原有的打印时获取运单号逻辑

### 2.3 全屏预览（已有实现，确认正确）

- **单个打印**：点击「全屏预览」→ 新窗口打开面单
- **批量打印**：点击「全部预览（横排）」→ 新窗口横排展示所有就绪面单
- 新窗口右上角固定工具栏：
  - 显示"共 N 张面单，每张一页纸"
  - **「🖨️ 打印全部面单」按钮**，点击触发 `window.print()`
  - 打印时工具栏自动隐藏（`@media print` 规则）
- 一个面单一张纸（`page-break-after: always`）

### 2.4 打印后确认发货（已有实现，确认正确）

流程：
1. 用户点击「打印面单」→ 弹出确认框（显示物流公司、运单号）
2. 确认后打开打印窗口 → 调用 `window.print()`
3. 打印窗口关闭后 → 弹出 **「确认打印结果」对话框**：
   - ✅ 已成功打印，标记发货 → 调用 `orderApi.update()` 更新订单状态为 `shipped`
   - ❌ 未打印 / 打印失败 → 订单状态不变更
4. 批量打印同理，逐个更新订单状态并汇报结果

### 2.5 新窗口渲染速度优化

**优化 `printService.ts` 的 `printLabels()` 函数：**
- 所有条形码/二维码在主窗口通过 `prepareLabelCodes()` 前置生成
- 条形码生成为内联 SVG 字符串
- 二维码生成为 DataURL 的 `<img>` 标签
- 新窗口只写入纯 HTML，无任何外部资源加载
- 增加备用触发机制：`onload` 300ms 内未触发則直接执行打印，避免卡死

## 四、功能流程图

### 单个打印面单流程
```
选择订单 → 打开打印面单弹窗
  → 选择物流公司
  → 点击「获取单号」（或手动输入）
  → 运单号填充 → 面单预览自动更新
  → [可选] 点击「全屏预览」→ 新窗口预览 → 右上角「打印全部面单」
  → 点击「打印面单」（需运单号就绪）
  → 确认打印信息
  → 打印窗口弹出 → 执行打印
  → 确认是否已打印
  → ✅ 已打印 → 订单标记为已发货
  → ❌ 未打印 → 状态不变
```

### 批量打印面单流程
```
勾选多个订单 → 打开批量打印弹窗
  → 批量设置物流公司（应用到全部/选中）
  → 点击「批量获取运单号」（或逐个获取/手动输入）
  → 状态标签显示就绪数量
  → [可选] 点击「全部预览（横排）」→ 新窗口横排预览 → 右上角「打印全部面单」
  → 点击「打印就绪订单 (N)」（需至少1个就绪）
  → 确认打印信息
  → 打印窗口弹出 → 执行打印
  → 确认是否已打印
  → ✅ 已打印 → 所有就绪订单标记为已发货
  → ❌ 未打印 → 状态不变
```

## 五、技术要点

1. **前置编码生成**：`prepareLabelCodes()` 使用本地 `jsbarcode` + `qrcode` 包，生成内联 SVG/DataURL，无CDN依赖
2. **新窗口秒开**：所有资源已内联到 HTML 中，`document.write()` 后即可渲染
3. **打印分页**：CSS `page-break-after: always` 确保每张面单独立一页
4. **状态管理**：打印确认后通过 `orderApi.update()` + `orderStore.updateOrder()` 双重更新
5. **API调用链**：`checkCreateOrderSupport()` → `createOrder()` → 返回运单号

## 六、第二轮优化（2026-04-08 下午）

### 6.1 问题1：批量打印订单列表横向铺满 + 列宽自适应

**改动：**
- 对话框宽度从 `950px` → `95vw`，充分利用屏幕宽度
- 所有表格列从 `width` 固定宽度改为 `min-width` 最小宽度，自动铺满
- 列宽调整：订单号 `min-width: 130`、客户 `min-width: 80`、物流公司 `min-width: 150`、运单号 `min-width: 200`、面单模板 `min-width: 160`、状态 `min-width: 90`
- header-right 支持 flex-wrap 防止按钮换行挤压

### 6.2 问题2：全屏预览打印后弹确认发货弹窗

**问题本质：** 用户在预览新窗口点击"打印全部面单"后，浏览器 `window.print()` 执行打印，但父窗口不知道这个事件，无法弹出确认发货的对话框。

**解决方案：**
- 在 `printService.ts` 的 `generatePrintPageHTML` 中，给预览窗口注入 `window.onafterprint` 事件处理器
- 打印完成后通过 `window.opener.postMessage({ type: 'label-print-done', count: N }, '*')` 通知父窗口
- 父窗口（PrintLabelDialog / BatchPrintLabelDialog）通过 `window.addEventListener('message', handler)` 监听消息
- 收到打印完成消息后，弹出 `ElMessageBox.confirm` 确认是否已发货
- 确认 → 更新订单状态为 `shipped`；取消 → 状态不变

**修改文件：**
- `src/utils/printService.ts` — 工具栏打印按钮增加 `onafterprint` + `postMessage`
- `src/views/Logistics/components/PrintLabelDialog.vue` — `handlePreview` 增加 message 监听
- `src/views/Logistics/components/BatchPrintLabelDialog.vue` — `handlePreviewAll` 增加 message 监听

### 6.3 问题3：自定义模板正确映射到选择器

**问题本质：** 在面单模板管理页面创建的自定义模板，保存到 localStorage 后，打印面板的模板下拉列表没有刷新。

**解决方案：**
- 增加 `templateRefreshKey` 响应式变量作为依赖触发器
- `filteredTemplates` 和 `currentTemplate` 计算属性中读取 `templateRefreshKey.value`
- 模板管理弹窗关闭时（`watch(templateDialogVisible)`）和 `onTemplateSelected` 时自增触发器
- 确保 `getAllTemplates()` 每次重新从 `PRESET_TEMPLATES + localStorage` 读取

### 6.4 问题4：模板支持应用到选中订单 + 每行可下拉选择

**改动：**
- 批量打印订单列表的"模板"列从纯文本展示改为 `<el-select>` 下拉选择器
- 每个订单行可以独立选择不同的面单模板
- "默认模板"旁增加**「应用到选中/全部」**按钮（`applyBatchTemplate` 方法）
- 批量应用物流公司时如果开启"自动匹配物流公司模板"，会自动匹配对应模板

### 6.5 问题5：面单"收"和"寄"标签对比度优化

**问题：** 收件人标签 `#000` 黑底白字还行，但寄件人标签 `#555` 灰底白字看不清。

**解决方案：**
- 收件标签 `.sl-tag-recv`：`background: #1a1a1a` 深黑色 + 白色文字
- 寄件标签 `.sl-tag-send`：`background: #2b6cb0` 深蓝色 + 白色文字（高反差）
- 标签 padding 从 `1px 6px` 增大到 `2px 8px`，增加 `letter-spacing: 1px`
- 三个地方同步修改：PrintLabelDialog.vue、BatchPrintLabelDialog.vue、printService.ts（实际打印输出）
