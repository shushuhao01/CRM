## 后续维护建议

1. **新文档**：所有新的文档都应该直接创建在 `docs/` 目录下
2. **临时文件**：开发过程中的临时文件应该放在 `docs/临时文件/` 或 `docs/kiro-session-notes/`
3. **定期清理**：定期检查并清理不再需要的临时文件
4. **文档分类**：建议在 `docs/` 下创建子目录进行分类管理：
   - `docs/部署文档/` - 部署相关
   - `docs/开发文档/` - 开发相关
   - `docs/系统说明/` - 系统说明
   - `docs/配置文档/` - 配置说明
