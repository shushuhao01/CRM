# 三系统服务管理指南

## 📋 系统概述

本项目包含三个独立的系统：

1. **CRM系统** - 客户关系管理系统（主系统）
   - 前端端口: 8080
   - 后端端口: 3000

2. **Admin后台** - 管理员后台系统
   - 前端端口: 8081

3. **Website官网** - 官方网站
   - 前端端口: 8082

## 🔧 常用PM2命令

```bash

# 删除单个服务
pm2 delete crm-backend
pm2 delete crm-frontend
pm2 delete admin-frontend
```

```bash
# 停止所有服务
pm2 stop all

# 重启所有服务
pm2 restart all

# 删除所有服务
pm2 delete all

# 保存当前配置
```

## 📝 环境配置

### 环境变量文件

每个系统都需要对应的 `.env` 文件：

```
项目根目录/.env          # CRM前端配置
backend/.env             # 后端配置
admin/.env               # Admin后台配置
website/.env             # Website官网配置
```

如果 `.env` 文件不存在，启动脚本会自动尝试使用 `.env.production` 文件。

### 端口配置

如需修改端口，请编辑对应的配置文件：

**CRM前端** (vue.config.js):
```javascript
devServer: {
  port: 8080
}
```

**Admin后台** (admin/vite.config.ts):
```javascript
server: {
  port: 8081
}
```

**Website官网** (website/vite.config.ts):
```javascript
server: {
  port: 8082
}
```

**后端** (backend/.env):
```
PORT=3000
```

## 🐛 故障排查

### 服务无法启动

1. **检查端口占用**
   ```bash
   # Linux/Mac
   lsof -i :8080
   lsof -i :8081
   lsof -i :8082
   lsof -i :3000
   
   # Windows
   netstat -ano | findstr :8080
   netstat -ano | findstr :8081
   netstat -ano | findstr :8082
   netstat -ano | findstr :3000
   ```

2. **检查依赖安装**
   ```bash
   # 重新安装依赖
   cd backend && npm install
   cd ../admin && npm install
   cd ../website && npm install
   cd .. && npm install
   ```

3. **检查环境变量**
   - 确保所有 `.env` 文件存在且配置正确
   - 检查数据库连接配置
   - 检查API地址配置

### 服务频繁重启

```bash
# 查看错误日志
pm2 logs crm-backend --err
pm2 logs crm-frontend --err

# 查看详细信息
pm2 show crm-backend
```

### 内存占用过高

```bash
# 查看资源使用
pm2 monit

# 重启服务释放内存
pm2 restart all

# 设置内存限制
pm2 start app.js --max-memory-restart 500M
```

## 🔐 生产环境部署

### 使用PM2生态系统文件

创建 `ecosystem.config.js`:

```javascript
module.exports = {
  apps: [
    {
      name: 'crm-backend',
      cwd: './backend',
      script: 'npm',
      args: 'run start:prod',
      env: {
        NODE_ENV: 'production'
      }
    },
    {
      name: 'crm-frontend',
      cwd: './',
      script: 'npm',
      args: 'run serve',
      env: {
        NODE_ENV: 'production'
      }
    },
    {
      name: 'admin-frontend',
      cwd: './admin',
      script: 'npm',
      args: 'run serve',
      env: {
        NODE_ENV: 'production'
      }
    },
    {
      name: 'website-frontend',
      cwd: './website',
      script: 'npm',
      args: 'run serve',
      env: {
        NODE_ENV: 'production'
      }
    }
  ]
};
```

使用生态系统文件启动：

```bash
pm2 start ecosystem.config.js
```

## 🆘 常见问题

### Q1: PM2命令找不到？

```bash
# 全局安装PM2
npm install -g pm2

# 或使用npx
npx pm2 list
```

### Q2: 服务启动后立即停止？

检查日志查看错误原因：
```bash
pm2 logs crm-backend --lines 50
```

常见原因：
- 端口被占用
- 环境变量配置错误
- 依赖未安装
- 数据库连接失败

### Q3: 如何更新代码后重启？

```bash
# 拉取最新代码
git pull

# 安装新依赖（如果有）
npm install

# 重启服务
pm2 restart all
```

### Q4: 如何查看某个时间段的日志？

```bash
# PM2日志文件位置
~/.pm2/logs/

# 使用grep过滤
pm2 logs crm-backend | grep "ERROR"

# 查看最近100行
pm2 logs crm-backend --lines 100
```
