## 二、部署方式总览

### 2.1 部署模式

| 模式 | 说明 | 配置方式 |
|------|------|----------|
| **私有部署** | 单租户，所有数据 `tenant_id = NULL` | `DEPLOY_MODE=private` |
| **SaaS模式** | 多租户，数据按 `tenant_id` 隔离 | `DEPLOY_MODE=saas` + RSA签名验证 |

### 2.2 一键部署脚本

| 脚本 | 平台 | 功能 |
|------|------|------|
| `deploy.ps1` | Windows | 一键构建+部署全部服务 |
| `deploy.sh` | Linux/macOS | 一键构建+部署全部服务 |
| `start-all-services.bat` | Windows | 启动所有服务 |
| `start-all-services.sh` | Linux | 启动所有服务 |
| `stop-all-services.bat/sh` | 全平台 | 停止所有服务 |
| `restart-all-services.bat/sh` | 全平台 | 重启所有服务 |
| `private-deploy/install.bat` | Windows | 私有部署安装 |
| `private-deploy/install.sh` | Linux | 私有部署安装 |

### 2.3 部署流程

```
1. 配置 backend/.env（数据库、JWT密钥等）
2. 运行 deploy.ps1 或 deploy.sh
   → 自动安装依赖
   → 构建CRM前端 + Admin后台 + 官网
   → 编译后端TypeScript
   → PM2启动服务
3. 配置Nginx反向代理（可选，参考 private-deploy/nginx.conf.template）
```

## 三、多租户数据隔离审查（核心安全）

### 3.1 隔离机制架构

```
请求 → JWT认证 → TenantContext设置 → getTenantRepo() Proxy拦截
                                      ↓
                              自动注入 tenant_id 过滤/写入
```

**三层保护机制**：

| 层级 | 组件 | 功能 |
|------|------|------|
| 第1层 | `tenantContextMiddleware` | AsyncLocalStorage 设置租户上下文 |
| 第2层 | `getTenantRepo()` Proxy | 自动拦截 find/save/update/delete/createQueryBuilder，注入 `tenant_id` |
| 第3层 | `tenantSQL()` / `tenantWHERE()` | 原生SQL场景的租户过滤辅助函数 |
