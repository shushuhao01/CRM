## 一、项目全景与产品定位

### 1.1 产品定位

云客CRM是一个 **SaaS + 私有部署双模式** 的完整CRM商业产品体系，覆盖 **6个前端终端 + 1个统一后端**：

| 子系统 | 技术栈 | 规模 | 端口 | 定位 |
|--------|--------|------|------|------|
| **Backend** | Node.js + Express + TypeORM + MySQL | 435个TS文件 | 3000 | 统一后端API服务 |
| **CRM前端** | Vue 3 + TS + Element Plus + Pinia | 373个Vue + 178个TS | 5173 | 主业务系统（员工日常使用） |
| **Admin后台** | Vue 3 + TS + Element Plus | 96个Vue + 15个TS | 5174 | 平台管理后台（运营方使用） |
| **Website官网** | Vue 3 + TS + 纯手写SCSS | 52个Vue + 11个TS | 8080 | 产品展示 + 注册 + 支付 + 会员中心 |
| **crmAPP** | uni-app + Vue 3 + TS | 18个Vue + 16个TS | — | 电销外呼移动APP |
| **H5服务商应用** | Vue 3 + Vant 4 + Vite 5 + TS | 11个Vue + 4个API | 5175 | 企微内嵌工作台H5 |

**全项目代码总量：约 450,000+ 行**

### 1.2 商业闭环

```
潜在客户访问官网 --> 选择套餐 --> 注册/支付 --> 自动开通租户 --> 下发登录信息
                                                                    |
管理后台运营管理 <-- 授权/续费/模块控制/扩容 <-- 管理员 <-- 登录CRM系统
                                                             |
                      +------------------+-------------------+-------------------+
                      |                  |                   |                   |
                 CRM日常使用        移动端APP            会员中心         企微H5工作台
                客户/订单/物流     外呼/录音/跟进       续费/扩容/账单    客户/统计/话术
                企微/虚拟商品/AI   签到/日程/任务     增值服务购买     成员绑定/数据融合
```

### 4.4 JS-SDK鉴权流程验收

```
H5页面加载
    │
    ▼
① 调用 /h5/jssdk-config → 获取企业级签名
    │
    ▼
② wx.config({ corpId, timestamp, nonceStr, signature })
    │
    ▼
③ wx.ready → 企业级SDK就绪
    │
    ▼
④ 调用 /h5/agent-config → 获取应用级签名
    │
    ▼
⑤ wx.agentConfig({ corpid, agentid, timestamp, nonceStr, signature })
    │
    ▼
⑥ 应用级SDK就绪 → 可调用getCurExternalContact/sendChatMessage
```

**JS-SDK封装文件（wecom-sdk.ts）功能**：
- `initJsSdk()` — 企业级SDK初始化
- `initAgentConfig()` — 应用级SDK初始化
- `getContext()` — 获取入口上下文
- `getCurExternalContact()` — 获取当前聊天外部联系人ID
- `sendChatMessage()` — 发送消息到聊天窗口
- `isWecomEnv()` — 检测企微环境

## 六、多端一体集成分析

### 6.1 统一后端架构

所有6个前端终端共享同一个Backend API服务，通过不同的路由前缀区分：

| 路由前缀 | 服务终端 | 认证方式 |
|----------|----------|----------|
| `/api/v1/*` | CRM主系统 | JWT（标准用户Token） |
| `/api/v1/admin/*` | Admin管理后台 | JWT（管理员Token） |
| `/api/v1/public/*` | Website官网/会员中心 | 部分公开 + JWT |
| `/api/v1/mobile/*` | crmAPP移动端 | JWT（移动端Token） |
| `/api/v1/wecom/*` | CRM企微模块 | JWT（标准用户Token） |
| `/api/v1/wecom/h5/*` | H5服务商应用 | JWT（Sidebar Token） |
| `/api/v1/wecom/suite/*` | 企微服务商回调 | 企微签名验证 |

### 6.2 统一数据层

| 数据层 | 说明 |
|--------|------|
| 同一数据库 | 所有终端共享同一MySQL数据库（170+张表） |
| 统一实体定义 | 118个TypeORM实体文件，所有终端使用同一数据模型 |
| 租户隔离 | 所有业务表均含tenantId字段，中间件自动过滤 |
| 企微数据融合 | WecomCustomer ↔ Customer 双向关联 |
