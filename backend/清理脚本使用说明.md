# 宝塔服务器清理脚本使用说明

## 📋 脚本列表

### 🔍 诊断工具

#### find-large-files.sh（必看！）
**用途：** 找出真正占用空间的大户

**功能：**
- 分析最大的目录
- 查找大于100MB的文件
- 检查日志、备份、数据库大小
- 生成详细的空间占用报告

**使用方法：**
```bash
chmod +x find-large-files.sh
./find-large-files.sh
```

---

### ⚡ 快速清理（推荐首选）

#### quick-fix-disk.sh（一键清理）
**安全级别：⭐⭐⭐⭐ 安全**

自动找出并清理最占空间的内容，无需手动确认。

**清理内容：**
- 宝塔回收站
- 旧备份（保留最新3个）
- 大日志文件（>100MB）
- 7天前的日志
- MySQL二进制日志（保留3天）
- 临时文件
- 包管理器缓存

**使用方法：**
```bash
chmod +x quick-fix-disk.sh
./quick-fix-disk.sh
```

---

### 🧹 标准清理

#### 1. clean-server-cache.sh
**安全级别：⭐⭐⭐⭐⭐ 非常安全**

只清理缓存和垃圾文件，不会影响任何重要数据。

**清理内容：**
- 系统临时文件（7天前）
- 宝塔临时文件
- 宝塔回收站
- 网站日志（保留7天）
- npm/yarn缓存
- Docker缓存
- 数据库备份（保留7天）
- 网站备份（保留7天）
- 大型系统日志（>100MB）

**使用方法：**
```bash
cd /www/wwwroot/你的项目/backend
chmod +x clean-server-cache.sh
./clean-server-cache.sh
```

---

#### 2. clean-mysql-logs.sh
**安全级别：⭐⭐⭐⭐ 安全**

专门清理MySQL日志文件。

**清理内容：**
- MySQL二进制日志
- 错误日志
- 慢查询日志
- 数据库表优化

**使用方法：**
```bash
chmod +x clean-mysql-logs.sh
./clean-mysql-logs.sh
```

---

### 💣 激进清理（谨慎使用）

#### 1. aggressive-clean.sh
**安全级别：⭐⭐ 需要确认**

删除所有备份和日志，释放最大空间。

**清理内容：**
- 所有数据库备份
- 所有网站备份
- 所有日志文件
- MySQL二进制日志
- 系统缓存

**使用方法：**
```bash
chmod +x aggressive-clean.sh
./aggressive-clean.sh
# 输入 DELETE 确认执行
```

---

#### 2. deep-clean-server.sh
**安全级别：⭐⭐⭐ 需要确认**

深度清理，会清理更多内容，建议先备份。

**额外清理内容：**
- APT/YUM包管理器缓存
- MySQL二进制日志
- 旧内核文件
- systemd日志
- 邮件队列
- core dump文件

**使用方法：**
```bash
cd /www/wwwroot/你的项目/backend
chmod +x deep-clean-server.sh
./deep-clean-server.sh
# 输入 yes 确认执行
```

---

### 3. clean-server-cache.bat（Windows本地）
**用途：** 清理本地开发环境

**使用方法：**
```cmd
cd backend
clean-server-cache.bat
```

## 🚀 推荐使用流程

### 第一步：诊断（必做）

先找出真正占用空间的大户：

```bash
cd /www/wwwroot/你的项目名/backend
chmod +x find-large-files.sh
./find-large-files.sh
```

查看输出，了解哪些目录/文件占用最多。

---

### 第二步：快速清理（推荐）

使用一键清理脚本：

```bash
chmod +x quick-fix-disk.sh
./quick-fix-disk.sh
```

这个脚本会自动清理最常见的垃圾文件，通常可以释放5-15GB空间。

---

### 第三步：如果还不够

如果清理后空间还是不够，根据第一步的诊断结果：

**情况A：备份文件占用大**
```bash
# 删除所有备份（谨慎！）
rm -rf /www/backup/database/*
rm -rf /www/backup/site/*
```

**情况B：日志文件占用大**
```bash
# 删除所有日志
rm -rf /www/wwwlogs/*
```

**情况C：MySQL日志占用大**
```bash
chmod +x clean-mysql-logs.sh
./clean-mysql-logs.sh
```

**情况D：需要全面清理**
```bash
chmod +x aggressive-clean.sh
./aggressive-clean.sh
# 输入 DELETE 确认
```

---

## 🎯 快速使用（最简单）

如果你不想看那么多，直接执行这两个命令：

```bash
# 1. 进入项目目录
cd /www/wwwroot/你的项目名/backend

# 2. 一键清理
bash quick-fix-disk.sh
```

如果还不够，再执行：

```bash
bash aggressive-clean.sh
# 输入 DELETE 确认
```

## 📊 清理效果预期

根据服务器运行时间，通常可以释放：

- **运行1-3个月：** 2-5GB
- **运行3-6个月：** 5-10GB
- **运行6个月以上：** 10-20GB+

主要释放空间来源：
1. 日志文件（通常占最多）
2. 旧备份文件
3. npm/Docker缓存
4. 临时文件

## ⚠️ 注意事项

### 安全提示

1. **首次使用建议：**
   - 先执行 `clean-server-cache.sh`（安全）
   - 观察效果后再考虑 `deep-clean-server.sh`

2. **备份建议：**
   - 清理前建议备份数据库
   - 重要文件做好备份

3. **不会删除的内容：**
   - ✅ 数据库数据
   - ✅ 网站源代码
   - ✅ 上传的图片/文件
   - ✅ 配置文件
   - ✅ 最近7天的备份

4. **会删除的内容：**
   - ❌ 7天前的日志
   - ❌ 7天前的备份
   - ❌ 临时文件
   - ❌ 缓存文件

## 🔧 手动清理补充

如果脚本清理后空间还不够，可以手动检查：

### 1. 查找大文件
```bash
# 查找大于500MB的文件
find /www -type f -size +500M -exec ls -lh {} \;

# 查看目录占用排行
du -h --max-depth=1 /www | sort -hr | head -10
```

### 2. 清理特定目录
```bash
# 清理上传的旧文件（谨慎！）
find /www/wwwroot/项目名/backend/uploads -type f -mtime +90 -ls

# 清理node_modules（可重新安装）
find /www/wwwroot -name "node_modules" -type d -mtime +90
```

### 3. 数据库优化
```bash
# 优化所有数据库表
mysqlcheck -o --all-databases -u root -p
```

## 📅 定期清理建议

### 设置定时任务

在宝塔面板 → 计划任务 → 添加任务：

**任务类型：** Shell脚本
**任务名称：** 自动清理缓存
**执行周期：** 每周
**脚本内容：**
```bash
cd /www/wwwroot/你的项目名/backend
bash clean-server-cache.sh
```

## 🆘 常见问题

### Q1: 执行脚本提示权限不足？
```bash
chmod +x clean-server-cache.sh
# 或使用 sudo
sudo bash clean-server-cache.sh
```

### Q2: 清理后服务异常？
```bash
# 重启相关服务
systemctl restart nginx
systemctl restart mysql
systemctl restart pm2
```

### Q3: 想恢复删除的备份？
备份文件一旦删除无法恢复，建议清理前先下载重要备份到本地。

### Q4: 如何查看清理了多少空间？
脚本执行时会显示清理前后的磁盘使用情况。
